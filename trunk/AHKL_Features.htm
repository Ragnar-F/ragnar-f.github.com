Änderungen von Befehlen

Textkodierungen

FileRead, FileReadLine, Loop Read und FileAppend unterstützen eine Mehrzahl an Textkodierungen, die von Windows unterstützt werden, nicht nur den System-Standard-ANSI-Zeichensatz. Mit FileEncoding kann die Standardkodierung bestimmt werden, die für FileRead und FileAppend wie folgt überschrieben werden kann:

FileRead, Ausgabevariable, *Pnnn Dateiname
FileAppend [, Text, Dateiname, Kodierung]

Während nnn eine numerische Zeichensatz-Kennung sein muss, ist die Kodierung im selben Format wie FileEncoding.

Siehe auch: Script-Kompatibilität

Verbesserungen der Icon-Unterstützung

Ungewöhnliche Größen

Icon-Ressourcen jeglicher unterstützten Größe des Betriebssystems können aus einer ausführbaren Datei extrahiert werden. Wenn Icon-Ressourcen in mehrere Größen innerhalb einer Icon-Gruppe vorhanden sind, dann wird die geeignetste Größe verwendet. Vor Revision 17 wurde willkürlich eine beliebige Icon-Ressource vom System ausgewählt, dass zuerst auf die Standardgröße des Systems und danach wieder auf die gewünschte Größe skaliert wurde.

Ressourcen-Bezeichner

Negative Iconnummern können verwendet werden, um eine Icon-Ressource einer Gruppe innerhalb einer ausführbaren Datei zu kennzeichnen. Im folgenden Beispiel wird das Tray-Icon auf das Standard-Icon von AHK gesetzt:

Menu, Tray, Icon, %A_AhkPath%, -160
