ControlGetText

Ermittelt den Text eines Steuerelements.

ControlGetText, Ausgabevariable [, Steuerelement, Fenstertitel, Fenstertext, Titelausnahme, Textausnahme]

Parameter

Ausgabevariable

Der Name der Variable, die den ermittelten Text enthält.
Steuerelement

Möglich ist entweder ClassNN (Klassenname und Instanznummer des Steuerelements) oder Name/Text des Steuerelements, die beide mit Windows Spy ermittelt werden können. Wenn Name/Text verwendet wird, wird das Übereinstimmungsverhalten mit SetTitleMatchMode bestimmt. Wenn dieser Parameter nicht vorhanden oder leer ist, dann wird das oberste Steuerelement des Fensters benutzt.

Für das HWND eines Steuerelements (Windows Handle) wird der Steuerelement-Parameter leer gelassen und ahk_id %SteuerelementHwnd% im Fenstertitel-Parameter angegeben (funktioniert auch bei versteckten Steuerelementen, selbst wenn DetectHiddenWindows deaktiviert ist). Das HWND eines Steuerelements wird üblicherweise mittels ControlGet Hwnd, MouseGetPos oder DllCall abgerufen.
Fenstertitel Der Titel oder Teiltitel des Fensters (das Übereinstimmungsverhalten wird mit SetTitleMatchMode bestimmt). Wenn dieser und die nächsten 3 Parameter nicht vorhanden sind, dann wird das zuletzt gefundene Fenster verwendet. Wenn dieser den Buchstaben A enthält und die nächsten 3 Parameter nicht vorhanden sind, dann wird das aktive Fenster verwendet. Bei Verwendung einer Fensterklasse muss ahk_class ExakterKlassenname angegeben werden (wird von Windows Spy angezeigt). Bei Verwendung einer Prozess-Kennung (PID) muss ahk_pid %VariableMitPID% angegeben werden. Bei Verwendung einer Fenstergruppe muss ahk_group Gruppenname angegeben werden. Bei Verwendung einer eindeutigen ID-Nummer eines Fensters muss ahk_id %VariableMitID% angegeben werden. Die Suche kann durch Angabe mehrerer Kriterien eingeschränkt werden. Zum Beispiel: Eigene Datei.txt ahk_class Notepad
Fenstertext Wenn vorhanden, dann muss dieser Parameter einen Teil der einzelnen Textelemente des Fensters enthalten (wird von Windows Spy angezeigt). Versteckte Textelemente werden erfasst, wenn DetectHiddenText aktiv ist.
Titelausnahme Fenstertitel mit diesem Wert werden nicht berücksichtigt.
Textausnahme Fenstertexte mit diesem Wert werden nicht berücksichtigt.

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

Bei Problemen wird ErrorLevel auf 1 gesetzt, ansonsten enthält ErrorLevel eine 0.

Bemerkungen

Hinweis: Um Text aus einem ListView-, ListBox- oder ComboBox-Steuerelement zu ermitteln, verwendet stattdessen ControlGet List.

Wenn der ermittelte Text gekürzt (unvollständig) erscheint, dann muss VarSetCapacity(Ausgabevariable, 55) vor ControlGetText verwenden werden [55 sollte durch eine wesentlich größere Zahl als die Länge des Textes ersetzt werden]. Das ist notwendig, weil einige Anwendungen nicht richtig auf die WM_GETTEXTLENGTH-Nachricht reagieren, dass zur Folge hat, dass AutoHotkey die Ausgabevariable zu klein macht, um den ganzen Text zu speichern.

Die Menge des ermittelten Textes ist auf die Maximalkapazität einer Variable begrenzt (die über die #MaxMem-Anweisung geändert werden kann). Folglich könnte dieser Befehl eine große Menge an RAM erzeugen, wenn das Steuerelement (z. B. ein Editor mit einem geöffneten umfangreichen Dokument) eine große Menge an Text beinhaltet. Allerdings kann der Speicher der Variable geleert werden, indem man ihr nichts zuteilt, d. h. Ausgabevariable := ""

Die ermittelten Texte von den meisten Steuerelementen verwenden am Ende jeder Zeile eher CR und LF (`r`n), als ein einzelnes LF (`n).

Es ist nicht notwendig, SetTitleMatchMode Slow zu verwenden, weil ControlGetText den Text immer mithilfe der langsamen Methode ermittelt (da es mit einem breiteren Bereich von Steuerelementarten funktioniert).

Um eine Liste mit Steuerelementen von einem Fenster zu erhalten, verwendet WinGet ControlList.

Fenstertitel und -texte sind von der Groß- und Kleinschreibung abhängig. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.
