Überschreiben oder Deaktivieren von Hotkeys

Alle internen Windows-Hotkeys außer Win+L und Win+U können deaktiviert werden, wenn folgende Änderungen an der Registrierung vorgenommen werden (sollte normalerweise in allen Betriebssystemen funktionieren, erfordert aber möglicherweise ein Neustart):

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer
NoWinKeys REG_DWORD 0x00000001 (1)

Lest weiter, wenn alle Hotkeys mehr als nur deaktiviert werden sollen. Beachtet, dass die meisten folgenden Beispiele nicht in Windows Me/98/95 unterstützt werden.

Hotkeys von anderen Anwendungen können einfach überschrieben oder deaktiviert werden, wenn ihnen eine Aktion im Script zugewiesen wird. Häufig werden dabei die Hotkeys geändert, die selbst in Windows integriert sind. Wenn zum Beispiel der Wunsch besteht, Win+E neu zu belegen (die Tastenkombination für das Starten des Windows Explorers), verwendet dies:

#e::
MsgBox Dieser Hotkey gehört nun zum Script.
Return

Im nächsten Beispiel wird der Hotkey Win+R (für das Öffnen des Ausführen-Fensters) vollständig deaktiviert:

#r::Return

Um auf ähnliche Weise die beiden Win-Modifikatoren zu deaktivieren, verwendet dies:

Lwin::Return
Rwin::Return

Um einen lokalen Hotkey einer Anwendung zu deaktivieren oder zu ändern (eine Tastenkombination, die nur funktioniert, wenn die Anwendung das aktive Fenster ist), zieht das folgende Beispiel in Betracht, dass Steuerung+P (Drucken) nur für den Editor deaktiviert:

$^p::
IfWinActive ahk_class Notepad
    Return  ; nichts tun, damit Steuerung+P im Editor deaktiviert ist.
Send ^p
Return

Im obigen Beispiel wird das $-Präfix benötigt, damit der Hotkey sich selbst senden kann, ohne dabei aktiviert zu werden (ansonsten wird eine Fehlermeldung wegen einer Endlosschleife ausgelöst). Siehe auch: kontextabhängige Hotkeys.

Die oben genannten Beispiele können ausprobiert werden, indem sie in einer neuen Textdatei z. B. namens "Override.ahk" kopiert werden und die Datei danach ausgeführt wird.
