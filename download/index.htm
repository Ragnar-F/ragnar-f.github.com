AutoHotkey_L - Aktuelle Version 1.1.07.03

Installationspaket
Umfasst Binärdateien, englische Dokumentation und Script-Compiler

Während der Installation können folgende Versionen ausgewählt werden:

  • Unicode x86 - Für neue Scripts empfohlen.
  • Unicode x64 - Für höhere Performance auf 64-Bit-Systeme.
  • ANSI x86 - Bessere Kompatibilität mit älteren Scripts.

Compiler - aktualisierte Version von Ahk2Exe, für die Kompilierung von x64-Scripts notwendig.

Dokumentation -  kompilierte Hilfe-Datei auf Basis der originalen AutoHotkey.chm:

  • Englisch (English)
  • Chinesisch (中文)
  • Deutsch (siehe Forumsbeitrag für mehr Informationen)

Einige ältere Versionen können in den Archiven gefunden werden.

Der Quellcode für AutoHotkey_L und Ahk2Exe kann mittels Git oder HTTP-Download auf GitHub eingesehen werden.

Weitere Ressourcen

  • AutoHotkey Basic v1.0.48.05: Das ist die ursprüngliche Version von AutoHotkey, die seit 2009 nicht mehr aktualisiert wird. Die jetzige Dokumentation kann ohne Bedenken für diese Version verwendet werden. Der Quellcode kann auf GitHub eingesehen werden.
  • SciTE4AutoHotkey: Ein frei verfügbarer Editor für AutoHotkey-Scripts. Dieser Editor unterstützt neben DBGp auch Syntaxhervorhebung, Calltips/Parameterinformation, automatische Vervollständigung und weitere nützliche Bearbeitungsfunktionen und Script-Werkzeuge.
  • AutoHotkey.dll: Mit dieser Bibliothek kann der AutoHotkey-Interpreter in jede beliebige Anwendung und Sprache integriert werden. Dadurch wird AutoHotkey-Funktionialität bereitgestellt, die möglicherweise in anderen Sprachen schwieriger zu implementieren ist.
  • Script-Decompiler: Dieses Befehlszeilenprogramm dekompiliert ein kompiliertes Script. Das heißt, dass das originale AHK-Script als Klartextdatei extrahiert wird (falls das Passwort bekannt ist). Allerdings gehen dabei alle ursprünglichen Kommentare (Semikolon oder /* */) verloren. Dieses Programm sollte von der Befehlszeile aus gestartet werden. Zum Beispiel: exe2ahk "Eigenes Script.exe" "Eigenes Passwort".
