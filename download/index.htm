AutoHotkey_L (empfohlen)

Installationspaket - umfasst Binärdateien, englische Dokumentation und Script-Compiler. AutoHotkey_L bietet viele neue Funktionen und wird im Gegensatz zu Basic ständig weiterentwickelt. Es wird empfohlen, für neue Scripts diese Version zu verwenden. Während der Installation können folgende Versionen ausgewählt werden:

  • Unicode x86 - für neue Scripts empfohlen.
  • Unicode x64 - für höhere Performance auf 64-Bit-Systeme.
  • ANSI x86 - bessere Kompatibilität mit älteren Scripts.

Compiler - aktualisierte Version von Ahk2Exe, für die Kompilierung von x64-Scripts notwendig.

Dokumentation -  kompilierte Hilfe-Datei auf Basis der originalen AutoHotkey.chm:

  • Englisch (English)
  • Chinesisch (中文)
  • Deutsch (siehe Forumsbeitrag für mehr Informationen)

Einige ältere Versionen können in den Archiven gefunden werden.

Der Quellcode für AutoHotkey_L und Ahk2Exe kann mittels Git oder HTTP-Download auf GitHub eingesehen werden.

AutoHotkey Basic

Installationspaket: Das ist die originale Version von AutoHotkey, die weder Unicode, Objekte/Arrays noch andere neue Funktionen unterstützt. Diese Version wurde zuletzt 2009 aktualisiert und wird nicht länger weiterentwickelt. Es existiert auch eine portable Version davon, um die Installation zu vermeiden.

Die deutsche Dokumentation von AutoHotkey_L kann ohne Bedenken für diese Version verwendet werden.

Einige ältere Versionen können in den Archiven gefunden werden.

Der Quellcode für AutoHotkey Basic und das originale Ahk2Exe können als selbst-extrahierende RAR-Dateien runtergeladen werden.

Sonstiges

SciTE4AutoHotkey: Ein frei verfügbarer Editor für AutoHotkey-Scripts. Dieser Editor unterstützt neben DBGp auch Syntaxhervorhebung, Calltips/Parameterinformation, automatische Vervollständigung und weitere nützliche Bearbeitungsfunktionen und Script-Werkzeuge.

SmartGUI Creator 4.0: Ein visuelles Layout-Tool, das automatisch GUI-Scripts erzeugt.

AutoHotkey.dll: Mit dieser Bibliothek kann der AutoHotkey-Interpreter in jede beliebige Anwendung und Sprache integriert werden. Dadurch wird AutoHotkey-Funktionialität bereitgestellt, die möglicherweise in anderen Sprachen schwieriger zu implementieren ist.

Exe2Ahk: Dieses Befehlszeilenprogramm dekompiliert ein kompiliertes Script. Das heißt, dass das originale AHK-Script als Klartextdatei extrahiert wird (falls das Passwort bekannt ist). Allerdings gehen dabei alle ursprünglichen Kommentare (Semikolon oder /**/) verloren. Dieses Programm sollte von der Befehlszeile aus gestartet werden. Zum Beispiel: exe2ahk "Eigenes Script.exe" "Eigenes Passwort".
