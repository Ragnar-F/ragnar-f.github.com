Script-Performance

Um die Performance zu maximieren, vermeidet die Verwendung von SetFormat (außer im schnellen Modus) und fügt folgende Zeilen am Anfang des Scripts hinzu:

#NoEnv
SetBatchLines -1
ListLines Off

Darüber hinaus können die folgenden Befehle auch die Performance je nach Art des Scripts beeinflussen: SendMode, SetKeyDelay, SetMouseDelay, SetWinDelay, SetControlDelay und SetDefaultMouseSpeed.

Interne Performance-Funktionen

Jedes Script wird teilweise kompiliert, während es geladen und dessen Syntax überprüft wird. Neben der Verringerung des Speicherverbrauchs wird dadurch auch die Laufzeit-Performance deutlich verbessert.

Hier die technischen Details des Optimierungsprozesses (Semi-Kompilierung):

  • Eingabe- und Ausgabevariablen (wenn deren Namen keine Referenzen auf anderen Variablen enthalten) und Gruppennamen werden in Speicheradressen aufgelöst.
  • Schleifen, Blöcke, IF- und ELSE-Anweisungen bekommen die Speicheradressen in Bezug auf ihre Sprungpunkte im Script.
  • Das Ziel jeden Hotkey-, Gosub- und Goto-Befehls wird in eine Speicheradresse aufgelöst, solange es keine Variable ist.
  • Jeder Befehlsname in der Sprungtabelle wird durch eine Adresse ersetzt.
  • Jede Zeile wird vorher in eine Liste von Parametern aufgeteilt und jeder Parameter wird vorher in eine Liste von Variablen aufgeteilt.
  • Jeder Ausdruck wird in Tokens übersetzt und von Infix zu Postfix umgewandelt.
  • Jede Referenz auf einer Variable oder Funktion wird in eine Speicheradresse aufgelöst.
  • Direkt vorhandene Integer in Ausdrücken und Mathematik-/Vergleichsbefehle werden mit binären Integer-Werten ersetzt.

Darüber hinaus werden binäre Zahlen während der Script-Ausführung in Variablen zwischengespeichert, um Umwandlungen zu/von Strings zu vermeiden. Siehe SetFormat für weitere Details.
