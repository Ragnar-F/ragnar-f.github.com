Öffnet das aktuelle Script mit dem Standard-Editor.

Edit

Mit dem Befehl Edit wird das aktuelle Script im Editor geöffnet, der mit dem Edit-Verb in der Registrierung verknüpft ist (oder Windows-Editor, wenn kein Verb vorhanden ist). Wenn ein Editor das Script allerdings bereits geöffnet hat (abhängig vom Fenstertitel), dann wird das Fenster aktiviert, anstatt eine neue Instanz des Editors zu öffnen.

Dieser Befehl hat keine Wirkung, wenn er innerhalb eines kompilierten Scripts zum Einsatz kommt.

Außerdem kann die Syntax-Hervorhebung von AutoHotkey für verschiedene Editoren aktiviert werden (siehe unten). Darüber hinaus kann die kontextabhängige Hilfe für AutoHotkey-Befehle in einem beliebigen Editor mit diesem Beispiel aktiviert werden. Die Produktivität des Editors kann mit der Benutzung von einer Auto-Vervollständigung (z. B. ISense) verbessert werden. ISense beobachtet, was der Benutzer schreibt und zeigt Menüs und Parameterlisten an, die ihm das Schreiben teilweise abnehmen und ihn daran erinnern, wie die Reihenfolge der Parameter ist.

Verwandte Befehle

Reload

Beispiel

Edit ; Das Script wird zur Bearbeitung geöffnet. 
; Wenn die Befehlszeilenverwendung des Editors wie folgt aussieht,
; dann kann dieses Script verwendet werden, um ihn als Standard-Editor für ahk-Dateien zu verwenden:
;
;   Editor.exe "Vollständiger Pfad des Scripts.ahk"
;
; Wenn der Benutzer das Script ausführt, dann wird der Benutzer dazu aufgefordert,
; die ausführbare Datei des gewünschten Editors auszuwählen.
;
FileSelectFile Editor, 2,, Editor auswählen, Programme (*.exe)
If ErrorLevel
    ExitApp
RegWrite REG_SZ, HKCR, AutoHotkeyScript\Shell\Edit\Command,, "%Editor%" "`%1"

Editoren mit AutoHotkey-Unterstützung

SciTE4AutoHotkey ist eine angepasste Version des Text-Editors SciTE, der für die Bearbeitung von AutoHotkey-Scripts angepasst ist. Seine Eigenschaften umfassen:

  • Syntax-Hervorhebung
  • Intelligente automatische Einrückung
  • Automatische Vervollständigung
  • Aufruftipps (auch als IntelliSense bekannt)
  • Klappbarer Code
  • Unterstützung für interaktive Fehlerbehebung.
  • Weitere Tools für AutoHotkey-Scripts

SciTE4AutoHotkey kann hier gefunden werden: http://www.autohotkey.com/forum/topic58820.html
