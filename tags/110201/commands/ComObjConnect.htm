ComObjConnect() [AHK_L 53+]

Verbindet die Ereignisquellen eines COM-Objekts mit Funktionen, die einen vorgegeben Präfix enthalten.

ComObjConnect(ComObject [, Präfix])

Parameter

ComObject Ein Objekt, das Ereignisse auslöst.
Präfix

Ein Präfix für den Ereignisnamen, um festzustellen, welche Funktion aufgerufen werden soll, wenn ein Ereignis auftritt.

Wenn nicht vorhanden, dann wird das Objekt "unterbrochen", d. h. das Script empfängt nicht länger Meldungen von ihren Ereignissen.

[v1.1.01+]: Dieser Parameter kann ein definiertes Objekt vom Script sein. Sobald ein Ereignis ausgelöst wird, wird die entsprechende Methode aufgerufen. Der erste Parameter, der für gewöhnlich der versteckte this-Parameter ist, bezieht sich auf das script-definierte Objekt, nicht auf das COM-Objekt. Um alle Ereignisse abzufangen, ohne jedesmal eine Methode zu definieren, definiert eine __Call-Meta-Funktion.

Verwendung

Um ComObjConnect effektiv zu nutzen, müssen zuerst Funktionen in das Script geschrieben werden, damit alle interessanten Ereignisse behandelt werden können. Solche Funktionen oder "Event-Handler" haben die folgende Struktur:

PräfixEreignisname([Parameter..., ComObject])
{
    ... event-handling code ...
    Return Rückgabewert
}

Für Präfix kann ein beliebiger Präfix angegeben werden, während der Ereignisname der Name des Ereignisses ist, das von der Funktion behandelt werden soll.

Die Parameter sind die Parameter, die das Ereignis hat. Wenn das Ereignis keine Parameter hat, dann sollten die Parameter ganz weggelassen werden. ComObject ist optional und kann nur verwendet werden, wenn die korrekte Anzahl an Parameter definiert ist. Es enthält eine Referenz auf das originale Wrapper-Objekt, das an ComObjConnect übergeben wurde. "ComObject" sollte mit einem Namen ersetzt werden, der sinnvoller für den Kontext des Scripts ist.

Beachtet, dass die Event-Handler möglicherweise Rückgabewerte haben. Um einen COM-spezifischen Werttyp zurückzugeben, benutzt ComObjParameter, das auch als ComObjRetVal(Typ, Wert) oder einfach ComObj(Typ, Wert) geschrieben werden kann.

Um das Event-Handling zu aktivieren, verwendet ComObjConnect(BeliebigesObjekt, "Präfix").

Um das Objekt zu trennen (Event-Handling anhalten), verwendet ComObjConnect(BeliebigesObjekt).

Wenn die Anzahl der Parameter unbekannt ist, dann kann eine variadische Funktion verwendet werden.

Verwandte Befehle

ComObjCreate, ComObjGet, ComObjActive, ComObjConnect, ComObjError, WScript.ConnectObject (MSDN)

Beispiele

ie := ComObjCreate("InternetExplorer.Application")

; Verbindet Ereignisse zu den entsprechenden Script-Funktionen mit dem Präfix "IE_".
ComObjConnect(ie, "IE_")

ie.Visible := true  ; Funktioniert bekanntlich nicht mit IE7.
ie.Navigate("http://www.autohotkey.net/~Lexikos/AutoHotkey_L/")
#Persistent

IE_DocumentComplete(ieEventParam, url, ieFinalParam) {
    global ie
    If (ie != ieEventParam)
        s .= "Erster Parameter ist ein neues Wrapper-Objekt.`n"
    If (ie == ieFinalParam)
        s .= "Letzter Parameter ist das originale Wrapper-Objekt.`n"
    If ((disp1:=ComObjUnwrap(ieEventParam)) == (disp2:=ComObjUnwrap(ieFinalParam)))
        s .= "Beide Wrapper-Objekte beziehen sich auf die gleiche IDispatch-Instanz.`n"
    ObjRelease(disp1), ObjRelease(disp2)
    MsgBox % s . "Das Laden von " ie.Document.title " @ " url " wurde abgeschlossen."
    ie.Quit()
    ExitApp
}
