StatusBarGetText

Ermittelt den Text einer Standard-Statusleiste.

StatusBarGetText, Ausgabevariable [, Bereichsnummer, Fenstertitel, Fenstertext, Titelausnahme, Textausnahme]

Parameter

Ausgabevariable Der Name der Variable, die den ermittelten Text enthält.
Bereichsnummer Welche Bereichsnummer der Leiste abgerufen wird (kann auch ein Ausdruck sein kann). Standard ist 1, für gewöhnlich der Bereich mit dem interessantesten Text.
Fenstertitel Der Titel oder Teiltitel des Fensters (das Übereinstimmungsverhalten wird mit SetTitleMatchMode bestimmt). Wenn dieser und die anderen 3 Fensterparameter weggelassen werden, wird das zuletzt gefundene Fenster verwendet. Sobald dieser den Buchstaben A enthält und die anderen 3 Fensterparameter weggelassen werden, wird das aktive Fenster verwendet. Bei Verwendung einer Fensterklasse muss ahk_class ExakterKlassenname angegeben werden (wird von Windows Spy angezeigt). Bei Verwendung einer Prozess-Kennung (PID) muss ahk_pid %VariableMitPID% angegeben werden. Bei Verwendung einer Fenstergruppe muss ahk_group Gruppenname angegeben werden. Bei Verwendung einer eindeutigen ID-Nummer eines Fensters muss ahk_id %VariableMitID% angegeben werden. Die Suche kann durch Angabe mehrerer Kriterien eingeschränkt werden. Zum Beispiel: Eigene Datei.txt ahk_class Notepad
Fenstertext In diesem Parameter kann ein Teilstring angegeben werden, der in einem Textelement des Zielfensters vorkommt (werden mit Windows Spy angezeigt). Versteckte Textelemente werden auch erkannt, sofern DetectHiddenText aktiv ist.
Titelausnahme Ignoriert Fenster, deren Titel diesen Wert enthalten.
Textausnahme Ignoriert Fenster, deren Textelemente diesen Wert enthalten.

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt. Bei Problemen wird auch die Ausgabevariable leer gemacht.

Bemerkungen

Dieser Befehl versucht, die erste Standard-Statusleiste eines Fensters zu lesen (allgemeines Steuerelement von Microsoft: msctls_statusbar32).  Einige Programme verwenden ihre eigenen Statusleisten oder spezielle Versionen des allgemeinen Steuerelements von Microsoft, wodurch der Text nicht abgerufen werden kann.

Anstatt diesen Befehl in einer Schleife zu verwenden, ist die Verwendung von StatusBarWait meist effizienter, um einen Overhead durch wiederholende Aufrufe von StatusBarGetText zu verhindern.

Fenstertitel und -texte sind von der Groß- und Kleinschreibung abhängig. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.

Verwandte Befehle

StatusBarWait, WinGetTitle, WinGetText, ControlGetText

Beispiel

StatusBarGetText, AbgerufenerText, 1, Suchergebnisse
IfInString, AbgerufenerText, Gefunden, MsgBox, Suchergebnisse wurden gefunden.
