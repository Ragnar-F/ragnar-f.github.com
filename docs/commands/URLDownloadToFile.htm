UrlDownloadToFile

Lädt eine Datei aus dem Internet herunter.

UrlDownloadToFile, URL, Dateiname

Parameter

URL URL der Datei, die heruntergeladen werden soll. Mit http://someorg.org beispielsweise wird die Willkommensseite der Organisation abgerufen.
Dateiname

Inhalt in eine Datei herunterladen: Der Name der lokal erstellten Datei, die sich im A_WorkingDir-Verzeichnis befindet, solange kein absoluter Pfad angegeben ist. Jede vorhandene Datei wird durch die neue Datei überschrieben.

Inhalt in eine Variable herunterladen: Es gibt eine aufrufbare Funktion, die in das Script eingefügt werden kann: www.autohotkey.com/forum/topic10466.html.

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

Bemerkungen

Der Download ist möglicherweise erfolgreich, selbst wenn die entfernt liegende Datei nicht existiert. Denn die meisten Webserver senden eine Fehlerseite anstelle der fehlenden Datei. Stattdessen wird daher einfach die Fehlerseite in Dateiname gespeichert.

Internet Explorer 3 oder höher muss installiert sein, damit diese Funktion korrekt arbeitet. Wenn Firewalls oder mehrere Netzwerkkarten vorhanden sind, dann schlägt die Funktion möglicherweise fehl. Außerdem können einige Webseiten solche Downloads blockieren.

Caching:

  • Seit v1.0.44.07 wird die URL direkt vom entfernt liegenden Server abgerufen (also nie vom Cache des Internet Explorers). Um das Caching zuzulassen, setzt *0 vor der URL, gefolgt von einem Leerzeichen; zum Beispiel: *0 http://someorg.org. Die Null nach dem Sternchen kann mit einer gültigen dwFlags-Nummer ersetzt werden; für weitere Details, sucht auf www.microsoft.com nach InternetOpenUrl.
  • In allen vorherigen Versionen wird die Datei sobald wie möglich direkt vom Cache abgerufen. Um das zu verhindern, verwendet einen Abfragestring am Ende der URL. Zum Beispiel: http://www.someorg.org/doc.html?fakeParam=42. Hinweis: Wenn die Datei mehrmals heruntergeladen wird, dann sollte der Abfragestring variieren.

Proxies: UrlDownloadToFile wird einen Proxy-Server verwenden, um auf das Internet zuzugreifen, wenn dieser Proxy in den Einstellungen des Internet Explorers konfiguriert wurde.

FTP und Gopher: Ab v1.0.48.04 werden FTP- und Gopher-URLs unterstützt. Zum Beispiel:

UrlDownloadToFile, ftp://example.com/home/Eigene Datei.zip, C:\Eigener Ordner\Eigene Datei.zip  ; Anonym anmelden.
UrlDownloadToFile, ftp://user:pass@example.com:21/home/Eigene Datei.zip, C:\Eigener Ordner\Eigene Datei.zip  ; Bestimmten Benutzer anmelden.
UrlDownloadToFile, ftp://benutzer:passwort@beispiel.com/Eigenes Verzeichnis, C:\Verzeichnisliste.html  ; Ermittelt eine Verzeichnisauflistung im HTML-Format.
