ToolTip

Erstellt ein Fenster irgendwo auf dem Bildschirm, das immer im Vordergrund ist.

ToolTip [, Text, X, Y, WelcherToolTip]

Parameter

Text

Ist dieser Parameter leer oder nicht vorhanden, wird der vorhandene Tooltip versteckt (falls vorhanden). Ansonsten enthält dieser Parameter den Anzeigetext im Tooltip. Um einen mehrzeiligen ToolTip zu erstellen, verwendet das LF-Zeichen (`n) zwischen jeder Zeile, zum Beispiel Zeile1`nZeile2.

Wenn Text lang ist, dann kann er in mehreren kurzen Zeilen aufgeteilt werden (Fortsetzungsbereich), die die Lesbarkeit und Wartbarkeit verbessern könnten.
X, Y Die Koordinaten X und Y des Tooltips beziehen sich auf das aktive Fenster (verwendet "CoordMode, ToolTip", um auf Bildschirmkoordinaten umzuschalten). Wenn die Koordinaten weggelassen werden, dann wird der Tooltip beim Mauszeiger angezeigt. X und Y können Ausdrücke sein.
WelcherToolTip

Lasst diesen Parameter weg, wenn mehrere Tooltips gleichzeitig nicht erwünscht sind. Ansonsten eine Nummer zwischen 1 und 20, um anzuzeigen, welches Tooltip-Fenster bearbeitet wird. Wenn nicht angegeben, dann standardmäßig die Nummer 1 (das erste Fenster). Dieser Parameter kann ein Ausdruck sein.

Bemerkungen

Wenn sich die X und Y-Koordinaten des Tooltips außerhalb des Bildschirms befinden, dann wird er neu positioniert, um vollständig sichtbar zubleiben.

Der Tooltip wird solange angezeigt, bis Folgendes auftritt:

  • Das Script wird beendet.
  • Der ToolTip-Befehl wird erneut ausgeführt, aber mit einem leeren Text.
  • Der Benutzer klickt auf den Tooltip (dieses Verhalten variiert je nach Betriebssystem).

Ein GUI-Fenster kann mithilfe von Gui +OwnDialogs einen Tooltip übergeordnet werden. Dieser Tooltip wird automatisch zerstört, wenn das dazugehörige GUI-Fenster zerstört wird.

Verwandte Befehle

CoordMode, TrayTip, GUI, Progress, SplashTextOn, MsgBox, InputBox, FileSelectFile, FileSelectFolder

Beispiel

ToolTip, Mehrzeiliger`nTooltip, 100, 150
