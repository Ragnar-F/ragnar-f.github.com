Input

Wartet auf die Tastatureingabe des Benutzers.

Input [, AusgabeVar, Optionen, Endtasten, Vergleichsliste]

Parameter

AusgabeVar

Name der Variable, in der die Benutzereingabe gespeichert werden soll (standardmäßig werden künstliche Eingaben auch erfasst).

Sobald dieser und die nachfolgenden Parameter weggelassen werden, wird jeder aktive Input-Befehl in anderen Threads beendet und ErrorLevel auf das Wort NewInput gesetzt. Der ErrorLevel des aktuellen Befehls wird hingegen auf 0 gesetzt, wenn ein vorheriger Input-Befehl beendet wurde, oder auf 1, wenn kein vorheriger Input-Befehl vorhanden war.

AusgabeVar speichert an sich keine Tastenanschläge. Stattdessen werden Zeichen gespeichert, die gemäß der Tastaturbelegung/Sprache des aktiven Fensters durch Tastatureingaben erzeugt wurden. Folglich werden Tastenschläge, die keine Zeichen erzeugen (z. B. PageUp und Escape), nicht gespeichert (obwohl sie mit dem unten genannten Parameter Endtasten erkannt werden können).

Leerraumzeichen wie TAB (`t) werden literal gespeichert. ENTER als Zeilenvorschub (`n).

Optionen

Ein String mit beliebig vielen Buchstaben (in beliebiger Reihenfolge, optional mit Leerzeichen getrennt):

B: Rücktaste (Backspace) wird ignoriert. Normalerweise wird durch Drücken der Rücktaste bei einem Input-Befehl das zuletzt gedrückte Zeichen vom String entfernt. Hinweis: Falls bei einem sichtbaren Eingabetext (wie in einem Editor) Pfeiltasten oder ähnliches zur Navigation verwendet werden, wird durch die Rücktaste weiterhin das letzte Zeichen entfernt, anstatt das Zeichen hinter der Einfügemarke.

C: Zeichengenauigkeit. Normalerweise unterscheidet Vergleichsliste nicht zwischen Groß- und Kleinschreibung (In den Versionen vor 1.0.43.03 wurden nur die Groß- und Kleinbuchstaben von A bis Z erkannt, aber nicht Buchstaben wie ü/Ü).

I: Ignoriert Eingaben, die vom AutoHotkey-Script erzeugt werden, wie z. B. der SendEvent-Befehl. SendInput und SendPlay werden allerdings immer ignoriert, unabhängig dieser Einstellung.

L: Länge begrenzen (z. B. L5). Die maximal erlaubte Eingabelänge. Sobald der Text diese Länge erreicht, wird Input beendet und ErrorLevel auf das Wort Max gesetzt. Stimmt der Text mit einem Element aus der Vergleichsliste überein, wird ErrorLevel auf das Wort Match gesetzt. Durch Weglassen der Zahl wird die Länge automatisch auf 16383 Zeichen begrenzt (der höchste zulässige Wert).

M: Tastenkombinationen wie STRG+A bis STRG+Z werden erkannt und in reale ASCII-Zeichen umgewandelt, falls vorhanden. Das folgende Beispiel zeigt, wie STRG+C erkannt werden kann:

Transform, StrgC, Chr, 3 ; Speichert das Zeichen für STRG+C in die Variable StrgC.
Input, AusgabeVar, L1 M
if AusgabeVar = %StrgC%
    MsgBox, Sie haben STRG+C gedrückt.
ExitApp

Hinweis: STRG+A bis STRG+Z sind die Zeichen von Chr(1) bis Chr(26). Zudem kann die M-Option dazu führen, dass sich einige Tastenkürzel wie STRG+NACH-LINKS bei einem Input-Befehl falsch verhalten.

T: Zeitlimit (z. B. T3). Die Wartezeit in Sekunden, bevor der Input-Befehl beendet und ErrorLevel auf das Wort Timeout gesetzt wird. Sobald die Zeit abgelaufen ist, enthält AusgabeVar den bis dahin eingegebenen Text des Benutzers. Dieser Wert kann eine Fließkommazahl wie 2.5 sein.

V: Sichtbar. Normalerweise wird die Benutzereingabe blockiert (kann nicht vom Betriebssystem erkannt werden). Mit dieser Option werden die Benutzereingaben direkt an das aktive Fenster gesendet.

*: Platzhalter (überall finden). Normalerweise muss die Benutzereingabe exakt mit einem Element aus der Vergleichsliste übereinstimmen. Mit dieser Option kann eine Übereinstimmung öfters gefunden werden, da die gesamte Länge des Eingabetextes durchsucht wird.

Endtasten

Eine Liste mit beliebig vielen Tasten, die den Input-Befehl beenden, sobald eine von denen gedrückt wird (die Endtaste selbst wird nicht in AusgabeVar geschrieben). Falls ein Input-Befehl auf diese Weise beendet wird, enthält ErrorLevel das Wort EndKey, gefolgt von einem Doppelpunkt und den Namen der Endtaste. Beispiele: EndKey:., EndKey:Escape.

Die Liste im Parameter Endtaste verwendet ein ähnliches Format wie das vom Send-Befehl. Zum Beispiel würde {Enter}.{Esc} bewirken, dass der Input-Befehl durch Drücken der ENTER-, PUNKT- oder ESC-Taste beendet werden kann. Damit die geschweiften Klammern selbst als Endtasten verwendet werden können, muss {{} und/oder {}} angegeben werden.

Um die STRG-, ALT- oder UMSCHALT-Taste als Endtaste zu verwenden, muss anstelle der neutralen die linke und/oder rechte Version der Taste angegeben werden. Zum Beispiel muss {LControl}{RControl} anstelle von {Control} angegeben werden.

Obwohl Tastenkombinationen wie STRG+C (^c) nicht unterstützt werden, können bestimmte Tasten, bei den die UMSCHALT-Taste gedrückt gehalten werden muss (nämlich die Satzzeichen !"§$%&/=?), seit Version 1.0.14 verwendet werden.

Alternativ kann ein eindeutiger virtueller Tastencode wie {vkFF} angegeben werden. Das ist vorallem dann nützlich, wenn die Taste weder einen Namen hat, noch beim Drücken ein sichtbares Zeichen erzeugt. Der virtuelle Tastencode kann ermittelt werden, wenn die Schritte im unteren Abschnitt der Tastenliste befolgt werden.

Vergleichsliste

Eine kommagetrennte Liste mit Schlüsselwörtern, die den Input-Befehl beenden werden (dabei wird ErrorLevel auf das Wort Match gesetzt). Die Benutzereingabe muss exakt mit einem Element aus der Liste übereinstimmen (sofern die *-Option nicht vorhanden ist). Alle Leer- und Tabulatorzeichen an den Trennkommas sind von Bedeutung - das heißt, sie gehören zum übereinstimmenden String. Wenn Vergleichsliste beispielsweise "ABC , XYZ" enthält, dann muss Var entweder ABC mit einem Leerzeichen danach oder XYZ mit einem Leerzeichen davor enthalten, damit eine Übereinstimmung erfolgen kann.

Zwei aufeinanderfolgende Kommas stellen ein literales Komma dar. Im folgenden Beispiel wird ein literales Komma nach dem Wort string1 erzeugt: "string1,,,string2". Ebenso enthält die folgende Liste nur ein einzelnes Element, in dem sich ein literales Komma befindet: "einzelnes,,Element".

Da die Elemente in Vergleichsliste nicht als einzelne Parameter behandelt werden, kann sich die komplette Liste in einer Variable befinden. Tatsächlich muss sich alles oder ein Teil davon in der Variable befinden, wenn eine Länge von 16383 Zeichen überschritten wird (die maximale Länge einer Script-Zeile). Zum Beispiel könnte Vergleichsliste aus %Liste1%,%Liste2%,%Liste3% bestehen, von der jede Unterliste eine lange Liste mit Elementen enthält.

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, eine Ausnahme auszulösen, wenn er ohne Parameter aufgerufen wird, oder wenn kein Input in Bearbeitung ist. Für mehr Informationen, siehe Laufzeitfehler.

1 oder 0

Sobald der Befehl ohne Parameter verwendet wird, wird ErrorLevel auf 0 gesetzt, wenn er erfolgreich ein vorheriges Input beendet, oder auf 1, falls kein Input aktiv ist.
NewInput Die Benutzereingabe wurde von einem anderen Thread unterbrochen, der den Input-Befehl benutzte.
Max Die Benutzereingabe hat die maximal erlaubte Länge erreicht und stimmt mit keinem Elementen aus Vergleichsliste überein.
Zeitlimit Die Benutzereingabe hat die Zeit überschritten.
Match Die Benutzereingabe stimmt mit einem Element aus der Vergleichsliste überein.
EndKey:name Eine Endtaste wurde gedrückt, um den Input-Befehl zu beenden. In diesem Fall beinhaltet ErrorLevel das Wort EndKey, gefolgt von einem Doppelpunkt und den Namen der Taste ohne Klammern, z. B. "EndKey:Enter", "EndKey:Escape", etc.

Bemerkungen

Wenn dieser Befehl verwendet wird, während ein Input-Befehl in einem anderen Thread bereits aktiv ist, wird dieser beendet und ErrorLevel auf das Wort NewInput gesetzt. Danach wird der neue Input-Befehl eingeleitet (wenn Parameter angegeben sind).

Wenn ein Input-Befehl aktiv ist, können weiterhin neue Threads wie benutzerdefinierte Menüelemente oder zeitgesteuerte Subroutinen erstellt werden. Ebenso sind Tastatur-Hotkeys auch dann aktiv, wenn Input sichtbar ist. Falls Input nicht sichtbar ist, können nur Hook-Hotkeys ausgelöst werden.

Sobald ein Script diesen Befehl erstmalig verwendet, wird der Tastatur-Hook installiert (wenn es nicht bereits geschehen ist). Darüber hinaus wird das Script persistent - das heißt, dass ExitApp zum Beenden des Scripts verwendet werden sollte. Der Tastatur-Hook bleibt bis zum nächsten Suspend- oder Hotkey-Befehl bestehen - wird der Hook bei diesem Befehl nicht mehr benötigt, wird er wieder entfernt.

Falls mehrere Sprachen oder Tastaturbelegungen vorhanden sind, greift der Input-Befehl auf die Tastaturbelegung des aktiven Fensters zurück, anstatt auf die des Scripts (egal ob die Eingabe sichtbar ist oder nicht). In den Versionen vor 1.0.44.03 wurde die Belegung des Scripts verwendet.

Obwohl Hotstrings im Gegensatz zu einem Input-Befehl nicht so flexibel sind, sind sie dennoch einfacher zu handhaben.

; Wartet darauf, dass der Benutzer eine Taste drückt.  Tasten, die keine sichtbaren Zeichen
; wie Modifikatoren, Funktionstasten und Pfeiltasten erzeugen, sind als Endtasten aufgelistet,
; damit auch diese erkannt werden.
Input, Einzeltaste, L1, {LControl}{RControl}{LAlt}{RAlt}{LShift}{RShift}{LWin}{RWin}{AppsKey}{F1}{F2}{F3}{F4}{F5}{F6}{F7}{F8}{F9}{F10}{F11}{F12}{Left}{Right}{Up}{Down}{Home}{End}{PgUp}{PgDn}{Del}{Ins}{BS}{Capslock}{Numlock}{PrintScreen}{Pause}

 

; Ein Hotkey-Beispiel.  Da der Hotkey den Tilde-Präfix (~) hat,
; wird seine Tastatureingabe selbst an das aktive Fenster übergeben.
; Schreibt man daher [btw (oder einen anderen
; Begriff) in einem Editor, kann das Script eine beliebige Aktion
; durchführen (z. B. das Ersetzen des Eingabetextes):

~[::
Input, Benutzereingabe, V T5 L4 C, {enter}.{esc}{tab}, btw,otoh,fl,ahk,ca
if (ErrorLevel = "Max")
{
    MsgBox, Ihre Eingabe "%Benutzereingabe%" hat die maximale Textlänge erreicht.
    return
}
if (ErrorLevel = "Timeout")
{
    MsgBox, Sie haben "%Benutzereingabe%" innerhalb der verfügbaren Zeit eingegeben.
    return
}
if (ErrorLevel = "NewInput")
    return
If InStr(ErrorLevel, "EndKey:")
{
    MsgBox, Sie haben "%Benutzereingabe%" eingegeben und Eingabe mit %ErrorLevel% beendet.
    return
}
; Ansonsten wurde eine Übereinstimmung gefunden.
if (Benutzereingabe = "btw")
    Send, {backspace 4}by the way
else if (Benutzereingabe = "otoh")
    Send, {backspace 5}on the other hand
else if (Benutzereingabe = "fl")
    Send, {backspace 3}Florida
else if (Benutzereingabe = "ca")
    Send, {backspace 3}California
else if (Benutzereingabe = "ahk")
    Run, http://ahkscript.org
return
