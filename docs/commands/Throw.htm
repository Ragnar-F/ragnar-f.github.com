Throw [v1.1.04+]

Signalisiert das Vorkommen eines Fehlers. Dieses Signal kann mit einer Try-Catch-Anweisung abgefangen werden.

Throw [, Ausdruck]

Parameter

Ausdruck

Der zu speichernde Wert in AusgabeVar von Catch.

Da dieser Parameter ein Ausdruck ist, sind alle folgenden Beispiele gültig:

throw 3
throw "direkter String"
throw EigeneVar
throw i + 1
throw { what: "Benutzerdefinierter Fehler", file: A_LineFile, line: A_LineNumber } ; Übergibt ein Objekt

Dieser Parameter ist immer ein Ausdruck, daher sollten Variablenverweise nicht von Prozentzeichen umschlossen werden, außer, um eine Doppeldereferenzierung durchzuführen.

[v1.1.05+]: Ist dieser Parameter nicht vorhanden, wird ein Ausnahme-Objekt mit einer Standardnachricht ausgelöst.

Exception(Meldung [, What, Extra])

Erstellt ein Ausnahme-Objekt.

Sobald What weggelassen wird, gilt standardmäßig der Name der aktuellen Funktion oder Subroutine. Ansonsten kann ein String oder ein negativer Offset vom Anfang der Aufrufliste angegeben werden. Zum Beispiel bewirkt ein Wert von -1, dass Exception.What auf die aktuelle Funktion oder Subroutine gesetzt wird und dass Exception.Line auf die Zeile gesetzt wird, die diese aufgerufen hat.

try
    SchlechtProgrammierteFunktion()
catch e
    MsgBox % "Fehler in " e.What ", das auf Zeile " e.Line " aufgerufen wurde.

SchlechtProgrammierteFunktion() {
    throw Exception("Fehlschlag", -1)
}

Verwandte Befehle

Try, Catch, Finally

Beispiele

Siehe Try.
