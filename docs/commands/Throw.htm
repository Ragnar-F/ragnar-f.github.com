Throw [v1.1.04+]

Signalisiert das Vorkommen eines Fehlers. Dieses Signal kann mit einer Try-Catch-Anweisung abgefangen werden.

Throw [, Ausdruck]

Parameter

Ausdruck

Ein Wert, der in die Ausgabevariable von Catch gespeichert wird.

Da dieser Parameter ein Ausdruck ist, sind alle folgenden Beispiele gültig:

throw 3
throw "direkter String"
throw EigeneVar
throw i + 1
throw { what: "Benutzerdefinierter Fehler", file: A_LineFile, line: A_LineNumber } ; Übergibt ein Objekt

Dieser Parameter ist immer ein Ausdruck, daher sollten Variablenverweise nicht von Prozentzeichen umschlossen werden, außer für Doppeldereferenzen.

[v1.1.05+]: Wenn weggelassen, dann wird ein Ausnahmeobjekt mit einer Standardnachricht ausgelöst.

Exception(Meldung [, Was, Extra])

Erstellt ein Ausnahme-Objekt.

Wenn Was weggelassen wird, dann wird standardmäßig der Name der aktuellen Funktion oder Subroutine verwendet. Ansonsten kann ein String oder der negative Offset vom Anfang der Aufrufliste angegeben werden. Zum Beispiel wird mit einem Wert von -1 für Exception.What die aktuelle Funktion oder Subroutine verwendet und für Exception.Line die Zeile, die sie ausgeführt hat.

try
    SchlechtProgrammierteFunktion()
catch e
    MsgBox % "Fehler in " e.What ", die bei Zeile " e.Line " aufgerufen wurde.

SchlechtProgrammierteFunktion() {
    throw Exception("Fehlschlag", -1)
}

Verwandte Befehle

Try, Catch

Beispiele

Siehe Try.
