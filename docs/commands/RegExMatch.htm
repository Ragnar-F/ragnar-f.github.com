RegExMatch() [v1.0.45+]

Überprüft, ob ein Muster (regulärer Ausdruck) im String vorkommt.

Fundposition := RegExMatch(Heuhaufen, NadelRegEx [, AusgabeVar = "", Startposition = 1])

Parameter

Fundposition

RegExMatch() gibt die Position von NadelRegEx zurück, das ganz links im Heuhaufen-String vorkommt. Position 1 ist das erste Zeichen. Eine 0 wird zurückgegeben, wenn der Ausdruck nicht gefunden wird. Falls ein Fehler auftritt (z. B. ein Syntaxfehler innerhalb von NadelRegEx), wird ein leerer String zurückgegeben und ErrorLevel auf einen der unteren Werte gesetzt, anstatt auf 0.

Heuhaufen

Der String, dessen Inhalt durchsucht wird.

NadelRegEx

Das zu suchende Muster, welches ein Perl-kompatibler regulärer Ausdruck (PCRE) ist. Die Optionen des Ausdrucks (falls vorhanden) müssen am Anfang des Strings stehen, gefolgt von einer schließenden Klammer. Zum Beispiel wird mit dem Ausdruck "i)abc.*123" die Unterscheidung von Groß- und Kleinschreibung deaktiviert und "abc" gesucht, gefolgt von null oder mehr beliebige Zeichen, gefolgt von "123". Wenn keine Optionen vorhanden sind, ist das ")" optional; zum Beispiel ist ")abc" das gleiche wie "abc".

AusgabeVar

Modus 1 (Standard): Name von AusgabeVar ohne Anführungszeichen, in der eine Zeichenfolge von Heuhaufen gespeichert werden soll, die mit dem angegeben Ausdruck übereinstimmt. Falls der Ausdruck nicht gefunden wird (also wenn die Funktion 0 zurückgibt), wird diese Variable sowie alle dazugehörigen Array-Elemente leer gemacht.

Sind Teilausdrücke in NadelRegEx vorhanden, werden ihre Übereinstimmungen in ein Pseudo-Array gespeichert, dessen Basisname AusgabeVar ist. Wenn die Variable zum Beispiel Match heißt, wird der Teilstring, der mit dem ersten Teilausdruck übereinstimmt, in Match1 gespeichert, der zweite in Match2 und so weiter. Benannte Teilausdrücke sind dabei eine Ausnahme: anstelle der Zahl wird deren Namen verwendet. Zum Beispiel wird der Teilstring, der mit dem benannten Teilausdruck "(?P<Jahr>\d{4})" übereinstimmt, in MatchJahr gespeichert. Wenn ein bestimmter Teilausdruck mit gar nichts übereinstimmt (oder die Funktion 0 zurückgibt), wird die entsprechende Variable leer gemacht.

Um innerhalb einer Funktion ein Pseudo-Array zu erstellen, das nicht lokal sondern global ist, muss der Basisname des Pseudo-Arrays zuvor als globale Variable deklariert werden. Bei global-behandelnde Funktionen gilt das Gegenteil.

Modus 2 (Position und Länge): Wenn der Großbuchstabe P in den Optionen von RegEx vorhanden ist (z. B. "P)abc.*123"), wird die Länge der Übereinstimmung in AusgabeVar gespeichert (oder 0, wenn nichts gefunden wird). Sind Teilausdrücke vorhanden, wird deren Position und Länge in zwei Pseudo-Arrays gespeichert: AusgabeVarPos und AusgabeVarLen. Wenn der Basisname der Variable zum Beispiel Match ist, wird die einsbasierte Position des ersten Teilausdrucks (falls gefunden) in MatchPos1 sowie dessen Länge in MatchLen1 gespeichert (beide sind leer, wenn keine Übereinstimmung gefunden wird oder die Funktion eine 0 zurückgibt). Benannte Teilausdrücke sind dabei eine Ausnahme: anstelle der Zahl wird deren Namen verwendet (z. B. MatchPosJahr und MatchLenJahr).

Modus 3 (Match-Objekt) [v1.1.05+]: Wenn der Großbuchstabe O in den Optionen von RegEx vorhanden ist (z. B. "O)abc.*123"), wird ein Match-Objekt in AusgabeVar gespeichert. Dieses Objekt kann dazu verwendet werden, Position, Länge und Wert der gesamten Übereinstimmung sowie von jedem Teilausdruck abzurufen, falls vorhanden.

Startposition

Ist Startposition nicht vorhanden, wird standardmäßig eine 1 verwendet (der Anfang von Heuhaufen). Ansonsten beginnt die Suche mit einer 2 beim zweiten Zeichen, mit einer 3 beim dritten Zeichen und so weiter. Wenn Startposition die Länge von Heuhaufen überschreitet, beginnt die Suche bei einem leeren String am Ende des Heuhaufens (das natürlich zur Folge hat, dass keine Übereinstimmung gefunden wird).

Sobald Startposition kleiner als 1 ist, wird dieser Parameter als Offset vom Ende des Heuhaufens angesehen. Zum Beispiel beginnt die Suche mit einer 0 beim letzten Zeichen und mit -1 beim vorletzten Zeichen. Wenn Startposition das linke Ende des Heuhaufens überschreitet, wird alles vom Heuhaufen durchsucht.

Unabhängig von Startposition bezieht sich der Rückgabewert immer auf das erste Zeichen des Heuhaufens. Zum Beispiel wäre die Position von "abc" in "123abc789" immer 4.

ErrorLevel

[v1.1.04+] Diese Funktion ist in der Lage, bei Misserfolg eine Ausnahme auszulösen (nicht zu verwechseln mit "keine Übereinstimmung gefunden"). Für mehr Informationen, siehe Laufzeitfehler.

ErrorLevel kann Folgendes enthalten:

  • Eine 0 - das heißt, dass keine Fehler aufgetreten sind.
  • Einen String in folgender Form: Compile error N at offset M: Beschreibung. In diesem String ist N die PCRE-Fehlernummer, M die Position des betroffenen Zeichens innerhalb des regulären Ausdrucks und Beschreibung ein Text, der den Fehler beschreibt.
  • Eine negative Zahl - das heißt, dass ein Fehler beim Ausführen des regulären Ausdrucks aufgetreten ist. Obwohl solche Fehler selten sind, sind die am häufigsten auftretenden Fehler "zu viele mögliche Übereinstimmungen mit leeren Strings" (-22), "Rekursion zu tief" (-21) und "Limit der Übereinstimmung erreicht" (-8). In so einem Fall sollte das Muster restriktiver gemacht werden. Zum Beispiel kann je nach Möglichkeit das Sternchen (*) mit einem Fragezeichen (?), Pluszeichen (+) oder einer Begrenzung wie {0,3} ersetzen werden.

Optionen

Siehe Optionen für Modifikatoren wie "i)abc", die die Unterscheidung von Groß- und Kleinschreibung im Muster "abc" deaktiviert.

Match-Objekt v1.1.05+

Wenn der Großbuchstabe O in den Optionen des Ausdrucks vorhanden ist, wird ein Match-Objekt in AusgabeVar gespeichert. Dieses Objekt hat folgende Eigenschaften:

Match.Pos(N): Gibt die Position der gesamten Übereinstimmung oder eines Teilausdrucks zurück.

Match.Len(N): Gibt die Länge der gesamten Übereinstimmung oder eines Teilausdrucks zurück.

Match.Value(N): Gibt die gesamte Übereinstimmung oder ein Teilausdruck zurück.

Match.Name(N): Gibt den Namen eines benannten Teilausdrucks zurück, falls vorhanden.

Match.Count(): Gibt die Gesamtanzahl an Teilausdrücken zurück.

Match.Mark(): Gibt NAME des zuletzt gefundenen Teilausdrucks (*MARK:NAME) zurück, falls vorhanden.

Match[N]: Wenn N eine 0 oder Nummer/Name eines gültigen Teilausdrucks ist, dann ist diese Schreibweise mit Match.Value(N) vergleichbar. Ansonsten kann N der Name einer der oben genannten Eigenschaften sein. Zum Beispiel sind Match["Pos"] und Match.Pos das gleiche wie Match.Pos(). Gibt es aber einen Teilausdruck mit dem Namen "Pos", sind sie das gleiche wie Match.Value("Pos").

Match.N: Das Gleiche wie oben, nur dass N eine Zahl oder ein Name ohne Anführungszeichen ist.

Bei allen oben genannten Eigenschaften kann N Folgendes sein:

  • 0 für die gesamte Übereinstimmung.
  • Die Nummer eines Teilausdrucks, auch einen mit Namen.
  • Der Name eines Teilausdrucks.

Es können eckige Klammern anstelle von runden Klammern verwendet werden, wenn N angegeben wurde.

Performance

Um einen einfachen Teilstring innerhalb eines größeren Strings zu suchen, ist es besser, InStr() zu benutzen, weil es schneller als RegExMatch() ist.

Zur Verbesserung der Performance werden die 100 aktuellsten regulären Ausdrücke im Speicher zwischengespeichert (in kompilierter Form).

Die S-Option kann mitunter dabei helfen, die Performance eines häufig benutzten regulären Ausdrucks zu verbessern (z. B. in einer Schleife).

Bemerkungen

Es ist möglich, einen Teilausdruck zu benennen, wie z. B. das Wort Jahr im Ausdruck "(?P<Jahr>\d{4})". Solche Namen können maximal 32 Zeichen lang sein. Zudem sind nur alphanumerische Zeichen und Unterstriche erlaubt. Die folgende Einschränkung gilt nicht für den O-Modus (Match-Objekt): Obwohl benannte Teilausdrücke während der RegEx-Operation selbst auch per Nummer zugänglich sind (z. B. ist \1 eine Rückreferenz auf den String, der zurzeit mit dem ersten Teilausdruck übereinstimmt), werden diese nur per Name in das Ausgabe-Pseudo-Array gespeichert (nicht per Nummer). Wenn der erste Teilausdruck zum Beispiel "Jahr" heißt, wird der übereinstimmende Teilstring in AusgabeVarJahr gespeichert, aber AusgabeVar1 bleibt unverändert (sie enthält weiterhin den vorherigen Wert, falls vorhanden). Wenn jedoch ein unbenannter Teilausdruck nach "Jahr" erfolgt, wird dieser in AusgabeVar2 gespeichert, anstatt in AusgabeVar1.

Die meisten Zeichen wie abc123 können literal in einem regulären Ausdruck verwendet werden. Bei den Zeichen \.*?+[{|()^$ muss allerdings ein umgekehrter Schrägstrich davor gesetzt werden, damit auch diese als literal angesehen werden. Zum Beispiel wird \. als literaler Punkt und \\ als literaler umgekehrter Schrägstrich angesehen. Das Escapen kann mit \Q...\E verhindert werden. Zum Beispiel: \QLiteraler Text\E.

Innerhalb eines regulären Ausdrucks können bei bestimmten Sonderzeichen wie Tabulatorzeichen und Zeilenumbrüche sowohl umgekehrte Häkchen (`) als auch umgekehrte Schrägstriche (\) als Escapezeichen verwendet werden. Zum Beispiel ist `t das gleiche wie \t, sofern die x-Option nicht verwendet wird.

Um die Grundlagen von regulären Ausdrücken kennenzulernen (oder das Gedächtnis ein wenig aufzufrischen), siehe RegEx-Kurzübersicht.

Beispiele

Fundposition := RegExMatch("xxxabc123xyz", "abc.*xyz")  ; Gibt 4 zurück - die Position, auf der die Übereinstimmung gefunden wurde.
Fundposition := RegExMatch("abc123123", "123$")  ; Gibt 7 zurück, weil das $ die Übereinstimmung dazu zwingt, am Ende vorzukommen.
Fundposition := RegExMatch("abc123", "i)^ABC")  ; Gibt 1 zurück, da eine Übereinstimmung durch i-Option (keine Unterscheidung zwischen Groß- und Kleinschreibung) gefunden wurde.
Fundposition := RegExMatch("abcXYZ123", "abc(.*)123", SubPat)  ; Gibt 1 zurück und speichert "XYZ" in SubPat1.
Fundposition := RegExMatch("abc123abc456", "abc\d+", "", 2)  ; Gibt 7 anstelle von 1 zurück, wegen der 2 in der Startposition.
