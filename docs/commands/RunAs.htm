RunAs

Bestimmt eine Reihe von Benutzeranmeldeinformationen, die für nachfolgende RunWait- und Run-Befehle gelten sollen. Benötigt Windows 2000/XP oder höher.

RunAs [, Benutzer, Passwort, Domäne]

Parameter

Benutzer Sind dieser und alle anderen Parameter nicht vorhanden, wird das RunAs-Feature deaktiviert, um das Standardverhalten von Run und RunWait wiederherzustellen. Ansonsten kann hier ein Benutzername angegeben werden, der beim Erstellen neuer Prozesse verwendet weren soll.
Passwort Benutzer-Passwort.
Domäne

Benutzer-Domäne. Um einen lokalen Account zu verwenden, muss dieser Parameter leer gelassen werden. Wenn das nicht funktioniert, versuche @DeinComputerName.

Bemerkungen

Nur Windows XP/2000 oder höher werden unterstützt. Dieser Befehl funktioniert nicht bei anderen Betriebssystemen. NT4-Benutzer sollten stattdessen den SU-Befehlssatz vom NT Resource Kit installieren und verwenden.

Dieser Befehl macht nichts anderes als AutoHotkey mitzuteilen, dass alternative Benutzeranmeldeinformationen für nachfolgende Run- und RunWait-Befehle verwendet (oder nicht verwendet) werden sollen.

ErrorLevel wird nicht von diesem Befehl geändert. Wenn Benutzer, Passwort und/oder Domäne ungültig sind, wird Run und RunWait eine Fehlermeldung über die aktuelle Problematik anzeigen (sofern UseErrorLevel nicht aktiv ist).

Es ist nicht möglich, Dokumente, URLs oder Systemverben per Run und RunWait zu starten, während das RunAs-Feature aktiv ist. Das heißt, dass die zu startende Datei eine ausführbare Datei sein muss.

Damit dieser Befehl funktioniert, muss der Dienst "Sekundäre Anmeldung" auf "Manuell" oder "Automatisch" gestellt sein (das Betriebssystem sollte den Dienst bei Bedarf automatisch starten, falls dieser auf "Manuell" gestellt ist).

Verwandte Befehle

Run, RunWait

Beispiel

RunAs, Administrator, MeinPasswort
Run, RegEdit.exe
RunAs  ; Normales Verhalten wiederherstellen.
