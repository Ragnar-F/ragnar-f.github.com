RunAs

Bestimmt mehrere Benutzeranmeldeinformationen für nachfolgende RunWait- und Run-Befehle. Benötigt Windows 2000/XP oder höher.

RunAs [, Benutzer, Passwort, Domäne]

Parameter

Benutzer Wenn dieser und alle anderen Parameter nicht vorhanden sind, wird das RunAs-Feature deaktiviert, um das Standardverhalten von Run und RunWait wiederherzustellen. Ansonsten kann ein Benutzername angegeben werden, der für neu erstellte Prozesse gilt.
Passwort Passwort des Benutzers.
Domäne

Domäne des Benutzers. Ist dieser Parameter leer, wird das lokale Konto verwendet. Funktioniert das nicht, kann @DeinComputername ausprobiert werden.

Bemerkungen

Nur Windows XP/2000 oder höher werden unterstützt. Dieser Befehl hat keine Auswirkung auf andere Betriebssysteme. NT4-Benutzer sollten stattdessen den SU-Befehlssatz vom NT Resource Kit installieren und verwenden.

Dieser Befehl macht nichts anderes, als AutoHotkey mitzuteilen, alternative Benutzeranmeldeinformationen für nachfolgende Run- und RunWait-Befehle zu verwenden (oder nicht zu verwenden).

ErrorLevel wird nicht von diesem Befehl geändert. Wenn Benutzer, Passwort oder Domäne ungültig ist, wird Run und RunWait eine Fehlermeldung anzeigen, die das Problem näher beschreibt (sofern UseErrorLevel nicht aktiv ist).

Während das RunAs-Feature aktiv ist, können Dokumente, URLs oder Systemverben nicht über Run und RunWait ausgeführt werden. Das heißt, dass die betroffene Datei eine ausführbare Datei (EXE, BAT usw.) sein muss.

Damit dieser Befehl funktioniert, muss der Dienst "Sekundäre Anmeldung" auf "Manuell" oder "Automatisch" gestellt sein (das Betriebssystem sollte den Dienst bei Bedarf automatisch starten, falls dieser auf "Manuell" gestellt ist).

Verwandte Befehle

Run, RunWait

Beispiel

RunAs, Administrator, EigenesPasswort
Run, RegEdit.exe
RunAs  ; Normales Verhalten wiederherstellen.
