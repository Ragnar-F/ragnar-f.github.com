RunAs

Bestimmt eine Reihe von Benutzerdaten für nachfolgende RunWait- und Run-Befehle.

RunAs [, Benutzer, Passwort, Domäne]

Parameter

Benutzer

Wenn alle Parameter weggelassen werden, dann wird die RunAs-Funktion deaktiviert, welches das Standardverhalten von Run und RunWait wiederherstellt. Ansonsten enthält dieser Parameter den Benutzernamen, der für neu erstellte Prozesse gilt.

Passwort

Passwort des Benutzers.

Domäne

Domäne des Benutzers. Um das lokale Konto zu verwenden, lasst diesen Parameter leer. Wenn das nicht funktioniert, versucht @DeinComputerName.

Bemerkungen

Dieser Befehl macht nichts anderes, als AutoHotkey zu benachrichtigen, alternative Benutzerdaten für alle nachfolgenden Run- und RunWait-Befehle zu verwenden (oder nicht zu verwenden).

ErrorLevel wird nicht durch diesen Befehl beeinflusst. Wenn die Angabe des Benutzers, des Passworts oder der Domäne ungültig ist, dann zeigt Run und RunWait eine Fehlermeldung an, um das Problem näher zu beschreiben (es sei denn, UseErrorLevel ist aktiv).

Während die RunAs-Funktion aktiv ist, sind Run und RunWait nicht in der Lage, Dokumente, URLs oder Systemverben auszuführen. Das heißt, dass eine ausführbare Datei angegeben werden muss.

Der Dienst "Sekundäre Anmeldung" muss auf manuell oder automatisch gestellt werden, damit dieser Befehl funktioniert (das Betriebssystem sollte den Dienst bei Bedarf automatisch starten, wenn es manuell gestellt ist).

Verwandte Befehle

Run, RunWait

Beispiel

RunAs, Administrator, EigenesPasswort
Run, RegEdit.exe
RunAs  ; Normales Verhalten wiederherstellen.
