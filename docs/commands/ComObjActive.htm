ComObjActive() [AHK_L 53+]

Ermittelt ein laufendes Objekt, das mit OLE registriert wurde.

ComObject := ComObjActive(CLSID)

Erstellt ein Objekt, das einen eingegebenen Wert repräsentiert, um es als Parameter oder Rückgabewert zu übergeben.

ParamObj := ComObjParameter(Variablentyp, Wert [, Flags])

Erstellt ein Objekt, das anstelle des Standardwertes eines optionalen Parameters verwendet werden kann, wenn eine Methode von einem COM-Objekt aufgerufen wird.

ParamObj := ComObjMissing()

Fortgeschritten

Packt einen unbearbeiteten IDispatch-Pointer in oder aus einem verwendbaren Objekt.

ComObject := ComObjEnwrap(DispPtr)
DispPtr := ComObjUnwrap(ComObject)

Parameter

CLSID CLSID oder eine visuell lesbare ProgID des COM-Objekts.
ComObject COM-Objekt, das mit der Objekt-Syntax verwendet werden kann.
Variablentyp Ein Integer-Wert, der den Typ des Wertes kennzeichnet. Siehe ComObjType() für eine Liste von Typen.
Wert Ein Wert, der eingepackt wird. Zurzeit werden nur Integer- oder Pointer-Werte unterstützt.
Flags Flags beeinflussen das Verhalten dieser Funktion und des Wrapper-Objekts (siehe unten).
DispPtr Unbearbeiteter IDispatch-Pointer.

Flags

 0  Standardverhalten. AddRef wird automatisch bei einem IUnknown- und IDispatch-Pointer aufgerufen, daher sollte der Aufrufer ObjRelease verwenden, um die Kopie des Pointers je nach Bedarf freizugeben.
 1  Übernimmt einen IUnknown-, IDispatch- oder SAFEARRAY-Pointer. AddRef wird nicht aufgerufen. Wenn das Wrapper-Objekt einen SAFEARRAY enthält (ohne VT_BYREF), dann wird SafeArrayDestroy automatisch aufgerufen, wenn das Wrapper-Objekt freigegeben wird.

Allgemeine Bemerkungen

ComObjActive ist polymorph, d. h. jeder Funktionsaufruf, der mit "ComObj" beginnt und nicht mit den anderen COM-Funktionen übereinstimmt, ruft auch ComObjActive auf. Zum Beispiel sind ComObj(9, DispPtr) und ComObjActive(DispPtr) das gleiche wie ComObjEnwrap(DispPtr). Dieses Verhalten könnte jedoch in einer zukünftigen Version geändert werden (oder neue Funktionen implementiert werden), daher ist es am besten, nur die hier aufgeführten Formen zu verwenden, wobei zusätzlich gilt, dass "ComObject" anstelle von ComObjEnwrap oder ComObjParameter verwendet werden kann.

Wenn diese Funktion dazu verwendet wird, einen IDispatch- oder eine IUnknown-Interface-Pointer einzupacken oder abzurufen, dann wird standardmäßig die Referenzzählung des COM-Objekts erhöht. Allgemein gilt, dass die originale Kopie des Pointers eine separate Referenz berücksichtigt, die manuell vom Script freigegeben werden muss, wenn sie nicht länger benötigt wird. Wenn das Wrapper-Objekt freigegeben wird, dann wird auch dessen Referenz auf das COM-Objekt freigegeben.

Bekannte Einschränkung: Jedes Mal, wenn ein COM-Objekt eingepackt wird, wird ein neues Wrapper-Objekt erstellt. Vergleiche und Zuweisungen wie obj1 == obj2 und array[obj1] := Wert behandeln die zwei Wrapper-Objekte einzigartig, selbst wenn sie das gleiche COM-Objekt enthalten.

Verwandte Befehle

ComObjCreate, ComObjGet, ComObjConnect, ComObjError, ComObjFlags, ObjAddRef/ObjRelease, ComObjQuery, GetActiveObject (MSDN)

Beispiele

ComObjUnwrap: Siehe ComObjConnect.

; Beispiel: Übergibt VARIANT ByRef an die COM-Funktion.

code =
(
Sub Beispiel(Var)
    MsgBox Var
    Var = "Ausgabewert!"
End Sub
)
sc := ComObjCreate("ScriptControl"), sc.Language := "VBScript", sc.AddCode(code)

var := ComVar()
var[] := "Eingabewert"
sc.Run("Beispiel", var.ref)
MsgBox % var[]

; ComVar: Erstellt ein Objekt, dass zur Übergabe eines ByRef-Wertes verwendet wird.
;   "ComVar[]" ermittelt den Wert.
;   "ComVar[] := Wert" bestimmt den Wert.
;   ComVar.ref ermittelt ein ByRef-Objekt, damit es zur COM-Funktion übergeben werden kann.
ComVar(Type=0xC)
{
    static base := { __Get: "ComVarGet", __Set: "ComVarSet", __Delete: "ComVarDel" }
    ; Erstellt ein Array von 1 VARIANT.  Diese Methode erlaubt dem eingebauten Code,
    ; auf alle Konvertierungen zwischen VARIANT und ahk-interne Typen zu achten.
    arr := ComObjArray(Type, 1)
    ; Sperrt das Array und ermittelt einen Pointer auf VARIANT.
    DllCall("oleaut32\SafeArrayAccessData", "ptr", ComObjValue(arr), "ptr*", arr_data)
    ; Speichert das Array und einen Objekt, die an VARIANT-ByRef übergeben werden können.
    Return { ref: ComObjParameter(0x4000|Type, arr_data), _: arr, base: base }
}
ComVarGet(cv, p*) { ; Wird aufgerufen, wenn das Script auf einem unbekannten Feld zugreift.
    If p.MaxIndex() = "" ; Keine Namen/Parameter, also cv[]
        Return cv._[0]
}
ComVarSet(cv, v, p*) { ; Wird aufgerufen, wenn das Script ein unbekanntes Feld setzt.
    If p.MaxIndex() = "" ; Keine Namen/Parameter, also cv[]:=v
        Return cv._[0] := v
}
ComVarDel(cv) { ; Wird aufgerufen, wenn das Objekt freigegeben wird.
    ; Das muss gemacht werden, damit das interne Array freigegebenen werden kann.
    DllCall("oleaut32\SafeArrayUnaccessData", "ptr", ComObjValue(cv._))
}
