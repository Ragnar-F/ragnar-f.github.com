#MaxThreadsPerHotkey

Bestimmt die maximale Anzahl an gleichzeitig vorhandenen Threads pro Hotkey oder Hotstring.

#MaxThreadsPerHotkey, Wert

Parameter

Wert Die maximale Anzahl an Threads, die von einer Hotkey/Hotstring-Subroutine gestartet werden kann (maximal 20).

Bemerkungen

Diese Einstellung kontrolliert, wie viele "Instanzen" einer bestimmten Hotkey- oder Hotstring-Subroutine gleichzeitig vorhanden sein dürfen. Wenn dem Hotkey ein Maximum von 1 zugewiesen wurde, und wird er nochmals gedrückt, während seine Subroutine läuft, wird dieses Drücken ignoriert. Dies ist nützlich, um zu verhindern, dass ein Hotkey doppelt gedrückt werden kann. Wenn solche Tastendrücke stattdessen zwischengespeichert werden sollen (um beispielsweise die Reaktionsfähigkeit der Autowiederholungsfunktion der Tastatur zu erhöhen), verwende #MaxThreadsBuffer.

Im Gegensatz zu #MaxThreads ist diese Einstellung nicht global. Stattdessen sollte man die Direktive vor dem Hotkey-Label positionieren, das als erstes von ihr beeinflusst werden soll. Das führt dazu, dass alle nachfolgenden Hotkeys diesen Wert verwenden, bis eine andere Instanz von dieser Direktive vorkommt.

Jede Hotkey-Subroutine, auf deren erste Zeile ExitApp, Pause, Edit, Reload, KeyHistory, ListLines, ListVars oder ListHotkeys steht, wird immer ausgeführt, unabhängig von dieser Einstellung.

Die Einstellung von #MaxThreads hat Vorrang, falls sie niedriger ist als diese Einstellung.

Ist diese Direktive im Script nicht angegeben, gilt standardmäßig 1.

Verwandte Befehle

#MaxThreads, #MaxThreadsBuffer, Critical, Threads, Hotkey, #MaxHotkeysPerInterval, #HotkeyInterval, ListHotkeys

Beispiel

#MaxThreadsPerHotkey 3
