StrPut() / StrGet() [AHK_L 46+]

Kopiert einen String in oder aus einem Speicherbereich unter Beachtung des angegebenen Zeichensatzes.

StrPut(String [, Codierung = Keine ] )
StrPut(String, Adresse [, Länge] [, Codierung = Keine ] )
StrGet(Adresse [, Länge] [, Codierung = Keine ] )

Parameter

String Ein beliebiger String. Zahlen werden auch akzeptiert.
Adresse Die Adresse des Speicherbereichs, wo der angegeben String gelesen/geschrieben werden soll.
Länge Die maximale Anzahl der zu lesenden/schreibenden Zeichen, einschließlich der Null-Terminator, falls notwendig. Siehe Rückgabewert.
Codierung Die Quellcodierung bei StrGet oder die Zielcodierung bei StrPut, zum Beispiel: "UTF-8", "UTF-16" oder "CP936". Sind Adresse und Länge nicht angegeben, müssen numerische Kennungen mit "CP" vorangestellt werden. Ein leerer String oder "CP0" kennzeichnet den Standard-ANSI-Zeichensatz des Systems.

Rückgabewert

Bei beiden Funktion wird ein leerer String zurückgegeben, sobald einer der Parameter ungültig ist.

Wenn StrPut erfolgreich ausgeführt wird, gibt sie die Anzahl der geschriebenen Zeichen zurück. Wenn keine Adresse angegeben ist, wird StrPut die erforderliche Puffergröße in Zeichen zurückgeben. Bei Misserfolg wird StrPut eine 0 zurückgeben. Wenn Länge kleiner als die Länge des Quellstrings ist, wird die Funktion fehlschlagen und eine 0 zurückgeben. Wenn Länge gleich die Länge des Quellstrings ist, wird der String nicht null-terminiert; ansonsten beinhaltet die zurückgegebene Anzahl den Null-Terminator.

StrGet wird den angeforderten String zurückgeben, nach Durchführen notwendiger Umwandlungen.

Bemerkungen

Beachte, dass der String-Parameter von StrPut und der Rückgabewert von StrGet immer in der nativen Codierung der aktuell ausführbaren Datei sind, wohingegen Codierung die Codierung des geschrieben oder gelesenen Strings spezifiziert. Ist keine Codierung angegeben, wird der String gemessen oder kopiert, ohne dass eine Umwandlung stattfindet.

Wenn eine Umwandlung zwischen den Zeichensätzen notwendig ist, könnte die erforderliche Puffergröße von der Größe des Quellstrings abweichen.

Scripts, die kompatibel mit AutoHotkey Basic sein müssen, können trotzdem StrPut und StrGet nutzen, wenn die entsprechenden Script-Dateien in eine Funktionsbibliothek installiert werden. Solche Scripts können im AutoHotkey Community Forum gefunden werden.

Verwandte Befehle

Script-Kompatibilität, FileEncoding, VarSetCapacity()

Beispiele

Direkt nach Adresse kann Codierung angegeben werden, dabei darf Codierung nicht numerisch sein:

StrA := StrGet(AdresseA, "cp0")     ; OK
StrA := StrGet(AdresseA, Länge, 0)  ; OK
StrA := StrGet(AdresseA, 0)         ; Fehler

StrPut kann einmal aufgerufen werden, um die erforderliche Puffergröße für einen String in einer bestimmten Codierung zu berechnen, und danach nochmal, um den String zu codieren und ihn in den Puffer zu schreiben. Wenn du öfters Variablen mit StrPut verwendest, solltest du folgende Funktion in deine Bibliothek aufnehmen:

StrPutVar(String, ByRef Var, Codierung)
{
    ; Kapazität gewährleisten.
    VarSetCapacity(Var, StrPut(String, Codierung)
        ; StrPut gibt die Zeichenanzahl zurück, aber VarSetCapacity benötigt Bytes.
        * ((Codierung = "utf-16" || Codierung = "cp1200") ? 2 : 1) )
    ; Kopiert oder wandelt den String um.
    Return StrPut(String, &Var, Codierung)
}
