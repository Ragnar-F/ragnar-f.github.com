SetWorkingDir

Ändert das aktuelle Arbeitsverzeichnis des Scripts.

SetWorkingDir, VerzName

Parameter

VerzName

Name des neuen Arbeitsverzeichnisses, das ein Unterordner im aktuellen A_WorkingDir-Verzeichnis ist, solange kein absoluter Pfad angegeben ist.

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

Bemerkungen

Das Arbeitsverzeichnis des Scripts ist das Standardverzeichnis, das zum Zugreifen auf Dateien und Ordner verwendet wird, falls kein absoluter Pfad angegeben ist. Im folgenden Beispiel wird davon ausgegangen, das sich die Datei Dateiname.txt in %A_WorkingDir% befindet: FileAppend, Eine Textzeile., Dateiname.txt.

Das anfängliche Arbeitsverzeichnis richtet sich danach, wie das Script gestartet wurde. Wurde es beispielsweise über eine Verknüpfung gestartet (z. B. im Startmenü), richtet sich dessen Arbeitsverzeichnis nach dem Feld "Ausführen in", das im Eigenschaftsfenster der Verknüpfung zu finden ist.

Um ein Script dazu zu bringen, bedingungslos seinen eigenen Ordner als Arbeitsverzeichnis zu verwenden, muss auf dessen ersten Zeile folgendes stehen:

SetWorkingDir %A_ScriptDir%

Diese Änderung wirkt sich sofort auf das gesamte Script aus. Alle unterbrochenen, pausierten und neu ausgeführten Threads sind davon betroffen, einschließlich Timer.

Siehe auch

A_WorkingDir, A_ScriptDir, FileSelectFolder

Beispiel

SetWorkingDir %A_ScriptDir%
SetWorkingDir, D:\Mein Ordner\Temp
