SetWorkingDir

Ändert das Arbeitsverzeichnis des Scripts.

SetWorkingDir, Verzeichnisname

Parameter

Verzeichnisname Der Name des neuen Arbeitsverzeichnisses, welches ein Unterordner im aktuellen A_WorkingDir-Verzeichnis ist, solange kein absoluter Pfad angegeben ist.

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

Bemerkungen

Das Arbeitsverzeichnis des Scripts ist das Standardverzeichnis, dass verwendet wird, um auf Dateien und Ordner zuzugreifen, wenn kein absoluter Pfad angegeben wurde. Im folgenden Beispiel wird davon ausgegangen, das sich die Datei Dateiname.txt in %A_WorkingDir% befindet: FileAppend, Eine Textzeile., Dateiname.txt

Das ursprüngliche Arbeitsverzeichnis des Scripts wird davon bestimmt, wie es gestartet wurde. Wenn zum Beispiel ein Script mittels einer Verknüpfung gestartet wird (beispielsweise im Startmenü), dann wird das Arbeitsverzeichnis vom Feld "Ausführen in" in den Eigenschaften der Verknüpfung bestimmt.

Damit ein Script seinen Ordner bedingungslos als Arbeitsverzeichnis verwendet, schreibt Folgendes in die erste Zeile:
SetWorkingDir %A_ScriptDir%

Sobald geändert tritt das neue Arbeitsverzeichnis sofort und global in Kraft. Alle unterbrochene, pausierte und neu ausgeführte Threads sind davon betroffen, einschließlich Timers.

Verwandte Befehle

A_WorkingDir, A_ScriptDir, FileSelectFolder

Beispiel

SetWorkingDir %A_ScriptDir%
SetWorkingDir, D:\Eigener Ordner\Temp
