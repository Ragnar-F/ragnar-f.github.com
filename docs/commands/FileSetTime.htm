FileSetTime

Ändert den Zeitstempel von einer oder mehreren Dateien (auch Ordner). Platzhalter werden unterstützt.

FileSetTime [, YYYYMMDDHH24MISS, Dateimuster, Zeitstempeltyp, OrdnerEinbeziehen?, Rekursiv?]

Parameter

YYYYMMDDHH24MISS

Ist dieser Parameter leer oder nicht vorhanden, gilt standardmäßig die aktuelle Zeit. Ansonsten kann eine Zeit angegeben werden, die für diese Operation verwendet werden soll (siehe Bemerkungen zum Format). Jahre vor 1601 werden nicht unterstützt.

Dieser Parameter ist ein Ausdruck. Das heißt, müssen mehrere Variablen zu einem Zeitstempel verkettet werden, sollte der Punkt-Operator anstatt das Prozentzeichen verwendet werden. Zum Beispiel: FileSetTime, Jahr . Monat . Tag, C:\Eigene Datei.txt.
Dateimuster

Name einer Datei, eines Ordners oder einer musterbasierten Suche wie C:\Temp\*.tmp. Dateimuster befindet sich im A_WorkingDir-Verzeichnis, solange kein absoluter Pfad angegeben ist.

Ist dieser Parameter nicht vorhanden, wird die aktuelle Datei vom innersten Loop (Dateien & Ordner) verwendet.
Zeitstempeltyp Welcher Zeitstempel gesetzt werden soll:
M = Änderungsdatum (Standard, wenn Parameter nicht vorhanden ist)
C = Erstellungsdatum
A = Zugriffsdatum
OrdnerEinbeziehen?

0 (Standard) Ordner werden nicht mit einbezogen (nur Dateien).
1 Alle Dateien und Ordner, die mit der musterbasierten Suche übereinstimmen, werden mit einbezogen.
2 Nur Ordner werden mit einbezogen (keine Dateien).

Hinweis: Falls Dateimuster ein einzelner Ordner anstatt eine musterbasierte Suche ist, wird er immer mit einbezogen, unabhängig von dieser Einstellung.

Dieser Parameter kann ein Ausdruck sein.
Rekursiv?

0 (Standard) Unterordner werden nicht rekursiv behandelt.
1 Unterordner werden rekursiv behandelt, damit die enthaltenen Dateien und Unterordner mit einbezogen werden, wenn sie mit dem Dateimuster übereinstimmen. Alle Unterordner werden rekursiv behandelt, nicht nur solche, die mit dem Dateimuster übereinstimmen. Allerdings werden Dateien und Ordner mit einem kompletten Pfad länger als 259 Zeichen ignoriert, als würde sie nicht existieren. Solche Dateien sind selten, weil normalerweise das Betriebssystem das Erstellen solcher Dateien verbietet.

Dieser Parameter kann ein Ausdruck sein.

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

Bei Misserfolg wird ErrorLevel auf die Anzahl der Dateien gesetzt, die nicht geändert werden konnten, ansonsten auf 0. Wenn der angegebene Zeitstempel ungültig ist oder Dateimuster in einen leeren Wert aufgelöst wird, wird ErrorLevel auf 1 gesetzt.

Falls Dateien gefunden wurden, wird A_LastError auf 0 gesetzt, oder auf den aktuellsten Rückgabewert der Systemfunktion GetLastError(). Ansonsten enthält A_LastError einen Fehlercode, der darauf hinweist, warum keine Dateien gefunden wurden.

Bemerkungen

In Windows 95/98/ME wird das Ändern des Zeitstempels von Ordnern nicht unterstützt. Solche Versuche werden ignoriert.

Das Zugriffsdatum einer Datei ist auf FAT16- & FAT32-Datenträger ungenauer als auf NTFS-Datenträger.

YYYYMMDDHH24MISS besteht aus folgenden Elementen:

YYYY 4-stelliges Jahr
MM 2-stelliger Monat (01-12)
DD 2-stelliger Tag des Monats (01-31)
HH24 2-stellige Stunde im 24-Stunden-Format (00-23). Zum Beispiel: 09 ist 9am und 21 ist 9pm.
MI 2-stellige Minute (00-59)
SS 2-stellige Sekunde (00-59)

Falls der YYYYMMDDHH24MISS-String nur teilweise angegeben ist (z. B. 200403), werden die restlichen Elemente, welche weggelassen worden sind, mit folgenden Standardwerten ersetzt: 01 für MM, 01 für DD, 00 für HH24, 00 für MI und 00 für SS.

Die interne Variable A_Now enthält die aktuelle Ortszeit im YYYYMMDDHH24MISS-Format. Ebenso enthält A_NowUTC die aktuelle koordinierte Weltzeit.

Hinweis: Datum-Zeit-Werte können per EnvAdd und EnvSub verglichen, addiert oder subtrahiert werden. Haben die Zeitwerte beim Vergleich unterschiedliche Stringlängen, sollten diese nicht mit "größer als" oder "kleiner als" verglichen werden, weil sie in diesem Fall als Zahlen behandelt werden. Zum Beispiel ist 20040201 aus numerischer Sicht immer kleiner (aber chronologisch größer) als 200401010533. Verwende stattdessen EnvSub, um herauszufinden, ob die Differenz zwischen denen positiv oder negativ ist.

Verwandte Befehle

FileGetTime, FileGetAttrib, FileSetAttrib, FileGetSize, FileGetVersion, FormatTime, Loop (Dateien & Ordner), EnvAdd, EnvSub

Beispiel

; Stellt das Änderungsdatum aller übereinstimmenden Dateien auf das aktuelle Datum um:
FileSetTime, , C:\temp\*.txt

; Stellt das Änderungsdatum um (Zeit wird auf Mitternacht gesetzt):
FileSetTime, 20040122, C:\Eigene Dokumente\test.doc

; Ändert das Erstelldatum. Die Zeit wird auf 16:55 Uhr umgestellt:
FileSetTime, 200401221655, C:\Eigene Dokumente\test.doc, C

; Stellt das Änderungsdatum aller übereinstimmenden Dateien um:
; Jeder übereinstimmende Ordner wird auch durch den letzten Parameter geändert:
FileSetTime, 20040122165500, C:\Temp\*.*, M, 1
