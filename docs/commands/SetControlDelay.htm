SetControlDelay

Bestimmt die Verzögerung, die nach jedem Steuerelement-Befehl auftreten wird.

SetControlDelay, Verzögerung

Parameter

Verzögerung Zeit in Millisekunden (kann ein Ausdruck sein). Durch -1 erfolgt überhaupt keine Verzögerung und mit 0 die kleinst mögliche Verzögerung. Ist dieser Parameter nicht vorhanden, gilt standardmäßig 20.

Bemerkungen

Eine kurze Verzögerung (Ruhephase) erfolgt automatisch nach jedem Befehl, der ein Steuerelement ändert, dazu zählt Control, ControlMove, ControlClick, ControlFocus und ControlSetText (ControlSend verwendet SetKeyDelay). Das geschieht, um die Zuverlässigkeit der Scripts zu verbessern, weil ein Steuerelement manchmal eine kurze Ruhephase benötigt, nachdem es von einem dieser Befehle geändert wurde. Dadurch hat das Steuerelement genug Zeit, sich selbst zu aktualisieren und auf den nächsten vom Script gesendeten Befehl zu reagieren.

Obwohl eine Verzögerung von -1 (keine Verzögerung) erlaubt ist, sollte sicherheitshalber lieber eine 0 verwendet werden, damit das Script auch unter hoher CPU-Auslastung noch richtig ausgeführt wird.

Eine Verzögerung von 0 führt intern die Funktion Sleep(0) aus, die den Restwert vom Script-Zeitintervall für einen anderen Prozess zurückgibt. Wenn es keine gibt, ermöglicht Sleep(0) keine Ruhephase.

Wenn die CPU langsam ist oder belastet wird, oder die Fensteranimation aktiviert ist, werden möglicherweise höhere Verzögerungswerte benötigt.

Die interne Variable A_ControlDelay enthält die aktuelle Einstellung.

Jeder neu gestartete Thread (z. B. ein Hotkey, ein benutzerdefinierter Menüpunkt oder eine zeitlich festgelegte Subroutine) beginnt mit der Standardeinstellung des Befehls. Diese Standardeinstellung kann geändert werden, indem dieser Befehl im automatischen Ausführungsbereich (am Anfang des Scripts) verwendet wird.

Verwandte Befehle

Control, ControlMove, ControlClick, ControlFocus, ControlSetText, SetWinDelay, SetKeyDelay, SetMouseDelay, SetBatchLines

Beispiel

SetControlDelay, 0
