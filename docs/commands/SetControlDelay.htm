SetControlDelay

Bestimmt die Verzögerung, die nach jedem Steuerelement-Befehl auftreten wird.

SetControlDelay, Verzögerung

Parameter

Verzögerung Zeit in Millisekunden (kann ein Ausdruck sein). Verwendet -1 für keine Verzögerung und 0 für die kleinste mögliche Verzögerung. Wenn nicht gesetzt, dann wird eine Standardverzögerung von 20 verwendet.

Bemerkungen

Eine kurze Verzögerung (Ruhephase) erfolgt automatisch nach jedem Befehl, der ein Steuerelement geändert hat, dazu zählen Control, ControlMove, ControlClick, ControlFocus und ControlSetText (ControlSend verwendet SetKeyDelay). Das geschieht, um die Zuverlässigkeit der Scripts zu verbessern, weil ein Steuerelement manchmal eine kurze Ruhephase benötigt, nachdem es von einem folgenden Befehl geändert wurde. Die Ruhephase ermöglicht dem Steuerelement, sich selbst zu aktualisieren und auf dem nächsten Befehl zu reagieren, der vom Script versuchsweise gesendet wird.

Obwohl eine Verzögerung von -1 (keine Verzögerung) erlaubt ist, wird mindestens ein Wert von 0 empfohlen, um das Vertrauen zu stärken, dass das Script auch unter hoher CPU-Auslastung noch richtig ausgeführt wird.

Eine Verzögerung von 0 führt Sleep(0) intern aus, die den Rest der Script-Zeitscheibe für einen anderen Prozess zurückgibt. Wenn es keine gibt, dann ermöglicht Sleep(0) keine Ruhephase.

Wenn die CPU langsam ist oder belastet wird, oder die Fensteranimation aktiviert ist, dann werden möglicherweise höhere Verzögerungswerte benötigt.

Die interne Variable A_ControlDelay enthält die aktuelle Einstellung.

Jeder neu gestartete Thread (z. B. ein Hotkey, ein benutzerdefinierter Menüpunkt oder eine zeitlich festgelegte Subroutine) beginnt mit der Standardeinstellung des Befehls. Diese Standardeinstellung kann geändert werden, indem dieser Befehl im automatischen Ausführungsbereich (am Anfang des Scripts) verwendet wird.

Verwandte Befehle

Control, ControlMove, ControlClick, ControlFocus, ControlSetText, SetWinDelay, SetKeyDelay, SetMouseDelay, SetBatchLines

Beispiel

SetControlDelay, 0
