SetControlDelay

Bestimmt die Verzögerung, die nach jedem steuerelement-modifizierenden Befehl erfolgen soll.

SetControlDelay, Verzögerung

Parameter

Verzögerung

Zeit in Millisekunden (kann ein Ausdruck sein). Verwende -1, um die Verzögerung vollständig zu deaktivieren, oder 0, um die kleinst mögliche Verzögerung zu veranlassen. Ist dieser Parameter nicht vorhanden, gilt standardmäßig 20.

Bemerkungen

Eine kurze Verzögerung (Pause) erfolgt automatisch nach jedem Befehl, der ein Steuerelement verändern kann, dazu zählt Control, ControlMove, ControlClick, ControlFocus und ControlSetText (ControlSend verwendet SetKeyDelay). Das verbessert die Zuverlässigkeit der Scripts, weil einige Steuerelemente eine Pause benötigen, um von einem dieser Befehle geändert werden zu können. Diese Pause gibt dem Steuerelement genug Zeit, sich selbst aktualisieren und auf den nächsten Befehl reagieren zu können, den das Script für diesen Steuerelement durchführen will.

Es ist möglich, eine Verzögerung von -1 anzugeben, um sie vollständig zu deaktivieren. Allerdings ist es besser, mindestens 0 zu verwenden, um sicherstellen zu können, dass das Script auch unter starker CPU-Belastung weiterhin problemlos läuft.

Eine Verzögerung von 0 führt intern die Sleep(0) aus. Diese Funktion kann bei Bedarf den Restwert vom Script-Zeitintervall an andere Prozesse übergeben. Besteht kein Bedarf, wird Sleep(0) keine Verzögerung erzeugen.

Wenn die CPU langsam oder unter starker Belastung ist, oder die Fensteranimation aktiviert ist, werden möglicherweise höhere Verzögerungswerte benötigt.

Die integrierte Variable A_ControlDelay enthält die aktuelle Einstellung.

Jeder neu gestartete Thread (z. B. ein Hotkey, ein benutzerdefinierter Menüpunkt oder eine zeitgesteuerte Subroutine) beginnt mit der Standardeinstellung des Befehls. Diese Standardeinstellung kann man ändern, wenn dieser Befehl bereits im automatischen Ausführungsbereich (am Anfang des Scripts) definiert wird.
