RegWrite

Schreibt einen Wert in die Registrierung.

RegWrite, Werttyp, Hauptschlüssel, Unterschlüssel [, Wertname, Wert]

Parameter

Werttyp Muss entweder REG_SZ, REG_EXPAND_SZ REG_MULTI_SZ, REG_DWORD oder REG_BINARY sein.
Hauptschlüssel Muss entweder HKEY_LOCAL_MACHINE (HKLM), HKEY_USERS (HKU), HKEY_CURRENT_USER (HKCU), HKEY_CLASSES_ROOT (HKCR) oder HKEY_CURRENT_CONFIG (HKCC) sein. Für den Zugriff auf eine Remoteregistrierung muss der Computername und ein Doppelpunkt wie folgt vorangestellt werden: \\workstation01:HKEY_LOCAL_MACHINE
Unterschlüssel Der Name des Unterschlüssels (z. B. Software\BeliebigeAnwendung). Falls der Unterschlüssel nicht vorhanden ist, wird er erstellt (einschließlich seine Elternelemente, falls nötig). Ist der Unterschlüssel leer, wird der Wert direkt in den Hauptschlüssel geschrieben (obwohl einige Betriebssysteme sich weigern könnten, in die oberste Ebene von HKEY_CURRENT_USER zu schreiben).
Wertname Der Name des zu schreibenden Wertes. Ist dieser Parameter nicht vorhanden, wird der Standardwert von Unterschlüssel verwendet. Dieser wird in RegEdit als "(Standard)" angezeigt.
Wert Der zu schreibende Wert. Ist dieser Parameter nicht vorhanden, wird je nach Werttyp standardmäßig ein leerer String oder eine 0 verwendet. Falls der Text zu lang ist, kann dieser in mehreren kurzen Zeilen aufgeteilt werden (Fortsetzungsbereich), um die Lesbarkeit und Wartbarkeit zu verbessern.

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

A_LastError wird auf das Ergebnis der Systemfunktion GetLastError() gesetzt.

Bemerkungen

Wenn der Werttyp REG_DWORD enthält, sollte der Wert zwischen -2147483648 und 4294967295 (0xFFFFFFFF) liegen.

Auch wenn nicht mehr als 64 KB an Daten aus REG_BINARY-Werten gelesen werden können, sind die anderen Typen unbegrenzt. In Windows 95/98/ME sind allerdings alle Typen auf 64 KB begrenzt. Wenn das Limit von 64 KB gilt, werden Daten über dem Limit nicht geschrieben. Das heißt, dass nur die ersten 64 KB eines sehr großen Strings in die Registrierung gespeichert werden.

Der zu schreibende Wert eines REG_BINARY-Schlüssels muss ein String mit Hex-Zeichen sein. Zum Beispiel ist es mithilfe des Strings 01A9FF77 möglich, einen REG_BINARY-Wert von 01,a9,ff,77 zu schreiben.

Der zu schreibende Wert eines REG_MULTI_SZ-Schlüssels muss aus mehreren Komponenten bestehen, die jeweils mit einen Zeilenumbruchzeichen (`n) enden. Optional kann die letzte Komponente auch mit Zeilenumbruchzeichen enden. Leere Komponenten sind nicht erlaubt. Das heißt, dass zwei Zeilenumbruchzeichen nicht hintereinander erfolgen dürfen (`n`n), denn das würde bedeuten, dass ein Wert in die Registrierung geschrieben wird, der kürzer als erwartet ist. Hinweis: Windows 95 unterstützt nicht den REG_MULTI_SZ-Werttyp.

Mit der Registrierungs-Schleife können mehrere Registrierungsschlüssel und -werte abgerufen und bearbeitet werden.

Bei den Bemerkungen zur Registrierungs-Schleife wird näher gebracht, wie man auf die Registrierung eines Remotecomputers zugreifen kann.

Mit SetRegView können Einträge von den 64-Bit-Bereichen der Registrierung in einem 32-Bit-Script oder umgekehrt gelesen und geschrieben werden.

Verwandte Befehle

RegDelete, RegRead, Registrierungs-Schleife, SetRegView, IniWrite

Beispiele

RegWrite, REG_SZ, HKEY_LOCAL_MACHINE, SOFTWARE\Testschlüssel, Wertname, Testwert
RegWrite, REG_BINARY, HKEY_CURRENT_USER, Software\TEST_APP, TEST_NAME, 01A9FF77
RegWrite, REG_MULTI_SZ, HKEY_CURRENT_USER, Software\TEST_APP, TEST_NAME, Zeile1`nZeile2
