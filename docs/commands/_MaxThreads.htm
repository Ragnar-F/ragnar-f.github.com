#MaxThreads

Bestimmt die maximale Anzahl an gleichzeitigen Threads.

#MaxThreads Wert

Parameter

Wert Die maximale Anzahl an Threads, die gleichzeitig vorhanden sein können. Ein Wert größer als 255 bleibt auf 255 (vor der Version 1.0.48 lag der Grenzwert auf 20).

Bemerkungen

Diese Einstellung ist global, daher muss diese Direktive irgendwo im Script nur einmal angegeben werden, um das Verhalten des ganzen Scripts zu beeinflussen.

Ein Wert von 1 ist nicht empfehlenswert, weil dadurch neue Hotkeys nicht ausgeführt werden können, wenn das Script eine MsgBox oder ein Dialogfenster anzeigt. Dadurch können Timer auch nicht ausgeführt werden, wenn ein anderer Thread ruht oder wartet.

Bis zu zwei der folgenden Thread-Typen können erstellt werden, auch wenn die maximale Anzahl an Threads erreicht wurde: Ein Hotkey, Hotstring, OnClipboardChange oder GUI-Ereignis, wenn die erste Zeile der Subroutine ein ExitApp, Pause, Edit, Reload, KeyHistory, ListLines, ListVars, oder ListHotkeys ist. Außerdem wird die OnExit-Subroutine auch immer ausgeführt, unabhängig von der Anzahl an vorhandenen Threads.

Wenn diese Einstellung niedriger als #MaxThreadsPerHotkey ist, dann wird diese Einstellung vom MaxThreadsPerHotkey-Wert tatsächlich überschrieben.

Wenn diese Direktive nicht angegeben ist, dann wird standardmäßig 10 verwendet.

Verwandte Befehle

#MaxThreadsPerHotkey, Threads, #MaxHotkeysPerInterval, #HotkeyInterval, ListHotkeys, #MaxMem

Beispiel

#MaxThreads 2
