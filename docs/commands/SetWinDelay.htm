SetWinDelay

Bestimmt die Verzögerung, die nach jedem Fenster-Befehl wie WinActivate auftreten wird.

SetWinDelay, Verzögerung

Parameter

Verzögerung Zeit in Millisekunden (kann ein Ausdruck sein). Mit -1 erfolgt überhaupt keine Verzögerung und mit 0 die kleinst mögliche Verzögerung. Ist dieser Parameter nicht vorhanden, gilt standardmäßig eine Verzögerung von 100.

Bemerkungen

Nach jedem Fenster-Befehl, außer IfWinActive und IfWinExist, erfolgt automatisch eine kurze Verzögerung (Sleep), um die Zuverlässigkeit der Scripts zu verbessern. Manchmal benötigt ein Fenster eine Ruhephase, nachdem es erstellt, aktiviert, minimiert usw. wurde, damit das Fenster die Chance hat, sich selbst zu aktualisieren und auf dem nächsten Befehl reagieren zu können, den das Script versucht hat, zu senden.

Obwohl eine Verzögerung von -1 (keine Verzögerung) erlaubt ist, ist es empfohlen, einen Wert größer gleich 0 zu verwenden, damit das Script auch unter hoher CPU-Last korrekt ausgeführt wird.

Eine Verzögerung von 0 bewirkt die interne Ausführung von Sleep(0), die den Rest der Script-Zeitscheibe für einen anderen Prozess zurückgibt. Ist keiner vorhanden, wird Sleep(0) überhaupt nicht ausgeführt.

Wenn die CPU langsam oder unter hoher Last ist, oder die Fensteranimation aktiviert ist, sind möglicherweise höhere Verzögerungswerte notwendig.

Die interne Variable A_WinDelay enthält die aktuelle Einstellung.

Jeder neu gestartete Thread (z. B. ein Hotkey, ein benutzerdefinierter Menüpunkt oder eine zeitlich festgelegte Subroutine) beginnt mit der Standardeinstellung des Befehls. Diese Standardeinstellung kann geändert werden, indem dieser Befehl im automatischen Ausführungsbereich (am Anfang des Scripts) verwendet wird.

Verwandte Befehle

SetControlDelay, SetKeyDelay, SetMouseDelay, SetBatchLines, SendMode

Beispiel

SetWinDelay, 10
