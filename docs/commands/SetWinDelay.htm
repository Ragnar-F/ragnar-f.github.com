SetWinDelay

Bestimmt die Verzögerung, die nach jedem fenster-spezifischen Befehl wie WinActivate erfolgen soll.

SetWinDelay, Verzögerung

Parameter

Verzögerung Zeit in Millisekunden (kann ein Ausdruck sein). Verwende -1, um die Verzögerung vollständig zu deaktivieren, oder 0, um die kleinst mögliche Verzögerung zu veranlassen. Ist dieser Parameter nicht vorhanden, gilt standardmäßig 100.

Bemerkungen

Nach jedem fenster-spezifischen Befehl, außer IfWinActive und IfWinExist, erfolgt automatisch eine kurze Verzögerung (Pause). Das verbessert die Zuverlässigkeit der Scripts, weil einige Fenster eine Pause benötigen, um erstellt, aktiviert, minimiert usw. werden zu können. Diese Pause gibt dem Fenster genug Zeit, sich selbst aktualisieren und auf dem nächsten Befehl reagieren zu können, den das Script für dieses Fenster durchführen will.

Es ist möglich, eine Verzögerung von -1 anzugeben, um sie vollständig zu deaktivieren. Allerdings ist es besser, mindestens 0 zu verwenden, um sicherstellen zu können, dass das Script auch unter starker CPU-Belastung weiterhin problemlos läuft.

Eine Verzögerung von 0 führt intern die Sleep(0) aus. Diese Funktion kann bei Bedarf den Restwert vom Script-Zeitintervall an andere Prozesse übergeben. Besteht kein Bedarf, wird Sleep(0) keine Verzögerung erzeugen.

Wenn die CPU langsam oder unter starker Belastung ist, oder die Fensteranimation aktiviert ist, werden möglicherweise höhere Verzögerungswerte benötigt.

Die interne Variable A_WinDelay enthält die aktuelle Einstellung.

Jeder neu gestartete Thread (z. B. ein Hotkey, ein benutzerdefinierter Menüpunkt oder eine zeitlich festgelegte Subroutine) beginnt mit der Standardeinstellung des Befehls. Diese Standardeinstellung kann man ändern, wenn man diesen Befehl bereits im automatischen Ausführungsbereich (am Anfang des Scripts) definiert.

Verwandte Befehle

SetControlDelay, SetKeyDelay, SetMouseDelay, SetBatchLines, SendMode

Beispiel

SetWinDelay, 10
