GroupDeactivate

Ähnelt GroupActivate, nur wird das nächste Fenster aktiviert, welches nicht zur Gruppe gehört.

GroupDeactivate, Gruppenname [, R]

Parameter

Gruppenname Der Name der Gruppe, ursprünglich von GroupAdd definiert.
R Damit wird festgelegt, ob das älteste oder das neueste Fenster außerhalb der Gruppe aktiviert wird, sobald kein Fenster der Gruppe aktiv ist. Wenn nicht vorhanden, dann wird immer das älteste Fenster außerhalb der Gruppe aktiviert. Mit dem Buchstaben R wird das neueste (zuletzt aktive) Fenster außerhalb der Gruppe aktiviert, aber mit der Bedingung, dass kein Fenster der Gruppe aktiv ist, wenn der Befehl ausgeführt wird. "R" ist beispielsweise nützlich, falls der Benutzer sich vorübergehend mit einer anderen Aufgabe beschäftigt. Kehrt er dann mittels GroupActivate, GroupDeactivate oder GroupClose zur Gruppe zurück, wird das am aktuellsten genutzte Fenster aktiviert.

Bemerkungen

Dieser Befehl aktiviert das erste Fenster, das nicht mit den Spezifikationen der Fenstergruppe übereinstimmt. Beim nochmaligen Verwenden wird das nächste Fenster in der Reihe aktiviert und so weiter. Normalerweise wird dieser Befehl einem Hotkey zugewiesen, damit dieses Verhalten beim Drücken einer Taste automatisiert wird.

Dieser Befehl ist nützlich, falls der Benutzer Lieblingsfenster hat, die immer vorhanden sein sollen. Durch das Hinzufügen dieser Fenster zu einer Gruppe kann der Benutzer mit dem Befehl GroupDeactivate jedes Fenster aufsuchen, das nicht zu den Favoriten gehört und dann entscheiden, ob dieses Fenster geschlossen werden soll oder nicht. Damit kann der Benutzer den Desktop im Gegensatz zur manuellen Vorgehensweise schnell aufräumen.

Siehe GroupAdd für mehr Details über Fenstergruppen.

Verwandte Befehle

GroupAdd, GroupActivate, GroupClose

Beispiel

GroupDeactivate, EigeneLieblingsfenster  ; Nicht favorisierte Fenster schließen.
