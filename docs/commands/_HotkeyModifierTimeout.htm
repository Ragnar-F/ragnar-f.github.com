#HotkeyModifierTimeout

Beeinflusst das Verhalten von Hotkey-Modifikatoren: CTRL, ALT, WIN und SHIFT.

#HotkeyModifierTimeout Millisekunden

Parameter

Millisekunden Die Länge des Intervalls in Millisekunden. Der Wert kann -1 sein, so dass es nie zur Zeitüberschreitung kommt (Modifikatortasten werden nach Send immer wieder nach unten gedrückt), oder 0, so dass immer eine Zeitüberschreitung erfolgt (Modifikatortasten werden nicht wieder nach unten gedrückt).

Bemerkungen

Diese Direktive muss nicht verwendet werden, wenn:

  • Hotkeys ihre Tastatureingaben mit den Methoden SendInput oder SendPlay senden. Das liegt daran, dass solche Methoden das physische Drücken und Loslassen der Tasten aufschieben, bis der Send-Befehl durchgeführt wurde.
  • das Script den Tastatur-Hook installiert hat (kann mit "View-Key history" in der Menüleiste des Hauptfensters oder mit dem KeyHistory-Befehl überprüft werden). Das liegt daran, dass der Hook nachverfolgen kann, welche Modifikatortasten (ALT/STRG/WIN/UMSCHALT) der Benutzer physisch gedrückt hält, ohne die Zeitüberschreitung verwenden zu müssen.

Zur Veranschaulichung dieser Direktive wird folgendes Beispiel in Betracht gezogen:

^!a::Send, abc

Beim Ausführen des Send-Befehls wird als erstes die STRG- und ALT-Taste losgelassen, so dass die Zeichen richtig gesendet werden können. Der Befehl hat danach keine Ahnung, ob er STRG und ALT sicher wieder nach unten drücken kann (um damit übereinzustimmen, ob der Benutzer diese Tasten weiterhin gedrückt hält). Wenn allerdings weniger als die angegebene Anzahl von Millisekunden vergangen sind, nimmt der Befehl an, dass der Benutzer nicht die Chance hatte, die Tasten loszulassen und wird sie daher wieder nach unten drücken, damit sie ihrem physischen Status entsprechen. Ansonsten werden die Modifikatortasten nicht wieder nach unten gedrückt. Das heißt, dass der Benutzer solche Tasten loslassen und erneut drücken muss, damit sie die gleiche oder eine andere Taste modifizieren können.

Die Zeitüberschreitung sollte auf einen Wert gesetzt werden, der kleiner ist als die vom Benutzer benötigte Zeit, Modifikatortasten eines Hotkeys zu drücken, bevor diese wieder losgelassen werden. Ansonsten werden die Modifikatortasten auf die gedrückte Position zurückgesetzt (bleiben stecken), auch dann, wenn der Benutzer diese nicht physisch gedrückt hält.

Die Notwendigkeit dieser Direktive kann wie folgt reduziert oder beseitigt werden:

  • Installiere den Tastatur-Hook durch Hinzufügen der Zeile #InstallKeybdHook irgendwo im Script (allerdings wird dieser Hook nicht in Win9x unterstützt).
  • Verwende die Methoden SendInput oder SendPlay, anstatt die traditionelle SendEvent-Methode.
  • Reduziere bei Benutzung der traditionellen SendEvent-Methode SetKeyDelay auf 0 oder -1, das dabei helfen sollte, die Tastatureingaben schneller zu senden.

Ist diese Direktive im Script nicht angegeben, gilt standardmäßig 50.

Verwandte Befehle

GetKeyState

Beispiel

#HotkeyModifierTimeout 100
