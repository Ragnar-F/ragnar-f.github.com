#HotkeyModifierTimeout

Beeinflusst das Verhalten von Hotkey-Modifikatoren: CTRL, ALT, WIN und SHIFT.

#HotkeyModifierTimeout Millisekunden

Parameter

Millisekunden Die Länge des Intervalls in Millisekunden. Der Wert kann -1 sein, damit es nie zur Zeitüberschreitung kommt (Modifikatoren werden immer wieder nach hinten verschoben), oder 0, damit es immer zur Zeitüberschreitung kommt (Modifikatoren werden nicht hinten verschoben).

Bemerkungen

Diese Direktive muss nicht verwendet werden, wenn:

  • Hotkeys ihre Tastatureingaben mit den Methoden SendInput oder SendPlay senden. Weil diese Methoden das physische Drücken und Loslassen der Tasten aufschieben, bis der Send-Befehl durchgeführt wurde.
  • das Script den Tastatur-Hook installiert hat (kann mit "View-Key history" in der Menüleiste des Hauptfensters oder mit dem KeyHistory-Befehl überprüft werden). Weil der Hook verfolgen kann, welche Modifikatoren (ALT/STRG/WIN/UMSCHALT) der Benutzer physisch gedrückt hält und nicht die Zeitüberschreibung verwenden müssen.

Zur Veranschaulichung dieser Direktive wird folgendes Beispiel in Betracht gezogen:
^!a::Send, abc

Wenn der Send-Befehl ausgeführt wird, dann lässt er zuerst die STRG- und ALT-Tasten los, damit die Zeichen richtig gesendet werden. Nachdem alle Tasten gesendet wurden, weiß der Befehl nicht mehr, ob er STRG und ALT sicher nach hinten verschieben kann (damit sie passen, falls der Benutzer sie weiterhin gedrückt hält). Aber wenn weniger als die angegebene Anzahl von Millisekunden vergangen sind, nimmt der Befehl an, dass der Benutzer nicht die Chance hatte, die Tasten loszulassen und wird sie daher nach hinter verschieben, damit sie ihrem physischen Status entsprechen. Ansonsten werden die Modifikatoren nicht hinten verschoben und der Benutzer muss die Tasten loslassen und erneut drücken, damit sie die gleiche oder eine andere Taste modifizieren können.

Die Zeitüberschreitung sollte auf einen Wert gesetzt werden, der kleiner als die Zeitmenge ist, die der Benutzer benötigt, Modifikatoren eines Hotkeys zudrücken, bis sie wieder losgelassen werden. Ansonsten wird der gedrückte Zustand von den Modifikatoren möglicherweise wiederhergestellt, auch wenn der Benutzer sie nicht physisch drückt.

Die Notwendigkeit dieser Direktive kann wie folgt reduziert oder beseitigt werden:

  • Installiert den Tastatur-Hook mit der Zeile #InstallKeybdHook irgendwo im Script (der Hook wird allerdings nicht von Win9x unterstützt).
  • Verwendet die Methoden SendInput oder SendPlay anstelle der traditionellen Methode SendEvent.
  • Wenn die traditionelle Methode SendEvent verwendet wird, dann reduziert SetKeyDelay von 0 auf -1, damit die Tastatureingaben schneller gesendet werden.

Wenn diese Direktive nicht angegeben ist, dann wird standardmäßig 50 verwendet.

Verwandte Befehle

GetKeyState

Beispiel

#HotkeyModifierTimeout 100
