RegDelete

Löscht einen Unterschlüssel oder Wert von der Registry.

RegDelete, Hauptschlüssel, Unterschlüssel [, Wertname]

Parameter

Hauptschlüssel

Entweder HKEY_LOCAL_MACHINE (oder HKLM), HKEY_USERS (oder HKU), HKEY_CURRENT_USER (oder HKCU), HKEY_CLASSES_ROOT (oder HKCR) oder HKEY_CURRENT_CONFIG (oder HKCC). Um den Zugriff auf eine Remote-Registry zu ermöglichen, muss der Computername und ein Doppelpunkt wie im folgenden Beispiel vorangestellt werden: \\workstation01:HKEY_LOCAL_MACHINE

Unterschlüssel

Der Name des Unterschlüssels (z. B. Software\BeliebigeAnwendung).

Wertname

Der Name des zu löschenden Wertes. Ist dieser Parameter nicht vorhanden, wird der ganze Unterschlüssel gelöscht. Um den Standardwert von Unterschlüssel (in RegEdit als "(Standard)" angezeigt) zu löschen, sollte in diesem Parameter AHK_DEFAULT angegeben werden.

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

Die Variable A_LastError enthält den Rückgabewert der Systemfunktion GetLastError().

Bemerkungen

Löschen in der Registry kann erhebliche Probleme verursachen - bitte Vorsicht walten lassen!

Mit der Registry-Schleife können mehrere Registry-Schlüssel und -werte abgerufen und bearbeitet werden.

Bei den Bemerkungen zur Registry-Schleife wird näher gebracht, wie man auf die Registry eines Remotecomputers zugreifen kann.

Mit SetRegView können Einträge von den 64-Bit-Bereichen der Registry in einem 32-Bit-Script oder umgekehrt entfernt werden.
