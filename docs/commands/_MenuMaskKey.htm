#MenuMaskKey [AHK_L 38+]

Bestimmt die Taste, die dazu verwendet wird, um Win- oder Alt-Up-Ereignisse zu verbergen.

#MenuMaskKey Tastenname

Parameter

Tastenname Ein Tastenname oder eine vkNN-Sequenz, die einen virtuellen Tastencode ungleich 0 bestimmt. Scan-Codes können nicht verwendet werden.

Bemerkungen

Diese Einstellung ist global, daher muss diese Direktive irgendwo im Script nur einmal angegeben werden, um das Verhalten des ganzen Scripts zu beeinflussen.

Wenn ein Hotkey mit dem Tastatur- oder Maus-Hook implementiert ist, dann ist die letzte Tastatureingabe auf dem aktiven Fenster oder System möglicherweise unsichtbar. Wenn Hotkeys die Modifikatoren Win oder Alt verwenden und diese losgelassen werden, dann wird möglicherweise das Startmenü oder die Menüleiste des aktiven Fensters aktiviert. Um das zu verhindern, "versteckt" AutoHotkey die Up-Ereignisse der gesendeten Tastatureingabe. Vor Revision 38 war es immer eine Strg-Tastatureingabe, die Probleme bei bestimmten Anwendungen verursachte.

Wenn diese Direktive nicht angegeben ist, dann wird standardmäßig die Strg-Taste verwendet.

Verwandte Befehle

Siehe diesen Thread für Hintergrundinformationen.

Beispiele

#MenuMaskKey vk07  ; vk07 ist unbelegt.
#UseHook
#Space::
!Space::
    KeyWait LWin
    KeyWait RWin
    KeyWait Alt
    KeyHistory
