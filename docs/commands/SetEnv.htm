SetEnv (Var = Wert)

Speichert den angegebenen Wert in eine Variable.

SetEnv, Var, Wert
Var = Wert

Parameter

Var Name der Variable, in der Wert gespeichert werden soll.
Wert Der zu speichernde Wert (String oder Zahl). Falls der String zu lang ist, kann dieser in mehreren kurzen Zeilen aufgeteilt werden (Fortsetzungsbereich), um die Lesbarkeit und Wartbarkeit zu verbessern.

Bemerkungen

Standardmäßig werden Leer- oder Tabulatorzeichen in Var entfernt, die sich am Anfang oder Ende des Werts befinden. Um das zu verhindern, können die beschriebenen Methoden in AutoTrim Off verwendet werden.

Der Name "SetEnv" ist irreführend und ein Überbleibsel von AutoIt v2. Im Gegensatz zu AutoIt v2 speichert AutoHotkey seine Variablen nicht in die Systemumgebung. Das liegt daran, dass die Performance darunter leiden würde und dass die Größe jeder Umgebungsvariable auf 32 KB begrenzt ist. Es ist möglich, EnvSet anstelle von SetEnv zu verwenden, um eine Umgebungsvariable zu beschreiben.

Um den Speicher einer großen Variable freizugeben, muss ihr nichts zugewiesen werden (z. B. Var =).

Es ist möglich, einen Array mit diesem Befehl zu erstellen, sowie mit anderen Befehlen, die AusgabeVar akzeptieren. Das wird erreicht, indem AusgabeVar einen Verweis auf eine andere Variable enthält (z. B. Array%i% = 123). Siehe Arrays für mehr Details.

Verwandte Befehle

AutoTrim, EnvSet, EnvAdd, EnvSub, EnvMult, EnvDiv, If, Arrays

Beispiel

Var1 = Das ist ein String.
Farbe2 = 450
Farbe3 = %Var1%
Array%i% = %A_TICKCOUNT%
