SetEnv (Var = Wert)

Speichert den angegebenen Wert in eine Variable.

SetEnv, Variable, Wert
Variable = Wert

Parameter

Variable Der Name der Variable, die den Wert enthält.
Wert Der String oder die Zahl, die gespeichert werden soll. Wenn der String lang ist, dann kann dieser in mehreren kurzen Zeilen aufgeteilt werden (Fortsetzungsbereich), die die Lesbarkeit und Wartbarkeit verbessern könnten.

Bemerkungen

Standardmäßig werden Leerzeichen oder Tabulatoren im Wert am Anfang und Ende der Variable weggelassen. Um das zu verhindern, verwendet die Methoden wie in AutoTrim Off beschrieben.

Der Name "SetEnv" ist irreführend und wird nur für die Abwärtskompatibilität mit AutoIt v2 verwendet. Im Gegensatz zu AutoIt v2 werden AutoHotkey-Variablen nicht in die Systemumgebung gespeichert. Denn die Performance wird schlechter und das Betriebssystem begrenzt die Größe jeder Umgebungsvariable auf 32 KB. Verwendet EnvSet anstelle von SetEnv, um in eine Umgebungsvariable zu schreiben.

Der Speicher einer großen Variable kann freigegeben werden, wenn ihr nichts zugewiesen wird (z. B. Variable =)

Es ist möglich, einen Array mit diesem Befehl zu erstellen, sowie mit anderen Befehlen, die eine Ausgabevariable akzeptieren. Dazu enthält die Ausgabevariable eine Referenz auf eine andere Variable (z. B. Array%i% = 123). Siehe Arrays für mehr Details.

Verwandte Befehle

AutoTrim, EnvSet, EnvAdd, EnvSub, EnvMult, EnvDiv, If, Arrays

Beispiel

Var1 = Das ist ein String.
Color2 = 450
Color3 = %Var1%
Array%i% = %A_TICKCOUNT%
