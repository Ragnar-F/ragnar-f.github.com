Transform

Führt verschiedene mathematische Funktionen, bitweise Operationen und Aufgaben wie ASCII/Unicode-Konvertierung aus.

Transform, Ausgabevariable, Befehl, Wert1 [, Wert2]

Parameter

Ausgabevariable Name der Variable, in die der Rückgabewert von Befehl gespeichert wird. SetFormat bestimmt, ob Integer hexadezimal oder dezimal gespeichert werden.
Befehl, Wert1/2 Siehe untere Liste.

Befehl, Wert1, Wert2

Die Parameter Befehl, Wert1 und Wert2 sind voneinander abhängig und können wie folgt verwendet werden.

Unicode [, String]: (Dieser Befehl steht in der Unicode-Version von AutoHotkey_L nicht zur Verfügung.) Ermittelt oder speichert den Unicode-Text in die Zwischenablage. Hinweis: Die gesamte Zwischenablage kann mit ClipboardAll gespeichert und wiederhergestellt werden, damit "Transform Unicode" ohne Verlust des originalen Inhalts der Zwischenablage durchgeführt wird.

Es gibt zwei verschiedene Methoden, den Befehl auszuführen:

Transform, Ausgabevariable, Unicode  ; Ermittelt den Unicode-Text aus der Zwischenablage als UTF-8-String.
Transform, Clipboard, Unicode, %EigenerUTF_String%  ; Speichert den Unicode-Text in die Zwischenablage.

Im zweiten oberen Beispiel besteht die Möglichkeit, anstelle von %EigenerUTF_String% einen direkten UTF-8-String anzugeben.

Mit dem folgenden Hotkey kann der UTF-8-String ermittelt werden, der zu einem angegebenen Unicode-String gehört:

^!u::  ; Hotkey Strg+Alt+U.
MsgBox Kopiere einen beliebigen Unicode-Text, kehre danach zu diesem Fenster zurück und drücke auf OK, um fortzufahren.
Transform, ClipUTF, Unicode
Clipboard = Transform, Clipboard, Unicode, %ClipUTF%`r`n
MsgBox Die Zwischenablage enthält nun die folgende Zeile, die du in deinem Script einfügen kannst. Sobald diese Zeile ausgeführt wird, wird der originale Unicode-Text, den du kopiert hast, in die Zwischenablage gespeichert:`n`n%Clipboard%
Return

Hinweise:

  • Windows 95 benötigt Microsoft Layer for Unicode (MSLU), um diesen Befehl unterstützen zu können (Windows 98/Me/NT4 oder höher haben eine integrierte Unterstützung).
  • Mit "Send {ASC nnnnn}" können auch Unicode-Zeichen erzeugt werden, allerdings funktioniert dieser Befehl nicht in jeder Anwendung.

Asc, String: Ermittelt den ASCII-Code (eine Zahl zwischen 1 und 255) für das erste Zeichen im String. Falls der String leer ist, wird die Ausgabevariable auch leer gemacht. Zum Beispiel: Transform, Ausgabevariable, Asc, %VarMitString%. Dazugehörige Funktion: Asc(String).

Chr, Wert1: Ermittelt ein einzelnes Zeichen, das zum angegebenen ASCII-Code in Wert1 gehört. Falls Wert1 nicht zwischen 1 und 255 liegt, wird die Ausgabevariable leer gemacht, um das Problem zu kennzeichnen. Zum Beispiel: Transform, Ausgabevariable, Chr, 130. Dazugehörige Funktion: Chr(Zahl).

Deref, String: Erweitert Variablenverweise und Schutzsequenzen, die sich in anderen Variablen befinden. Jeder schlecht formatierte Variablenverweis wird im erweiterten Ergebnis weggelassen. Das gleiche gilt auch, wenn sich die Ausgabevariable in sich selbst erweitert. Das heißt, dass Verweise auf Ausgabevariable innerhalb der Variablen von String beim Erweitern weggelassen werden (allerdings kann String selbst die %Ausgabevariable% sein). Wenn im folgenden Beispiel Var1 den String "Test" und Var2 den direkten String "%Var1%" enthält, wird Ausgabevariable auf den String "Test" gesetzt: Transform, Ausgabevariable, deref, %Var2%. Innerhalb einer Funktion wird jede Variable in String immer in eine lokale Variable aufgelöst, sofern so eine Variable vorhanden ist, ansonsten wird sie in eine globale Variable aufgelöst (oder leer, wenn keine vorhanden ist).

HTML, String [, Flags]:

In AutoHotkey und AutoHotkey_L ANSI wird der Flags-Parameter ignoriert. Dieser Befehl wandelt den String in dessen HTML-Äquivalent um, indem er Zeichen, deren ASCII-Werte über 127 liegen, in ihre HTML-Namen übersetzt (z. B. wird £ zu &pound;). Zudem werden die vier Zeichen "&<> in &quot;&amp;&lt;&gt; und zu guter Letzt jedes LF-Zeichen (`n) in <br>`n übersetzt (<br>, gefolgt von einem LF-Zeilenumbruchszeichen).

In AutoHotkey_L Unicode kann zusätzlich der Flags-Parameter verwendet werden, der entweder eine 0 oder eine Kombination (Summe) der folgenden Werte sein kann. Ist dieser Parameter nicht vorhanden, gilt standardmäßig eine 1.

  • 1: Wandelt bestimmte Zeichen in benannte Ausdrücke um. Zum Beispiel wird € zu &euro;.
  • 2: Wandelt bestimmte Zeichen in nummerierte Ausdrücke um. Zum Beispiel wird € zu &#8364;.

Nur Zeichen, die nicht zum ASCII-Zeichensatz gehören, sind davon betroffen. Falls Flags = 3 ist, werden nummerierte Ausdrücke nur verwendet, wenn ein benannter Ausdruck nicht verfügbar ist. Die folgenden Zeichen werden immer umgewandelt: <>"& und `n (LF-Zeilenumbruchszeichen).

FromCodePage / ToCodePage: Veraltet. Verwende stattdessen StrPut() / StrGet().

Mod, Dividend, Divisor: Ermittelt den Rest von Dividend, geteilt durch Divisor. Falls Divisor eine 0 ist, wird die Ausgabevariable leer gemacht. Sowohl Dividend als auch Divisor können einen Dezimalpunkt enthalten. Falls Divisor negativ ist, wird er für die Berechnung als positiv angesehen. Im folgenden Beispiel ist das Ergebnis eine 2: Transform, Ausgabevariable, mod, 5, 3. Dazugehörige Funktion: Mod(Dividend, Divisor).

Pow, Basis, Exponent: Ermittelt Basis, potenziert mit Exponent. Sowohl Basis als auch Exponent können einen Dezimalpunkt enthalten. Falls Exponent negativ ist, wird die Ausgabevariable in eine Fließkommazahl umgewandelt, auch dann, wenn Basis und Exponent Integer sind. Eine negative Basis, kombiniert mit einem gebrochenen Exponent wie 1.5, wird nicht unterstützt und führt dazu, dass die Ausgabevariable leer gemacht wird. Siehe auch: **-Operator.

Exp, N: Ermittelt e (ungefähr 2.71828182845905), potenziert mit N. N kann negativ sein und einen Dezimalpunkt enthalten. Dazugehörige Funktion: Exp(N).

Sqrt, Wert1: Ermittelt die Quadratwurzel von Wert1. Falls Wert1 negativ ist, wird die Ausgabevariable leer gemacht. Dazugehörige Funktion: Sqrt(Zahl).

Log, Wert1: Ermittelt den Logarithmus (Basis 10) von Wert1. Falls Wert1 negativ ist, wird die Ausgabevariable leer gemacht. Dazugehörige Funktion: Log(Zahl).

Ln, Wert1: Ermittelt den Logarithmus (Basis e) von Wert1. Falls Wert1 negativ ist, wird die Ausgabevariable leer gemacht. Dazugehörige Funktion: Ln(Zahl).

Round, Wert1 [, N]: Ist N nicht vorhanden, wird die Ausgabevariable auf Wert1 gesetzt, gerundet auf den nächsten Integer. Wenn N eine positive Zahl ist, wird Wert1 auf N Stellen nach dem Komma gerundet. Falls N negativ ist, wird Wert1 auf N Stellen vor dem Komma gerundet. Zum Beispiel rundet -1 auf Einerstelle, -2 auf Zehnerstelle und -3 auf Hunderterstelle. Hinweis: Round kann beim Runden von Dezimalstellen nachfolgende Nullen nicht entfernen. Zum Beispiel wird 12.333 zu 12.300000 beim Runden auf eine Dezimalstelle. Dieses Verhalten kann geändert werden, indem so etwas wie SetFormat, float, 0.1 vor der Operation verwendet wird (tatsächlich könnte SetFormat das Verwenden von Round an erster Stelle überflüssig machen). Dazugehörige Funktion: Round(Zahl [, N]).

Ceil, Wert1: Ermittelt Wert1, auf den nächst höheren Integer gerundet. Dazugehörige Funktion: Ceil(Zahl).

Floor, Wert1: Ermittelt Wert1, auf den nächst niedrigeren Integer gerundet. Dazugehörige Funktion: Floor(Zahl).

Abs, Wert1: Ermittelt den absoluten Wert von Wert1, indem das führende Minuszeichen (Bindestrich), falls vorhanden, von Wert1 entfernt wird. Dazugehörige Funktion: Abs(Zahl).

Sin, Wert1: Ermittelt den trigonometrischen Sinus von Wert1. Wert1 muss als Bogenmaß angegeben werden. Dazugehörige Funktion: Sin(Zahl).

Cos, Wert1: Ermittelt den trigonometrischen Kosinus von Wert1. Wert1 muss als Bogenmaß angegeben werden. Dazugehörige Funktion: Cos(Zahl).

Tan, Wert1: Ermittelt den trigonometrischen Tangens von Wert1. Wert1 muss als Bogenmaß angegeben werden. Dazugehörige Funktion: Tan(Zahl).

ASin, Wert1: Ermittelt den Arkussinus (die Zahl, deren Sinus Wert1 ist) als Bogenmaß. Falls Wert1 kleiner als -1 oder größer als 1 ist, wird die Ausgabevariable leer gemacht. Dazugehörige Funktion: ASin(Zahl).

ACos, Wert1: Ermittelt den Arkuskosinus (die Zahl, deren Kosinus Wert1 ist) als Bogenmaß. Falls Wert1 kleiner als -1 oder größer als 1 ist, wird die Ausgabevariable leer gemacht. Dazugehörige Funktion: ACos(Zahl).

ATan, Wert1: Ermittelt den Arkustangens (die Zahl, deren Tangens Wert1 ist) als Bogenmaß. Dazugehörige Funktion: ATan(Zahl).

HINWEIS: Jede nachfolgende bitweise Operation kann innerhalb von Ausdrücken als bitweiser Operator angegeben werden.

BitNot, Wert1: Speichert die bit-invertierte Version von Wert1 in die Ausgabevariable (falls Wert1 eine Fließkommazahl ist, wird sie zu einem Integer gekürzt, bevor die Berechnung erfolgt). Wenn Wert1 ein Zahl zwischen 0 und 4294967295 (0xffffffff) ist, wird sie als vorzeichenlosen 32-Bit-Wert behandelt. Ansonsten wird sie als vorzeichenbehafteten 64-Bit-Wert behandelt. Im folgenden Beispiel ist das Ergebnis 0xfffff0f0 (4294963440): Transform, Ausgabevariable, BitNot, 0xf0f

BitAnd, Wert1, Wert2: Ermittelt das Ergebnis des bitweisen UND's von Wert1 und Wert2 (Fließkommazahlen werden zu Integer gekürzt, bevor die Berechnung erfolgt). Im folgenden Beispiel ist das Ergebnis 0xff00 (65280): Transform, Ausgabevariable, BitAnd, 0xff0f, 0xfff0

BitOr, Wert1, Wert2: Ermittelt das Ergebnis des bitweisen ODER's von Wert1 und Wert2 (Fließkommazahlen werden zu Integer gekürzt, bevor die Berechnung erfolgt). Im folgenden Beispiel ist das Ergebnis 0xf0f0 (61680): Transform, Ausgabevariable, BitOr, 0xf000, 0x00f0

BitXOr, Wert1, Wert2: Ermittelt das Ergebnis des bitweisen Exklusiv-ODER's von Wert1 und Wert2 (Fließkommazahlen werden zu Integer gekürzt, bevor die Berechnung erfolgt). Im folgenden Beispiel ist das Ergebnis 0xff00 (65280): Transform, Ausgabevariable, BitXOr, 0xf00f, 0x0f0f

BitShiftLeft, Wert1, Wert2: Ermittelt das Ergebnis durch Verschieben von Wert1 um Wert2 Bit-Positionen nach links, das dem Multiplizieren von Wert1 mit "2 potenziert mit Wert2" entspricht (Fließkommazahlen werden zu Integer gekürzt, bevor die Berechnung erfolgt). Im folgenden Beispiel ist das Ergebnis eine 8: Transform, Ausgabevariable, BitShiftLeft, 1, 3

BitShiftRight, Wert1, Wert2: Ermittelt das Ergebnis durch Verschieben von Wert1 um Wert2 Bit-Positionen nach rechts, das dem Dividieren von Wert1 mit "2 potenziert mit Wert2" entspricht, bei dem der Rest gekürzt wird (Fließkommazahlen werden zu Integer gekürzt, bevor die Berechnung erfolgt). Im folgenden Beispiel ist das Ergebnis eine 2: Transform, Ausgabevariable, BitShiftRight, 17, 3

Bemerkungen

Unterbefehle, die numerische Parameter akzeptieren, können auch Ausdrücke für solche Parameter verwenden.

Sobald Wert1 oder Wert2 eine Fließkommazahl ist, ermitteln folgende Befehle eine Fließkommazahl, anstatt ein Integer: Mod, Pow, Round und Abs. Die Anzahl der abgerufenen Dezimalstellen wird über SetFormat bestimmt.

Um ein Bogenmaß in Grad umzuwandeln, kann es mit 180/pi (ungefähr 57.29578) multipliziert werden. Um ein Grad-Wert in Bogenmaß umzuwandeln, kann es mit pi/180 (ungefähr 0.01745329252) multipliziert werden.

Der Wert von pi (ungefähr 3,141592653589793) ist viermal der Arkustangens von 1.

Verwandte Befehle

SetFormat, Ausdrücke, EnvMult, EnvDiv, StringLower, If Var is Typ

Beispiel

Transform, Ausgabevariable, Asc, A  ; Ermittelt den ASCII-Code vom Buchstaben A.
