WinWaitActive / WinWaitNotActive

Wartet, bis das angegebene Fenster aktiv oder nicht mehr aktiv ist.

WinWaitActive [, Fenstertitel, Fenstertext, Sekunden, Titelausnahme, Textausnahme]
WinWaitNotActive [, Fenstertitel, Fenstertext, Sekunden, Titelausnahme, Textausnahme]

Parameter

Fenstertitel Der Titel oder Teiltitel des Fensters (das Übereinstimmungsverhalten wird mit SetTitleMatchMode bestimmt). Wenn dieser und die anderen 3 Fensterparameter weggelassen werden, wird das zuletzt gefundene Fenster verwendet. Bei Verwendung einer Fensterklasse muss ahk_class ExakterKlassenname angegeben werden (wird von Windows Spy angezeigt). Bei Verwendung einer Prozess-Kennung (PID) muss ahk_pid %VariableMitPID% angegeben werden. Bei Verwendung einer Fenstergruppe muss ahk_group Gruppenname angegeben werden. Bei Verwendung einer eindeutigen ID-Nummer eines Fensters muss ahk_id %VariableMitID% angegeben werden. Die Suche kann durch Angabe mehrerer Kriterien eingeschränkt werden. Zum Beispiel: Eigene Datei.txt ahk_class Notepad
Fenstertext In diesem Parameter kann ein Teilstring angegeben werden, der in einem Textelement des Zielfensters vorkommt (werden mit Windows Spy angezeigt). Versteckte Textelemente werden auch erkannt, sofern DetectHiddenText aktiv ist.
Sekunden Die Wartezeit in Sekunden, bis der Befehl beendet und ErrorLevel auf 1 gesetzt wird. Kann leer gelassen werden, um unendlich lang zuwarten. Der Wert 0 entspricht 0.5. Dieser Parameter kann ein Ausdruck sein.
Titelausnahme Ignoriert Fenster, deren Titel diesen Wert enthalten.
Textausnahme Ignoriert Fenster, deren Textelemente diesen Wert enthalten.

ErrorLevel

Wenn die Wartezeit abgelaufen ist, dann wird ErrorLevel auf 1 gesetzt, ansonsten enthält ErrorLevel eine 0.

Bemerkungen

Wenn ein passendes Fenster die Erwartung des Befehls zufriedenstellt, dann wartet der Befehl nicht, bis die Sekunden abgelaufen sind. Stattdessen wird ErrorLevel sofort auf 0 gesetzt und das Script weiter ausgeführt.

Sowohl WinWaitActive als auch WinWaitNotActive aktualisieren das zuletzt gefundene Fenster, falls ein qualifiziertes Fenster aktiv ist, sobald der Befehl beginnt. Zusätzlich aktualisiert WinWaitActive das zuletzt gefundene Fenster, falls ein qualifiziertes Fenster aktiv wird, bevor der Befehl die Zeit überschreitet.

Während der Befehl in einem Wartezustand ist, können neue Threads mittels Hotkey, benutzerdefinierte Menüeinträge oder Timer gestartet werden.

Wenn ein anderer Thread den Inhalt der Variable(n) verändert, die für die Parameter des Befehls verwenden wurden, dann sieht dieser Befehl die Veränderung nicht. Er verwendet weiterhin den Titel und Text, die original in den Variablen vorhandenen sind, wenn der Befehl wartet.

Fenstertitel und -texte sind von der Groß- und Kleinschreibung abhängig. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.

Verwandte Befehle

WinWait, WinWaitClose, IfWinExist, IfWinActive, SetTitleMatchMode, DetectHiddenWindows

Beispiel

Run, notepad.exe
WinWaitActive, Unbenannt - Editor, , 2
If ErrorLevel
{
    MsgBox, Zeit von WinWait überschritten.
    Return
}
Else
    WinMinimize  ; Minimiert das zuletzt gefundene Fenster.
