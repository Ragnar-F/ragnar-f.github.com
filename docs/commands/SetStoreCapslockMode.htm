SetStoreCapslockMode

Bestimmt, ob der Zustand der FESTSTELL-Taste nach einem Send-Befehl wiederhergestellt werden soll.

SetStoreCapslockMode, On|Off

Parameter

On|Off

On: Die Standardeinstellung für alle Scripts: Die FESTSTELL-Taste wird auf den vorherigen Zustand zurückgesetzt, falls Send diese Taste für die eigene Operation vorübergehend ändern musste.

Off: Der Zustand der FESTSTELL-Taste wird überhaupt nicht geändert. Dadurch wird Send die Groß- und Kleinschreibung der Zeichen umkehren, wenn die FESTSTELL-Taste während einer Operation aktiviert wird.

Bemerkungen

In Windows Me/98/95 ist dieser Befehl zurzeit augrund von Einschränkungen nicht hundertprozentig zuverlässig. Das heißt, dass die FESTSTELL-Taste bei Send oder ControlSend nicht immer ausgeschaltet wird. Selbst bei erfolgreicher Deaktivierung der FESTSTELL-Taste ist es möglich, dass sie nicht wieder zurückgesetzt wird, nachdem die Tasten gesendet wurden.

Dieser Befehl wird selten verwendet, da das Standardverhalten in den meisten Fällen genügt.

Jeder neu gestartete Thread (z. B. ein Hotkey, ein benutzerdefinierter Menüpunkt oder eine zeitlich festgelegte Subroutine) beginnt mit der Standardeinstellung des Befehls. Diese Standardeinstellung kann geändert werden, indem dieser Befehl im automatischen Ausführungsbereich (am Anfang des Scripts) verwendet wird.

Verwandte Befehle

SetCaps/Num/ScrollLockState, Send, ControlSend

Beispiel

SetStoreCapslockMode, off
