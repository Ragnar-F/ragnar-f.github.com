WinExist()

Überprüft, ob das angegebene Fenster existiert.

EindeutigeID := WinExist("Fenstertitel", "Fenstertext", "Titelausnahme", "Textausnahme")

Parameter

Fenstertitel

Der Titel oder Teiltitel des Fensters (das Übereinstimmungsverhalten wird mit SetTitleMatchMode bestimmt). Wenn dieser und die anderen 3 Fensterparameter nicht vorhanden sind, dann wird das zuletzt gefundene Fenster verwendet (z. B. wird das bestimmte Fenster überprüft, ob es immer noch existiert). Wenn dieser den Buchstaben A enthält und die anderen 3 Fensterparameter nicht vorhanden sind, dann wird das aktive Fenster verwendet. Bei Verwendung einer Fensterklasse muss ahk_class ExakterKlassenname angegeben werden (wird von Windows Spy angezeigt). Bei Verwendung einer Prozess-Kennung (PID) muss ahk_pid %VariableMitPID% angegeben werden. Bei Verwendung einer Fenstergruppe muss ahk_group Gruppenname angegeben werden. Bei Verwendung einer eindeutigen ID-Nummer eines Fensters muss ahk_id %VariableMitID% angegeben werden. Die Suche kann durch Angabe mehrerer Kriterien eingeschränkt werden. Zum Beispiel: Eigene Datei.txt ahk_class Notepad

Fenstertext

Wenn vorhanden, dann muss dieser Parameter einen Teil der einzelnen Textelemente des Fensters enthalten (wird von Windows Spy angezeigt). Versteckte Textelemente werden erfasst, wenn DetectHiddenText aktiv ist.

Titelausnahme

Fenstertitel mit diesem Wert werden nicht berücksichtigt. Hinweis: Aufgrund der Abwärtskompatibilität mit .aut-Scripts wird dieser Parameter als Befehl interpretiert, wenn eine exakte Übereinstimmung mit dem Befehlsnamen vorhanden ist. Als Übergangslösung kann die Funktion WinExist() benutzt werden.

Textausnahme

Fenstertexte mit diesem Wert werden nicht berücksichtigt.

Rückgabewert

Gibt die eindeutige ID (HWND) des ersten übereinstimmenden Fensters zurück (0, wenn nicht).

Da alle Zahlen außer 0 als "wahr" angesehen werden, ist die Anweisung If WinExist("Fenstertitel") immer wahr, sobald der Fenstertitel vorhanden ist.

Bemerkungen

Sobald alle Parameter weggelassen werden, wird das zuletzt gefundene Fenster überprüft, ob es noch vorhanden ist.

Wenn ein geeignetes Fenster vorhanden ist, wird das zuletzt gefundene Fenster auf dieses Fenster aktualisiert.

Um das HWND eines Steuerelements abzurufen (zur Verwendung mit Post/SendMessage oder DllCall), benutzt ControlGet Hwnd oder MouseGetPos.

Fenstertitel und -texte sind von der Groß- und Kleinschreibung abhängig. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.

Verwandte Befehle

WinActive(), SetTitleMatchMode, DetectHiddenWindows, zuletzt gefundenes Fenster, Process, WinActivate, WinWaitActive, WinWait, WinWaitClose, #IfWinActive/Exist

Beispiele

If WinExist("Unbenannt - Editor")
{
    WinActivate  ; Verwendet das oben gefundene Fenster.
    WinMaximize  ; dasselbe
    Send, Dies ist ein Text.{Enter}
    Return
}

If !WinExist("Rechner")
    Return
Else
{
    WinActivate  ; Das obere "IfWinNotExist" setzt das zuletzt gefundene Fenster für uns.
    WinMove, 40, 40  ; Zur neuen Position verschieben.
    Return
}

If WinExist("ahk_class Notepad") or WinExist("ahk_class" . ClassName)
    WinActivate  ; Verwendet das zuletzt gefundene Fenster.

MsgBox % "Die ID des aktiven Fensters ist " . WinExist("A")
