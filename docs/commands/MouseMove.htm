MouseMove

Bewegt den Mauszeiger.

MouseMove, X, Y [, Geschwindigkeit, R]

Parameter

X, Y Die X/Y-Koordinaten der neuen Mausposition (können Ausdrücke sein). Koordinaten beziehen sich auf das aktive Fenster, es sei denn, CoordMode wurde dazu verwendet, dass zu ändern.
Geschwindigkeit

Die Bewegungsgeschwindigkeit der Maus von 0 (am schnellsten) bis 100 (am langsamsten), das auch ein Ausdruck sein kann.  Hinweis: mit der Geschwindigkeit von 0 wird die Maus sofort bewegt.  Wenn weggelassen, dann wird die Standard-Geschwindigkeit von SetDefaultMouseSpeed (wenn nicht vorhanden, dann die 2) verwendet.

Geschwindigkeit wird bei der SendInput- oder SendPlay-Methode ignoriert: damit wird die Maus sofort bewegt (obwohl SetMouseDelay einen Modus hat, der für SendPlay gilt). Um die Maus sichtbar langsamer zu bewegen (z. B. für ein Script, dass eine Demonstration für das Publikum vorführt), verwendet SendEvent {Click 100, 200} oder SendMode Event (optional in Verbindung mit BlockInput).
R Wenn dieser Parameter den Buchstabe R enthält, dann werden Koordinaten X und Y als Offsets von der aktuellen Mausposition behandelt. In anderen Worten wird der Zeiger von seiner aktuellen Position um X Pixel nach rechts (nach links ist negativ) und um Y Pixel nach unten (nach oben ist negativ) verschoben.

Bemerkungen

Dieser Befehl verwendet die vordefinierte Methode von SendMode.

Mit dem SendPlay-Modus ist man in der Lage, Mausaktionen erfolgreich in einer breiteren Auswahl an Spielen durchzuführen. Darüber hinaus können einige Anwendungen und Spiele Probleme haben, die Maus richtig zu erkennen, wenn sie zu schnell bewegt wird. Der Geschwindigkeit-Parameter oder SetDefaultMouseSpeed kann verwendet werden, um die Geschwindigkeit zu reduzieren (nur im Standard-SendEvent-Modus).

Mit dem BlockInput-Befehl ist es möglich, Unterbrechungen von simulierten Mausaktionen durch physische Mausaktionen des Benutzers zu verhindern. Allerdings ist dieser Befehl im SendInput- oder SendPlay-Modus nicht notwendig, weil die physischen Mausaktionen des Benutzers automatisch bis zum Schluss zwischengespeichert werden.

Nach jeder Mausbewegung wird automatisch eine Verzögerung erzeugt (außer im SendInput-Modus). Die Dauer der Verzögerung kann mit SetMouseDelay geändert werden.

Das folgende Beispiel ist ein alternativer Weg, den Mauszeiger zu bewegen, dass bei bestimmten Multi-Monitor-Konfigurationen besser funktionieren könnte:

DllCall("SetCursorPos", int, 100, int, 400)  ; Die erste Nummer ist die X-Koordinate und die zweite Nummer die Y-Koordinate (bezogen auf dem Bildschirm).

Beachtet zudem, dass es möglich ist, den Mauszeiger vorübergehend zu verstecken, siehe dazu das DllCall-Beispiel.

Verwandte Befehle

CoordMode, SendMode, SetDefaultMouseSpeed, SetMouseDelay, Click, MouseClick, MouseClickDrag, MouseGetPos, BlockInput

Beispiel

; Bewegt die Maus auf eine neue Position:
MouseMove, 200, 100

; Bewegt die Maus von der aktuellen Position langsam (Geschwindigkeit 50 vs. 2)
; 20 Pixel nach rechts und 30 Pixel nach unten:
MouseMove, 20, 30, 50, R
