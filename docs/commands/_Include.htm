#Include / #IncludeAgain

Bewirkt, dass der Inhalt der angegebenen Datei exakt an dieser Position ausgeführt wird.

#Include DateiOderVerzeichnis
#Include <Bibliotheksname>
#IncludeAgain DateiOderVerzeichnis

Parameter

DateiOderVerzeichnis

Der Pfad einer Datei oder eines Verzeichnisses, wie unten beschrieben. Der Dateiname darf keine Anführungszeichen, Platzhalter oder Variablenverweise außer %A_ScriptDir%, %A_AppData%, %A_AppDataCommon% und (seit v1.1.11) %A_LineFile% enthalten. Escapesequenzen außer Semikolon (`;) dürfen weder verwendet werden, noch sind sie notwendig, da solche Zeichen (z. B. Prozentzeichen) direkt behandelt werden.

Datei: Name der Datei, die eingefügt werden soll und sich voraussichtlich im Start- oder Arbeitsverzeichnis befinden wird, solange kein absoluter Pfad angegeben ist (außer bei ahk2exe, das die Datei im eigenen Verzeichnis des Scripts vermutet). Hinweis: SetWorkingDir hat keine Auswirkung auf #Include, da die Direktive verarbeitet wird, bevor das Script ausgeführt wird.

Verzeichnis: Es ist möglich, ein Verzeichnis anstelle einer Datei anzugeben, um das Arbeitsverzeichnis zu ändern, welches für alle nachfolgende Vorkommnisse von #Include und FileInstall gilt. Hinweis: Das Ändern des Arbeitsverzeichnisses auf diese Weise hat keinen Einfluss auf das Arbeitsverzeichnis des Scripts, das am Anfang gesetzt wird (A_WorkingDir). Verwende dafür SetWorkingDir im oberen Bereich des Scripts.
Bibliotheksname [AHK_L 57+]: Bibliotheksdatei oder Funktionsname. Zum Beispiel würden sowohl #Include <Bibliothek> als auch #Include <Bibliothek_Funktion> die Datei Bibliothek.ahk aus einem der Ordner mit Funktionsbibliotheken einfügen.

Bemerkungen

Ein Script verhält sich so, als befände sich der Inhalt der eingefügten Datei exakt auf der Position von #Include (als ob der Inhalt der Datei kopiert und eingefügt wurde). Folglich können zwei isolierte Scripts in der Regel nicht zu einem funktionierenden Script zusammengefügt werden (um das zu erreichen, siehe www.autohotkey.com/forum/topic18545.html).

#Include stellt sicher, dass Dateiname nur einmal eingefügt wird, selbst wenn es mehrere Einfügungen findet. Im Gegensatz dazu ermöglicht #IncludeAgain mehrere Einfügungen von der gleichen Datei, ansonsten ist sie vergleichbar mit #Include.

Der Dateiname-Parameter kann optional mit *i und einem einzelnen Leerzeichen vorangestellt werden, wodurch das Programm jeden Fehler beim Lesen der eingefügten Datei ignoriert. Zum Beispiel: #Include *i Spezialoptionen.ahk. Diese Option sollte nur verwendet werden, wenn die Inhalte der eingefügten Datei nicht ausschlaggebend für die Operation des Hauptscripts sind.

Zeilen, die im Hauptfenster per ListLines oder dem Menü "View-Lines" angezeigt werden, sind immer nach ihrer physischen Reihenfolge innerhalb des eigenen Scripts nummeriert. Das heißt, dass das Einfügen einer neuen Datei die Zeilennummerierung des Hauptscripts um 1 erhöht, nämlich von der #Include-Zeile selbst (außer bei kompilierten Scripts, da eingefügte Dateien zu einem großen Script während der Kompilierung zusammengefügt werden).

#Include wird oft verwendet, um Funktionen aus einer externen Datei zu laden. Im Gegensatz zu Subroutinen können Funktionen ganz am Anfang des Scripts eingefügt werden, ohne dabei den automatischen Ausführungsbereich zu beeinflussen.

Wie bei anderen Direktiven kann #Include nicht bedingt ausgeführt werden. Das heißt, dass das folgende Beispiel nicht funktionieren wird:

If x = 1
    #Include BeliebigeDatei.ahk  ; Die Zeile ist wirksam, unabhängig vom X-Wert.
    y = 2  ; Und diese Zeile bezieht sich auf das obere IF, da Direktiven sich nicht auf IFs beziehen können.

Dateien können automatisch eingefügt werden (ohne dabei #Include zu verwenden), wenn eine Bibliotheksfunktion mit dem Namen aufgerufen wird.

[v1.1.11+]: Verwende %A_LineFile%\.., um auf das Verzeichnis zu verweisen, das die aktuelle Datei enthält, selbst wenn sie nicht die Hauptscript-Datei ist. Zum Beispiel #Include %A_LineFile%\..\script.ahk.

Verwandte Befehle

Funktionsbibliotheken, Funktionen, FileInstall

Beispiel

#Include C:\Eigene Dokumente\Scripts\Tool-Subroutinen.ahk
#Include %A_ScriptDir%  ; Ändert das Arbeitsverzeichnis aller nachfolgenden #Includes und FileInstalls.
#Include C:\Eigene Scripts  ; Wie oben, aber ein direkt angegebenes Verzeichnis.
