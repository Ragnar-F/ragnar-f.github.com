#Include / #IncludeAgain

Bewirkt, dass der Inhalt der angegebenen Datei exakt an dieser Position ausgeführt wird.

#Include DateiOderVerzeichnis
#Include <Bibliotheksname>
#IncludeAgain DateiOderVerzeichnis

Parameter

DateiOderVerzeichnis

Datei: Der Name der eingefügten Datei, die sich im Start- oder Arbeitsverzeichnis befindet, solange kein absoluter Pfad angegeben ist (außer bei ahk2exe, das die Datei im eigenen Verzeichnis des Scripts sucht). Der Dateiname darf keine Variablenverweise (außer %A_ScriptDir%, %A_AppData% und %A_AppDataCommon%), Anführungszeichen oder Platzhalter beinhalten. Schutzsequenzen außer Semikolon (`;) dürfen weder verwendet werden, noch sind sie notwendig, da solche Zeichen (z. B. Prozentzeichen) direkt behandelt werden. Hinweis: SetWorkingDir hat keine Auswirkung auf #Include, weil #Include verarbeitet wird, bevor das Script ausgeführt wird.

Verzeichnis: Verwendet ein Verzeichnis anstelle einer Datei, um das Arbeitsverzeichnis zu ändern, welches für alle nachfolgende Vorkommnisse von #Include und FileInstall gilt. Der Verzeichnisname darf keine Variablen beinhalten, außer %A_ScriptDir%, %A_AppData% und %A_AppDataCommon%. Hinweis: Wenn das Arbeitsverzeichnis auf diese Weise verändert wird, dann wird das anfängliche Arbeitsverzeichnis des Scripts nicht davon beeinflusst, wenn es ausgeführt wird (A_WorkingDir). Um das zu ändern, verwendet SetWorkingDir am Anfang des Scripts.
Bibliotheksname [AHK_L 57+]: Eine Bibliotheksdatei oder ein Funktionsname. Mit #include <Bibliothek> und #include <Bibliothek_Funktion> wird beispielsweise Bibliothek.ahk von einem Ordner mit Funktionsbibliotheken eingefügt.

Bemerkungen

Die #Include-Direktive bewirkt, dass sich der Inhalt der eingefügten Datei exakt an dieser Position befindet (als wäre die Datei kopiert und eingefügt). Folglich können zwei isolierte Scripts in der Regel nicht zu einem funktionierenden Script zusammengefügt werden (um das zu erreichen, siehe www.autohotkey.com/forum/topic18545.html).

#Include stellt sicher, dass der Dateiname nur einmal eingefügt wird, selbst wenn mehrere Einfügungen gefunden werden. #IncludeAgain ermöglicht hingegen mehrere Einfügungen derselben Datei, ansonsten ist sie vergleichbar mit #Include.

Der Dateiname-Parameter kann optional mit *i und einem Leerzeichen vorangestellt werden, damit das Programm jeden Fehler beim Laden der eingefügten Datei ignoriert. Zum Beispiel: #Include *i Spezialoptionen.ahk. Diese Option sollte nur verwendet werden, wenn die Inhalte der eingefügten Datei nicht unbedingt für die Operation des Hauptscripts erforderlich sind.

Zeilen, die im Hauptfenster mit ListLines oder mit dem Menü "View-Lines" angezeigt werden, sind immer nach ihrer physischen Reihenfolge innerhalb des eigenen Scripts nummeriert. Das heißt, dass das Einfügen einer neuen Datei die Zeilennummerierung des Hauptscripts um 1 erhöht, nämlich von der #Include-Zeile selbst (außer bei kompilierten Scripts, da eingefügte Dateien zu einem großen Script während der Kompilierung zusammenfügt werden).

#Include wird oft verwendet, um Funktionen einer externen Datei zu laden. Im Gegensatz zu Subroutinen können Funktionen ganz am Anfang des Scripts eingefügt werden, ohne dabei den automatischen Ausführungsbereich zu beeinflussen.

Wie bei anderen Direktiven kann #Include nicht bedingt ausgeführt werden. Das heißt, dass das folgende Beispiel nicht funktionieren wird:

If x = 1
    #Include BeliebigeDatei.ahk  ; Die Zeile ist wirksam, unabhängig vom X-Wert.
    y = 2  ; Und diese Zeile bezieht sich auf das obere IF, da Direktiven sich nicht auf IFs beziehen können.

Dateien können automatisch eingefügt werden (ohne dabei #Include zu verwenden), wenn eine Bibliotheksfunktion mit dem Namen aufgerufen wird.

Verwandte Befehle

Funktionsbibliotheken, Funktionen, FileInstall

Beispiel

#Include C:\Eigene Dokumente\Scripts\Tool-Subroutinen.ahk
#Include %A_ScriptDir%  ; Verändert das Arbeitsverzeichnis aller nachfolgenden #Includes und FileInstalls.
#Include C:\Eigene Scripts  ; Wie oben, aber für ein direkt angegebenes Verzeichnis.
