ComObjConnect() [AHK_L 53+]

Verbindet die Ereignisquellen eines COM-Objekts mit Funktionen, die einen vorgegeben Präfix enthalten.

ComObjConnect(ComObject [, Präfix])

Parameter

ComObject Ein Objekt, das Ereignisse auslöst.
Präfix

Ein Präfix für den Ereignisnamen, um festzustellen, welche Funktion aufgerufen werden soll, wenn ein Ereignis auftritt.

Wenn es weggelassen wird, dann wird das Objekt "getrennt", d. h. das Script empfängt nicht länger Benachrichtigungen von dessen Ereignissen.

[v1.1.01+]: Dieser Parameter kann ein definiertes Objekt vom Script sein. Beim Auslösen eines Ereignisses wird die entsprechende Methode aufgerufen. Der erste Parameter, der für gewöhnlich der versteckte this-Parameter ist, bezieht sich auf das script-definierte Objekt und nicht auf das COM-Objekt. Um jedes Ereignis ohne Methodendefinition abzufangen, definiert eine __Call-Meta-Funktion.

Verwendung

Zur effektiven Nutzung von ComObjConnect müssen zuerst Funktionen in das Script geschrieben werden, um relevante Ereignisse zu bearbeiten. Solche Funktionen, auch "Event-Handler" genannt, haben die folgende Struktur:

PräfixEreignisname([Parameter..., ComObject])
{
    ... Code zur Bearbeitung von Ereignissen ...
    Return Rückgabewert
}

Für Präfix kann ein beliebiger Präfix angegeben werden, während der Ereignisname der Name des Ereignisses ist, das von der Funktion bearbeitet werden soll.

Parameter entspricht den Parametern, die das Ereignis hat. Wenn das Ereignis keine Parameter hat, dann sollten die Parameter ganz weggelassen werden. ComObject ist optional und kann nur verwendet werden, wenn die korrekte Anzahl an Parameter definiert ist. Es enthält einen Verweis auf das originale Wrapper-Objekt, das an ComObjConnect übergeben wurde. "ComObject" sollte mit einem Namen ersetzt werden, der eher dem Kontext des Scripts entspricht.

Beachtet, dass die Event-Handler möglicherweise Rückgabewerte haben. Damit ein COM-spezifischer Werttyp zurückgegeben wird, benutzt ComObjParameter, das auch als ComObjRetVal(Typ, Wert) oder einfach als ComObj(Typ, Wert) geschrieben werden kann.

Um die Bearbeitung von Ereignissen zu aktivieren, verwendet ComObjConnect(BeliebigesObjekt, "Präfix").

Um das Objekt zu trennen (Ereignisbearbeitung anhalten), verwendet ComObjConnect(BeliebigesObjekt).

Wenn die Anzahl der Parameter unbekannt ist, dann kann eine variadische Funktion verwendet werden.

Verwandte Befehle

ComObjCreate, ComObjGet, ComObjActive, ComObjError, WScript.ConnectObject (MSDN)

Beispiele

ie := ComObjCreate("InternetExplorer.Application")

; Verbindet Ereignisse mit entsprechenden Script-Funktionen, die den Präfix "IE_" haben.
ComObjConnect(ie, "IE_")

ie.Visible := true  ; Funktioniert bekanntlich nicht mit IE7.
ie.Navigate("http://l.autohotkey.net/")
#Persistent

IE_DocumentComplete(ieEventParam, url, ieFinalParam) {
    global ie
    If (ie != ieEventParam)
        s .= "Erster Parameter ist ein neues Wrapper-Objekt.`n"
    If (ie == ieFinalParam)
        s .= "Letzter Parameter ist das originale Wrapper-Objekt.`n"
    If ((disp1:=ComObjUnwrap(ieEventParam)) == (disp2:=ComObjUnwrap(ieFinalParam)))
        s .= "Beide Wrapper-Objekte beziehen sich auf die gleiche IDispatch-Instanz.`n"
    ObjRelease(disp1), ObjRelease(disp2)
    MsgBox % s . "Das Laden von " ie.Document.title " @ " url " wurde abgeschlossen."
    ie.Quit()
    ExitApp
}
