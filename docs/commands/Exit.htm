Exit

Beendet den aktuellen Thread oder (wenn das Script nicht persistent ist und keine Hotkeys enthält) das gesamte Script.

Exit [, ExitCode]

Parameter

ExitCode Ein Integer-Wert zwischen -2147483648 und 2147483647 (kann ein Ausdruck sein), welcher zurückgegeben wird, wenn das Script beendet wird. Dieser Code ist für jedes Programm zugänglich, welches das Script startet, wie beispielsweise durch ein anderes Script (mittels RunWait) oder einer Batch-Datei (.bat). Wenn nicht vorhanden, dann enthält ExitCode standardmäßig eine Null. Eine Null wird normalerweise verwendet, um Erfolg zu zeigen. Hinweis: In Windows 95 kann es sein, dass die Größe von ExitCode eingeschränkt ist.

Bemerkungen

Ein Script wird sofort mit Exit beendet, solange keine Hotkeys aktiv sind oder die Befehle #Persistent, SetCapsLockState, SetNumLockState oder SetScrollLockState nicht verwendet werden (außer mit einer OnExit-Subroutine).

Ansonsten wird der aktuelle Thread mit dem Exit-Befehl beendet. Mit anderen Worten wird ein Stapel mehrerer Subroutinen, die direkt oder indirekt von einem Menu, Timer oder Hotkey aufgerufen werden, sofort beendet, als wurde jeweils ein Return in den Subroutinen ausgeführt. Exit hat die gleiche Bedeutung wie Return, wenn der Befehl direkt in einer Subroutine verwendet wird.

Mit ExitApp kann ein Script vollständig beendet werden, dass persistent ist oder Hotkeys enthält.

Verwandte Befehle

ExitApp, OnExit, Funktionen, Gosub, Return, Threads, #Persistent

Beispiel

#z::
Gosub, Sub2
MsgBox, Diese MsgBox erscheint wegen Exit nicht.
Return

Sub2:
Exit  ; Beendet diese Subroutine, sowie die aufrufende Subroutine.
