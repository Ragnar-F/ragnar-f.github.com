Exit

Beendet den aktuellen Thread oder (wenn das Script nicht persistent ist und keine Hotkeys enthält) das gesamte Script.

Exit [, ExitCode]

Parameter

ExitCode Ein Integer-Wert zwischen -2147483648 und 2147483647 (kann ein Ausdruck sein), welcher zurückgegeben wird, wenn das Script beendet wird. Dieser Code ist für jedes Programm zugänglich, welches das Script startet, wie beispielsweise durch ein anderes Script (mittels RunWait) oder einer Batch-Datei (.bat). Sobald dieser Parameter weggelassen wird, enthält er standardmäßig eine 0. Eine Null wird normalerweise verwendet, um Erfolg zu zeigen. Hinweis: In Windows 95 könnte ExitCode auf eine maximale Größe beschränkt sein.

Bemerkungen

Bei einem Exit wird das Script sofort beendet, solange keine Hotkeys vorhanden sind, das Script nicht persistent ist und nicht der AlwaysOn- oder AlwaysOff-Status für die Num-/Scroll-/Feststelltaste angefordert wurde (es sei denn, es hat eine OnExit-Subroutine).

Ansonsten wird der aktuelle Thread beendet. Das heißt, dass die Subroutinen, die direkt oder indirekt durch ein Menü, Timer oder Hotkey aufgerufen wurden, sofort mit einem Return beendet werden. Sobald dieser Befehl direkt in so einer Subroutine verwendet wird (anstatt in einer Subroutine, die indirekt von ihr aufgerufen wurde), hat Exit die gleiche Bedeutung wie Return.

Mit ExitApp kann ein Script auch vollständig beendet werden, wenn es persistent ist oder Hotkeys enthält.

Verwandte Befehle

ExitApp, OnExit, Funktionen, Gosub, Return, Threads, #Persistent

Beispiel

#z::
Gosub, Sub2
MsgBox, Diese MsgBox erscheint wegen Exit nicht.
Return

Sub2:
Exit  ; Beendet diese Subroutine, sowie auch die aufrufende Subroutine.
