ControlGet

Ruft verschiedene Informationen zu einem Steuerelement ab.

ControlGet, Ausgabevariable, Befehl [, Wert, Steuerelement, Fenstertitel, Fenstertext, Titelausnahme, Textausnahme]

Parameter

Ausgabevariable Der Name der Variable, in der der Rückgabewert vom Befehl gespeichert wird.
Befehl, Wert Siehe untere Liste.
Steuerelement

Möglich ist entweder ClassNN (Klassenname und Instanznummer des Steuerelements) oder Name/Text des Steuerelements, die beide mit Windows Spy ermittelt werden können. Beim Name/Text wird das Übereinstimmungsverhalten mit SetTitleMatchMode bestimmt. Wenn dieser Parameter leer ist, wird das oberste Steuerelement des Fensters benutzt.

Um das HWND (Windows Handle) eines Steuerelements geltend zu machen, muss dieser Parameter leer gelassen und ahk_id %SteuerelementHwnd% im Fenstertitel-Parameter angegeben werden (funktioniert auch bei versteckten Steuerelementen, selbst wenn DetectHiddenWindows deaktiviert ist). Das HWND eines Steuerelements wird üblicherweise mittels ControlGet Hwnd, MouseGetPos oder DllCall abgerufen.
Fenstertitel Der Titel oder Teiltitel des Fensters (das Übereinstimmungsverhalten wird mit SetTitleMatchMode bestimmt). Wenn dieser und die nächsten 3 Parameter weggelassen werden, wird das zuletzt gefundene Fenster verwendet. Sobald dieser den Buchstaben A enthält und die nächsten 3 Parameter weggelassen werden, wird das aktive Fenster verwendet. Bei Verwendung einer Fensterklasse muss ahk_class ExakterKlassenname angegeben werden (wird von Windows Spy angezeigt). Bei Verwendung einer Prozess-Kennung (PID) muss ahk_pid %VariableMitPID% angegeben werden. Bei Verwendung einer Fenstergruppe muss ahk_group Gruppenname angegeben werden. Bei Verwendung einer eindeutigen ID-Nummer eines Fensters muss ahk_id %VariableMitID% angegeben werden. Die Suche kann durch Angabe mehrerer Kriterien eingeschränkt werden. Zum Beispiel: Eigene Datei.txt ahk_class Notepad
Fenstertext In diesem Parameter kann ein Teilstring angegeben werden, der in einem Textelement des Zielfensters vorkommt (werden mit Windows Spy angezeigt). Versteckte Textelemente werden auch erkannt, sofern DetectHiddenText aktiv ist.
Titelausnahme Ignoriert Fenster, deren Titel diesen Wert enthalten.
Textausnahme Ignoriert Fenster, deren Textelemente diesen Wert enthalten.

Befehl, Wert

Die Parameter Befehl und Wert sind voneinander abhängig und können wie folgt verwendet werden.

List: Ermittelt eine Liste mit Einträgen von ListView, ListBox, ComboBox, oder DropDownList.

ListView: Das bekannteste Beispiel für ListView ist die Liste mit Dateien und Ordnern vom Explorer (der Desktop ist auch ein ListView). Die Syntax für eine ListView-Abfrage ist:

ControlGet, Ausgabevariable, List, Optionen, SysListView321, Fenstertitel, Fenstertext

Ist der Optionen-Parameter leer oder nicht vorhanden, wird der ganze Text vom Steuerelement abgerufen. Jede Reihe, außer die letzte, endet mit einem Zeilenumbruch (`n). Innerhalb jeder Reihe endet jedes Feld (Spalte), außer das letzte, mit einem Tabulatorzeichen (`t).

Die Optionen können entweder leer sein oder mehrere folgende Wörter enthalten, getrennt durch Leer- oder Tabulatorzeichen:

Selected: Ermittelt nur die markierten (hervorgehobenen) Reihen. Falls das nicht zutrifft, wird die Ausgabevariable leer gemacht.
Focused: Ermittelt nur die fokussierte Reihe. Falls das nicht zutrifft, wird die Ausgabevariable leer gemacht.
ColN: Ermittelt nur die N-te Spalte (Feld).
Count: Ermittelt die Gesamtanzahl der Reihen im Steuerelement.
Count Selected: Ermittelt die Gesamtanzahl der markierten (hervorgehobenen) Reihen.
Count Focused: Ermittelt die Reihennummer (Position) der fokussierten Reihe (0, wenn nicht).
Count Col: Ermittelt die Spaltenanzahl des Steuerelements (oder -1, falls die Anzahl nicht ermittelt werden kann).

HINWEIS: Einige Anwendungen speichern ihren ListView-Text vertraulich. Folglich kann der Text nicht abgerufen werden. In diesem Fall wird ErrorLevel üblicherweise auf 0 gesetzt (signalisiert Erfolg) und alle abgerufenen Felder leer gemacht. Beachtet außerdem, dass #MaxMem keinen Einfluss darauf hat, wie viel Text aus einem ListView abgerufen werden kann.

Bei Erfolg wird ErrorLevel auf 0 gesetzt. Bei Misserfolg wird ErrorLevel auf 1 gesetzt und die Ausgabevariable leer gemacht. Fehler treten auf, wenn:

  1. das Fenster oder Steuerelement nicht existiert
  2. das Steuerelement nicht vom Typ SysListView32 ist
  3. der Prozess mit dem ListView nicht geöffnet werden konnte, da er gesperrt ist oder Benutzerrechte fehlen
  4. die ColN-Option auf eine nicht vorhandene Spalte verweist

Um die Reihen und Felder aus einem ListView einzeln zu extrahieren, verwendet eine Parsing-Schleife:

ControlGet, Liste, List, Selected, SysListView321, Fenstertitel
Loop, Parse, Liste, `n  ; Reihen sind durch Zeilenvorschübe (`n) abgegrenzt.
{
    Reihennummer := A_Index
    Loop, Parse, A_LoopField, %A_Tab%  ; Felder (Spalten) sind in jeder Reihe durch Tabulatoren (A_Tab) abgegrenzt.
        MsgBox Reihe #%Reihennummer% Spalte #%A_Index% enthält %A_LoopField%.
}

Beachtet zudem, dass in einer ListView die Spaltengröße mit SendMessage geändert werden kann:

SendMessage, 4126, 0, 80, SysListView321, Fenstertitel  ; 4126 ist die Nachricht LVM_SETCOLUMNWIDTH.
; Die 0 in der oberen Zeile kennzeichnet die erste Spalte (1 die Zweite, 2 die Dritte, etc.) Ebenso ist 80 die neue Breite.
; Ersetzt 80 mit -1, um die Spaltenbreite automatisch anzupassen. Ersetzt 80 mit -2, um die Spaltenbreite unter Berücksichtigung der Spaltenkopfbreite automatisch anzupassen.

ListBox, ComboBox, DropDownList: Der gesamte Text wird vom Steuerelement ermittelt (das heißt, dass die oben genannten ListView-Optionen wie Count und Selected nicht unterstützt werden).

Jede Reihe, außer die letzte, endet mit einem Zeilenumbruch (`n). Um die Einträge einzeln abzurufen, verwendet eine Parsing-Schleife:

ControlGet, List, List,, ComboBox1, Fenstertitel
Loop, Parse, List, `n
    MsgBox Item number %A_Index% is %A_LoopField%.

Checked: Setzt die Ausgabevariable auf 1, wenn ein Kontrollkästchen oder Optionsfeld markiert ist, andernfalls 0.

Enabled: Setzt die Ausgabevariable auf 1, wenn das Steuerelement aktiv ist, andernfalls 0.

Visible: Setzt die Ausgabevariable auf 1, wenn das Steuerelement sichtbar ist, andernfalls 0.

Tab: Die Ausgabevariable beinhaltet die Tab-Nummer eines SysTabControl32-Steuerelements. Beachtet, dass der erste Tab mit der Positionsnummer 0 beginnt. Um zu erfahren, wie viele Tabs (Seiten) ein Tab-Steuerelement hat, siehe folgendes Beispiel:

SendMessage, 0x1304,,, SysTabControl321, Fenstertitel  ; 0x1304 ist TCM_GETITEMCOUNT.
Tabanzahl = %ErrorLevel%

FindString, String: Die Ausgabevariable beinhaltet die Eintragsnummer eines ListBox- oder ComboBox-Steuerelements, die mit String genau übereinstimmt. Der erste Eintrag im Steuerelement ist die 1, der zweite die 2 und so weiter. Wenn keine Übereinstimmung gefunden wurde, dann wird die Ausgabevariable leer gemacht und ErrorLevel auf 1 gesetzt.

Choice: Die Ausgabevariable beinhaltet den Namen des derzeit ausgewählten Eintrags in einem ListBox- oder ComboBox-Steuerelement. Um stattdessen die Positionsnummer des ausgewählten Eintrags zu ermitteln, siehe folgendes Beispiel (verwendet aber nur eine der beiden Zeilen):

SendMessage, 0x188, 0, 0, ListBox1, Fenstertitel  ; 0x188 ist LB_GETCURSEL (für eine ListBox).
SendMessage, 0x147, 0, 0, ComboBox1, Fenstertitel  ; 0x147 ist CB_GETCURSEL (für eine DropDownList oder ComboBox).
ChoicePos = %ErrorLevel%  ; Ergebnis lautet -1, wenn kein Eintrag ausgewählt ist.
ChoicePos += 1  ; Wandelt 0 in 1 um, damit der erste Eintrag mit 1 beginnt, nicht 0.

LineCount: Die Ausgabevariable beinhaltet die Zeilenanzahl eines Eingabefeldes. Alle Eingabefelder haben mindestens eine Zeile, selbst wenn das Steuerelement leer ist.

CurrentLine: Die Ausgabevariable beinhaltet die Zeilennummer eines Eingabefeldes, auf der sich der Text-Cursor befindet. Die erste Zeile beginnt bei 1. Falls Text im Steuerelement markiert ist, wird die Ausgabevariable auf die erste markierte Zeilennummer gesetzt.

CurrentCol: Die Ausgabevariable beinhaltet die Spaltennummer eines Eingabefeldes, auf der sich der Text-Cursor befindet. Die erste Spalte beginnt bei 1. Falls Text im Steuerelement markiert ist, wird die Ausgabevariable auf die erste markierte Spaltennummer gesetzt.

Line, N: Die Ausgabevariable beinhaltet den Text, der sich auf der N-ten Zeile in einem Eingabefeld befindet. Zeile 1 ist die erste Zeile. Je nach Beschaffenheit des Steuerelements kann die Ausgabevariable mit einem CR (`r) oder einem CR + LF (`r`n) enden. Wenn die angegebene Zeilennummer leer oder nicht vorhanden ist, wird ErrorLevel auf 1 gesetzt und die Ausgabevariable leer gemacht.

Selected: Die Ausgabevariable beinhaltet den markierten Text eines Eingabefeldes. Wenn kein Text markiert ist, wird die Ausgabevariable leer gemacht und ErrorLevel auf 0 gesetzt (signalisiert Erfolg). Es ist möglich, dass bestimmte Steuerelementarten wie RichEdit20A den Text nicht richtig darstellen (z. B. Metapad).

Style: Ermittelt den Style des Steuerelements als 8-stellige Hexadezimalzahl. Die Style-Tabelle beinhaltet eine Auflistung einiger Styles.

ExStyle: Ermittelt den erweiterten Style als 8-stellige Hexadezimalzahl.

Hwnd [v1.0.43.06+]: Ermittelt das HWND (Window Handle) des angegebenen Steuerelements. Zum Beispiel: ControlGet, Ausgabevariable, Hwnd,, Edit1, Fenstertitel. Das HWND eines Steuerelements wird oft bei PostMessage, SendMessage und DllCall verwendet. Beachtet zudem, dass das HWND eines Steuerelements auch mit MouseGetPos abgerufen werden kann. Dieses HWND kann im Fenstertitel-Parameter direkt mit ahk_id verwendet werden (das selbst bei versteckten Steuerelementen funktioniert, auch wenn DetectHiddenWindows deaktiviert ist).

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

Bei Erfolg wird ErrorLevel auf 0 gesetzt. Bei Misserfolg (wenn das Fenster oder Steuerelement beispielsweise nicht vorhanden ist), wird ErrorLevel auf 1 gesetzt und die Ausgabevariable leer gemacht.

Bemerkungen

Im Gegensatz zu Befehlen, die ein Steuerelement ändern, hat ControlGet keine automatische Verzögerung (d. h. ControlGet wird nicht von SetControlDelay beeinflusst).

Zur Erfassung des Steuerelementnamens unter dem Mauszeiger wird MouseGetPos verwendet. Um eine Liste mit Steuerelementen von einem Fenster zu erhalten, verwendet WinGet ControlList.

Fenstertitel und -texte sind von der Groß- und Kleinschreibung abhängig. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.

Verwandte Befehle

Control, GuiControlGet, ControlMove, ControlGetText, ControlSetText, ControlGetPos, ControlClick, ControlFocus, ControlSend, WinGet

Beispiel

ControlGet, Ausgabevariable, Line, 1, Edit1, Beliebiger Fenstertitel

ControlGet, WhichTab, Tab, , SysTabControl321, Beliebiger Fenstertitel
If ErrorLevel
    MsgBox Es wurde ein Problem festgestellt.
Else
    MsgBox Tab #%WhichTab% ist aktiv.
