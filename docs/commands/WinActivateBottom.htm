WinActivateBottom

Das Gleiche wie WinActivate, nur mit der Ausnahme, dass das unterste (am wenigsten verwendete) übereinstimmende Fenster anstelle des obersten Fensters aktiviert wird.

WinActivateBottom [, Fenstertitel, Fenstertext, Titelausnahme, Textausnahme]

Parameter

Fenstertitel Der Titel oder Teiltitel des Fensters (das Übereinstimmungsverhalten wird mit SetTitleMatchMode bestimmt). Bei Verwendung einer Fensterklasse muss ahk_class ExakterKlassenname angegeben werden (wird von Windows Spy angezeigt). Bei Verwendung einer Prozess-Kennung (PID) muss ahk_pid %VariableMitPID% angegeben werden. Bei Verwendung einer Fenstergruppe muss ahk_group Gruppenname angegeben werden. Bei Verwendung einer eindeutigen ID-Nummer eines Fensters muss ahk_id %VariableMitID% angegeben werden. Die Suche kann durch Angabe mehrerer Kriterien eingeschränkt werden. Zum Beispiel: Eigene Datei.txt ahk_class Notepad
Fenstertext Wenn vorhanden, dann muss dieser Parameter einen Teil der einzelnen Textelemente des Fensters enthalten (wird von Windows Spy angezeigt). Versteckte Textelemente werden erfasst, wenn DetectHiddenText aktiv ist.
Titelausnahme Fenstertitel mit diesem Wert werden nicht berücksichtigt.
Textausnahme Fenstertexte mit diesem Wert werden nicht berücksichtigt.

Bemerkungen

Wenn nur ein Fenster übereinstimmt, dann verhält sich WinActivateBottom identisch zu WinActivate.

Fenstergruppen sind nützlicher als dieser Befehl, weil sie mehr Funktionen und Flexibilität bieten.

Wenn das Fenster minimiert ist, dann wird es automatisch wiederhergestellt, bevor es aktiviert wird.

Sechs Versuche werden innerhalb von 60ms unternommen, um das Zielfenster zu aktivieren. Daher ist es normalerweise nicht notwendig, WinWaitActive darauf folgend auszuführen.

Im Gegensatz zu WinActivate kann das zuletzt gefundene Fenster nicht verwendet werden, weil es möglicherweise nicht das unterste Fenster ist. Deshalb darf mindestens ein Parameter nicht leer gelassen werden.

Wenn ein Fenster aktiviert wird, nachdem ein anderes Fenster aktiviert wurde, dann fangen die Schaltflächen der Taskleiste an zu blinken (abhängig vom Betriebssystem und von den Einstellungen). Mit #WinActivateForce kann das verhindert werden.

Fenstertitel und -texte sind von der Groß- und Kleinschreibung abhängig. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.

Verwandte Befehle

WinActivate, #WinActivateForce, SetTitleMatchMode, DetectHiddenWindows, IfWinExist, IfWinActive, WinWaitActive, WinWait, WinWaitClose, GroupActivate

Beispiel

; Mit diesem Hotkey werden alle offenen Browser-Fenster von alt zu neu besucht:
#i::
SetTitleMatchMode, 2
WinActivateBottom, - Microsoft Internet Explorer
