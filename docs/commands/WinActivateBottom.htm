WinActivateBottom

Das gleiche wie WinActivate, außer dass das unterste (am wenigsten verwendete) übereinstimmende Fenster aktiviert wird, anstatt das oberste.

WinActivateBottom [, Fenstertitel, Fenstertext, Titelausnahme, Textausnahme]

Parameter

Fenstertitel Der Titel oder Teiltitel des Fensters (das Übereinstimmungsverhalten wird mit SetTitleMatchMode bestimmt). Bei Verwendung einer Fensterklasse muss ahk_class ExakterKlassenname angegeben werden (wird von Windows Spy angezeigt). Bei Verwendung einer Prozess-Kennung (PID) muss ahk_pid %VariableMitPID% angegeben werden. Bei Verwendung einer Fenstergruppe muss ahk_group Gruppenname angegeben werden. Bei Verwendung einer eindeutigen ID-Nummer eines Fensters muss ahk_id %VariableMitID% angegeben werden. Die Suche kann durch Angabe mehrerer Kriterien eingeschränkt werden. Zum Beispiel: Eigene Datei.txt ahk_class Notepad
Fenstertext In diesem Parameter kann ein Teilstring angegeben werden, der in einem Textelement des Zielfensters vorkommt (werden mit Windows Spy angezeigt). Versteckte Textelemente werden auch erkannt, sofern DetectHiddenText aktiv ist.
Titelausnahme Ignoriert Fenster, deren Titel diesen Wert enthalten.
Textausnahme Ignoriert Fenster, deren Textelemente diesen Wert enthalten.

Bemerkungen

Falls nur ein Fenster übereinstimmt, verhält sich WinActivateBottom genauso wie WinActivate.

Fenstergruppen sind aufgrund von mehr Features und Flexibilität fortschrittlicher als dieser Befehl, daher sind diese eher in Betracht zu ziehen.

Falls das Fenster minimiert ist, wird es vor dem Aktivieren automatisch wiederhergestellt.

Es werden 6 Versuche innerhalb von 60 ms unternommen, um das Zielfenster zu aktivieren. Daher ist es üblicherweise nicht notwendig, nach diesem Befehl WinWaitActive auszuführen.

Im Gegensatz zu WinActivate kann hier das zuletzt gefundene Fenster nicht verwendet werden, weil es möglicherweise nicht das unterste Fenster ist. Deshalb muss mindestens einer der Parameter nicht leer sein.

Wenn ein Fenster unmittelbar nach Aktivieren eines anderen Fensters aktiviert wird, kann es passieren, dass die Schaltflächen der Taskleiste anfangen zu blinken (abhängig vom Betriebssystem und von den Einstellungen). Mit #WinActivateForce kann das verhindert werden.

Fenstertitel und -texte unterscheiden zwischen Groß- und Kleinschreibung. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.

Verwandte Befehle

WinActivate, #WinActivateForce, SetTitleMatchMode, DetectHiddenWindows, IfWinExist, IfWinActive, WinWaitActive, WinWait, WinWaitClose, GroupActivate

Beispiel

; Dieser Hotkey erlaubt dir, alle offenen Browser-Fenster durchzugehen, beginnend mit dem ältesten Fenster:
#i::
SetTitleMatchMode, 2
WinActivateBottom, - Microsoft Internet Explorer
