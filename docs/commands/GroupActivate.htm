GroupActivate

Aktiviert das nächste Fenster in einer Fenstergruppe, die mit GroupAdd definiert worden ist.

GroupActivate, Gruppenname [, R]

Parameter

Gruppenname Name der zu aktivierenden Gruppe, die ursprünglich mit GroupAdd definiert worden ist.
R Legt fest, ob das älteste oder das neueste Fenster aktiviert werden soll, wenn kein einziges Fenster aus der Gruppe aktiv ist. Ist dieser Parameter nicht vorhanden, wird immer das älteste Fenster aktiviert. Ist der Buchstabe R angegeben, wird das neueste (zuletzt aktive) Fenster aktiviert, aber nur, wenn beim Ausführen des Befehls kein Fenster aus der Gruppe aktiv ist. "R" ist beispielsweise nützlich, wenn der Benutzer vorübergehend eine andere Aufgabe durchführen will. Kehrt er dann mit GroupActivate, GroupDeactivate oder GroupClose zur Gruppe zurück, wird das am aktuellsten genutzte Fenster aktiviert.

Bemerkungen

Dieser Befehl bewirkt, dass das erste Fenster, das mit den Spezifikationen der Fenstergruppe übereinstimmt, aktiviert wird. Verwendet man den Befehl danach nochmals, wird das nächste Fenster in der Reihe aktiviert und so weiter. Normalerweise weist man diesen Befehl einem Hotkey zu, so dass diese Fensterdurchschaltung durch Drücken dieser Taste automatisiert werden kann.

Aktiviert man ein Fenster unmittelbar nach Aktivierung eines anderen Fensters, fangen die Schaltflächen der Taskleiste an zu blinken (abhängig vom Betriebssystem und von den Einstellungen). Mit #WinActivateForce kann das verhindert werden.

Siehe GroupAdd für mehr Details über Fenstergruppen.

ErrorLevel [AHK_L 54+]

ErrorLevel wird auf 1 gesetzt, wenn kein Fenster zum Aktivieren gefunden werden kann, ansonsten enthält ErrorLevel immer eine 0.

Verwandte Befehle

GroupAdd, GroupDeactivate, GroupClose, #WinActivateForce

Beispiel

GroupActivate, EigeneGruppe, R
