GUI

Erstellt und verwaltet Fenster und deren Steuerelemente. Solche Fenster können zur Dateneingabe oder als individuelle Benutzerschnittstellen verwendet werden.

Gui, Unterbefehl [, Param2, Param3, Param4]

Inhaltsverzeichnis

  • New: Erstellt ein neues Fenster.
  • Add: Erstellt ein Steuerelement wie z. B. Text, Schaltfläche oder Kontrollkästchen.
  • Show: Zeigt das Fenster an. Kann ein Fenster auch minimieren, maximieren oder verschieben.
  • Submit: Speichert die Benutzereingaben und versteckt optional das Fenster.
  • Cancel (oder Hide): Versteckt das Fenster.
  • Destroy: Entfernt das Fenster.
  • Font: Bestimmt die Schriftart, die -Schriftgröße, den Schriftstil und die Textfarbe für danach erstellte Steuerelemente.
  • Color: Bestimmt die Hintergrundfarbe des Fensters und/oder dessen Steuerelemente.
  • Margin: Bestimmt den Abstand, der immer dann verwendet wird, wenn keine eindeutige Position für ein Steuerelement definiert ist.
  • Optionen und Styles für ein Fenster: Legt verschiedene Optionen zum Aussehen und Verhalten des Fensters fest.
  • Menu: Hinzufügen oder Entfernen einer Menüleiste.
  • Minimize / Maximize / Restore: Minimiert, Maximiert oder stellt ein Fenster wieder her.
  • Flash: Führt dazu, dass die Taskleistenschaltfläche des Fensters anfängt zu blinken.
  • Default: Ändert den Standard-GUI-Fensternamen des aktuellen Threads.
  • Position und Größe von Steuerelementen anpassen
  • Speichern von und Reagieren auf Benutzereingaben: Variablen und g-Label
  • Optionen und Styles für Steuerelemente
  • Fensterereignisse: GuiClose | GuiEscape | GuiSize | GuiContextMenu | GuiDropFiles
  • Mehrere GUI-Fenster erstellen
  • GUI-Ereignisse, Threads und Subroutinen
  • Sonstiges: Navigation mit der Tastatur | Fensterdarstellung | Allgemeine Bemerkungen
  • Beispiele: Enthält funktionierende Scripts, die GUI-Fenster und Steuerelemente zeigen.

Gui, New [, Optionen, Titel] [v1.1.04]

Erstellt ein neues Fenster.

Gui, New  ; Erstellt ein neues unbenanntes GUI.
Gui, Name:New  ; Erstellt ein neues GUI und zerstört jedes vorhandene GUI mit diesem Namen.

Zur Vereinfachung wird das neue Fenster als Standard für den aktuellen Thread definiert.

Wenn kein Name angegeben ist, haben spezielle Label wie GuiClose standardmäßig das Präfix "Gui", sofern das nicht in den Optionen mit +LabelPräfix überschrieben wird. Wenn das GUI, das den aktuellen Thread gestartet hat, auf diese Weise erstellt wurde, enthält A_Gui ein HWND, anstatt einen Namen. Desweiteren kann das HWND des neuen Fensters in eine Ausgabevariable gespeichert werden, sobald +HwndAusgabevariable in den Optionen verwendet wird.

Falls ein Name angegeben ist, wird jedes vorhandene GUI mit diesem Namen zerstört.

Der Optionen-Parameter kann Optionen enthalten, die vom Haupt-GUI-Befehl unterstützt werden.

Falls der Titel-Parameter nicht vorhanden ist, wird der Dateiname des Scripts verwendet.

Obwohl dieser Befehl dazu dient, Klarheit zu verschaffen, muss er nicht zwingend benutzt werden. Wenn einer der anderen Gui-Unterbefehle (z. B. Gui, Add) verwendet wird und das GUI nicht vorhanden ist, wird es automatisch erstellt.

Hinweis: Vor v1.1.08 war dieser Befehl nicht in der Lage, das Standard-Gui zu setzen, falls ein Name angegeben war.

Gui, Add, Steuerelementart [, Optionen, Text]

Fügt ein Steuerelement in einem GUI-Fenster hinzu (falls notwendig, wird das GUI-Fenster erst erstellt).

Folgende Steuerelementart kann verwendet werden:

  • Text, Edit, UpDown, Picture
  • Button, Checkbox, Radio
  • DropDownList, ComboBox
  • ListBox, ListView, TreeView
  • Link, Hotkey, DateTime
  • MonthCal, Slider, Progress
  • GroupBox, Tab, StatusBar
  • ActiveX (z. B. Internet Explorer Control)
  • Custom

Zum Beispiel:

Gui, Add, Text,, Bitte Name eingeben:
Gui, Add, Edit, vName
Gui, Show

Gui, Show [, Optionen, Titel]

Sofern nicht anders in Optionen angegeben, wird dieser Befehl das Fenster sichtbar machen, dessen Minimierung aufheben (falls notwendig), aktivieren und dessen Titel festlegen. Wenn Titel nicht angegeben ist, wird der vorherige Titel weiterverwendet (ist keiner vorhanden, dann der Dateiname des Scripts).

Du kannst die unten genannten Optionen X, Y, W und H weglassen, damit das Fenster seine vorherige Größe und Position beibehält. Falls keine vorherige Position vorhanden ist, wird das Fenster automatisch auf eine der beiden Abmessungen zentriert, wenn die unten erwähnten Optionen X und/oder Y fehlen. Falls eine vorherige Größe nicht wiederhergestellt werden kann, wird die Fenstergröße automatisch an den vorhandenen Steuerelementen angepasst.

Folgende Strings können in den Optionen angegeben werden (jede Zahl muss dezimal und in Pixeln angegeben werden):

Wn: Ersetze n mit der Breite des Client-Bereichs im Fenster (der Client-Bereich schließt Rahmen, Titelleiste und Menüleiste des Fensters aus).

Hn: Ersetze n mit der Höhe des Client-Bereichs im Fenster.

Xn: Ersetze n mit der X-Position des Fensters. Position 0 ist die erste linke sichtbare Pixelspalte auf dem Bildschirm.

Yn: Ersetze n mit der Y-Position des Fensters. Position 0 ist die erste obere sichtbare Pixelzeile auf dem Bildschirm.

Center: Zentriert das Fenster horizontal und vertikal auf dem Bildschirm.

xCenter: Zentriert das Fenster horizontal auf dem Bildschirm. Zum Beispiel: Gui, Show, xCenter y0.

yCenter: Zentriert das Fenster vertikal auf dem Bildschirm.

AutoSize: Passt die Fenstergröße an den sichtbaren Steuerelementen an. Das ist nützlich, um die Fenstergröße anzupassen, nachdem neue Steuerelemente hinzugefügt oder Größe und Sichtbarkeit vorhandener Steuerelemente geändert wurden. Zum Beispiel: Gui, Show, AutoSize Center.


Eine der folgenden Optionen kann auch vorhanden sein:

Minimize: Minimiert das Fenster und aktiviert das darunterliegende Fenster.

Maximize: Maximiert und aktiviert das Fenster.

Restore: Hebt die Maximierung oder Minimierung des Fensters auf, falls notwendig. Zudem wird das Fenster je nach Bedarf angezeigt und aktiviert.

NoActivate: Hebt die Maximierung oder Minimierung des Fensters auf, falls notwendig. Zudem wird das Fenster angezeigt, ohne es zu aktivieren.

NA: Zeigt das Fenster an, ohne es zu aktivieren. Ist das Fenster minimiert, wird das beibehalten, steigt aber in der Z-Rangordnung nach oben (die Rangordnung kann im Alt-Tab-Fenster überprüft werden). Wurde das Fenster vorher versteckt, führt das wahrscheinlich dazu, dass es über dem aktiven Fenster erscheint, selbst wenn das aktive Fenster nicht deaktiviert ist.

Hide: Versteckt das Fenster und aktiviert das darunterliegende Fenster. Prinzipiell funktioniert es wie Gui Cancel. Allerdings hat diese Option den Vorteil, dass ein verstecktes Fenster verschoben, skaliert oder dessen Titel geändert werden kann, ohne dass es angezeigt werden muss. Zum Beispiel: Gui, Show, Hide x55 y66 w300 h200, Neuer Titel.

Gui, Submit [, NoHide]

Speichert den Inhalt von jedem Steuerelement in die zugewiesene Variable (falls vorhanden) und versteckt das Fenster, solange die NoHide-Option nicht vorhanden ist. Bei Steuerelementen mit mehreren Ausgabefeldern (z. B. eine mehrfach ausgewählte ListBox) enthält die Ausgabe das aktuelle Begrenzungszeichen des Fensters. Wenn das Fenster nicht vorhanden ist (da es z. B. mittels Gui Destroy zerstört wurde), bleibt dieser Befehl unwirksam.

Gui, Cancel

Versteckt das Fenster, ohne dabei die Inhalte der Steuerelemente in die jeweils zugewiesene Variable zu speichern. Wenn das Fenster nicht vorhanden ist (da es z. B. mittels Gui Destroy zerstört wurde), bleibt dieser Befehl unwirksam.

Gui, Destroy

Entfernt das Fenster (falls vorhanden) samt aller dazugehörigen Steuerelemente und gibt den entsprechenden Arbeitsspeicher sowie die Systemressourcen frei. Falls das Script später das Fenster neu erstellt, beginnen alle Fenstereigenschaften wie Farbe und Schriftart wieder mit ihren Standardwerten (als hätte das Fenster nie existiert). Wird Gui Destroy nicht verwendet, werden alle GUI-Fenster automatisch zerstört, sobald das Script beendet wird.

Gui, Font [, Optionen, Schriftart]

Bestimmt die Schriftart, die Schriftgröße, den Schriftstil und/oder die Textfarbe für Steuerelemente, die nach diesem Befehl erstellt werden. Zum Beispiel:

Gui, Font, s10, Verdana  ; Schriftart Verdana mit Größe 10.

Lasse die letzten beiden Parameter weg, um die Standard-Schriftart, die Standard-Schriftgröße und die Standard-Textfarbe des Betriebssystems wiederherzustellen.

Schriftart kann der Name einer beliebigen Schriftart sein (siehe Tabelle für Schriftarten). Falls Schriftart weggelassen wird oder nicht im Betriebssystem vorhanden ist, wird die zuvor verwendete Schriftart weiter verwendet (bzw. die Standardschriftart des Betriebssystems, wenn keine vorhanden ist). Das hat den Vorteil, dass ein GUI-Fenster bei mehreren Betriebssystemen die gleiche Schriftart verwendet, selbst wenn die bevorzugte Schriftart auf einigen dieser Systeme fehlt. Im folgenden Beispiel hat Verdana Vorrang vor Arial, welches wiederum Vorrang vor MS Sans Serif hat:

Gui, Font,, MS sans serif
Gui, Font,, Arial
Gui, Font,, Verdana  ; Bevorzugte Schriftart.

Ist der Optionen-Parameter leer, werden die zuvor benutzten Schriftartattribute weiter verwendet. Ansonsten können ersatzweise beliebig viele der folgenden Optionsbuchstaben angegeben werden:

C: Namen der Farbe (siehe Farbtabelle), ein RGB-Wert oder das Wort "Default", um die Standardfarbe des Betriebssystems wiederherzustellen (meistens schwarz). Beispielwerte: cRed, cFFFFAA, cDefault. Hinweis: Schaltflächen ignorieren benutzerdefinierte Farben. Durch Einfügen der C-Option ist es möglich, ein bestimmtes Steuerelement zu erstellen, das eine andere Schriftfarbe hat. Zum Beispiel: Gui, Add, Text, cRed, Eigener Text.

S: Größe (in Punkten). Zum Beispiel: s12 (muss dezimal sein).

W: Breite (Dicke). Eine Nummer zwischen 1 und 1000 (400 ist normal und 700 ist fett). Zum Beispiel: w600 (muss dezimal sein).

Zudem werden folgende Wörter unterstützt: bold, italic, strike, underline und norm (übersetzt: fett, kursiv, durchgestrichen, unterstrichen und normal). Norm stellt die normale Breite/Dicke der Schrift wieder her und deaktiviert italic, strike und underline (Farbe und Größe bleiben jedoch erhalten). Es ist möglich, erst alle Attribute mit norm zu deaktivieren und danach neue Attribute hinzuzufügen. Mit norm italic beispielsweise wird die Schrift normalisiert und danach kursiv gemacht.

Sind mehr als eine Option notwendig, müssen diese jeweils mit einem Leerzeichen getrennt werden. Zum Beispiel: cBlue s12 bold.

Erstellt ein Script mehrere GUI-Fenster, wird sich jedes Fenster seine eigene "aktuelle Schriftart" zwecks Erstellung neuer Steuerelemente merken.

Das Betriebssystem bietet zudem Standard-Dialogfenster, die den Benutzer auffordern können, eine Farbe, eine Schriftart oder ein Icon auszuwählen. Diese Dialogfenster können per DllCall() angezeigt werden, siehe dazu www.autohotkey.com/forum/topic17230.html.

Q: [AHK_L 19+]: Textdarstellungsqualität. Zum Beispiel: q3. Nach Q sollte eine Zahl aus der folgenden Tabelle erfolgen:

0 = DEFAULT_QUALITY Die Darstellung der Schriftart spielt keine Rolle.
1 = DRAFT_QUALITY Die Darstellung der Schriftart ist ein wenig unwichtiger als wie bei PROOF_QUALITY.
2 = PROOF_QUALITY Die Zeichenqualität der Schriftart ist wichtiger als die exakte Übereinstimmung mit logischen Schriftattributen (z. B. fett, kursiv und so weiter).
3 = NONANTIALIASED_QUALITY Schrift wird nicht geglättet.
4 = ANTIALIASED_QUALITY Schrift wird geglättet, wenn die Schriftart so etwas unterstützt und deren Größe nicht zu klein und nicht zu groß ist.
5 = CLEARTYPE_QUALITY Windows XP und höher: Der Text wird gegebenenfalls mithilfe der ClearType-Kantenglättung dargestellt.

Um mehr darüber zu erfahren, was diese Werte bedeuten, siehe MSDN: CreateFont.

Standardmäßig gilt die höchste Qualitätseinstellung. Deshalb wird dieses Feature eher dazu verwendet, Kantenglättung zu deaktivieren, um den Text bei bestimmten Fällen besser lesbar zu machen.

Gui, Color [, Fensterfarbe, Steuerelementfarbe]

Bestimmt die Hintergrundfarbe des Fensters und/oder dessen Steuerelemente. Mit Fensterfarbe wird der Hintergrund des Fensters selbst bestimmt. Steuerelementfarbe gilt für alle gegenwärtigen und zukünftigen Steuerelemente im Fenster (beachte dabei, dass einige Steuerelemente benutzerdefinierte Farben nicht unterstützen). Obwohl Steuerelementfarbe anfangs von ListView- und TreeView-Steuerelemente beachtet wird, sind nachfolgende Änderungen der Steuerelementfarbe nicht mehr möglich. In solchen Fällen kann GuiControl +BackgroundFF9977, EigeneListview verwendet werden, um die Farbe direkt zu ändern.

Lass den entsprechenden Parameter leer, wenn du die aktuelle Farbe beibehalten willst. Ansonsten kannst du eine Hauptfarbe aus der HTML-Farbtabelle, ein 6-stelliger RGB-Wert (0x-Präfix ist optional) oder das Wort "Default" für die Standardfarbe angegeben. Beispielwerte: Silver, FFFFAA, 0xFFFFAA, Default.

Standardmäßig ist die Hintergrundfarbe des Fensters die Systemfarbe der Schaltflächen, und die Hintergrundfarbe des Steuerelements standardmäßig die Systemfarbe des Fensters (üblicherweise weiß).

Die Farbe der Menüleiste und deren Untermenüs kann wie folgt geändert werden: Menu, EigeneMenüleiste, Color, White.

Um den Hintergrund in Windows 2000/XP oder höher transparent zumachen, verwende WinSet TransColor. Machst du das allerdings, ohne vorher ein benutzerdefiniertes Fenster per Gui Color zugewiesen zu haben, werden die Schaltflächen ebenfalls transparent. Du kannst das verhindern, wenn du zunächst eine benutzerdefinierte Fensterfarbe zuweist und diese dann transparent machst. Zum Beispiel:

Gui, Color, EEAA99
Gui +LastFound  ; Macht das GUI-Fenster zum zuletzt gefundenen Fenster.
WinSet, TransColor, EEAA99

Möchtest du zusätzlich den Rahmen und die Titelleiste eines Fensters mit transparentem Hintergrund entfernen, solltest du folgendes verwenden, nachdem das Fenster transparent gemacht worden ist:

Gui -Caption  ; Oder "Gui, GuiName:-Caption", falls es nicht das Standardfenster ist.

Die oben genannte Methode wird durch die Bildschirmanzeige (OSD) weiter unten nochmals näher gebracht.

Gui, Margin [, X, Y]

X und Y bestimmen die Anzahl an Pixeln, die einen Freiraum an der linken/rechten und oberen/unteren Seite bilden, falls Steuerelemente automatisch positioniert werden, denen eindeutig die Koordinaten X und Y fehlen. Außerdem werden diese Abstände dazu verwendet, den vertikalen und horizontalen Abstand zwischen den automatisch positionierten Steuerelementen festzulegen. Die Abstände zur Berechnung der Fenstergröße werden berücksichtigt, sobald Gui Show das erste Mal verwendet wird (falls keine eindeutige Größe angegeben worden ist).

Falls dieser Befehl nicht verwendet wird, erhält das Fenster automatisch an jeder Seite einen Standardabstand, sobald das erste Steuerelement hinzugefügt wird. Dieser Abstand wird proportional zur Größe der aktuell ausgewählten Schriftart sein (0.75 x Schrifthöhe für Unten & Oben, 1.25 x Schriftbreite für Links & Rechts).

Die Abstände können geändert werden, während Steuerelemente hinzugefügt werden. Allerdings werden solche Änderungen nur für Steuerelemente wirksam, die nachträglich hinzugefügt werden, nicht für solche, die bereits vorhanden sind. Du kannst entweder X oder Y leer lassen, um den entsprechenden Abstand unverändert zu lassen.

Gui, +/-Option1 +/-Option2 ...

Nach dem GUI-Befehl können eine oder mehrere Optionen angegeben werden. Aus Performance-Gründen ist es besser, alle Optionen auf einer Zeile zu setzen, bevor das Fenster erstellt wird (das heißt, bevor andere Unterbefehle wie Gui Add verwendet werden).

Der Effekt des Befehls ist kumulativ (summierend). Das heißt, dass dieser Befehl nur Einstellungen ändert, die direkt angegeben worden sind, alle anderen bleiben unverändert.

Mit einem Plus- oder Minuszeichen kann eine Option jeweils hinzugefügt oder entfernt werden. Zum Beispiel:

Gui +Resize -MaximizeBox  ; Ändert die Einstellungen des Standard-GUI-Fensters.
Gui EigenesGui:+Resize -MaximizeBox  ; Ändert die Einstellungen des GUI's namens EigenesGui.

AlwaysOnTop: Bewirkt, dass das Fenster immer über allen anderen Fenstern bleibt, was den gleichen Effekt hat wie WinSet AlwaysOnTop.

Border: Stattet das Fenster mit einem dünnen Rahmen aus. Das ist unüblich.

Caption (standardmäßig vorhanden): Stattet das Fenster mit einer Titelleiste und einem dicken Rahmen aus. Falls du vorhast, die Titelleiste eines Fensters zu entfernen, das WinSet TransColor verwenden wird, solltest du diese erst entfernen, wenn TransColor bereits gesetzt worden ist.

Delimiter: Bestimmt, dass das Fenster anstelle des senkrechten Striches (|) ein anderes Zeichen verwenden soll, sobald Inhalte von Steuerelementen mit Gui Add hinzugefügt, mit GuiControl bearbeitet oder mit Gui Submit bzw. GuiControlGet ermittelt werden. Füge das gewünschte Trennzeichen unmittelbar nach dem Wort Delimiter an. Zum Beispiel würde Gui +Delimiter`n ein Zeilenumbruchszeichen verwenden, das sich besonders gut für Fortsetzungsbereiche eignen könnte. Gui +Delimiter| würde das Standardtrennzeichen wiederherstellen. Möchtest du ein Leer- oder Tabulatorzeichen verwenden, solltest du Gui +DelimiterSpace oder Gui +DelimiterTab angeben. Wurde das Trennzeichen einmal geändert, beeinflusst diese Option alle vorhandenen und nachfolgenden Threads, die dieses bestimmte Fenster bearbeiten.

Disabled: Deaktiviert das Fenster, wodurch der Benutzer nicht länger mit den Steuerelementen interagieren kann. Diese Option wird oft bei Fenstern benutzt, die untergeordnete Fenster haben (siehe Owner).

DPIScale [v1.1.11+]: Verwende Gui -DPIScale, um die DPI-Skalierung zu deaktivieren, welche standardmäßig aktiviert ist. Falls die DPI-Skalierung auf einem System mit nicht-standardmäßiger DPI-Einstellung aktiviert ist, werden die Gui-Befehle Koordinaten und Größen automatisch skalieren, um den Steuerelementen die scheinbar fast gleiche Größe (aber höhere Auflösung) zuzuweisen. Bei einer DPI von beispielsweise 144 (150%) würde Gui Show, w100 das Gui 150 Pixel breit machen, allerdings würde A_GuiWidth weiterhin 100 zurückgeben. A_ScreenDPI enthält die aktuelle DPI des Systems.

HwndAusgabevariable [v1.1.04+]: Diese Option speichert das Fenster-Handle (HWND) des GUIs in Ausgabevariable. Zum Beispiel: Gui +HwndEigenesGuiHwnd. Innerhalb einer Funktion wird EigenesGuiHwnd als dynamische Variable behandelt. Das HWND eines GUIs wird oft in Verbindung mit PostMessage, SendMessage und DllCall verwendet. Außerdem kann es direkt als ahk_id-Fenstertitel oder anstelle vom GUI-Namen benutzt werden; zum Beispiel Gui %EigenesGuiHwnd%:Destroy.

Label [v1.0.44.09+]: Bestimmt benutzerdefinierte Namen für die speziellen Label des Fensters. Zum Beispiel würde Gui EigenesGui:+LabelEigenesGui_On die Label EigenesGui_OnClose und EigenesGui_OnSize verwenden (falls diese existieren), anstatt EigenesGuiGuiClose und EigenesGuiGuiSize. Das heißt, dass innerhalb der Namen aller speziellen Label der String "EigenesGuiGui" mit "EigenesGui_On" ersetzt wird. Dadurch können auch mehrere Fenster dazu gebracht werden, dass sie sich die gleichen Label teilen (in diesem Fall könnte das Script auf A_Gui zurückgreifen, um zu ermitteln, welches Fenster die Subroutine gestartet hat).

LastFound: Markiert das aktuelle Fenster als zuletzt gefundenes Fenster (das in einem GUI-Thread unnötig ist, weil das automatisch geschieht), wodurch Befehle wie WinSet die Möglichkeit haben, das Fenster zu bearbeiten, selbst wenn es versteckt ist (das heißt, dass DetectHiddenWindows nicht notwendig ist). Das ist besonders dann nützlich, wenn die Eigenschaften des Fensters geändert werden sollen, bevor es angezeigt wird. Zum Beispiel:

Gui +LastFound
WinSet, TransColor, %BeliebigeFarbe% 150
Gui Show

LastFoundExist [v1.0.43.09+]: Im Gegensatz zu den anderen Optionen wird LastFoundExist nur erkannt, wenn keine anderen Optionen auf derselben Zeile vorkommen. +LastFoundExist ist das gleiche wie +LastFound, außer dass das Fenster nicht erstellt wird, falls es noch nicht vorhanden ist. Der Hauptnutzen davon ist es, zu erkennen, ob ein bestimmtes GUI-Fenster bereits vorhanden ist. Zum Beispiel:

Gui EigenesGui:+LastFoundExist
IfWinExist
    MsgBox GUI "EigenesGui" existiert bereits.

MaximizeBox: Aktiviert die Maximieren-Schaltfläche in der Titelleiste. Diese Option ist bei Resize inbegriffen.

MinimizeBox (standardmäßig vorhanden): Aktiviert die Minimieren-Schaltfläche in der Titelleiste.

MinSize und MaxSize [v1.0.44.13+]: Bestimmt die minimale und/oder maximale Größe des Fensters, wenn der Benutzer beispielsweise die Größe des Fensters durch Ziehen an den Ecken ändern will. Du kannst die Wörter MinSize und/oder MaxSize ohne Suffix angeben, um die aktuelle Fenstergröße als Begrenzung zu verwenden (hat das Fenster keine aktuelle Größe, gilt die Größe, welche von Gui Show erstmalig verwendet wurde). Ansonsten kannst du Breite mal Höhe angeben: Gui +Resize +MinSize640x480. Die Abmessungen (in Pixel) bestimmen die Größe des Client-Bereichs im Fenster (der Client-Bereich schließt Rahmen, Titelleiste und Menüleiste des Fensters aus). Jede Zahlenangabe muss dezimal sein, nicht hexadezimal.

Du kannst eine der Abmessungen weglassen, um diese unverändert zu lassen (z. B. +MinSize640x oder +MinSizex480). Weiterhin kannst du Min/MaxSize mehrmals angeben, um die aktuelle Fenstergröße für eine Abmessung und eine eindeutige Größe für die andere Abmessung zu verwenden. Zum Beispiel würde +MinSize +MinSize640x die aktuelle Fenstergröße für die Höhe und 640 für die Breite verwenden.

Solltest du MinSize und MaxSize nie verwenden, gelten die Standardwerte des Betriebssystems (ähnlich wie Gui -MinSize -MaxSize, das diese Standardwerte wiederherstellen kann). Hinweis: das Fenster muss +Resize haben, damit dessen Größe vom Benutzer geändert werden kann.

OwnDialogs: Gui +OwnDialogs sollte in einem Thread (z. B. ButtonOK-Subroutine) angegeben werden, bei dem darauffolgende MsgBox-, InputBox-, FileSelectFile- oder FileSelectFolder-Dialogfenster einem Fenster untergeordnet werden sollen. Solche Dialogfenster werden dabei modal. Das heißt, dass der Benutzer erst wieder mit dem GUI-Fenster interagieren kann, wenn das Dialogfenster geschlossen wird. ToolTip-, Progress- und SplashImage-Fenster können hingegen nicht modal sein, selbst wenn sie untergeordnet werden. Sie bleiben lediglich immer im Vordergrund. In jedem Fall werden untergeordnete Dialogfenster automatisch zerstört, sobald ihr GUI-Fenster zerstört wird.

Normalerweise ist es nicht notwendig, diese Einstellung wieder zu deaktivieren, weil sie keine Auswirkung auf andere Threads hat. Muss ein Thread sowohl untergeordnete als auch nicht-untergeordnete Dialogfenster anzeigen, kann er diese Einstellung per Gui -OwnDialogs deaktivieren.

Falls du keinen Fensternamepräfix angegeben hast (weil du beispielsweise Gui +OwnDialogs anstatt Gui EigenesGui:+OwnDialogs verwendest), werden die Dialogfenster dem Standardfenster des Threads untergeordnet.

Owner: Verwende +Owner, um das Fenster einem anderen unterzuordnen. Ein untergeordnetes Fenster hat standardmäßig keine eigenen Taskleistenschaltflächen. Ist das Fenster sichtbar, befindet sich es immer im Vordergrund. Dieses Fenster wird automatisch zerstört, sobald das dazugehörige GUI-Fenster zerstört wird. +Owner darf erst nach der Erstellung des übergeordneten Fensters verwendet werden, und seit v1.1.05 auch vor und nach der Erstellung des untergeordneten Fensters. Es gibt zwei Möglichkeiten, +Owner wie folgt zu verwenden:

Gui, EigenesGui:+OwnerAnderesGui  ; EigenesGui wird AnderesGui untergeordnet.
Gui, EigenesGui:+Owner  ; EigenesGui wird dem Hauptfenster des Scripts untergeordnet, um die Schaltfläche in der Taskleiste zu unterdrücken.

[v1.1.03+]: Unmittelbar nach +Owner kann der Name oder die Nummer eines vorhandenen Gui-Fensters, oder das HWND eines eigenständigen Fensters erfolgen.

Hinweis zur Kompatibilität: Seit v1.1.03 bewirkt +Owner, dass der WS_CHILD-Style entfernt und der WS_POPUP-Style gesetzt wird. Möchtest du das übergeordnete Fenster eines Gui's setzen, müssen deine Scripts entweder die Option +Parent verwenden oder die entsprechenden Styles nach der Option +Owner überschreiben.

Falls du die Interaktion des Benutzers mit dem übergeordneten Fenster unterbinden willst, während ein untergeordnetes Fenster sichtbar ist, muss du das übergeordnete Fenster mit Gui +Disabled deaktivieren. Sobald das untergeordnete Fenster geschlossen oder zerstört wird, kann das übergeordnete Fenster wieder mit Gui -Disabled aktiviert werden. Mache das, bevor das Fenster geschlossen bzw. zerstört wird, so dass das übergeordnete Fenster wieder automatisch aktiviert werden kann.

Parent [v1.1.03+]: Unmittelbar nach +Parent kann der Name oder die Nummer eines vorhandenen Gui-Fensters, oder das HWND eines beliebigen Fensters oder Steuerelements erfolgen, um das Fenster einem anderen unterzuordnen. Möchtest du das Gui-Fenster wieder in ein eigenständiges Fenster umwandeln, verwende -Parent. Diese Option funktioniert auch, nachdem das Fenster erstellt wurde.

Resize: Macht das Fenster skalierbar und aktiviert die Maximieren-Schaltfläche in der Titelleiste. Um das Aktivieren der Schaltfläche zu unterdrücken, verwende +Resize -MaximizeBox.

SysMenu (standardmäßig vorhanden): Verwende -SysMenu, um das Systemmenü und Icon in der oberen linken Ecke des Fensters zu entfernen, und die Minimieren-, Maximieren- und Schließen-Schaltfläche auszublenden.

Theme: Verwende -Theme, um allen nachfolgenden Steuerelementen im Fenster das klassische Design zuzuweisen (nur ab Windows XP). Möchtest du danach weitere Steuerelemente unter Berücksichtigung des neuen Designs erstellen, verwende +Theme. Hinweis: Diese Option funktioniert nicht, wenn das Betriebssystem älter als Windows XP ist, oder wenn in Windows XP selbst das klassische Design aktiv ist. Soll diese Einstellung nur für ein bestimmtes Steuerelement geändert werden, schreibe +Theme oder -Theme in dessen Optionen, sobald es erstellt wird.

ToolWindow: Stattet das Fenster mit einer schmalen Titelleiste aus, aber ohne Taskleistenschaltfläche.

(Unbenannter Style): Verwende ein Plus- oder Minuszeichen, gefolgt von einer dezimalen oder hexadezimalen Style-Nummer.

(Unbenannter ExStyle): Verwende ein Plus- oder Minuszeichen, gefolgt von dem Buchstaben "E" und einer dezimalen oder hexadezimalen erweiterten Style-Nummer. Zum Beispiel würde +E0x40000 den WS_EX_APPWINDOW-Style hinzufügen, wodurch das Fenster eine Taskleistenschaltfläche erhält, die ansonsten fehlen würde. Weitere nicht-dokumentierte Styles findest du auf www.microsoft.com mit Suchbegriffen wie WS_EX_APPWINDOW.

Gui, Menu [, Menüname]

Befestigt eine Menüleiste an das Fenster. Verwende dazu den Menu-Befehl, um ein normales Menü zu erstellen. Zum Beispiel:

Menu, Dateimenü, Add, Ö&ffnen`tCtrl+F, MenüDateiÖffnen  ; Siehe untere Bemerkungen zu Ctrl+F.
Menu, Dateimenü, Add, &Beenden, MenuHandler
Menu, Hilfsmenü, Add, Inf&o, MenuHandler
Menu, EigeneMenüleiste, Add, &Datei, :Dateimenü  ; Fügt die oben erstellten Untermenüs hinzu.
Menu, EigeneMenüleiste, Add, &?, :Hilfsmenü
Gui, Menu, EigeneMenüleiste

Beachte in der obersten Zeile, dass nach Ö&ffnen die Tastenkombination Ctrl+F erfolgt (mit einem Tabulatorzeichen dazwischen). Anstatt zum Menüpunkt zu navigieren, könnte der Benutzer einfach diese Tastenkombination drücken. [v1.1.04+]: Verwendet die Tastenkombination nur die Standard-Modifikatortaste Ctrl, ALT oder Shift, wird sie automatisch als Keyboard Accelerator (Zugriffstaste) für das GUI registriert. Einzelzeichen-Zugriffstasten ohne Modifikatoren unterscheiden zwischen Groß- und Kleinschreibung und können durch ungewöhnliche Mitteln wie IME oder ALT+NNNN ausgelöst werden.

Sollte eine bestimmte Tastenkombination nicht automatisch funktionieren, musst du möglicherweise einen kontextabhängigen Hotkey verwenden. Beachte aber, dass solche Hotkeys üblicherweise nicht durch Send ausgelöst werden können und dass sie eher andere Scripts stören, anstatt eine Standard-Zugriffstaste zu sein.

Möchtest du die aktuelle Menüleiste eines Fensters entfernen, verwende Gui Menu (lasse den letzten Parameter weg).

Wurde ein Menü einmal als Menüleiste verwendet, sollte es nicht mehr als Popup-Menü oder Untermenü eingesetzt werden. Das liegt daran, dass Menüleisten intern ein anderes Format benötigen (allerdings gilt diese Einschränkung nur für die Menüleiste selbst, nicht für die Untermenüs). Musst du diese Tatsache umgehen, erstelle ein Menü, das die Menüleiste sein soll, und das gleiche Menü nochmals, das etwas anderes sein soll.

Die Verwendung von destruktiven Menu-Befehlen wie Delete und DeleteAll bei einem Menü, das zurzeit eine Menüleiste ist (und in einigen Fällen auch deren Untermenüs), wird nicht unterstützt und verursacht eine Fehlermeldung (sofern UseErrorLevel nicht aktiv ist). Führe die folgenden Schritte durch, um solche Änderungen vornehmen zu können:

  1. Entferne die Menüleiste mit Gui Menu (lasse Menüname weg)
  2. Führe die Änderungen durch
  3. Füge die Menüleiste mit Gui, Menu, EigeneMenüleiste wieder an

Gui, Hide / Minimize / Maximize / Restore

Gui Hide ist das gleiche wie Gui Cancel. Die anderen drei Befehle machen das Fenster wieder sichtbar (falls notwendig) und führen dann die entsprechenden Operationen aus. Ist das Fenster nicht vorhanden (da es beispielsweise durch Gui Destroy zerstört wurde), werden diese Befehle nicht funktionieren.

Gui, Flash [, Off]

Lässt die Taskleistenschaltfläche des Fensters aufblinken. Dazu wird die Farbe der Titelleiste und/oder Taskleistenschaltfläche (falls vorhanden) invertiert. Das optionale Wort "OFF" bewirkt, dass die Titelleiste und Taskleistenschaltfläche ihre ursprünglichen Farben wiederherstellen (das Verhalten ist allerdings vom jeweiligen Betriebssystem abhängig). Im folgenden Beispiel wird das Fenster dreimal aufblinken, weil jeweils ein Paar von "Gui Flash" die Darstellung invertiert und wiederherstellt:

Loop 6
{
    Gui Flash
    Sleep, 500  ; Dieser Wert ist ziemlich empfindlich; Änderungen können zu unerwarteten Ergebnissen führen.
}

Gui, GuiName:Default

Ändert den GUI-Standardfensternamen des aktuellen Threads, der immer dann verwendet wird, wenn bei GuiControl, GuiControlGet und dem Gui-Befehl selbst kein Fenstername angegeben ist. Im folgenden Beispiel wird der Standardname des Fensters auf "EigenesGui" gesetzt: Gui EigenesGui:Default. In "Standardnummer eines Threads" findest du mehr Informationen über das Standardfenster.

Position und Größe von Steuerelementen anpassen

Lässt du einige Abmessungen oder Koordinaten in Optionen weg, wird das Steuerelement relativ zum vorherigen Steuerelement positioniert und/oder dessen Größe abhängig vom Inhalt automatisch angepasst.

Die folgenden Optionen werden unterstützt:

R: Textzeilen (kann eine Fließkommazahl wie bei R2.5 enthalten). R wird gegenüber H (Höhe) meist bevorzugt. Sind sowohl R als auch H vorhanden, wird R Vorrang haben. Bei einer GroupBox bestimmt diese Einstellung die Menge an Steuerelementen, für denen innerhalb der Box Platz geschaffen werden soll. Bei DropDownLists, ComboBoxes und ListBoxes ist das die Anzahl an Einträgen, die gleichzeitig im Listenbereich sichtbar sein sollen (aber in Windows XP oder höher ist es oft besser, die Optionen R und H bei DropDownList und ComboBox wegzulassen, damit die aufklappbare Liste automatisch die verfügbare Höhe des Desktops ausnutzen kann). Bei allen anderen Steuerelementen ist R die Anzahl der Textzeilen, die sichtbar in das Steuerelement reinpassen sollen.

W: Breite in Pixeln. Ist diese Option nicht vorhanden, berechnet sich die Breite bei einigen Steuerelementarten automatisch, basierend auf deren Inhalt. Alle anderen Steuerelementarten haben standardmäßig folgende Breiten:
Tab-Steuerelemente: Das 30-fache der aktuellen Schriftgröße, plus das 3-fache des X-Abstands.
Vertikaler Fortschrittsbalken: Das 2-fache der aktuellen Schriftgröße.
Horizontaler Fortschrittsbalken, horizontaler Schieberegler, DropDownList, ComboBox, ListBox, GroupBox, Edit und Hotkey: Das 15-fache der aktuellen Schriftgröße (außer GroupBox, das mit 18 multipliziert wird, um innerhalb der Abstände Platz zu schaffen).

H: Höhe in Pixeln. Fehlt sowohl H als auch R, verwenden DropDownLists, ComboBoxes, ListBoxes und leere mehrzeilige Eingabefelder standardmäßig eine Höhe von 3 Zeilen; GroupBoxes standardmäßig 2 Zeilen; vertikale Schieberegler und Fortschrittsbalken standardmäßig 5 Zeilen; horizontale Schieberegler standardmäßig 30 Pixel (außer wenn eine Dicke angegeben wurde); horizontale Fortschrittsbalken standardmäßig das 2-fache der aktuellen Schriftgröße; Hotkey-Steuerelemente standardmäßig 1 Zeile; und Tab-Steuerelemente standardmäßig 10 Zeilen. Bei allen anderen Steuerelementen berechnet sich die Höhe automatisch, basierend auf deren Inhalt. Beachte, dass sich H bei DropDownLists und ComboBoxes aus dem immer sichtbaren Teil und dem dazugehörigen Listenbereich zusammensetzt (selbst wenn die Höhe zu niedrig ist, wird mindestens immer ein Eintrag in der Liste angezeigt). Auch hier gilt, dass man R anstelle von H bevorzugen sollte, weil es ein Steuerelement daran hindern kann, unvollständige Textzeilen darzustellen.

wp+n, hp+n, wp-n und hp-n (wobei n eine beliebige Zahl ist) können dazu verwendet werden, die Abmessungen eines Steuerelements an die Abmessungen des zuvor hinzugefügten Steuerelements anzupassen, optional mit einem Plus- oder Minuszeichen. Zum Beispiel würde wp die Breite eines Steuerelements auf die Breite des vorherigen Steuerelements setzen, und wp-50 würde die Breite des vorherigen Steuerelements verwenden, subtrahiert mit 50.

X: X-Position. Zum Beispiel würde x0 y0 das Steuerelement in die obere linke Ecke des Client-Bereichs positionieren (das ist der Fensterbereich ohne Rahmen, Titelleiste und Menüleiste). Ist Y vorhanden und X nicht, wird das Steuerelement auf die rechte Seite der zuvor eingefügten Steuerelementen positioniert, als würde eine neue "Spalte" beginnen.

Y: Y-Position. Ist X vorhanden und Y nicht, wird das Steuerelement auf die untere Seite der zuvor eingefügten Steuerelementen positioniert, als würde eine neue "Zeile" beginnen.

Das Weglassen von X und/oder Y hat den Vorteil, dass sich ein GUI-Layout später automatisch daran anpassen kann, wenn du die Größe eines Steuerelements oder die Schriftart änderst. Hast du dagegen bei jedem Steuerelement einen absoluten Wert angegeben, musst du die Position von allen Steuerelementen, die sich darunter oder daneben befinden, manuell verschieben.

Ist X und Y nicht vorhanden, wird das Steuerelement unterhalb des vorherigen Steuerelements positioniert, basierend auf den voreingestellten Innenabstand.

Bei Bedarf kann bei X und Y ein Pluszeichen eingefügt werden, um ein Steuerelement relativ zum rechten oder unteren Rand des zuvor eingefügten Steuerelements zu positionieren. Zum Beispiel würde Y+10 das Steuerelement 10 Pixel unter dem vorherigen Steuerelement positionieren, anstatt den voreingestellten Innenabstand zu verwenden. X+10 würde das Steuerelement 10 Pixel rechts neben dem vorherigen Steuerelement positionieren. Da negative Zahlen wie X-10 für die absolute Positionierung vorbehalten sind, ist es für einen negativen Offset notwendig, ein Pluszeichen davor einzufügen. Zum Beispiel: X+-10.

xp+n, yp+n, xp-n und yp-n (wobei n eine beliebige Zahl ist) können dazu verwendet werden, Steuerelemente relativ zur linken oberen Ecke des vorherigen Steuerelements zu positionieren, um beispielsweise Steuerelemente in einer GroupBox einzuschließen.

xm und ym können dazu verwendet werden, ein Steuerelement jeweils am linken und rechten Abstand des Fensters zu positionieren (nach diesen Optionen kann ein Plus-/Minuszeichen und eine Zahl folgen). Verwendest du ym, ohne eine X-Position angegeben zu haben, wird das Steuerelement am oberen Abstand positioniert, aber auf der rechten Seite von allen zuvor eingefügten Steuerelementen, als würde eine neue "Spalte" beginnen. Das Gegenteil davon (xm ohne Y-Position) gilt auch.

xs und ys sind ähnlich wie xm und ym, außer dass sie sich auf Koordinaten beziehen, die gespeichert wurden, während ein Steuerelement hinzugefügt worden ist, das das Wort Section in Option hat (das erste Steuerelement im Fenster beginnt immer eine neue Sektion, auch wenn dieses Wort nicht direkt angegeben wurde). Verwendest du ys, ohne eine X-Position angegeben zu haben, wird das Steuerelement auf die zuvor gespeicherte Y-Position positioniert, aber auf der rechten Seite von allen Steuerelementen, die seit der letzten Verwendung von Section hinzugefügt worden sind; als würde eine neue "Spalte" innerhalb der Sektion beginnen. Zum Beispiel:

Gui, Add, Edit, w600  ; Fügt ein breites Eingabefeld am oberen Rand des Fensters hinzu.
Gui, Add, Text, Section, Vorname:  ; Speichert die Position des Steuerelements und beginnt eine neue Sektion.
Gui, Add, Text,, Nachname:
Gui, Add, Edit, ys  ; Beginnt eine neue Spalte innerhalb dieser Sektion.
Gui, Add, Edit
Gui, Show

Das Gegenteil davon (xs ohne Y-Position) gilt auch.

Bei Bedarf kann nach xs und ys ein Plus-/Minuszeichen und eine Zahl erfolgen. Außerdem ist es möglich, gleichzeitig das Wort Section und xs/ys in den Optionen des Steuerelements anzugeben, wodurch es die vorherige Position noch für sich selbst verwendet, aber für nachfolgende Steuerelemente eine neue Sektion erstellt.

Speichern von und Reagieren auf Benutzereingaben

V: Variable. Verbindet eine Variable mit einem Steuerelement. Nach dem Buchstaben V erfolgt der Name einer globalen Variable (oder auch einer lokale ByRef-Variable, die sich auf eine globale Variable bezieht, oder seit v1.0.46.01 auch auf eine statische Variable). Zum Beispiel wird mit vEigenesEdit der Inhalt des Steuerelements in die EigenesEdit-Variable gespeichert, sobald Gui Submit verwendet wird. Falls ein Steuerelement nicht eingabefähig ist (z. B. Text oder GroupBox), könnte es dennoch mit einer Variable verbunden werden, weil der Variablenname bei GuiControl, GuiControlGet und A_GuiControl als eindeutige Steuerelement-ID dient. Hinweis: Gui Submit ändert nicht den Inhalt von Variablen, die mit eingabefähigen (z. B. Text oder GroupBox) oder sonstigen (z. B. ListView und TreeView) Steuerelementen verbunden wurden.

G: Gosub (g-Label). Startet automatisch eine Subroutine, sobald ein Steuerelement angeklickt oder geändert wird. Nach dem Buchstaben G erfolgt der Name des Labels, das ausgeführt werden soll. Desweiteren könnte gCancel angegeben werden, um Gui Cancel indirekt durchzuführen (sobald ein Label namens "Cancel" vorhanden ist, wird dieser stattdessen ausgeführt). Die Subroutine kann auf folgende interne Variablen zurückgreifen: A_Gui, A_GuiControl, A_GuiEvent und A_EventInfo.

Steuerelemente: Gewöhnliche Styles und Optionen

Hinweis: Vorzeichenlose Eigenschaften werden standardmäßig hinzugefügt; zum Beispiel ist Wrap das Gleiche wie +Wrap. Mit -Wrap wird hingegen die entsprechende Eigenschaft entfernt.

AltSubmit: Verwendet eine alternative Methode von Submit. Sobald der ausgewählte Eintrag einer DropDownList, ComboBox oder ListBox mithilfe von Gui Submit ermittelt wird, wird anstelle des Textes die Positionsnummer gespeichert. Falls kein Eintrag ausgewählt ist, wird die ComboBox den Text weiterhin in das Eingabefeld speichern; DropDownList oder ListBox machen ihre Ausgabevariablen weiterhin leer. Hinweis: AltSubmit beeinflusst auch das Verhalten von GuiControlGet, wenn der Inhalt von solchen Steuerelementen abgerufen wird.

C: Farbe des Textes (funktioniert nicht bei Schaltflächen). Nach dem Buchstaben C erfolgt der Name einer Farbe (siehe Farbtabelle) oder ein RGB-Wert (das 0x-Präfix ist optional). Beispiele: cRed, cFF2211, c0xFF2211, cDefault.

Disabled: Deaktiviert ein eingabefähiges Steuerelement. Das heißt, dass der Benutzer es nicht länger fokussieren oder dessen Inhalt ändern kann. Mit GuiControl Enable kann das Steuerelement wieder aktiviert werden. Hinweis: Um ein Eingabefeld stattdessen in den Schreibschutzmodus zu versetzen, verwendet den String ReadOnly. Nach dem Wort Disabled kann auch eine 0 oder 1 für den Anfangsstatus erfolgen (0 für Aktivieren, 1 für Deaktivieren). Das heißt, dass Disabled und Disabled%VarMitEins% identisch sind.

Hidden: Das Steuerelement wird sofort versteckt. Mit GuiControl Show kann das Steuerelement wieder sichtbar gemacht werden. Nach dem Wort Hidden kann auch eine 0 oder 1 für den Anfangsstatus erfolgen (0 für sichtbar, 1 für versteckt). Das heißt, dass Hidden und Hidden%VarMitEins% identisch sind.

Left: Der Text im Steuerelement wird linksbündig ausgerichtet.

Right: Der Text im Steuerelement wird rechtsbündig ausgerichtet. Bei Kontrollkästchen und Optionsfelder wird die Box auf die rechte Seite verlegt.

Center: Der Text im Steuerelement wird horizontal zentriert.

Section: Beginnt eine neue Sektion und speichert die Position des Steuerelements, damit sie danach mit xs und ys weiterverwendet werden kann (siehe oben).

Tabstop: Mit -Tabstop wird ein eingabefähiges Steuerelement übersprungen, sobald der Benutzer die Tabulatortaste drückt.

Wrap: Aktiviert den automatischen Zeilenumbruch im Steuerelement. Da nahezu alle Steuerelemente den automatischen Zeilenumbruch aktiv haben, ist eher Wrap mit einem Minuszeichen nötig, um den automatischen Zeilenumbruch zu deaktivieren.

VScroll: Aktiviert die vertikale Bildlaufleiste in einem Steuerelement, wenn möglich.

HScroll: Aktiviert die horizontale Bildlaufleiste in einem Steuerelement, wenn möglich. Der Rest des Abschnitts gilt nur für ListBox-Steuerelemente. Die horizontale Bildlaufbreite ist standardmäßig das 3-fache der ListBox-Breite. Für eine benutzerdefinierte Bildlaufbreite muss eine Zahl nach dem Wort HScroll erfolgen. Zum Beispiel ermöglicht HScroll500, innerhalb der ListBox mit 500 Pixel zu scrollen. Ist die Bildlaufbreite allerdings kleiner als die Breite der ListBox, dann wird keine Bildlaufleiste angezeigt (nur mit HScroll ist es später möglich, die horizontale Bildlaufleiste mittels GuiControl, +HScroll500, EigeneBildlaufleiste einzufügen).

Steuerelemente: Ungewöhnliche Styles und Optionen

BackgroundTrans: Verwendet einen transparenten Hintergrund. Dadurch können Steuerelemente, die sich hinter einem Text-, Picture- oder GroupBox-Steuerelement befinden, durchscheinen. Wenn zum Beispiel ein transparentes Text-Steuerelement auf ein Picture-Steuerelement gesetzt wird, dann sieht es so aus, als würde der Text zum Bild gehören. Danach kann diese Option mit GuiControl, +Background rückgängig gemacht werden. Im AltSubmit-Abschnitt des Picture-Steuerelements befinden sich weitere Informationen über transparente Bilder. Bekannte Einschränkung: BackgroundTrans funktioniert möglicherweise nicht richtig bei Steuerelementen, die sich in einem Tab-Steuerelement befinden, das eine ListView enthält.

-Background: Verwendet die Standard-Hintergrundfarbe, anstatt die Farbe, welche von Gui Color gesetzt wurde. Diese Option wird häufig eingesetzt, um einem Tab-Steuerelement seine Standardfarbe anstelle der Fensterfarbe zu verabreichen. Danach kann das mit GuiControl, +Background rückgängig gemacht werden.

Border: Das Steuerelement erhält einen dünnen Rahmen. Die meisten Steuerelemente benötigen diese Option nicht, da sie bereits einen typenspezifischen Rahmen haben. Sobald der Rahmen an ein vorhandenes Steuerelement angefügt wird, ist es wahrscheinlich notwendig, die Breite und Höhe um einen Pixel zu vergrößern.

HwndAusgabevariable [v1.0.46.01+]: Beim Verwenden dieser Option in Gui Add wird das Fenster-Handle (HWND) des neu erstellten Steuerelements in die Ausgabevariable gespeichert. Zum Beispiel: Gui, Add, Edit, vEigenesEdit HwndEigenesEditHwnd. Befindet sich dieses Beispiel in einer Funktion, wird EigenesEditHwnd als eine dynamische Variable behandelt. Das HWND eines Steuerelements wird oft mit PostMessage, SendMessage und DllCall verwendet. Es kann direkt als ahk_id-Fenstertitel (das auch bei versteckten Steuerelementen funktioniert, selbst wenn DetectHiddenWindows deaktiviert ist) oder [seit v1.1.04+] bei GuiControl und GuiControlGet als SteuerelementID-Parameter verwendet werden. Beachtet zudem, dass das HWND eines übergeordneten Fensters mit Gui EigenesGui:+HwndAusgabevariable abgerufen werden kann.

Theme: Diese Option kann dazu verwendet werden, das aktuelle Fenster-Design des neu erstellten Steuerelements zu überschreiben. Bei bereits vorhandenen Steuerelementen hat diese Option keinen Effekt; allerdings wird das möglicherweise in einer zukünftigen Version geändert. Siehe Gui +/-Theme für Details.

(Unbenannter Style): Verwendet ein Plus- oder Minuszeichen, gefolgt von einer dezimalen oder hexadezimalen Style-Nummer. Vorzeichenlose Styles werden standardmäßig hinzugefügt.

(Unbenannter ExStyle): Verwendet ein Plus- oder Minuszeichen, gefolgt von dem Buchstaben "E" und einer dezimalen oder hexadezimalen erweiterten Style-Nummer. Vorzeichenlose Styles werden standardmäßig hinzugefügt. Zum Beispiel würde E0x200 den WS_EX_CLIENTEDGE-Style hinzufügen, der einen Rahmen mit abgesenktem Rand bereitstellt, gut geeignet für Picture-Steuerelemente und Ähnliches. Weitere nicht-dokumentierte Styles findet man auf www.microsoft.com, z. B. mit der Suche nach WS_EX_CLIENTEDGE.

Fensterereignisse

Die folgenden Label (Subroutinen) werden automatisch mit einem GUI-Fenster verbunden, falls diese im Script vorhanden sind:

GuiClose: Wird ausgeführt, sobald entweder die X-Schaltfläche in der Titelleiste gedrückt, "Schließen" im Systemmenü ausgewählt oder WinClose verwendet wird, um das Fenster zu schließen. Falls dieses Label weggelassen wird, wird das Fenster einfach versteckt, was den gleichen Effekt wie Gui Cancel hat. Einer der häufigsten Maßnahmen für GuiClose ist ExitApp. Zum Beispiel:

GuiClose:
ExitApp

GuiEscape: Wird ausgeführt, sobald die ESC-Taste im aktiven Fenster gedrückt wird. Falls dieses Label weggelassen wird, passiert nichts beim Drücken dieser Taste. Bekannte Einschränkung: Sollte das erste Steuerelement im Fenster deaktiviert sein (wahrscheinlich abhängig vom Typ des Steuerelements), wird das GuiEscape-Label nicht ausgeführt. Dieser Effekt könnte auch durch andere Umstände ausgelöst werden.

GuiSize: Wird ausgeführt, sobald das Fenster skaliert, minimiert, maximiert oder wiederhergestellt wird. Die internen Variablen A_GuiWidth und A_GuiHeight enthalten jeweils die neue Breite und Höhe des Client-Bereichs (Der Client-Bereich schließt Rahmen, Titelleiste und Menüleiste des Fensters aus). Außerdem enthalten A_EventInfo und ErrorLevel jeweils eine der folgenden Zahlen:

0: Das Fenster wurde wiederhergestellt oder dessen Größe durch Ziehen des Rahmens geändert.
1: Das Fenster wurde minimiert.
2: Das Fenster wurde maximiert.

GuiSize kann dazu verwendet werden, die Position und Größe der Steuerelemente neu anzupassen, falls der Benutzer die Größe des Fensters ändert. Dieser Prozess kann z. B. mit dem "Anchor"-Script von Titan via #Include vereinfacht werden.

GuiContextMenu: Wird ausgeführt, sobald das Fenster mit der rechten Maustaste angeklickt wird (außer Titelleiste und Menüleiste). Zudem wird es ausgeführt, sobald die MENÜ-Taste oder UMSCHALT+F10 gedrückt wird. Im Gegensatz zu den meisten anderen GUI-Labels können mehrere Threads von diesem Label vorhanden sein. Innerhalb des GuiContextMenu-Labels sind folgende interne Variablen verfügbar:

  1. A_GuiControl, die den Text oder Variablennamen des Steuerelements enthält, zu dem das Ereignis gesendet wurde (leer, wenn nichts).
  2. A_EventInfo: Falls das Ziel des Kontextmenüs ein ListBox-, ListView- oder TreeView-Steuerelement ist (kann mit A_GuiControl ermittelt werden), enthält A_EventInfo den Eintrag des Steuerelements, der als Ziel dient:
    ListBox oder ListView: A_EventInfo enthält die Nummer der aktuell fokussierten Zeile (0, wenn keine).
    TreeView: Bei Rechtsklick enthält A_EventInfo die ID-Nummer des angeklickten Eintrags (0, wenn der Benutzer keinen Eintrag angeklickt hat). Drückst du die MENÜ-Taste oder UMSCHALT+F10, enthält A_EventInfo die ID-Nummer des ausgewählten Eintrags.
  3. A_GuiX und A_GuiY, die die Koordinaten X und Y des geöffneten Kontextmenüs enthalten (z. B. Menu, EigenerKontext, Show, %A_GuiX%, %A_GuiY%). Die Koordinaten beziehen sich auf die linke obere Ecke des Fensters.
  4. A_GuiEvent, die das Wort "RightClick" enthält, falls die rechte Maustaste geklickt wurde, oder "Normal", falls das Menü mit der MENÜ-Taste oder UMSCHALT+F10 geöffnet wurde.

Hinweis: Da die Steuerelemente Edit und MonthCal ihre eigenen Kontextmenüs haben, wird das GuiContextMenu-Label bei Rechtsklick nicht ausgeführt.

GuiDropFiles: Wird ausgeführt, sobald Dateien oder Ordner per Ziehen und Ablegen ins Fenster verschoben werden (falls das Label bereits ausgeführt wird, wird das Ablegen ignoriert). Innerhalb des GuiDropFiles-Labels sind folgende interne Variablen verfügbar:

  1. A_GuiControl, die den Text oder Variablennamen des Steuerelements enthält, auf dem die Dateien abgelegt wurden (leer, wenn keine).
  2. A_EventInfo und ErrorLevel, die die Anzahl an abgelegten Dateien enthalten.
  3. A_GuiX und A_GuiY, die die Koordinaten X und Y der Position enthalten, auf der die Dateien abgelegt wurden (bezogen auf die obere linke Ecke des Fensters).
  4. A_GuiEvent, die die Namen der abgelegten Dateien enthält. Jeder Dateiname mit Ausnahme des Letzten endet mit einem Zeilenumbruchszeichen (`n).

Verwendet eine Parsing-Schleife wie folgt, um einzelne Dateien zu extrahieren: 

; BEISPIEL #1:
Loop, parse, A_GuiEvent, `n
{
    MsgBox, 4,, Dateinummer %A_Index% ist:`n%A_LoopField%.`n`nWeiter?
    IfMsgBox, No, Break
}

; BEISPIEL #2: Nur die erste Datei extrahieren:
Loop, parse, A_GuiEvent, `n
{
    FirstFile = %A_LoopField%
    Break
}

; BEISPIEL #3: Dateien alphabetisch sortieren:
FileList = %A_GuiEvent%
Sort, FileList
Loop, parse, FileList, `n
    MsgBox Dateinummer %A_Index% ist:`n%A_LoopField%.

Um das Ziehen und Ablegen in einem Fenster vorübergehend zu deaktivieren, entfernt den WS_EX_ACCEPTFILES-Style mittels Gui -E0x10. Danach kann es wieder mit Gui +E0x10 aktiviert werden.

Erkennen von und reagieren auf andere Ereignisse: Mittels OnMessage() können andere Arten von GUI-Ereignissen erkannt und bearbeitet werden. Zum Beispiel ist es möglich, eine kontextabhängige Hilfe über den ToolTip anzuzeigen, sobald sich der Mauszeiger über ein Steuerelement im Fenster befindet. Siehe dazu das Beispiel mit GUI-ToolTip.

Mehrere GUI-Fenster erstellen

Abgesehen vom ersten Fenster müssen bei anderen Fenstern deren Namen oder Nummern (falls vorhanden) angegeben werden, damit sie die oben genannten Labels verwenden können. Zum Beispiel würden 2GuiEscape und 2GuiClose die Standard-Label des zweiten Fensters sein, während EigenesGuiGuiEscape und EigenesGuiGuiClose die Standard-Label für EigenesGui sind. Um benutzerdefinierte Labelnamen zu verwenden, siehe Gui +Label.

Um neben dem Standard-Fenster auch andere Fenster bearbeiten zu können, fügt dessen Name oder Nummer (oder seit v1.1.03 dessen HWND) gefolgt von einem Doppelpunkt vor dem Unterbefehl wie folgt ein:

Gui, EigenesGui:Add, Text,, Text für das Über-Fenster.
Gui, EigenesGui:Show

Mit Gui EigenesGui:Default kann verhindert werden, dass das oben genannte Präfix "EigenesGui:" angegeben werden muss. Innerhalb eines GUI-Threads, der das gleiche Fenster verarbeitet, welches den Thread ausgeführt hat, ist das Präfix außerdem nicht notwendig.

[v1.1.03+]: Für die Gui-Namen gelten die gleichen Regeln wie bei Variablennamen. Jeder Nummer größer als 99 oder länger als zwei Zeichen (wie z. B. 0x01) muss das HWND eines vorhandenen Gui-Fensters sein, denn ansonsten würde dieser Befehl fehlschlagen. Je nach verfügbaren Systemressourcen können beliebig viele Fenster erstellt werden.

[v1.1.04+]: Mit Gui, New können beliebig viele unbenannte GUI-Fenster erstellt werden.

GUI-Ereignisse, Threads und Subroutinen

Ein GUI-Thread wird wie jeder andere Thread definiert, der aufgrund einer GUI-Aktion gestartet wurde. Zu den GUI-Aktionen gehört beispielsweise das Auswählen eines Menüpunkts oder das Ausführen eines g-Labels (z. B. beim Drücken einer Schaltfläche).

Bei einem GUI-Thread ist der Standardfenstername der Name des Fensters, das diesen Thread gestartet hat. Nicht-GUI-Threads verwenden standardmäßig eine 1.

Sobald ein GUI-Thread ausgeführt wird, ist das zuletzt gefundene Fenster das GUI-Fenster selbst. Dadurch können Fenster- und Steuerelementbefehle (z. B. WinMove, WinHide, WinSet, WinSetTitle und ControlGetFocus) ihre Fenstertitel- und Fenstertext-Parameter weglassen, wenn das GUI-Fenster selbst bearbeitet wird (selbst wenn es versteckt ist).

Das Klicken auf ein Steuerelement zeigt keine Wirkung, falls das g-Label wegen einem vorherigen Klick noch ausgeführt wird. Verwendet bei der Subroutine als erste Zeile Critical, um dieses Problem zu umgehen (allerdings werden dadurch auch andere Threads zwischengespeichert/aufgeschoben, wie beispielsweise beim Drücken eines Hotkeys).

Die internen Variablen A_Gui und A_GuiControl enthalten den Fensternamen und die ID des Steuerelements, das den aktuellen Thread gestartet hat. Siehe A_Gui und A_GuiControl für mehr Details.

Es ist möglich, dass mehrere Ereignisse jeweils die gleiche Subroutine ausführen - setzt dazu ihre Label einfach übereinander. Zum Beispiel:

GuiEscape:
GuiClose:
ButtonCancel:
ExitApp  ; Alle oberen Labels führen diesen Befehl aus.

Alle neuen GUI-Threads beginnen mit Standardwerten bei Einstellungen wie SendMode. Diese können im automatischen Ausführungsbereich geändert werden.

Navigation mit der Tastatur

Es ist möglich, ein GUI-Fenster mit der TAB-Taste zu steuern. Dabei wird der Tastaturfokus auf das nächste eingabefähige Steuerelement geschoben (Steuerelemente mit deaktiviertem Tabstop-Style werden übersprungen). Die Reihenfolge der hinzugefügten Steuerelemente bestimmt die Reihenfolge der Navigation. Anfangs befindet sich der Tastaturfokus auf das erste eingabefähige Steuerelement, das den Tabstop-Style aktiv hat (was die meisten Steuerelemente standardmäßig haben).

Bei einigen Steuerelementen kann ein kaufmännisches Und (&) angegeben werden, um eine Tastenkombination zu erzeugen, die im Steuerelementtext als unterstrichenes Zeichen dargestellt wird (abhängig von den Systemeinstellungen). Der Benutzer kann diese Tastenkombination aktivieren, wenn er die ALT-Taste gedrückt hält und das entsprechende Zeichen auf der Tastatur drückt. Bei Schaltflächen, Kontrollkästchen und Optionsfeldern entspricht das Drücken der Tastenkombination dem Anklicken des Steuerelements. Bei GroupBox- und Text-Steuerelementen bewirkt das Drücken der Tastenkombination, dass der Tastaturfokus auf das erste eingabefähige Tabstop-Steuerelement springt, das danach erstellt wurde. Wenn allerdings mehrere Steuerelemente die gleiche Tastenkombination haben, springt der Tastaturfokus zwischen diesen Steuerelementen abwechselnd hin und her.

Zur direkten Darstellung eines kaufmännischen Und-Zeichens innerhalb der oben genannten Steuerelementarten müssen zwei aufeinanderfolgende kaufmännische Und-Zeichen wie folgt angegeben werden: Save && Exit.

Fensterdarstellung

Ein GUI-Fenster verwendet als Icon das Tray-Icon, das beim Erstellen des Scripts verwendet wurde. Für ein anderes Icon muss daher erst das Tray-Icon geändert werden, noch bevor das Fenster erstellt wird. Zum Beispiel: Menu, Tray, Icon, EigenesIcon.ico. Zudem ist es mit DllCall und SendMessage möglich, das große und kleine Icon unterschiedlich aussehen zu lassen (das große Icon wird in der Programmumschaltung via Alt+Tab angezeigt):

hIcon32 := DllCall("LoadImage", uint, 0
    , str, "Eigenes Icon.ico"  ; Dateiname des Icons (diese Datei kann mehrere Icons enthalten).
    , uint, 1  ; Bildtyp: IMAGE_ICON
    , int, 32, int, 32  ; gewünschte Breite und Höhe des Bilds (hilft LoadImage zu entscheiden, welches Icon am besten geeignet ist).
    , uint, 0x10)  ; Flags: LR_LOADFROMFILE
Gui +LastFound
SendMessage, 0x80, 1, hIcon32  ; 0x80 ist WM_SETICON; und 1 bedeutet ICON_BIG (0 für ICON_SMALL).
Gui Show

Aufgrund von Einschränkungen in Windows XP und höher übernehmen Kontrollkästchen, Optionsfelder und GroupBox-Steuerelemente mit einer benutzerdefinierten Textfarbe die klassische Design-Darstellung.

Verwandtes Thema: Abstand im Fenster.

Allgemeine Bemerkungen

Mit GuiControl und GuiControlGet können beliebige Steuerelemente in einem GUI-Fenster bearbeitet werden.

Jedes GUI-Fenster kann ca. 11000 Steuerelemente enthalten. Allerdings kann das System schon bei 5000 Steuerelementen instabil werden.
