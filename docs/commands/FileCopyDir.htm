FileCopyDir

Kopiert einen Ordner, einschließlich seine Unterordner und Dateien (ähnlich wie xcopy).

FileCopyDir, Quelle, Ziel [, Flag]

Parameter

Quelle Der Name des Quellverzeichnisses (ohne Backslash), das sich im A_WorkingDir-Verzeichnis befindet, solange kein absoluter Pfad angegeben ist. Zum Beispiel: C:\Eigener Ordner
Ziel Der Name des Zielverzeichnisses (ohne Backslash), das sich im A_WorkingDir-Verzeichnis befindet, solange kein absoluter Pfad angegeben ist. Zum Beispiel: C:\Kopie des eigenen Ordners
Flag

(Optional) Dieser Parameter bestimmt, ob Dateien überschrieben werden dürfen, falls sie bereits vorhanden sind:

0 (Standard): Vorhandene Dateien werden nicht überschrieben. Die Operation wird fehlschlagen und keinen Effekt haben, wenn das Ziel bereits als Datei oder Verzeichnis existiert.

1: Vorhandene Dateien werden überschrieben. Allerdings bleiben Dateien oder Unterordner innerhalb vom Ziel erhalten, die kein entsprechendes Gegenstück in der Quelle haben.

Dieser Parameter kann ein Ausdruck sein, auch einer, der als wahr oder falsch gewertet wird (weil True und False intern als 1 und 0 gespeichert sind).

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt. Wenn das Quellverzeichnis gespeicherte Webseiten enthält, bestehend aus Seitenname.htm und dem dazugehörigen Verzeichnis Seitenname_Dateien, könnte dennoch ein Fehler angezeigt werden, obwohl das Kopieren erfolgreich war.

Bemerkungen

Wenn die Verzeichnisstruktur des Ziels nicht vorhanden ist, wird sie erstellt, falls möglich.

Da die Operation einen Ordner mit seinen Unterordnern und Dateien rekursiv kopiert, ist das Ergebnis unbestimmt, wenn ein Ordner irgendwo in sich selbst kopiert wird. Um dieses Problem zu umgehen, muss zuerst eine Kopie des Ordners außerhalb von sich selbst erstellt werden, und diese Kopie dann mit FileMoveDir an die gewünschte Stelle verschoben werden.

FileCopyDir kopiert nur einen einzigen Ordner. Um stattdessen den Inhalt eines Ordners zu kopieren (einschließlich Dateien und Unterordner), siehe Beispiele bei FileCopy.

Verwandte Befehle

FileMoveDir, FileCopy, FileMove, FileDelete, Loop (Datei & Ordner), FileSelectFolder, SplitPath

Beispiele

FileCopyDir, C:\Eigener Ordner, C:\Kopie von Eigener Ordner

; Beispiel #2: Ein Script, der den Benutzer auffordert, einen Ordner zu kopieren.
FileSelectFolder, Quellordner, , 3, Wähle den Ordner`, der kopiert werden soll.
If Quellordner =
    Return
; Ansonsten fortfahren.
FileSelectFolder, Zielordner, , 3, Wähle den Ordner`, der die Kopie des Ordners enthalten soll.
If Zielordner =
    Return
; Ansonsten fortfahren.
MsgBox, 4, , Eine Kopie des Ordners "%Quellordner%" wird in "%Zielordner%" erstellt.  Weiter?
IfMsgBox, No
    Return
SplitPath, Quellordner, Quellordnername ; Nur den Ordnernamen aus dem vollständigen Pfad extrahieren.
FileCopyDir, %Quellordner%, %Zielordner%\%Quellordnername%
If ErrorLevel
    MsgBox Der Ordner konnte nicht kopiert werden, weil vielleicht ein Ordner mit dem selben Namen in "%Zielordner%" bereits existiert.
Return
