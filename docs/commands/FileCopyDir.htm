FileCopyDir

Kopiert einen Ordner, einschließlich seine Unterordner und Dateien (ähnlich wie xcopy).

FileCopyDir, Quelle, Ziel [, Flag]

Parameter

Quelle Name des Quellverzeichnisses (ohne Backslash), das sich im A_WorkingDir-Verzeichnis befindet, solange kein absoluter Pfad angegeben ist. Zum Beispiel: C:\Eigener Ordner
Ziel Name des Zielverzeichnisses (ohne Backslash), das sich im A_WorkingDir-Verzeichnis befindet, solange kein absoluter Pfad angegeben ist. Zum Beispiel: C:\Kopie des eigenen Ordners
Flag

(Optional) Dieser Parameter bestimmt, ob bereits vorhandene Dateien überschrieben werden dürfen:

0 (Standard): Vorhandene Dateien nicht überschreiben. Die Operation wird fehlschlagen und keinen Effekt haben, wenn Ziel bereits als Datei oder Verzeichnis existiert.

1: Vorhandene Dateien überschreiben. Allerdings werden Dateien oder Unterordner innerhalb von Ziel, die kein Gegenstück in Quelle haben, nicht gelöscht.

Dieser Parameter kann ein Ausdruck sein, auch einer, der als wahr oder falsch gewertet wird (weil True und False intern als 1 und 0 gespeichert sind).

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt. Enthält das Quellverzeichnis allerdings gespeicherte Webseiten, bestehend aus Seitenname.htm und dem dazugehörigen Verzeichnis Seitenname_Dateien, könnte dennoch ein Fehler angezeigt werden, auch wenn das Kopieren erfolgreich war.

Bemerkungen

Wenn die Verzeichnisstruktur des Ziels nicht vorhanden ist, wird sie erstellt, falls möglich.

Da die Operation einen Ordner mit seinen Unterordnern und Dateien rekursiv kopiert, ist es nicht definiert, was passieren wird, wenn sich ein Ordner in sich selbst kopiert. Um dieses Problem zu umgehen, muss zuerst eine Kopie des Ordners außerhalb von sich selbst erstellt werden, und diese Kopie dann mit FileMoveDir an die gewünschte Stelle verschoben werden.

FileCopyDir kopiert nur einen einzigen Ordner. Um stattdessen den Inhalt eines Ordners zu kopieren (einschließlich Dateien und Unterordner), siehe Beispiele bei FileCopy.

Verwandte Befehle

FileMoveDir, FileCopy, FileMove, FileDelete, Loop (Datei & Ordner), FileSelectFolder, SplitPath

Beispiele

FileCopyDir, C:\Eigener Ordner, C:\Kopie von Eigener Ordner

; Beispiel #2: Ein Script, der den Benutzer auffordert, einen Ordner zu kopieren.
FileSelectFolder, Quellordner, , 3, Wähle den Ordner`, der kopiert werden soll.
If Quellordner =
    Return
; Ansonsten fortfahren.
FileSelectFolder, Zielordner, , 3, Wähle den Ordner`, der die Kopie des Ordners enthalten soll.
If Zielordner =
    Return
; Ansonsten fortfahren.
MsgBox, 4, , Eine Kopie des Ordners "%Quellordner%" wird in "%Zielordner%" erstellt.  Weiter?
IfMsgBox, No
    Return
SplitPath, Quellordner, Quellordnername ; Nur den Ordnernamen aus dem vollständigen Pfad extrahieren.
FileCopyDir, %Quellordner%, %Zielordner%\%Quellordnername%
If ErrorLevel
    MsgBox Der Ordner konnte nicht kopiert werden, weil vielleicht ein Ordner mit dem selben Namen in "%Zielordner%" bereits existiert.
Return
