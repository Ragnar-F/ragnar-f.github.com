SplitPath

Teilt ein Dateiname oder eine URL in Name, Verzeichnis, Erweiterung und Laufwerk auf.

SplitPath, Eingabevariable [, Dateiname, Verzeichnis, Erweiterung, NameOhneErw, Laufwerk]

Parameter

Eingabevariable Name der Variable, die den Dateinamen zur Analyse enthält.
Dateiname Der Name der Ausgabevariable, in der der Dateiname ohne Pfad gespeichert wird. Die Dateierweiterung ist inbegriffen.
Verzeichnis Der Name der Ausgabevariable, in der das Verzeichnis der Datei, einschließlich Laufwerksbuchstabe oder Freigabename (falls vorhanden), gespeichert wird. Der letzte Backslash ist nicht enthalten, auch dann nicht, wenn sich die Datei im Wurzelverzeichnis des Laufwerks befindet.
Erweiterung Der Name der Ausgabevariable, in der die Dateierweiterung gespeichert wird (z. B. TXT, DOC oder EXE). Der Punkt (.) ist nicht inbegriffen.
NameOhneErw Der Name der Ausgabevariable, in der der Dateiname ohne Pfad, Punkt und Erweiterung gespeichert wird.
Laufwerk Der Name der Ausgabevariable, in der der Laufwerksbuchstabe oder Servername der Datei gespeichert wird. Wenn sich die Datei auf einem lokalen oder zugeordneten Laufwerk befindet, wird die Variable den Laufwerksbuchstaben gefolgt von einem Doppelpunkt enthalten (ohne Backslash). Befindet sich die Datei auf einem Netzwerkpfad (UNC), wird die Variable den Freigabenamen enthalten (z. B. \\Workstation01).

Bemerkungen

Falls eine bestimmte Information nicht benötigt wird, kann die entsprechende Ausgabevariable weggelassen werden.

Wenn Eingabevariable einen Dateinamen ohne Laufwerksbuchstaben enthält (also ohne Pfad oder nur einen relativen Pfad), wird Laufwerk leer gemacht, aber alle anderen Ausgabevariablen normal gesetzt. Ist kein Pfad vorhanden, wird Verzeichnis leer gemacht; ist ein Pfad, aber kein Dateiname vorhanden, wird Dateiname und NameOhneErw leer gemacht.

Derzeitige Dateien und Verzeichnisse im Dateisystem werden nicht von diesem Befehl überprüft. Er analysiert einfach nur den angegebenen String in Eingabevariable.

Platzhalter (* und ?) und andere Zeichen, die im Dateinamen nicht erlaubt sind, können in der Eingabevariable vorkommen. Das gilt nicht für Doppelpunkt, Backslash und Punkt, da diese als Abgrenzung für Laufwerksbuchstaben, Verzeichnisse und Erweiterungen dienen.

Unterstützung für URLs: Wenn Eingabevariable einen Doppelpunkt und zwei Schrägstriche enthält (z. B. http://domain.com oder ftp://domain.com), wird Verzeichnis auf Protokoll-Präfix + Domänenname + Verzeichnis (z. B. http://domain.com/images) und Laufwerk auf Protokoll-Präfix + Domännename (z. B. http://domain.com) gesetzt. Alle anderen Ausgabevariablen werden in Bezug auf ihren oben genannten Definitionen gesetzt.

Verwandte Befehle

A_LoopFileExt, StringSplit, StringGetPos, StringMid, StringTrimLeft, StringLeft, FileSelectFile, FileSelectFolder

Beispiel

KompletterDateiname = C:\Eigene Dokumente\Adressliste.txt

; Um nur den Dateinamen ohne Pfad abzurufen:
SplitPath, KompletterDateiname, Dateiname

; Um nur das Verzeichnis abzurufen:
SplitPath, KompletterDateiname,, Verzeichnis

; Um alle Informationen abzurufen:
SplitPath, KompletterDateiname, Name, Verz, Erw, NameOhneErw, Laufwerk

; Die oben genannten Variablen werden wie folgt gesetzt:
; Name = Adressliste.txt
; Verz = C:\Eigene Dokumente
; Erw = txt
; NameOhneErw = Adressliste
; Laufwerk = C:
