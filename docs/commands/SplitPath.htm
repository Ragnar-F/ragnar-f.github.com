SplitPath

Teilt ein Dateiname oder eine URL in Name, Verzeichnis, Erweiterung und Laufwerk auf.

SplitPath, Eingabevariable [, Dateiname, Verzeichnis, Erweiterung, NameOhneErw, Laufwerk]

Parameter

Eingabevariable Name der Variable, die den Dateinamen zur Analyse enthält.
Dateiname Der Name der Ausgabevariable, in der der Dateiname ohne Pfad gespeichert wird. Die Dateierweiterung ist inbegriffen.
Verzeichnis Der Name der Ausgabevariable, in der das Verzeichnis der Datei, einschließlich Laufwerksbuchstaben oder Freigabenamen (wenn vorhanden) gespeichert werden. Der letzte Backslash ist nicht enthalten, selbst wenn sich die Datei in einem Wurzelverzeichnis befindet.
Erweiterung Der Name der Ausgabevariable, in der die Dateierweiterung gespeichert wird (z. B. TXT, DOC oder EXE). Der Punkt (.) ist nicht inbegriffen.
NameOhneErw Der Name der Ausgabevariable, in der der Dateiname ohne Pfad, Punkt und Erweiterung gespeichert wird.
Laufwerk Der Name der Ausgabevariable, in der der Laufwerksbuchstabe oder Servername der Datei gespeichert wird. Wenn sich die Datei auf einem lokalen oder zugeordneten Laufwerk befindet, dann wird die Variable den Laufwerksbuchstaben gefolgt von einem Doppelpunkt enthalten (ohne Backslash). Wenn sich die Datei auf einem Netzwerkpfad (UNC) befindet, dann wird die Variable den Freigabenamen enthalten (z. B. \\Workstation01).

Bemerkungen

Jede Ausgabevariable kann weggelassen werden, wenn sie nicht gebraucht wird.

Wenn die Eingabevariable einen Dateinamen ohne Laufwerksbuchstaben enthält (also ohne Pfad oder nur einen relativen Pfad), dann wird das Laufwerk leer gemacht, alle anderen Ausgabevariablen jedoch richtig gesetzt. Wenn kein Pfad vorhanden ist, dann wird das Verzeichnis leer gemacht; wenn ein Pfad ohne Dateiname vorhanden ist, dann wird der Dateiname und NameOhneErw leer gemacht.

Gegenwärtige Dateien und Verzeichnisse im Dateisystem werden nicht von diesem Befehl überprüft. Er analysiert einfach den angegebenen String in der Eingabevariable.

Platzhalter (* und ?) und andere illegale Zeichen im Dateinamen werden wie legale Zeichen behandelt, wobei Doppelpunkte, Backslashs und Punkte eine Ausnahme bilden, da sie als Abgrenzung für Laufwerksbuchstaben, Verzeichnisse und Erweiterungen dienen.

Unterstützung für URLs: Wenn die Eingabevariable einen Doppelpunkt und zwei Schrägstriche enthält (z. B. http://domain.com oder ftp://domain.com), dann wird das Verzeichnis Protokoll-Präfix + Domänenname + Verzeichnis (z. B. http://domain.com/images) und das Laufwerk Protokoll-Präfix + Domännename (z. B. http://domain.com) enthalten. Alle anderen Ausgabevariablen werden nach ihrer Definition gesetzt.

Verwandte Befehle

A_LoopFileExt, StringSplit, StringGetPos, StringMid, StringTrimLeft, StringLeft, FileSelectFile, FileSelectFolder

Beispiel

KompletterDateiname = C:\Eigene Dokumente\Adressliste.txt

; Um nur den Dateinamen ohne Pfad abzurufen:
SplitPath, KompletterDateiname, Dateiname

; Um nur das Verzeichnis abzurufen:
SplitPath, KompletterDateiname,, Verzeichnis

; Um alle Informationen abzurufen:
SplitPath, KompletterDateiname, Name, Verz, Erw, NameOhneErw, Laufwerk

; Die oben genannten Variablen werden wie folgt gesetzt:
; Name = Adressliste.txt
; Verz = C:\Eigene Dokumente
; Erw = txt
; NameOhneErw = Adressliste
; Laufwerk = C:
