InputBox

Zeigt ein Fenster zur Eingabe eines Strings an.

InputBox, AusgabeVar [, Titel, Anzeigetext, HIDE, Breite, Höhe, X, Y, Schriftart, Zeitüberschreitung, Standardwert]

Parameter

AusgabeVar Name der Variable, in der die Benutzereingabe gespeichert werden soll.
Titel Der Titel der InputBox. Ist dieser Parameter leer oder nicht vorhanden, wird standardmäßig der aktuelle Name des Scripts verwendet.
Anzeigetext Text des Eingabefensters, der darauf hinweist, was eingegeben werden soll. Falls der Anzeigetext zu lang ist, kann dieser in mehreren kurzen Zeilen aufgeteilt werden (Fortsetzungsbereich), um die Lesbarkeit und Wartbarkeit zu verbessern.
HIDE Sobald dieser Parameter das Wort HIDE enthält, wird die Benutzereingabe wie bei einer Passworteingabe mit Maskierungszeichen ersetzt.
Breite Ist dieser Parameter leer oder nicht vorhanden, wird standardmäßig eine Breite von 375 Pixel verwendet. Dieser Parameter kann ein Ausdruck sein.
Höhe Ist dieser Parameter leer oder nicht vorhanden, wird standardmäßig eine Höhe von 189 Pixel verwendet. Dieser Parameter kann ein Ausdruck sein.
X, Y Die Koordinaten X und Y des Fensters (mit 0,0 wird das Fenster in die linke obere Ecke des Bildschirms geschoben), die Ausdrücke sein können. Sobald eine der Koordinaten leer oder nicht vorhanden ist, wird das Fenster in dieser Abmessung zentriert. Jede Koordinate kann negativ sein, um das Fenster teilweise oder vollständig über den Bildschirmrand hinaus zu positionieren.
Schriftart Noch nicht implementiert (leer lassen). In Zukunft könnte dieser Parameter so etwas ähnliches wie verdana:8 akzeptieren.
Zeitüberschreitung Zeitüberschreitung in Sekunden (kann einen Dezimalpunkt oder Ausdruck enthalten). Der Parameter wird automatisch auf 2147483 (24.8 Tage) zurückgesetzt, falls dieser Wert überschritten wird. Sobald die Zeit abgelaufen ist, wird das Eingabefenster automatisch geschlossen und ErrorLevel auf 2 gesetzt. AusgabeVar enthält weiterhin die Benutzereingabe.
Standardwert Ein String, der anfänglich im Eingabefeld des Dialogfensters angezeigt wird. Der Benutzer kann den String z. B. durch Drücken der Rücktaste ändern.

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

Siehe unten.

Bemerkungen

Mit diesem Dialogfenster kann der Benutzer einen Text eingeben und danach OK oder CANCEL (Abbrechen) drücken. Durch Ziehen der Rahmen kann dessen Größe geändert werden.

ErrorLevel wird auf 1 gesetzt, wenn der Benutzer die Schaltfläche CANCEL drückt, auf 0, wenn der Benutzer OK drückt, oder auf 2, wenn das Dialogfenster die Zeit überschritten hat. In allen drei Fällen wird AusgabeVar auf den eingegebenen Wert gesetzt. Dadurch kann die Schaltfläche CANCEL außer Abbrechen noch andere Funktionen ausführen, falls der Entwickler das wünscht.

Ein GUI-Fenster kann eine modale InputBox mittels Gui, +OwnDialogs anzeigen. Eine modale InputBox hindert den Benutzer daran, mit dem GUI-Fenster zu interagieren, bis die InputBox wieder geschlossen wird.

Verwandte Befehle

GUI, Input, MsgBox, FileSelectFile, FileSelectFolder, SplashTextOn, ToolTip

Beispiel

InputBox, Passwort, Passwort eingeben, (die Eingabe wird versteckt), hide
InputBox, Benutzereingabe, Telefonnummer, Bitte geben sie eine Telefonnummer ein., , 640, 480
If ErrorLevel
    MsgBox, Sie haben CANCEL gedrückt.
Else
    MsgBox, Sie haben "%Benutzereingabe%" eingegeben.
