WinActivate

Aktiviert das angegebene Fenster (setzt es in den Vordergrund).

WinActivate [, Fenstertitel, Fenstertext, Titelausnahme, Textausnahme]

Parameter

Fenstertitel Der Titel oder Teiltitel des Fensters (das Übereinstimmungsverhalten wird mit SetTitleMatchMode bestimmt). Wenn dieser und die anderen 3 Fensterparameter nicht vorhanden sind, dann wird das zuletzt gefundene Fenster aktiviert. Bei Verwendung einer Fensterklasse muss ahk_class ExakterKlassenname angegeben werden (wird von Windows Spy angezeigt). Bei Verwendung einer Prozess-Kennung (PID) muss ahk_pid %VariableMitPID% angegeben werden. Bei Verwendung einer Fenstergruppe muss ahk_group Gruppenname angegeben werden. Bei Verwendung einer eindeutigen ID-Nummer eines Fensters muss ahk_id %VariableMitID% angegeben werden. Die Suche kann durch Angabe mehrerer Kriterien eingeschränkt werden. Zum Beispiel: Eigene Datei.txt ahk_class Notepad
Fenstertext In diesem Parameter kann ein Teilstring angegeben werden, der in einem Textelement des Zielfensters vorkommt (werden mit Windows Spy angezeigt). Versteckte Textelemente werden auch erkannt, sofern DetectHiddenText aktiv ist.
Titelausnahme Ignoriert Fenster, deren Titel diesen Wert enthalten.
Textausnahme Ignoriert Fenster, deren Textelemente diesen Wert enthalten.

Bemerkungen

Wenn das Fenster minimiert ist, dann wird es automatisch wiederhergestellt, bevor es aktiviert wird.

Sechs Versuche werden innerhalb von 60ms unternommen, um das Zielfenster zu aktivieren. Daher ist es normalerweise nicht notwendig, WinWaitActive oder IfWinNotActive nach WinActivate auszuführen.

Wenn ein übereinstimmendes Fenster bereits aktiv ist, dann bleibt dieses Fenster aktiv, anstatt ein anderes übereinstimmendes Fenster darunter zu aktivieren. Wenn mehr als ein Fenster übereinstimmt, dann wird das oberste (aktuellste verwendete) Fenster aktiviert. Mit WinActivateBottom kann das unterste (am wenigsten verwendete) Fenster aktiviert werden.

Wenn ein Fenster aktiviert wird, nachdem ein anderes Fenster aktiviert wurde, dann fangen die Schaltflächen der Taskleiste an zu blinken (abhängig vom Betriebssystem und von den Einstellungen). Mit #WinActivateForce kann das verhindert werden.

Fenstertitel und -texte sind von der Groß- und Kleinschreibung abhängig. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.

Verwandte Befehle

WinActivateBottom, #WinActivateForce, SetTitleMatchMode, DetectHiddenWindows, Zuletzt gefundenes Fenster, IfWinExist, IfWinActive, WinWaitActive, WinWait, WinWaitClose, WinClose, GroupActivate, WinSet

Beispiel

IfWinExist, Unbenannt - Editor
    WinActivate ; zuletzt gefundenes Fenster verwenden
Else
    WinActivate, Rechner
