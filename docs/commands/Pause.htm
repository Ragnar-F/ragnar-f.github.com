Pause

Pausiert einen aktuellen Thread des Scripts.

#p::Pause ; Win+P einmal drücken, um Script zu pausieren. Nochmal drücken, um die Pause aufzuheben.
Pause [, On|Off|Toggle, DarunterLiegendenThreadEinbeziehen?]

Parameter

On|Off|Toggle

Ist dieser Parameter leer oder nicht vorhanden, wird standardmäßig Toggle verwendet. Ansonsten ein folgendes Wort angeben:

Toggle: Pausiert den aktuellen Thread, es sei denn, der Thread darunter ist pausiert, in diesem Fall wird die Pause vom darunter liegenden Thread aufgehoben.

On: Pausiert den aktuellen Thread.

Off: Wenn der Thread unter dem aktuellen Thread pausiert ist, dann wird dessen Pause aufgehoben, wenn er wieder aufgenommen wird. Ansonsten hat der Befehl keine Wirkung.
DarunterLiegendenThreadEinbeziehen?

Dieser Parameter wird bei "Pause Off" ignoriert, weil dabei immer der darunter liegende Thread bearbeitet wird. Bei den anderen wird er ignoriert, es sei denn, Pause ist aktiviert (auch über Toggle).

Verwendet eine folgende Zahl:

0 (oder weggelassen): Der Befehl pausiert den aktuellen Thread; das heißt, dass dieser Thread jetzt den Pause-Befehl ausführt.

1: Der Befehl setzt den Thread unter dem aktuellen Thread auf pausiert, damit der ausgeführte Befehl beendet wird (falls vorhanden), wenn er wieder aufgenommen wird. Gebt danach einen pausierten Zustand an. Wenn kein Thread unter dem aktuellen Thread ist, dann wird das Script selbst pausiert, wodurch Timer gestoppt werden (entspricht das Auswählen vom Menüpunkt "Pause Script", während das Script keine Threads hat).

Hinweis: A_IsPaused enthält den Zustand von Pause des darunter liegenden Threads.

Bemerkungen

Im Gegensatz zum Befehl Suspend (der Hotkeys und Hotstrings deaktiviert) wird der aktuelle Thread blockiert, wenn Pause aktiviert ist. Als Nebeneffekt ist jeder darunter liegende unterbrochene Thread untätig.

Sobald ein Thread pausiert wird, werden Timer nicht ausgeführt. Direkt gestartete Threads wie Hotkeys und Menüpunkte können weiterhin ausgeführt werden; sobald jedoch deren Threads enden, bleibt der darunter liegende Thread weiterhin pausiert. Das heißt, dass jeder Thread unabhängig von den anderen pausiert werden kann.

Die Farbe des Tray-Icons wechselt von Grün auf Rot, sobald der aktuelle Thread des Scripts pausiert wird. Diese Farbveränderung kann durch Blockieren des Icons verhindert werden, wenn eine 1 im letzten Parameter des Menu-Befehls verwendet wird. Zum Beispiel:
Menu, Tray, Icon, C:\Eigenes Icon.ico, , 1

Um Timer ohne Pause zu deaktivieren, verwendet "Thread, NoTimers".

Der Pause-Befehl ist mit dem Menüpunkt "Pause Script" vergleichbar.

Ein Script wird immer angehalten (wenn auch nicht offiziell pausiert), wenn ein Menü angezeigt wird (Tray-Menü, Menüleiste, GUI-Kontextmenü, usw.)

Verwandte Befehle

Suspend, Menu, ExitApp, Threads, SetTimer

Beispiele

Pause::Pause  ; Die Pause-Funktion wird der Pause-Taste zugewiesen ...
#p::Pause  ; ... oder der Tastenkombination Win+p zugewiesen.
