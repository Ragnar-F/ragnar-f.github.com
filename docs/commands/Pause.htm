Pause

Pausiert den aktuellen Thread vom Script.

#p::Pause ; Drückt einmal WIN+P, um das Script zu pausieren. Drückt nochmal, um die Pause aufzuheben.
Pause [, On|Off|Toggle, DarunterliegendenThreadEinbeziehen?]

Parameter

On|Off|Toggle

Ist dieser Parameter leer oder nicht vorhanden, wird standardmäßig Toggle verwendet. Ansonsten kann eins der folgenden Wörter angegeben werden:

Toggle: Pausiert den aktuellen Thread, es sei denn, der Thread darunter ist pausiert, in diesem Fall wird die Pause vom darunterliegenden Thread aufgehoben.

On: Pausiert den aktuellen Thread.

Off: Wenn der Thread unter dem aktuellen Thread pausiert ist, wird dessen Pause bei Wiederaufnahme aufgehoben. Ansonsten hat der Befehl keine Wirkung.
DarunterliegendenThreadEinbeziehen?

Dieser Parameter wird bei Pause Off ignoriert, weil dabei immer der darunterliegende Thread einbezogen wird. Bei den anderen wird er ignoriert, es sei denn, Pause ist aktiviert (auch über Toggle).

Es kann eine der folgenden Zahlen angegeben werden:

0 (oder weggelassen): Der Befehl pausiert den aktuellen Thread; das heißt, dass dieser Thread jetzt den Pause-Befehl betreibt.

1: Der Befehl markiert den Thread unter dem aktuellen Thread als pausiert, so dass dieser bei Wiederaufnahme den zurzeit laufenden Befehl (falls vorhanden) fertig ausführt und danach in einen pausierten Zustand versetzt wird. Falls unter dem aktuellen Thread kein weiterer Thread vorhanden ist, wird das Script selbst pausiert, wodurch Timer gestoppt werden (es wird das gleiche erreicht, wenn der Menüpunkt "Pause Script" ausgewählt wird, während das Script keine Threads hat).

Hinweis: A_IsPaused enthält den Pausezustand vom darunterliegenden Thread.

Bemerkungen

Im Gegensatz zum Befehl Suspend - der Hotkeys und Hotstrings deaktiviert - bewirkt das Pausieren, dass der aktuelle Thread eingefroren wird. Als Nebeneffekt werden alle unterbrochenen Thread darunter inaktiv sein.

Sobald ein Thread pausiert ist, werden Timer nicht ausgeführt. Ausdrücklich gestartete Threads wie Hotkeys und Menüpunkte können hingegen weiterhin ausgeführt werden; sobald jedoch deren Threads enden, wird der darunterliegende Thread weiterhin pausiert sein. Das heißt, dass jeder Thread unabhängig von den anderen pausiert werden kann.

Die Farbe des Tray-Icons wechselt von Grün auf Rot, sobald der aktuelle Thread vom Script im pausierten Zustand ist. Dieser Farbwechsel kann durch Einfrieren des Icons verhindert werden - das wird erreicht, wenn im letzten Parameter des Menu-Befehls eine 1 angegeben wird. Zum Beispiel:

Menu, Tray, Icon, C:\Eigenes Icon.ico, , 1

Mit Thread, NoTimers können Timer deaktiviert werden, ohne dabei das ganze Script pausieren zu müssen.

Der Pause-Befehl ist mit dem internen Menüpunkt "Pause Script" vergleichbar.

Ein Script wird immer angehalten (wenn auch nicht offiziell pausiert), wenn ein Menü angezeigt wird (Tray-Menü, Menüleiste, GUI-Kontextmenü und so weiter).

Verwandte Befehle

Suspend, Menu, ExitApp, Threads, SetTimer

Beispiele

Pause::Pause  ; Bewirkt, dass bei Drücken der PAUSE-Taste der Pausezustand des Scripts umgeschalten wird.
#p::Pause  ; Das gleiche wie oben, aber mit der Tastenkombination WIN+P.
; Sendet ein Pause-Befehl an ein anderes Script.
DetectHiddenWindows, On
WM_COMMAND := 0x111
ID_FILE_PAUSE := 65403
PostMessage, WM_COMMAND, ID_FILE_PAUSE,,, C:\EigenesScript.ahk ahk_class AutoHotkey
