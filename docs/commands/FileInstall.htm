FileInstall

Fügt die angegebene Datei in die kompilierte Version des Scripts ein.

FileInstall, Quelle, Ziel [, Flag]

Parameter

Quelle

Der Name der Datei, die zur kompilierten EXE-Datei hinzugefügt wird. Die Datei befindet sich im (oder ist relativ zum) Arbeitsverzeichnis des Scripts, solange kein absoluter Pfad angegeben ist.

Der Dateiname darf keine Anführungszeichen, Variablenverweise (z. B.%A_ProgramFiles%) oder Platzhalter enthalten. Darüber hinaus müssen Sonderzeichen wie direkte Prozentzeichen und Kommas geschützt werden (wie in jedem anderen Befehlsparameter auch). Schließlich muss dieser Parameter rechts von FileInstall erfolgen (d. h. nicht auf eine Fortsetzungszeile darunter).
Ziel Der neue Name der Datei, wenn die Quelle aus der EXE-Datei extrahiert wird. Die Datei befindet sich im A_WorkingDir-Verzeichnis, solange kein absoluter Pfad angegeben ist. Das Zielverzeichnis muss bereits vorhanden sein. Im Gegensatz zur Quelle können hier Variablenverweise eingesetzt werden.
Flag

(Optional) Dieser Parameter bestimmt, ob Dateien überschrieben werden dürfen, falls sie bereits vorhanden sind:

0 = (Standard) vorhandene Dateien nicht überschreiben
1 = vorhandene Dateien überschreiben

Dieser Parameter kann ein Ausdruck sein, auch einer, der als wahr oder falsch gewertet wird (weil True und False intern als 1 und 0 gespeichert sind).

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

Bemerkungen

Dieser Befehl ist eine Direktive für den Ahk2Exe-Compiler, die es erlaubt, zusätzliche Dateien in das kompilierte Script einzufügen. Sobald das kompilierte Script ausgeführt wird, werden diese Dateien wieder auf das Laufwerk extrahiert.

Die Datei wird während der Script-Kompilierung hinzugefügt. Sobald das kompilierte Script ausgeführt und FileInstall erreicht wird, wird die Datei ins Ziel extrahiert.

Dateien, die auf diese Weise hinzugefügt werden, sind komprimiert als auch verschlüsselt.

Wenn dieser Befehl in einem normalen (nicht kompilierten) Script verwendet wird, erfolgt stattdessen ein einfaches Kopieren der Datei (das hilft dabei, Scripts zu testen, die eventuell kompiliert werden).

Verwandte Befehle

FileCopy, #Include

Beispiel

FileInstall, C:\Eigene Dokumente\Eigene Datei.txt, %A_ProgramFiles%\Eigene Anwendung\Readme.txt, 1
