GroupAdd

Fügt eine Fensterspezifikation zu einer Fenstergruppe hinzu, und erstellt die Gruppe, falls erforderlich.

GroupAdd, Gruppenname [, Fenstertitel, Fenstertext, Label, Titelausnahme, Textausnahme]

Parameter

Gruppenname

Name der Gruppe, die die Fensterspezifikation erhalten soll. Falls die Gruppe nicht existiert, wird sie erstellt. Gruppennamen unterscheiden nicht zwischen Groß- und Kleinschreibung.

Fenstertitel

Ein Fenstertitel oder sonstige Kriterien, die das Zielfenster ermitteln sollen. Siehe Fenstertitel.

Fenstertext

In diesem Parameter kann ein Teilstring angegeben werden, der in einem Textelement des Zielfensters vorkommt (abrufbar mit Windows Spy). Versteckte Textelemente werden auch erkannt, sofern DetectHiddenText aktiv ist, während GroupActivate, GroupDeactivate und GroupClose verwendet wird.

Label

Label einer Subroutine, das ausgeführt werden soll, falls kein einziges Fenster beim Verwenden des GroupActivate-Befehls mit dieser Gruppe (oder mit dieser Fensterbeschreibung vor AHK_L 54) übereinstimmt. Der Sprung zum Label verhält sich wie bei einem Gosub. Dieser Parameter kann leer- oder weggelassen werden.

Titelausnahme

Ignoriert Fenster, deren Titel diesen Wert enthalten.

Textausnahme

Ignoriert Fenster, deren Textelemente diesen Wert enthalten.

Bemerkungen

Jedes Mal, wenn dieser Befehl benutzt wird, wird eine neue Regel hinzugefügt. Das heißt, dass eine Gruppe aus einer Reihe von Kriterien besteht, anstatt einer feste Liste von Fenstern. Wenn eine Gruppe später von Befehlen wie GroupActivate verwendet wird, wird jedes Fenster auf dem Desktop mit diesen Kriterien verglichen. Stimmt ein Fenster mit einem der Kriterien aus der Gruppe überein, ist die Übereinstimmung erfolgreich.

SetTitleMatchMode und DetectHiddenWindows haben keinen direkten Einfluss darauf, wie sich dieser Befehl verhalten soll. Allerdings beeinflussen sie die anderen Gruppenbefehle wie GroupActivate und GroupClose, und auch ahk_group, das in Fenstertitel-Parametern von anderen Befehlen verwendet wird.

Eine Fenstergruppe wird üblicherweise dazu verwendet, mehrere verwandte Fenster zu verbinden. Das ist besonders bei Aufgaben nützlich ist, die viele verwandte Fenster umfassen, oder bei einer Anwendung, die mehrere Unterfenster besitzt. Arbeitet man beispielsweise sehr oft mit mehreren Instanzen von Grafikprogrammen oder Texteditoren, könnte man GroupActivate in Verbindung mit einem Hotkey verwenden, um jede Instanz solcher Programme einzeln durchzugehen, ohne Alt+Tab oder die Taskleiste benutzen zu müssen.

Da die Einträge in jeder Gruppe nur einmal hinzugefügt werden müssen, wird dieser Befehl in der Regel im automatischen Ausführungsbereich verwendet (oberster Bereich des Scripts). Doppelt vorkommende Einträge werden ignoriert.

Um alle Fenster in einer Gruppe einzufügen (außer Programm-Manager), verwende dieses Beispiel:

GroupAdd, AllWindows

Alle Fensterbefehle können eine Fenstergruppe bearbeiten, wenn ahk_group MeinGruppenname im Fenstertitel-Parameter verwendet wird. Die Befehle WinMinimize, WinMaximize, WinRestore, WinHide, WinShow, WinClose und WinKill beeinflussen alle Fenster einer Gruppe. Um stattdessen nur das oberste Fenster zu beeinflussen, befolge dieses Beispiel:

WinHide % "ahk_id " . WinExist("ahk_group MeineGruppe")

Andere Fensterbefehle wie WinActivate und IfWinExist bearbeiten hingegen nur das oberste Fenster einer Gruppe.
