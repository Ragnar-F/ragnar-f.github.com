GroupAdd

Fügt eine Fensterspezifikation zur Fenstergruppe hinzu und erstellt je nach Bedarf die Gruppe.

GroupAdd, Gruppenname [, Fenstertitel, Fenstertext, Label, Titelausnahme, Textausnahme]

Parameter

Gruppenname Der Name der Gruppe, der die Fenster hinzugefügt werden. Wenn die Gruppe nicht existiert, dann wird sie erstellt. Gruppennamen sind nicht von der Groß- bzw. Kleinschreibung abhängig.
Fenstertitel

Ein Fenstertitel oder sonstige Kriterien, die das Zielfenster ermitteln sollen. Siehe Fenstertitel.
Fenstertext Wenn vorhanden, dann muss dieser Parameter einen Teil der einzelnen Textelemente des Fensters enthalten (wird von Windows Spy angezeigt). Versteckte Textelemente werden erfasst, wenn DetectHiddenText aktiv ist, während GroupActivate, GroupDeactivate und GroupClose verwendet wird.
Label Die Bezeichnung einer auszuführenden Subroutine, wenn kein Fenster zur Gruppe (oder zur Fensterbeschreibung vor AHK_L 54) passt, während der GroupActivate-Befehl verwendet wird. Der Sprung zum Label verhält sich wie bei einem Gosub. Dieser Parameter kann leer- oder weggelassen werden.
Titelausnahme Ignoriert Fenster, deren Titel diesen Wert enthalten.
Textausnahme Ignoriert Fenster, deren Textelemente diesen Wert enthalten.

Bemerkungen

Jedes Mal wird eine neue Regel hinzugefügt, wenn dieser Befehl benutzt wird. Das heißt, dass eine Gruppe aus einer Reihe von Kriterien besteht, anstatt einer feste Liste von Fenstern. Wenn eine Gruppe später von Befehlen wie GroupActivate verwendet wird, dann wird jedes Fenster auf dem Desktop mit diesen Kriterien verglichen. Wenn ein Fenster mit einem Kriterium der Gruppe übereinstimmt, dann ist die Übereinstimmung erfolgreich.

Obwohl SetTitleMatchMode und DetectHiddenWindows nicht direkt Einfluss auf das Verhalten diesen Befehls haben, beeinflussen sie dennoch andere Gruppenbefehle wie GroupActivate und GroupClose. Sie beeinflussen auch das Benutzen von ahk_group in Fenstertitel-Parametern von anderen Befehlen.

Für gewöhnlich werden verwandte Fenster miteinander verbunden, besonders für Aufgaben, die solche Fenster mit einbeziehen, oder für eine Anwendung, die mehrere Unterfenster besitzt. Wenn der Benutzer beispielsweise sehr oft mit mehreren Instanzen von Grafikprogrammen oder Texteditoren arbeitet, dann kann er GroupActivate in Verbindung mit einem Hotkey verwenden, um jede Instanz solcher Programme hintereinander aufzusuchen, ohne dass er dazu Alt+Tab oder die Taskleiste benutzt.

Da die Einträge in jeder Gruppe nur einmal aufgenommen werden müssen, wird dieser Befehl in der Regel im automatischen Ausführungsbereich verwendet (oberster Bereich des Scripts). Doppelt vorkommende Einträge werden ignoriert.

Um alle Fenster in einer Gruppe mit einzuschließen (außer Programm-Manager), verwendet dieses Beispiel:

GroupAdd, AllWindows

Alle Fensterbefehle können eine Fenstergruppe bearbeiten, wenn ahk_group EigenerGruppenname im Fenstertitel-Parameter verwendet wird. Die Befehle WinMinimize, WinMaximize, WinRestore, WinHide, WinShow, WinClose und WinKill beeinflussen alle Fenster einer Gruppe. Um stattdessen nur das oberste Fenster zu beeinflussen, befolgt dieses Beispiel:

WinHide % "ahk_id " . WinExist("ahk_group EigeneGruppe")

Andere Fensterbefehle wie WinActivate und IfWinExist bearbeiten hingegen nur das oberste Fenster einer Gruppe.

Verwandte Befehle

GroupActivate, GroupDeactivate, GroupClose
