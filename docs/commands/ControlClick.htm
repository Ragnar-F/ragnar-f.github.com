ControlClick

Sendet eine Maus- oder Mausradaktion an einem Steuerelement.

ControlClick [, Steuerelement-oder-Pos, Fenstertitel, Fenstertext, Taste, Klickanzahl, Optionen, Titelausnahme, Textausnahme]

Parameter

Steuerelement-oder-Pos

Wenn dieser Parameter leer ist, wird das oberste Steuerelement des Fensters benutzt (oder das Fenster selbst, falls keine Steuerelemente vorhanden sind). Ansonsten kann ein Modus von unten verwendet werden.

1. Modus (Position): Koordinaten X und Y beziehen sich auf die obere linke Ecke des Zielfensters. Die X-Koordinate muss vor der Y-Koordinate angegeben werden, mit mindestens einem Leer- oder Tabulatorzeichen dazwischen. Zum Beispiel: X55 Y33. Falls sich auf den angegebenen Koordinaten ein Steuerelement befindet, wird dieser Befehl eine Klickaktion auf genau diesen Koordinaten ausführen. Ist dort kein Steuerelement, wird die Aktion an das Fenster selbst gesendet (funktioniert nicht bei jedem Fenster). Hinweis: In diesem Modus werden die Optionsbuchstaben X und Y ignoriert.

2. Modus (ClassNN oder Text): Möglich ist entweder ClassNN (Klassenname und Instanznummer des Steuerelements) oder Name/Text des Steuerelements, die beide mit Windows Spy ermittelt werden können. Beim  Name/Text wird das Übereinstimmungsverhalten mit SetTitleMatchMode bestimmt.

Standardmäßig hat der zweite Modus Vorrang vor dem ersten Modus. Zum Beispiel würde bei einem Steuerelement, dessen Text und Klassenname das Format "Xnnn Ynnn" haben (eher unwahrscheinlich), der zweite Modus in Kraft treten. Um den ersten Modus dennoch zu erwirken, fügt das Wort "Pos" wie folgt in den Optionen ein: ControlClick, x255 y152, Fenstertitel,,,, Pos.

Um das HWND (Windows Handle) eines Steuerelements geltend zu machen, muss dieser Parameter leer gelassen und ahk_id %SteuerelementHwnd% im Fenstertitel-Parameter angegeben werden (funktioniert auch bei versteckten Steuerelementen, selbst wenn DetectHiddenWindows deaktiviert ist). Das HWND eines Steuerelements wird üblicherweise mittels ControlGet Hwnd, MouseGetPos oder DllCall abgerufen.
Fenstertitel Der Titel oder Teiltitel des Fensters (das Übereinstimmungsverhalten wird mit SetTitleMatchMode bestimmt). Wenn dieser und die anderen 3 Fensterparameter weggelassen werden, wird das zuletzt gefundene Fenster verwendet. Sobald dieser den Buchstaben A enthält und die anderen 3 Fensterparameter weggelassen werden, wird das aktive Fenster verwendet. Bei Verwendung einer Fensterklasse muss ahk_class ExakterKlassenname angegeben werden (wird von Windows Spy angezeigt). Bei Verwendung einer Prozess-Kennung (PID) muss ahk_pid %VariableMitPID% angegeben werden. Bei Verwendung einer Fenstergruppe muss ahk_group Gruppenname angegeben werden. Bei Verwendung einer eindeutigen ID-Nummer eines Fensters muss ahk_id %VariableMitID% angegeben werden. Die Suche kann durch Angabe mehrerer Kriterien eingeschränkt werden. Zum Beispiel: Eigene Datei.txt ahk_class Notepad
Fenstertext In diesem Parameter kann ein Teilstring angegeben werden, der in einem Textelement des Zielfensters vorkommt (werden mit Windows Spy angezeigt). Versteckte Textelemente werden auch erkannt, sofern DetectHiddenText aktiv ist.
Taste

Die zu klickende Taste: LEFT, RIGHT, MIDDLE (oder einfach der erste Buchstabe). Ist dieser Parameter leer oder nicht vorhanden, wird die Maustaste LEFT verwendet.

Benötigt Windows 2000/XP oder höher. X1 (XButton1, die 4. Maustaste) und X2 (XButton2, die 5. Maustaste) werden auch unterstützt.

Benötigt Windows NT/2000/XP oder höher. WheelUp (oder WU) und WheelDown (oder WD) werden auch unterstützt. Mit der Klickanzahl kann die Anzahl der Rasterungen zum Drehen des Rades bestimmt werden.

Windows Vista oder höher [v1.0.48+]: WheelLeft (WL) oder WheelRight (WR) werden auch unterstützt (funktioniert nicht bei älteren Betriebssystemen). Mit der Klickanzahl kann die Anzahl der Rasterungen zum Drehen des Rades bestimmt werden.
Klickanzahl Die Anzahl der Mausklicks, das auch ein Ausdruck sein kann. Ist dieser Parameter leer oder nicht vorhanden, wird ein Mausklick gesendet.
Optionen

Entweder leer oder mehrere folgende Optionsbuchstaben. Zum Beispiel: d x50 y25

NA [v1.0.45+]: Kann die Zuverlässigkeit verbessern. Siehe Zuverlässigkeit weiter unten.

D: Maustaste gedrückt halten (erzeugt ein Down-Ereignis). Fehlt sowohl die D- als auch U-Option, wird ein vollständiger Mausklick (drücken und loslassen) gesendet.

U: Maustaste loslassen (erzeugt ein Up-Ereignis). Diese Option sollte weggelassen werden, wenn die D-Option bereits aktiv ist (und umgekehrt).

Pos: Bewirkt, dass der X/Y-Positionierungsmodus bedingungslos verwendet wird, wie oben beim Parameter Steuerelement-oder-Pos beschrieben.

Xn: Ersetzt n mit der X-Position, auf die geklickt wird, bezogen auf die obere linke Ecke des Steuerelements. Ansonsten erfolgt der Mausklick auf der horizontalen Mitte des Steuerelements.

Yn: Ersetzt n mit der Y-Position, auf die geklickt wird, bezogen auf die obere linke Ecke des Steuerelements. Ansonsten erfolgt der Mausklick auf der vertikalen Mitte des Steuerelements.

Bei den Optionen X und Y können nur Dezimalzahlen verwendet werden (keine Hexadezimalzahlen).
Titelausnahme Ignoriert Fenster, deren Titel diesen Wert enthalten.
Textausnahme Ignoriert Fenster, deren Textelemente diesen Wert enthalten.

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

Zuverlässigkeit

Zur Verbesserung der Zuverlässigkeit (besonders, wenn der Benutzer die Maus während des ControlClick-Befehls physisch bewegt) könnte das eine oder andere vielleicht weiterhelfen:

  1. SetControlDelay, -1 vor ControlClick benutzen. Dadurch wird verhindert, dass die Maustaste beim Klicken gedrückt gehalten wird, dass wiederum die Störung von physischen Mausbewegungen des Benutzers reduziert.
  2. NA im sechsten Parameter (Optionen) einfügen. Zum Beispiel: ControlClick, Toolbar321, Fenstertitel,,,, NA. Dadurch wird verhindert, dass das Fenster als aktiv markiert wird und dass sich die Eingabeverarbeitungen des Fensters und des Scripts vermischen. Das hat zur Folge, dass die Störung von physischen Mausbewegungen unterdrückt wird (aber üblicherweise nur, wenn das Fenster inaktiv ist). Allerdings funktioniert diese Methode nicht mit allen Fenster- und Steuerelementarten.

Bemerkungen

Einige Anwendungen ignorieren eine Klickanzahl höher als 1 zur Drehung des Mausrads. Verwendet für solche Anwendungen eine Schleife wie folgt, um das Mausrad mehr als eine Rasterung zu drehen:

Loop, 5
    ControlClick, Steuerelement, Fenstertitel, Fenstertext, WheelUp

Fenstertitel und -texte sind von der Groß- und Kleinschreibung abhängig. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.

Verwandte Befehle

SetControlDelay, Control, ControlGet, ControlGetText, ControlMove, ControlGetPos, ControlFocus, ControlSetText, ControlSend, Click

Beispiele

ControlClick, OK, Beliebiger Fenstertitel  ; Klickt die OK-Schaltfläche
ControlClick, x55 y77, Fenstertitel  ; Klickt auf den Koordinaten. Beachtet, dass das Komma zwischen X und Y fehlt.

; Die folgende Methode kann die Zuverlässigkeit verbessern und Nebeneffekte reduzieren:
SetControlDelay -1
ControlClick, Toolbar321, Fenstertitel,,,, NA x192 y10  ; Klickt im NA-Modus auf Koordinaten, die sich auf dem angegebenen Steuerelement beziehen.
