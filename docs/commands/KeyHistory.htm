KeyHistory

Zeigt Script-Informationen und einen Verlauf der aktuellsten Tastatureingaben und Mausklicks.

KeyHistory

Bemerkungen

Dieser Befehl entspricht das Auswählen des Menüelements "View->Key history" im Hauptfenster.

Mit der folgenden Zeile irgendwo im Script kann der Tastenverlauf deaktiviert werden:
#KeyHistory 0

Mit #KeyHistory kann auch die maximale Anzahl an dargestellten Ereignissen geändert werden.

Diese Funktion hilft dabei, Scripts und Hotkeys zu debuggen. Damit ist es auch möglich, den Code von unbekannten Tasten zu ermitteln, wie im unteren Abschnitt der Tastenliste beschrieben (solche Codes können für Hotkeys verwendet werden).

Die virtuellen Tasten (VK) von Mausradereignissen (WheelDown, WheelUp, WheelLeft, und WheelRight) sind Platzhalterwerte, die außerhalb von AutoHotkey keine Bedeutung haben. Außerdem entspricht der Scan-Code der Mausradereignisse aktuell der Anzahl an Rastern (üblicherweise 1).

Wenn das Script keinen Tastatur-Hook installiert hat, dann zeigt das KeyHistory-Fenster nur die Tastaturereignisse an, die vom Script selbst erzeugt wurden (nicht vom Benutzer). Wenn das Script keinen Maus-Hook installiert hat, dann werden Maustastenereignisse nicht anzeigt. Mittels "View->Key History" im Hauptfenster (zugänglich mittels "Open" im Tray-Icon) kann außerdem herausgefunden werden, ob das Script einen Hook verwendet. Mit den folgenden Zeilen können Hooks erzwungen werden:
#InstallKeybdHook
#InstallMouseHook

Verwandte Befehle

#KeyHistory, #InstallKeybdHook, #InstallMouseHook, ListHotkeys, ListLines, ListVars, GetKeyState, KeyWait, A_PriorKey

Beispiele

KeyHistory ; Zeigt den Tastenverlauf in einem Fenster.
