KeyHistory

Zeigt Script-Informationen und einen Verlauf der aktuellsten Tastatureingaben und Mausklicks.

KeyHistory

Bemerkungen

Auch über den Menüpunkt "View->Key history" im Hauptfenster aufrufbar.

Mit #KeyHistory 0 kann der Tastenverlauf deaktiviert werden. Diese Direktive kann auch dazu verwendet werden, die maximale Anzahl an dargestellten Ereignissen zu ändern.

KeyHistory hilft dabei, Scripts und Hotkeys zu debuggen. Dieser Befehl kann auch dazu verwendet werden, den Scancode von unbekannten Tasten zu ermitteln, wie im unteren Abschnitt der Tastenliste beschrieben (dadurch kann so eine Taste z. B. als Hotkey verwendet werden).

Die virtuellen Tasten (VK) von Mausradereignissen (WheelDown, WheelUp, WheelLeft, und WheelRight) sind Platzhalterwerte, die außerhalb von AutoHotkey keine Bedeutung haben. Außerdem ist der Scancode von den Mausradereignissen aktuell die Anzahl an Rastern (üblicherweise 1).

Wenn das Script keinen Tastatur-Hook installiert hat, zeigt das KeyHistory-Fenster nur die Tastaturereignisse an, die vom Script selbst erzeugt wurden (nicht vom Benutzer). Wenn das Script keinen Maus-Hook installiert hat, werden Maustastenereignisse nicht angezeigt. Über "View->Key History" im Hauptfenster (zugänglich mittels "Open" im Tray-Icon) kann außerdem herausgefunden werden, ob das Script einen Hook verwendet. Mit #InstallKeybdHook und/oder #InstallMouseHook können Hooks erzwungen werden.

Verwandte Befehle

#KeyHistory, #InstallKeybdHook, #InstallMouseHook, ListHotkeys, ListLines, ListVars, GetKeyState, KeyWait, A_PriorKey

Beispiele

KeyHistory ; Zeigt den Tastenverlauf in einem Fenster.
