#If [AHK_L]

Erstellt kontextabhängige Hotkeys und Hotstrings. Solche Hotkeys führen eine andere Aktion aus (oder gar keine), abhängig vom Ergebnis des Ausdrucks.

#If [, Ausdruck]

Parameter

Ausdruck Beliebiger gültiger Ausdruck.

Grundlagen zur Bedienung

Jeder gültige Ausdruck kann verwendet werden, um den Kontext zu definieren, welcher Hotkey aktiv sein soll. Zum Beispiel:

#If WinActive("ahk_class Notepad") or WinActive(Fenstertitel)
#Space::MsgBox Win+Leerzeichen wurde im Editor oder %Fenstertitel% gedrückt.

Die #If-Direktive ist wie die #IfWin-Direktive von der Position abhängig; alle nachfolgende Hotkeys und Hotstrings im Script sind davon betroffen. #If und #IfWin schließen sich auch gegenseitig aus, d. h. nur das aktuellste #If oder #IfWin ist aktiv.

Um die Kontextabhängigkeit zu deaktivieren, verwendet #If oder #IfWin ohne Parameter. Zum Beispiel:

#If

Allgemeine Bemerkungen

Wenn eine Tastenkombination (Hotkey) gedrückt wird, dann wird der #If-Ausdruck ausgewertet, um festzustellen, ob der Hotkey aktiviert werden soll. Das System reagiert möglicherweise erst auf die Tastatur- oder Mauseingabe, wenn die Auswertung des Ausdrucks fertig ist oder die Zeit überschritten hat. Das Senden von Tastatureingaben oder Mausklicks, während der Ausdruck ausgewertet wird (z. B. von einer Funktion, die die Direktive aufgerufen hat), kann zu Komplikationen führen und sollte verhindert werden.

Aufgrund einer Eigenart von AutoHotkey wird der Ausdruck manchmal bis zu zweimal bei einem einzelnen Tastatur- oder Mausereignis ausgewertet.

Durch die oben genannten Gründe sollte der Ausdruck so konstruiert werden, dass dieser schnell und ohne Nebeneffekte durchgeführt werden kann.

[AHK_L 53+]: A_ThisHotkey und A_TimeSinceThisHotkey werden gesetzt, abhängig vom aktuell ausgewerteten Hotkey des #If-Ausdrucks.

[v1.0.95.00+]: A_PriorHotkey und A_TimeSincePriorHotkey enthalten vorübergehend die vorherigen Werte der entsprechenden "This"-Variablen.

Verwandte Befehle

Die meisten Verhaltenseigenschaften von #IfWin-Direktiven gelten auch für #If.

#IfTimeout kann verwendet werden, um den Standardwert der Zeitüberschreitung zu überschreiben.

Beispiele

; Beispiel #1: Passt die Lautstärke mit dem Mausrad an, wenn sich der Mauszeiger über der Task-Leiste befindet.
#If MouseIsOver("ahk_class Shell_TrayWnd")
WheelUp::Send {Volume_Up}
WheelDown::Send {Volume_Down}

MouseIsOver(Fenstertitel) {
    MouseGetPos,,, Fenster
    Return WinExist(Fenstertitel . " ahk_id " . Fenster)
}

; Beispiel #2: Einfache Tastaturbefehle, um Wörter in einem Eingabefeld zu löschen.
#If ActiveControlIsOfClass("Edit")
^BS::Send ^+{Left}{Del}
^Del::Send ^+{Right}{Del}

ActiveControlIsOfClass(Klasse) {
    ControlGetFocus, FokussiertesSteuerelement, A
    ControlGet, FokussiertesSteuerelementHwnd, Hwnd,, %FokussiertesSteuerelement%, A
    WinGetClass, FokussiertesSteuerelementKlasse, ahk_id %FokussiertesSteuerelementHwnd%
    Return (FokussiertesSteuerelementKlasse=Klasse)
}

; Beispiel #3: Kontextunabhängiger Hotkey.
#If
Esc::ExitApp

; Beispiel #4: Dynamische Hotkeys. Benötigt Beispiel #1.
NumpadAdd::
Hotkey, If, MouseIsOver("ahk_class Shell_TrayWnd")
If (doubleup := !doubleup)
    Hotkey, WheelUp, DoubleUp
Else
    Hotkey, WheelUp, WheelUp
Return

DoubleUp:
Send {Volume_Up 2}
Return
