#ClipboardTimeout

Ändert, wie lange das Script versuchen soll, auf die Zwischenablage zuzugreifen, falls der erste Versuch fehlschlägt.

#ClipboardTimeout Millisekunden

Parameter

Millisekunden Die Länge des Intervalls in Millisekunden. Mit -1 wird der Versuch unendlich oft durchgeführt. Mit 0 wird der Versuch nur einmal durchgeführt. Scripts, die diese Direktive nicht enthalten, verwenden eine Zeitüberschreitung von 1000 ms.

Bemerkungen

Einige Anwendungen halten die Zwischenablage für einen langen Zeitraum offen, um zum Beispiel eine große Datenmenge schreiben oder lesen zu können. In solchen Fällen bewirkt das Erhöhen dieser Einstellung, dass das Script länger wartet, bevor es aufgibt und eine Fehlermeldung anzeigt.

Diese Einstellungen gelten für alle Clipboard-Operationen, die einfachsten davon sind folgende Beispiele: Var = %Clipboard% und Clipboard = Neuer Text.

Immer wenn das Script auf die Verfügbarkeit der Zwischenablage wartet, können weder neue Threads ausgeführt werden noch Timer laufen. Wenn der Benutzer allerdings einen Hotkey drückt, einen benutzerdefinierten Menüpunkt auswählt oder eine GUI-Aktion (z. B. das Drücken einer Schaltfläche) durchführt, wird dieses Ereignis für später zwischengespeichert. Das heißt, dass dessen Subroutine durchgeführt wird, wenn die Zwischenablage wieder verfügbar ist.

Diese Direktive bewirkt zudem, dass die Zwischenablage nochmals gelesen wird, wenn der erste Versuch fehlschlägt (in den vorherigen Versionen wurde nur versucht, die Zwischenablage nochmals zu öffnen).

Verwandte Befehle

Clipboard, Thread

Beispiel

#ClipboardTimeout 2000
