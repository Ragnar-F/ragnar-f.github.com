#ClipboardTimeout

Verändert, wie lange das Script versucht, auf die Zwischenablage zuzugreifen, wenn der erste Versuch fehlschlägt.

#ClipboardTimeout Millisekunden

Parameter

Millisekunden Die Länge des Intervalls in Millisekunden. Verwendet -1, damit es unendlich lang versucht wird. Verwendet 0, damit nur ein Versuch durchgeführt wird. Scripts, die diese Direktive nicht enthalten, verwenden eine Zeitüberschreitung von 1000 ms.

Bemerkungen

Einige Anwendungen halten die Zwischenablage für einen langen Zeitraum offen, weil sie vielleicht eine große Datenmenge schreiben oder lesen. In solchen Fällen kann die Einstellung erhöht werden, damit das Script länger wartet, bevor es aufgibt und eine Fehlermeldung anzeigt.

Diese Einstellungen gelten für alle Clipboard-Operationen, die Einfachsten davon sind die folgenden Beispiele:
Var = %Clipboard%
Clipboard = Neuer Text

Immer wenn das Script auf die Verfügbarkeit der Zwischenablage wartet, können weder neue Threads ausgeführt werden noch Timer laufen. Wenn der Benutzer allerdings einen Hotkey drückt, einen benutzerdefinierten Menüpunkt auswählt oder eine GUI-Aktion (z. B. das Drücken einer Schaltfläche) durchführt, dann wird dieses Ereignis für später zwischengespeichert. Das heißt, dass die Subroutine durchgeführt wird, wenn die Zwischenablage wieder verfügbar ist.

Diese Direktive bewirkt, dass die Zwischenablage nochmals gelesen wird, wenn der erste Versuch fehlschlägt (in den vorherigen Versionen wurde nur versucht, die Zwischenablage nochmals zu öffnen).

Verwandte Befehle

Clipboard, Thread

Beispiel

#ClipboardTimeout 2000
