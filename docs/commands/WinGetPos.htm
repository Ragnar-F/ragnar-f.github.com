WinGetPos

Ermittelt die Position und Größe eines bestimmten Fensters.

WinGetPos [, X, Y, Breite, Höhe, Fenstertitel, Fenstertext, Titelausnahme, Textausnahme]

Parameter

X, Y Die Namen der Variablen, in denen die X- und Y-Koordinate der oberen linken Ecke des Zielfensters gespeichert werden. Sobald ein Parameter weggelassen wird, wird dessen Wert entsprechend nicht gespeichert.
Breite/Höhe Die Namen der Variablen, in denen die Breite und Höhe des Zielfensters gespeichert werden. Sobald ein Parameter weggelassen wird, wird dessen Wert entsprechend nicht gespeichert.
Fenstertitel Der Titel oder Teiltitel des Fensters (das Übereinstimmungsverhalten wird mit SetTitleMatchMode bestimmt). Wenn dieser und die nächsten 3 Parameter weggelassen werden, wird das zuletzt gefundene Fenster verwendet. Sobald dieser den Buchstaben A enthält und die nächsten 3 Parameter weggelassen werden, wird das aktive Fenster verwendet. Bei Verwendung einer Fensterklasse muss ahk_class ExakterKlassenname angegeben werden (wird von Windows Spy angezeigt). Bei Verwendung einer Prozess-Kennung (PID) muss ahk_pid %VariableMitPID% angegeben werden. Bei Verwendung einer Fenstergruppe muss ahk_group Gruppenname angegeben werden. Bei Verwendung einer eindeutigen ID-Nummer eines Fensters muss ahk_id %VariableMitID% angegeben werden. Die Suche kann durch Angabe mehrerer Kriterien eingeschränkt werden. Zum Beispiel: Eigene Datei.txt ahk_class Notepad
Fenstertext In diesem Parameter kann ein Teilstring angegeben werden, der in einem Textelement des Zielfensters vorkommt (werden mit Windows Spy angezeigt). Versteckte Textelemente werden auch erkannt, sofern DetectHiddenText aktiv ist.
Titelausnahme Ignoriert Fenster, deren Titel diesen Wert enthalten.
Textausnahme Ignoriert Fenster, deren Textelemente diesen Wert enthalten.

Bemerkungen

Wenn kein passendes Fenster gefunden wird, dann werden die Ausgabevariablen leer gemacht.

Wenn der Fenstertitel "Program Manager" verwendet wird, dann ermittelt der Befehl die Größe des Desktops, der der aktuellen Bildschirmauflösung entspricht.

Ein minimiertes Fenster besitzt weiterhin eine Position und Größe. Die zurückgegebenen Werte sind in diesem Fall vom Betriebssystem und den Konfigurationen abhängig.

Zur Erfassung des Fenster- und Steuerelementnamens unter dem Mauszeiger wird MouseGetPos verwendet.

Fenstertitel und -texte sind von der Groß- und Kleinschreibung abhängig. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.

Verwandte Befehle

WinMove, ControlGetPos, WinGetActiveStats, WinGetActiveTitle, WinGetTitle, WinGetText, ControlGetText

Beispiel

WinGetPos, X, Y, Breite, Höhe, Rechner
MsgBox, Der Rechner befindet sich auf %X%`,%Y%

WinGetPos, X, Y, , , A  ; "A", um die Position des aktiven Fensters zu bekommen.
MsgBox, Das aktive Fenster befindet sich auf %X%`,%Y%

IfWinExist, Unbenannt - Editor
{
    WinGetPos, Xpos, Ypos  ; Verwendet das zuletzt gefundene Fenster.
    MsgBox, Der Editor befindet sich auf %Xpos%`,%Ypos%
}
