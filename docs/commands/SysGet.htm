SysGet

Ermittelt die Bildschirmauflösung, Multi-Monitor-Informationen, Dimensionen von Systemobjekten und andere Systemeigenschaften.

SysGet, Ausgabevariable, Unterbefehl [, Param3]

Parameter

Ausgabevariable Der Name der Variable, in der das Ergebnis gespeichert wird.
Unterbefehl Siehe untere Liste.
Param3 Dieser Parameter wird weggelassen, sofern nicht anders angegeben.

Unterbefehle

MonitorCount: Ermittelt die Gesamtzahl an Monitoren. Im Gegensatz zum unten erwähnten SM_CMONITORS enthält MonitorCount alle Monitore, selbst solche, die nicht vom Desktop verwendet werden. In Windows 95/NT ist die Anzahl immer 1.

MonitorPrimary: Ermittelt die Nummer des primären Monitors, die eine 1 in einem Single-Monitor-System sein wird. In Windows 95/NT ist der primäre Monitor immer 1.

Monitor [, N] : Ermittelt die Begrenzungskoordinaten des Monitors mit der Nummer N (wenn N weggelassen wird, dann wird der primäre Monitor verwendet). Die Informationen werden in vier Variablen gespeichert und beginnen alle mit Ausgabevariable. Wenn N zu hoch ist oder ein Problem beim Abrufen der Informationen auftritt, dann werden Variablen leer gemacht. Zum Beispiel:
SysGet, Mon2, Monitor, 2
MsgBox, Links: %Mon2Left% -- Oben: %Mon2Top% -- Rechts: %Mon2Right% -- Unten: %Mon2Bottom%.

Um eine Reihe von globalen anstelle von lokalen Variablen innerhalb einer Funktion zu erstellen, deklariert Mon2 als globale Variable, bevor dieser Befehl verwendet wird (Das Gegenteil davon gilt für global-angesehene Funktionen).

MonitorWorkArea [, N]: Das Gleiche wie oben, nur mit der Ausnahme, dass der Bereich durch die Taskleiste und andere registrierte Desktop-Toolbars reduziert ist.

MonitorName [, N]: Ermittelt den Systemnamen des Monitors mit der Nummer N (wenn N weggelassen wird, dann wird der primäre Monitor verwendet).

(Numerisch): Gebt für den Unterbefehl eine Nummer aus der unteren Tabelle an, um den entsprechenden Wert abzurufen. Das folgende Beispiel speichert die Anzahl an Maustasten in eine Variable namens "Maustastenanzahl":
SysGet, Maustastenanzahl, 43

Häufig verwendet

Nummer Beschreibung
80 SM_CMONITORS: Anzahl der Anzeigemonitore auf dem Desktop (ohne "nichts anzeigende Pseudo-Monitore"). Windows 95/NT: Der abgerufene Wert ist immer 0.
43 SM_CMOUSEBUTTONS: Anzahl der Tasten auf der Maus (0, wenn keine Maus installiert ist).
16, 17 SM_CXFULLSCREEN, SM_CYFULLSCREEN: Breite und Höhe des Client-Bereichs eines Vollbild-Fensters auf dem primären Anzeigemonitor (in Pixel).
61, 62 SM_CXMAXIMIZED, SM_CYMAXIMIZED: Standard-Abmessungen eines maximierten im Vordergrund liegenden Fensters auf dem primären Anzeigemonitor (in Pixel).
59, 60 SM_CXMAXTRACK, SM_CYMAXTRACK: Maximale Standard-Abmessungen eines Fensters, dass eine Titelleiste und skalierbare Rahmen hat (in Pixel). Diese Maße beziehen sich auf dem gesamten Desktop. Beim Verkleinern des Fensterrahmens mit einem Ziehvorgang kann der Benutzer diese Abmessungen nicht überschreiten.
28, 29 SM_CXMIN, SM_CYMIN: Minimale Breite und Höhe des Fensters (in Pixel).
57, 58 SM_CXMINIMIZED, SM_CYMINIMIZED: Abmessungen eines minimierten Fensters (in Pixel).
34, 35 SM_CXMINTRACK, SM_CYMINTRACK: Die minimale Standardbreite und -höhe des Fensters (in Pixel). Beim Verkleinern des Fensterrahmens mit einem Ziehvorgang kann der Benutzer diese Abmessungen nicht unterschreiten. Ein Fenster kann diese Werte überschreiben, wenn die WM_GETMINMAXINFO-Nachricht verarbeitet wird.
0, 1 SM_CXSCREEN, SM_CYSCREEN: Breite und Höhe des Bildschirms vom primären Anzeigemonitor (in Pixel). Diese Werte sind das gleiche wie die internen Variablen A_ScreenWidth und A_ScreenHeight.
78, 79 SM_CXVIRTUALSCREEN, SM_CYVIRTUALSCREEN: Breite und Höhe des virtuellen Bildschirms (in Pixel). Der virtuelle Bildschirm ist ist das Begrenzungsrechteck aller Anzeigemonitore. Die Maße von SM_XVIRTUALSCREEN, SM_YVIRTUALSCREEN sind die Koordinaten der oberen linken Ecke des virtuellen Bildschirms. Windows NT, Windows 95: Der abgerufene Wert ist immer 0.
19 SM_MOUSEPRESENT: Ungleich 0, wenn eine Maus installiert ist, ansonsten 0.
75 SM_MOUSEWHEELPRESENT: Ungleich 0, wenn eine Maus mit Mausrad installiert ist, ansonsten 0. Windows 95: Der abgerufene Wert ist immer 0.
63 SM_NETWORK: Ist ein Netzwerk vorhanden, wird das am wenigsten signifikante Bit gesetzt, andernfalls wird es gelöscht. Die anderen Bits werden für die zukünftige Verwendung reserviert.
8193 SM_REMOTECONTROL: Diese Systemmaße werden in einer Terminaldienstumgebung verwendet. Ungleich 0, wenn die aktuelle Sitzung ferngesteuert wird, ansonsten 0. Windows 2000/NT, Windows Me/98/95: Der abgerufene Wert ist immer 0.
4096 SM_REMOTESESSION: Diese Systemmaße werden in einer Terminaldienstumgebung verwendet. Wenn der aufrufende Prozess mit einer Terminal-Services-Clientsitzung verbunden ist, dann ist der Rückgabewert ungleich 0. Wenn der aufrufende Prozess mit einer Terminal-Services-Konsolensitzung verbunden ist, dann ist der Rückgabewert 0. Die Konsolensitzung ist nicht unbedingt die physische Konsole. Windows NT 4.0 SP3 und niedriger, Windows Me/98/95: Der abgerufene Wert ist immer 0.
70 SM_SHOWSOUNDS: Ungleich 0, wenn der Benutzer wünscht, dass eine Anwendung die Informationen visuell zur Verfügung stellt, wo sie sonst nur in auditiver Form bereitgestellt würden, ansonsten 0.
8192 SM_SHUTTINGDOWN: Ungleich 0, wenn die aktuelle Sitzung heruntergefahren wird, ansonsten 0. Windows 2000/NT, Windows Me/98/95: Der abgerufene Wert ist immer 0.
23 SM_SWAPBUTTON: Ungleich 0, wenn die Funktionen für die linke und rechte Maustaste vertauscht sind, ansonsten 0.
76, 77 SM_XVIRTUALSCREEN, SM_YVIRTUALSCREEN: Koordinaten für die linke obere Ecke des virtuellen Bildschirms. Der virtuelle Bildschirm ist ist das Begrenzungsrechteck aller Anzeigemonitore. Die Maße von SM_CXVIRTUALSCREEN, SM_CYVIRTUALSCREEN (weiter oben) sind hingegen die Breite und Höhe des virtuellen Bildschirms. Windows NT, Windows 95: Der abgerufene Wert ist immer 0.

Die zwei Klicks müssen auch innerhalb einer bestimmten Zeit auftreten.

83, 84 SM_CXFOCUSBORDER, SM_CYFOCUSBORDER: Breite (in Pixel) vom linken und rechten Rand und die Höhe vom oberen und unteren Rand des Fokusrechtecks eines Steuerelements. Windows 2000/NT, Windows Me/98/95:  Der abgerufene Wert ist immer 0.
21, 3 SM_CXHSCROLL, SM_CYHSCROLL: Die Breite des Pfeil-Bitmaps auf einer horizontalen Bildlaufleiste und die Höhe einer horizontalen Bildlaufleiste (in Pixel).
10 SM_CXHTHUMB: Die Breite der Positionsanzeige auf einer horizontale Bildlaufleiste (in Pixel).
11, 12 SM_CXICON, SM_CYICON: Standard-Breite und -Höhe eines Icons (in Pixel).
38, 39 SM_CXICONSPACING, SM_CYICONSPACING: Die Abmessungen einer Gitterzelle für Elemente in der Ansicht für große Symbole (in Pixel). Jedes Element passt in ein Rechteck dieser Größe, wenn angeordnet. Diese Werte sind immer größer oder gleich SM_CXICON bzw. SM_CYICON.
71, 72 SM_CXMENUCHECK, SM_CYMENUCHECK: Die Abmessungen der Markierungs-Bitmap im Standardmenü (in Pixel).
54, 55 SM_CXMENUSIZE, SM_CYMENUSIZE: Die Abmessungen der Schaltflächen in der Menüleiste, wie z. B. die Schließen-Schaltfläche eines anhängenden Fensters in einem Multiple Document Interface (in Pixel).
47, 48 SM_CXMINSPACING SM_CYMINSPACING: Die Abmessungen einer Gitterzelle für für minimierte Fenster (in Pixel). Jedes minimierte Fenster passt in ein Rechteck dieser Größe, wenn angeordnet. Diese Werte sind immer größer oder gleich SM_CXMINIMIZED bzw. SM_CYMINIMIZED.
30, 31 SM_CXSIZE, SM_CYSIZE: Die Breite und Höhe der Schaltflächen in der Titelleiste (in Pixel).
32, 33 SM_CXSIZEFRAME, SM_CYSIZEFRAME: Rahmendicke eines Fensters, das skalierbar ist (in Pixel). SM_CXSIZEFRAME ist die Breite der horizontalen Abgrenzung und SM_CYSIZEFRAME die Höhe der vertikalen Abgrenzung. Gleichbedeutend mit SM_CXFRAME und SM_CYFRAME.
49, 50 SM_CXSMICON, SM_CYSMICON: Empfohlene Abmessungen eines kleinen Icons (in Pixel). Kleine Icons werden normalerweise in der Titelleiste und in der Ansicht für kleine Icons angezeigt.
52, 53 SM_CXSMSIZE, SM_CYSMSIZE: Abmessungen der Schaltflächen in der kleinen Titelleiste (in Pixel).
2, 20 SM_CXVSCROLL, SM_CYVSCROLL: Die Breite einer vertikalen Bildlaufleiste und die Höhe des Pfeil-Bitmaps auf einer horizontalen Bildlaufleiste (in Pixel).
4 SM_CYCAPTION: Die Höhe einer normalen Titelleiste (in Pixel).
18 SM_CYKANJIWINDOW: Für Windows-Versionen mit Doppel-Byte-Zeichensatz (DBCS) ist dies die Höhe des Kanji-Fensters unten auf dem Bildschirm.
15 SM_CYMENU: Die Höhe einer einzeiligen Menüleiste (in Pixel).
51 SM_CYSMCAPTION: Die Höhe einer kleinen Titelleiste (in Pixel).
9 SM_CYVTHUMB: Die Höhe der Positionsanzeige auf einer vertikalen Bildlaufleiste (in Pixel).
42 SM_DBCSENABLED: Ungleich 0, wenn User32.dll DBCS unterstützt, ansonsten 0. Windows Me/98/95:  Ungleich 0, wenn die Version mit dem Doppel-Byte-Zeichensatz (DBCS) von User.exe installiert ist, ansonsten 0.
22 SM_DEBUG: Ungleich 0, wenn die Debugging-Version von User.exe installiert ist, ansonsten 0.
82 SM_IMMENABLED: Ungleich 0, wenn die Funktionen vom Input Method Manager (IMM) / Input Method Editor (IME) aktiviert sind, ansonsten 0. Windows NT, Windows Me/98/95:  Der abgerufene Wert ist immer 0.

SM_IMMENABLED zeigt an, ob das System bereit ist, einen unicode-basierten IME auf eine Unicode-Anwendung anzuwenden. Um sicherzustellen, dass ein sprachabhängiger IME funktioniert, überprüft SM_DBCSENABLED und den ANSI-Zeichensatz des Systems. Andernfalls kann die ANSI-zu-Unicode-Konvertierung nicht richtig ausgeführt werden, oder einige Komponenten wie Schriftarten oder Registrierungseinstellungen sind nicht vorhanden.

87 SM_MEDIACENTER: Ungleich 0, wenn das aktuelle Betriebssystem die Windows XP Media Center Edition ist, ansonsten 0.
40 SM_MENUDROPALIGNMENT: Ungleich 0, wenn ausklappbare Menüs mit den entsprechenden Menüpunkten rechts ausgerichtet sind, ansonsten 0.
74 SM_MIDEASTENABLED: Ungleich 0, wenn die Sprachen Hebräisch und Arabisch im System aktiviert sind, ansonsten 0.
41 SM_PENWINDOWS: Ungleich 0, wenn Microsoft Windows für Pen-Programmier-Erweiterungen installiert ist, ansonsten 0.
44 SM_SECURE: Ungleich 0, wenn Sicherheitsfunktionen eingeschaltet sind, ansonsten 0.
81 SM_SAMEDISPLAYFORMAT: Ungleich 0, wenn alle Anzeigemonitore das gleiche Farbformat haben, ansonsten 0. Beachtet, dass zwei Anzeigemonitore die gleiche Farbtiefe haben können, aber unterschiedliche Farbformate. Zum Beispiel können die roten, grünen und blauen Pixel mit unterschiedlicher Anzahl von Bits kodiert werden, oder solche Bits können an verschiedenen Orten in einem Pixelfarbwert gefunden werden. Windows NT, Windows 95:  Der abgerufene Wert ist immer 0.
73 SM_SLOWMACHINE: Immer 0 in Windows NT/2000/XP oder höher. Ungleich 0 in Windows 95/98/ME, wenn der Computer mindestens eine folgende Eigenschaft hat: einen 386-Prozessor, weniger als 6MB RAM oder eine Grafikkarte, die scheinbar langsam ist.

Bemerkungen

Die internen Variablen A_ScreenWidth und A_ScreenHeight enthalten die Abmessungen des primären Monitors (in Pixel).

Verwandte Befehle

DllCall, WinGet

Beispiele

; Beispiel #1:
SysGet, Maustastenanzahl, 43
SysGet, VirtuelleBildschirmbreite, 78
SysGet, VirtuelleBildschirmhöhe, 79
