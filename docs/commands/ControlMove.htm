ControlMove

Verschiebt ein Steuerelement oder ändert dessen Größe.

ControlMove, Steuerelement, X, Y, Breite, Höhe [, Fenstertitel, Fenstertext, Titelausnahme, Textausnahme]

Parameter

Steuerelement

Möglich ist entweder ClassNN (Klassenname und Instanznummer des Steuerelements) oder Name/Text des Steuerelements, die beide mit Windows Spy ermittelt werden können. Beim Name/Text wird das Übereinstimmungsverhalten mit SetTitleMatchMode bestimmt. Wenn dieser Parameter leer ist, wird das oberste Steuerelement des Fensters benutzt.

Um ein Steuerelement per HWND (Windows Handle) anzusteuern, muss dieser Parameter leer gelassen und ahk_id %SteuerelementHwnd% im Fenstertitel-Parameter angegeben werden (funktioniert auch bei versteckten Steuerelementen, selbst wenn DetectHiddenWindows deaktiviert ist). Das HWND eines Steuerelements wird üblicherweise per ControlGet Hwnd, MouseGetPos oder DllCall abgerufen.
X, Y Die neuen Koordinaten X und Y der oberen linken Ecke des Steuerelements (in Pixel), die Ausdrücke sein können. Sobald eine Koordinate weggelassen wird, bleibt die Position des Steuerelements bei dieser Abmessung unverändert. Diese Koordinaten beziehen sich auf die linke obere Ecke des Fensters, in der sich das Steuerelement befindet; diese können mit ControlGetPos oder Window Spy ermittelt werden.
Breite, Höhe Die neue Breite und Höhe des Steuerelements (in Pixel), die Ausdrücke sein können. Sobald ein Parameter weggelassen wird, bleibt die Größe des Steuerelements bei dieser Abmessung unverändert.
Fenstertitel Ein Fenstertitel oder sonstige Kriterien, die das Zielfenster ermitteln sollen. Siehe Fenstertitel.
Fenstertext In diesem Parameter kann ein Teilstring angegeben werden, der in einem Textelement des Zielfensters vorkommt (werden mit Windows Spy angezeigt). Versteckte Textelemente werden auch erkannt, sofern DetectHiddenText aktiv ist.
Titelausnahme Ignoriert Fenster, deren Titel diesen Wert enthalten.
Textausnahme Ignoriert Fenster, deren Textelemente diesen Wert enthalten.

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

Bemerkungen

Um die Zuverlässigkeit des Befehls zu verbessern, erfolgt nach jeder Verwendung automatisch eine Verzögerung. Diese Verzögerung kann per SetControlDelay geändert werden.

Fenstertitel und -texte unterscheiden zwischen Groß- und Kleinschreibung. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.

Verwandte Befehle

ControlGetPos, WinMove, SetControlDelay, Control, ControlGet, ControlGetText, ControlSetText, ControlClick, ControlFocus, ControlSend

Beispiel

InputBox, Ausgabevariable, Eigenes Eingabefenster
Return

    Return
