IniWrite

Schreibt einen Wert in eine normierte INI-Datei.

IniWrite, Wert, Dateiname, Sektion [, Schlüssel]

Parameter

Wert

String oder Nummer, der auf der rechten Seite des Gleichheitszeichens (=) neben dem Schlüssel geschrieben wird. Falls der Text zu lang ist, kann er in mehreren kurzen Zeilen aufgeteilt werden (Fortsetzungsbereich), um ihn besser lesbar und verwaltbar zu machen.

[AHK_L 57+]: Ist Schlüssel nicht vorhanden, muss dieser Parameter den kompletten Inhalt einer Sektion enthalten (ohne [Sektionsname]-Kopf), die in die INI-Datei geschrieben werden soll.
Dateiname Name der Datei, die sich im A_WorkingDir-Verzeichnis befindet, solange kein absoluter Pfad angegeben ist.
Sektion Name der Sektion, die in der INI-Datei mit eckigen Klammern dargestellt wird (füge die Klammern nicht in diesem Parameter ein).
Schlüssel Name des Schlüssels in der INI-Datei.

ErrorLevel

[v1.1.04+] Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

Bemerkungen

Eine typische INI-Datei sieht wie folgt aus:

[Sektionsname]
Schlüssel=Wert

Unicode: IniRead und IniWrite sind auf die externen Funktionen GetPrivateProfileString und WritePrivateProfileString angewiesen, um Werte lesen oder schreiben zu können. Diese Funktionen unterstützen Unicode nur in UTF-16-Dateien; bei allen anderen Dateien wird vorausgesetzt, dass sie den Standard-ANSI-Zeichensatz des Systems verwenden. In Unicode-Scripts verwendet IniWrite das UTF-16-Format bei jeder neuen Datei. Wenn das unerwünscht ist, muss man sicherstellen, dass die Datei existiert, bevor IniWrite aufgerufen wird. Zum Beispiel:

FileAppend,, KeinUnicode.ini, CP0 ; Der letzte Parameter kann meistens weggelassen werden.

Verwandte Befehle

IniDelete, IniRead, RegWrite

Beispiel

IniWrite, Das ist ein neuer Wert, C:\Temp\MeineDatei.ini, Sektion2, Schlüssel
