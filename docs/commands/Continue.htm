Continue

Überspringt den Rest der aktuellen Schleife und beginnt einen neuen Durchlauf. Nur in einer Schleife gültig.

Continue [, LoopLabel]

[AHK_L 59+]: LoopLabel gibt an, bei welcher Schleife dieser Befehl angewendet werden soll. Dieser Parameter kann entweder der Name eines Labels oder eine numerische Verschachtelungsebene sein. Ist dieser Parameter nicht vorhanden oder 1, wird dieser Befehl auf die innerste Schleife angewendet, die den Befehl umschließt. LoopLabel muss ein konstanter Wert sein; das heißt, dass Variablen und Ausdrücke nicht unterstützt werden. Ist ein Label angegeben, muss es direkt auf ein Loop-Befehl verweisen.

Der Continue-Befehl verhält sich so, als würde er die schließende Klammer einer Schleife erreichen:

  1. Er erhöht A_Index um 1.
  2. Er überspringt den Rest des Schleifenkörpers.
  3. Die Bedingung einer Schleife (falls vorhanden) wird dahingehend untersucht, ob sie erfüllt wurde. Wenn das der Fall ist, beginnt ein neuer Durchlauf; ansonsten endet die Schleife.

Break und Continue sind besser geeignet als Goto, weil sie ein Script üblicherweise besser lesbar und verwaltbar machen.

Verwandte Befehle

Break, Loop, Until, While-Loop, For-Loop, Block, Label

Beispiel

; Dieses Beispiel zeigt vom 6. bis zum 10. Durchlauf 5 Dialogfenster.
; Bei den ersten 5 Durchläufen veranlasst der Continue-Befehl
; immer einen Neustart des Durchlaufs, bevor die MsgBox-Zeile erreicht wird.
Loop, 10
{
    If A_Index <= 5
        continue
    MsgBox %A_Index%
}
; Setzt die äußere Schleife fort.
outer:
Loop 3
{
    x := A_Index
    Loop 3
    {
        If (x*A_Index = 4)
            continue outer  ; Das gleiche wie continue 2 oder goto continue_outer.
        MsgBox %x%,%A_Index%
    }
    continue_outer: ; Für goto.
    ErrorLevel:=ErrorLevel ; Vor Revision 57 konnten Labels nicht auf das Ende eines Blocks verweisen.
}
