WinKill

Zwingt die Schließung des angegeben Fensters.

WinKill [, Fenstertitel, Fenstertext, Wartezeit, Titelausnahme, Textausnahme]

Parameter

Fenstertitel Der Titel oder Teiltitel des Fensters (das Übereinstimmungsverhalten wird mit SetTitleMatchMode bestimmt). Wenn dieser und die anderen 3 Fensterparameter weggelassen werden, wird das zuletzt gefundene Fenster verwendet. Sobald dieser den Buchstaben A enthält und die anderen 3 Fensterparameter weggelassen werden, wird das aktive Fenster verwendet. Bei Verwendung einer Fensterklasse muss ahk_class ExakterKlassenname angegeben werden (wird von Windows Spy angezeigt). Bei Verwendung einer Prozess-Kennung (PID) muss ahk_pid %VariableMitPID% angegeben werden. Um eine Fenstergruppe gewaltsam zuschließen, verwendet ahk_group EigenerGruppenname (in diesem Fall müssen Fenstertext, Titelausnahme und Textausnahme leer sein). Bei Verwendung einer eindeutigen ID-Nummer eines Fensters muss ahk_id %VariableMitID% angegeben werden. Die Suche kann durch Angabe mehrerer Kriterien eingeschränkt werden. Zum Beispiel: Eigene Datei.txt ahk_class Notepad
Fenstertext In diesem Parameter kann ein Teilstring angegeben werden, der in einem Textelement des Zielfensters vorkommt (werden mit Windows Spy angezeigt). Versteckte Textelemente werden auch erkannt, sofern DetectHiddenText aktiv ist.
Wartezeit Ist dieser Parameter leer oder nicht vorhanden, wartet der Befehl überhaupt nicht. Wenn 0, dann wartet er 500ms. Ansonsten wartet er die angegebene Anzahl an Sekunden (kann einen Dezimalpunkt oder einen Ausdruck enthalten), bis das Fenster geschlossen wird. Wenn das Fenster nicht innerhalb der Zeit geschlossen wird, dann wird das Script fortgesetzt. ErrorLevel wird nicht von diesem Befehl gesetzt, daher verwendet IfWinExist oder WinWaitClose, wenn geschlossene Fenster ermittelt werden müssen.
Titelausnahme Ignoriert Fenster, deren Titel diesen Wert enthalten.
Textausnahme Ignoriert Fenster, deren Textelemente diesen Wert enthalten.

Bemerkungen

Dieser Befehl versucht zunächst kurz, das Fenster normal zuschließen. Wenn das fehlschlägt, dann versucht der Befehl, das Fenster gewaltsam zuschließen, indem der dazugehörige Prozess beendet wird.

Wenn ein übereinstimmendes Fenster aktiv ist, dann wird vorrangig dieses Fenster anstelle anderer übereinstimmende Fenster darunter geschlossen. Wenn mehr als ein Fenster übereinstimmt, dann wird das oberste (aktuellste verwendete) Fenster geschlossen.

Dieser Befehl arbeitet immer nur mit dem ersten gefundenen Fenster, es sein denn, es befindet sich ahk_group Groupname in Fenstertitel. In dem Fall sind alle Fenster der Gruppe betroffen.

Fenstertitel und -texte sind von der Groß- und Kleinschreibung abhängig. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.

Verwandte Befehle

WinClose, WinWaitClose, Process, WinActivate, SetTitleMatchMode, DetectHiddenWindows, Zuletzt gefundenes Fenster, IfWinExist, IfWinActive, WinWaitActive, WinWait, GroupActivate

Beispiel

IfWinExist, Unbenannt - Editor
    WinKill ; zuletzt gefundenes Fenster verwenden
Else
    WinKill, Rechner
