Catch [v1.1.04+]

Bestimmt den Code, der ausgeführt werden soll, wenn eine Ausnahme während einer Try-Anweisung ausgelöst wird.

Catch [, Ausgabevariable]
    Anweisungen
Catch [, Ausgabevariable]
{
    Anweisungen
}

Parameter

Ausgabevariable (Optional) Der Name der Variable, in der der Wert von der Ausnahme gespeichert wird.
Anweisung(en) Befehle oder Ausdrücke, die ausgeführt werden sollen, wenn eine Ausnahme ausgelöst wird.

Bemerkungen

Jedes Catch muss mit einer vorherigen Try-Anweisung assoziiert werden. Ein Catch gehört immer einer nächstgelegenen nicht beanspruchten Try-Anweisung, es sei denn, Block wird dazu verwendet, dieses Verhalten zu ändern.

Der "One True Brace Style" kann gegebenenfalls verwendet werden. Zum Beispiel:

try {
    ...
} catch e {
    ...
}

Laufzeitfehler

Eine Try-Catch-Anweisung kann auch dazu verwendet werden, Laufzeitfehler zu verarbeiten. Es gibt zwei Arten von Laufzeitfehlern: solche, die ErrorLevel normal setzen, und solche, die den aktuellen Thread nach der Fehlermeldungsanzeige beenden. Ladezeitfehler können nicht verarbeitet werden, da sie vor dem Ausführen der Try-Anweisung auftauchen. Für die Abwärtskompatibilität (und in einigen Fällen der Einfachheit halber) lösen Laufzeitfehler nur Ausnahmen beim Ausführen eines Try-Blocks aus. Die meisten Befehle unterstützen das Benutzen von Try-Catch; allerdings wird ErrorLevel weiterhin auf 0 gesetzt, wenn der Befehl erfolgreich durchgeführt wurde.

Der gespeicherte Wert in der Ausgabevariable (falls vorhanden) ist ein Objekt, das folgende Felder enthält:

What: Der Name des Befehls oder der Funktion, der/die ausgeführt wurde oder dabei war, ausgeführt zu werden, als der Fehler auftrat.

File: Der vollständige Pfad der Script-Datei, die die fehlerhafte Zeile enthält.

Line: Die Nummer der fehlerhaften Zeile.

Message: Eine Fehlermeldung oder ein ErrorLevel-Wert.

Extra: Zusätzliche Informationen über den Fehler, falls verfügbar.

Hinweis: Diese Details sollten nur fürs Debuggen verwendet werden, da sie möglicherweise in einer zukünftigen Version geändert werden.

Verwandte Befehle

Try, Throw, Block

Beispiele

Siehe Try.
