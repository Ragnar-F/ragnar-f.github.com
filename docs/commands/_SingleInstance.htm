#SingleInstance

Bestimmt, ob ein bereits laufendes Script nochmals ausgeführt werden darf.

#SingleInstance [force|ignore|off]

Parameter

force|ignore|off

Dieser Parameter bestimmt, was geschieht, wenn ein Script gestartet wird, das eine frühere Instanz von sich selbst aktiv hat:

Das Wort FORCE überspringt das Dialogfenster und ersetzt die alte Instanz automatisch, das vergleichbar ist mit dem Reload-Befehl.

Das Wort IGNORE überspringt das Dialogfenster und lässt die alte Instanz weiterlaufen. Das heißt, dass der Versuch ignoriert wird, ein bereits laufendes Script zu starten.

Das Wort OFF erlaubt mehrere Instanzen des Scripts, damit sie gleichzeitig ausgeführt werden können.

Ist dieser Parameter nicht vorhanden, wird ein Dialogfenster mit der Anfrage angezeigt, ob die alte Instanz beibehalten oder mit einer neuen Instanz ersetzt werden soll.

Bemerkungen

Ein Script mit Hotkeys, Hotstrings, #Persistent, OnMessage() oder Gui ist standardmäßig eine Einzelinstanz (Dialogfenster & Eingabeaufforderung). Andere Scripts erlauben standardmäßig mehrere Instanzen. Dieses Verhalten kann wie oben beschrieben deaktiviert oder geändert werden.

Verwandte Befehle

Reload, #Persistent

Beispiel

#SingleInstance force
#SingleInstance ignore
#SingleInstance off
