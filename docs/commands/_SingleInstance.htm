#SingleInstance

Bestimmt, ob ein Script wieder ausgeführt werden kann, wenn es bereits läuft.

#SingleInstance [force|ignore|off]

Parameter

force|ignore|off

Dieser Parameter bestimmt, was geschieht, wenn ein Script gestartet wird, während eine frühere Instanz von ihr bereits aktiv ist:

Das Wort FORCE überspringt das Dialogfenster und ersetzt die alte Instanz automatisch, vergleichbar mit dem Reload-Befehl.

Das Wort IGNORE überspringt das Dialogfenster und erhält die alte Instanz aufrecht. Das heißt, dass der Versuch unterdrückt wird, ein laufendes Script zu starten.

Das Wort OFF erlaubt mehrere Instanzen des Scripts, damit sie gleichzeitig ausgeführt werden können.

Wenn der Parameter nicht vorhanden ist, wird ein Dialogfenster mit der Anfrage anzeigt, ob die alte Instanz erhalten bleiben soll oder mit einer neuen Instanz ersetzt werden soll.

Bemerkungen

Ein Script mit Hotkeys, Hotstrings, #Persistent, OnMessage() oder Gui ist standardmäßig eine Einzelinstanz (Dialogfenster & Eingabeaufforderung). Andere Scripts erlauben standardmäßig mehrere Instanzen. Dieses Verhalten kann wie oben beschrieben deaktiviert oder geändert werden.

Verwandte Befehle

Reload, #Persistent

Beispiel

#SingleInstance force
#SingleInstance ignore
#SingleInstance off
