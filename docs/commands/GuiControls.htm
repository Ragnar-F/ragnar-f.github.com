Text

Beschreibung: Ein Textfeld ohne Rand, den der Benutzer nicht bearbeiten kann. Oft verwendet, um andere Steuerelemente zu beschriften.

Beispiel: Gui, Add, Text,, Bitte Name eingeben:

In diesem Fall ist der letzte Parameter der String, der angezeigt wird. Der String kann Zeilenumbruchszeichen (`n) enthalten, um neue Zeilen zu beginnen. Mithilfe eines Fortsetzungsbereichs kann eine lange Zeile außerdem in mehreren kurzen Zeilen aufgeteilt werden.

Wenn in den Optionen die Breite (W) angegeben wird, aber die Zeilen (R) oder die Höhe (H) weggelassen werden, wird der Text wie vorgesehen automatisch umbrochen und die Höhe der Steuerelemente automatisch gesetzt.

Da der Inhalt des Steuerelements der letzte Parameter vom GUI-Befehl ist, müssen direkte Kommas nicht geschützt werden. Das gilt auch für den letzten Parameter aller anderen Befehle.

Es ist möglich, ein g-Label wie gEigeneSubroutine in den Optionen des Steuerelements anzugeben. Dadurch würde das Label EigeneSubroutine automatisch ausgeführt, sobald der Benutzer auf den Text klickt. Beispielsweise kann damit ein unterstrichener, blauer Hyperlink simuliert werden:

Gui, Font, underline
Gui, Add, Text, cBlue gGoogleStarten, Hier klicken, um Google zu starten.

; Alternativ kann auch das Link-Steuerelement verwendet werden:
Gui, Add, Link,, <a href="www.google.com">Hier</a> klicken, um Google zu starten.

Gui, Font, norm
Gui, Show
Return

GoogleStarten:
Run www.google.com
Return

Es kann auch ein Doppelklick erkannt werden, wenn überprüft wird, ob A_GuiEvent das Wort DoubleClick enthält.

Ein Und-Zeichen (&) kann im Text verwendet werden, um einen Buchstaben zu unterstreichen. Zum Beispiel:

Gui, Add, Text,, &Vorname:
Gui, Add, Edit

Im obigen Beispiel wird der Buchstabe F unterstrichen. Das ermöglicht den Benutzer, die Tastenkombination Alt+V zu drücken, um den Tastaturfokus auf das erste eingabefähige Steuerelement zu setzen, welches nach dem Text-Steuerelement hinzugefügt wurde. Um stattdessen ein Und-Zeichen direkt anzeigen zu lassen, verwendet zwei aufeinanderfolgende Und-Zeichen (&&). Um die Sonderbehandlung der Und-Zeichen zu deaktivieren, fügt 0x80 in den Optionen des Steuerelements ein.

Siehe allgemeine Optionen für weitere Optionen wie Right, Center und Hidden. Siehe auch: Position und Größe von Steuerelementen anpassen.

Edit

Beschreibung: Ein Eingabefeld, in dem formatloser Text eingegeben werden kann.

Beispiel: Gui, Add, Edit, r9 vEigenesEdit, Text erscheint innerhalb des Eingabefeldes (lasst diesen Parameter weg, um anfangs leer zu sein).

Das Steuerelement wird mehrzeilig, sobald es mehr als eine Textzeile enthält. Mit r3 in den Optionen wird zum Beispiel ein dreizeiliges Eingabefeld mit folgenden Standardeigenschaften erstellt:

  • eine vertikale Bildlaufleiste
  • automatischer Zeilenumbruch
  • Enter erzeugt einen Zeilenumbruch, anstatt die Standard-Schaltfläche des Fensters auszulösen.

Um eine neue Zeile in einem mehrzeiligen Eingabefeld zu beginnen, fügt im letzten Parameter ein Zeilenumbruchszeichen (`n) oder CR+LF (`r`n) ein. Beide Methoden erzeugen innerhalb des Eingabefeldes direkte CR+LF-Paare (`r`n). Falls das Steuerelement jedoch mit Gui Submit oder GuiControlGet in seine Variable gespeichert wird, wird jedes CR+LF-Paar (`r`n) immer in ein Zeilenumbruchszeichen (`n) umgewandelt. Der Text kann wie folgt in eine Datei geschrieben werden: FileAppend, %EigenesEingabefeld%, C:\Gespeicherte Datei.txt.

Bei einem Steuerelement mit automatischem Zeilenumbruch (Standard bei mehrzeiligen Eingabefeldern) wird jeder Zeilenumbruch nicht als Zeilenumbruchszeichen interpretiert (nur die Entertaste ist dazu fähig).

In Windows 95/98/Me sind mehrzeilige Eingabefelder auf 64 KB Text begrenzt, aber in Windows NT/2k/XP oder höher auf ungefähr 4 GB. Die zugewiesene Speichergröße passt sich je nach Benutzereingabe an.

Es ist möglich, ein g-Label wie gEigeneSubroutine in den Optionen des Steuerelements anzugeben. Dadurch würde das Label EigeneSubroutine automatisch ausgeführt, sobald der Benutzer oder das Script den Inhalt des Steuerelements ändert.

Um eine Textdatei in ein Eingabefeld zu laden, verwendet FileRead und GuiControl. Zum Beispiel:

Gui, Add, Edit, R20 vEigenesEdit
FileRead, Dateiinhalt, C:\Eigene Datei.txt
GuiControl,, EigenesEdit, %Dateiinhalt%

Optionen

Hinweis: Um eine Option stattdessen zu entfernen, setzt ein Minuszeichen davor.

Limit: Beschränkt die Benutzereingabe auf die sichtbare Breite des Eingabefeldes. Alternativ kann direkt danach eine Zahl angegeben werden, um die Eingabe auf eine bestimmte Anzahl an Zeichen zu begrenzen. Zum Beispiel würde Limit10 nicht mehr als 10 Zeichen erlauben.

Lowercase: Die Zeichen werden bei Benutzereingabe automatisch in Kleinbuchstaben umgewandelt.

Multi: Ermöglicht mehrere Textzeilen. Allerdings muss diese Option normalerweise nicht angegeben werden, weil sie automatisch erkannt wird, basierend auf Höhe (H), Zeilen (R) oder Inhalt (Text).

Number: Hindert den Benutzer daran, etwas anderes als Zahlen in das Feld einzugeben (allerdings ist es noch möglich, Nicht-Zahlen von der Zwischenablage einzufügen). Alternativ kann die numerische Eingabe auch erzwungen werden, wenn ein UpDown ans Eingabefeld angefügt wird.

Password: Versteckt die Benutzereingabe (z. B. für ein Passwort), indem die eingegebenen Zeichen mit Maskierungszeichen ersetzt werden. Es ist möglich, nach dem Wort Password ein benutzerdefiniertes Maskierungszeichen anzugeben. Zum Beispiel wird mit Password* das Standard-Maskierungszeichen (schwarzer Punkt in Windows XP) mit einem Sternchen ersetzt. Hinweis: Diese Option funktioniert nicht bei mehrzeiligen Eingabefeldern.

ReadOnly: Hindert den Benutzer daran, den Inhalt des Steuerelements zu ändern. Der Text kann aber weiterhin gescrollt, markiert und in die Zwischenablage kopiert werden.

Tn: Der Buchstabe T kann dazu verwendet werden, Tabstopps innerhalb eines mehrzeiligen Eingabefeldes zu setzen (da Tabstopps die Spaltenpositionen bestimmen, zu denen direkte TAB-Zeichen springen werden, können sie dazu verwendet werden, den Text in Spalten zu formatieren). Wenn der Buchstabe T nicht verwendet wird, werden die Tabstopps alle 32 Dialogeinheiten (DLU) gesetzt (die Breite jeder Dialogeinheit wird vom Betriebssystem bestimmt). Falls der Buchstabe T nur einmal verwendet wird, werden die Tabstopps auf je n Einheiten über die gesamte Breite des Steuerelements verteilt. Zum Beispiel wird mit Gui, Add, Edit, vEigenesEingabefeld r16 t64 der Standardabstand zwischen den Tabstopps verdoppelt. Zum Setzen benutzerdefinierter Tabstopps könnte der Buchstabe T wie folgt mehrmals verwendet werden: Gui, Add, Edit, vEigenesEingabefeld r16 t8 t16 t32 t64 t128. Ein Tabstopp ist jeweils eine absolute Spaltenposition in der Liste. Es können nur maximal 50 Tabstopps angegeben werden. Hinweis: Tabstopps benötigen ein mehrzeiliges Eingabefeld.

Uppercase: Die Zeichen werden bei Benutzereingabe automatisch in Großbuchstaben umgewandelt.

WantCtrlA [v1.0.44+]: Mit -WantCtrlA wird der Benutzer daran gehindert, den ganzen Text im Eingabefeld mit Strg+A zu markieren.

WantReturn: Mit -WantReturn ist ein mehrzeiliges Eingabefeld nicht mehr in der Lage, Enter für sich selbst zu nutzen. Folglich bewirkt das Drücken dieser Taste dann, dass die Standard-Schaltfläche im Fenster ausgelöst wird (falls vorhanden). Als Übergangslösung könnte der Benutzer Strg+Enter drücken, um eine neue Zeile zu beginnen.

WantTab: Führt dazu, dass Tab ein Tabulatorzeichen erzeugt, anstatt das nächste Steuerelement anzusteuern. Ohne diese Option kann der Benutzer Strg+Tab drücken, um ein neues Tabulatorzeichen innerhalb eines mehrzeiligen Eingabefeldes zu erzeugen. Hinweis: Obwohl WantTab auch in einem einzeiligen Eingabefeld funktioniert, werden Tabulatorzeichen als kastenförmige Zeichen dargestellt (die aber weiterhin gültige Tabulatorzeichen sind).

-Wrap: Deaktiviert den automatischen Zeilenumbruch in einem mehrzeiligen Eingabefeld. Da dieser Style nach dem Erstellen des Steuerelements nicht mehr geändert werden kann, verwendet eine der folgenden Methoden:

  1. Das Fenster und die dazugehörigen Steuerelemente zerstören und danach neu erstellen.
  2. Zwei übereinander liegende Eingabefelder erstellen, bei denen das erste den automatischen Zeilenumbruch aktiviert hat und beim anderen nicht. Dabei bleibt das nicht benutzte Eingabefeld leer und/oder versteckt.

Siehe allgemeine Optionen für weitere Optionen wie Right, Center und Hidden. Siehe auch: Position und Größe von Steuerelementen anpassen.

Eingabefeld mit mehr Funktionen: HiEdit ist ein frei verfügbares Eingabefeld, das unter anderem mehrere Tabs ermöglicht, große Dateien öffnet und sehr wenig Speicher verbraucht. Es kann sowohl Text- als auch Binärdateien bearbeiten. Für Details siehe Thread von majkinetor.

UpDown

Beschreibung: Zwei Pfeiltasten, die der Benutzer drücken kann, um einen Wert zu erhöhen oder zu verringern. Standardmäßig fügt sich ein UpDown automatisch an das zuvor hinzugefügte Steuerelement an. Dieses vorherige Steuerelement wird auch als Buddy bezeichnet. Das bekannteste Beispiel dafür ist der "Spinner" (Drehrad) - ein UpDown, verbunden mit einem Eingabefeld. Zum Beispiel:

Gui, Add, Edit
Gui, Add, UpDown, vEigenesUpDown Range1-10, 5

Im obigen Beispiel ist das Eingabefeld das Buddy von UpDown. Sobald der Benutzer eine der Pfeiltasten drückt, wird die Zahl im Eingabefeld automatisch erhöht oder verringert.

Ein Buddy von UpDown kann auch ein Textfeld oder Listenfeld sein. Aufgrund von Einschränkungen des Betriebssystems funktionieren alle anderen Steuerelemente (wie Kombinationsfeld oder Dropdownliste) möglicherweise nicht richtig mit g-Labels und anderen Funktionen.

Die Anfangsposition des UpDowns kann im letzten Parameter angegeben werden (wird dieser weggelassen, beginnt die Position bei 0 oder bei der nächstmöglichen Zahl).

Beim Verwenden des Befehls Gui Submit erhält die zugewiesene Ausgabevariable des Steuerelements (falls vorhanden) die numerische Position von UpDown. Falls das UpDown mit einem Eingabefeld verbunden ist und die Gültigkeit der Benutzereingabe irrelevant ist, dann sollte man am besten den Wert vom UpDown verwenden, anstatt den vom Eingabefeld. Das liegt daran, dass das UpDown immer eine Zahl im gültigen Bereich zurückgibt, auch dann, wenn der Benutzer im Eingabefeld keine Zahl oder eine Zahl außerhalb des Bereichs eingegeben hat. Beachtet zudem, dass Zahlen mit mehr als 3 Stellen standardmäßig einen Tausender-Trennzeichen (z. B. Komma) erhalten. Diese Trennzeichen werden in die Ausgabevariable des Eingabefeldes gespeichert, aber nicht in die vom UpDown.

Falls das UpDown ein g-Label hat, wird es ausgeführt, sobald der Benutzer eine der Pfeiltasten anklickt oder eine Pfeiltaste auf der Tastatur drückt. Beim Ausführen des g-Labels wird zudem die UpDown-Position in die zugewiesene Ausgabevariable gespeichert (falls vorhanden).

Optionen

Horz: Die Pfeiltasten zeigen nach links und rechts, anstatt nach oben und unten. Standardmäßig wird dabei das Steuerelement vom Buddy getrennt. Dieses Verhalten kann aber mit Horz 16 in den Optionen des Steuerelements überschrieben werden.

Left: Setzt das UpDown an die linke Seite des Buddys, anstatt an die rechte Seite.

Range: Bestimmt einen benutzerdefinierten Bereich (standardmäßig von 0 bis 100). Nach dem Wort Range erfolgt der Minimalwert, ein Bindestrich und der Maximalwert. Zum Beispiel würde Range1-1000 eine Zahl zwischen 1 und 1000 erlauben, Range-50-50 eine Zahl zwischen -50 und 50 und Range-10--5 eine Zahl zwischen -10 und -5. Die Minimal- und Maximalwerte können vertauscht werden, damit die Pfeiltasten in die entgegengesetzte Richtung wirken. Der größte zulässige Bereich geht von -2147483648 bis 2147483647. Windows 95 und NT4 benötigen jedoch Internet Explorer 5.0 oder höher, um einen größeren Bereich als -32767 bis 32767 zu unterstützen. Wenn das Buddy eine ListBox ist, gilt der Standardbereich von 32767 bis 0 für vertikal und umgekehrt für horizontal (Horz).

Wrap: Bewirkt, dass das Steuerelement auf das entgegengesetzte Bereichsende springt, falls der Minimal- oder Maximalwert überschritten wird. Ohne Wrap stoppt das Steuerelement beim Erreichen des Minimal- oder Maximalwertes.

-16: Bewirkt, dass ein vertikales UpDown vom Buddy getrennt wird. Das führt auch dazu, dass das Steuerelement jede angegebene Breite, Höhe und Position berücksichtigt, anstatt sich nach der Größe des Buddys zu richten. Darüber hinaus merkt sich ein isoliertes UpDown intern die eigene Position. Diese Position kann beispielsweise mit Gui Submit ermittelt werden.

0x80: Mit 0x80 in den Optionen werden die Tausender-Trennzeichen weggelassen, die jeweils zwischen drei Dezimalstellen im Buddy vorkommen. Normalerweise wird dieser Style nicht verwendet, weil die Trennzeichen weggelassen werden, sobald die Zahl vom UpDown selbst abgerufen wird (statt vom Buddy).

Beliebige Schrittweite: In diesem Script von NumEric wird deutlich, wie die UpDown-Schrittweite geändert werden kann (z. B. 5 oder 0.1).

Siehe auch: Position und Größe von Steuerelementen anpassen.

Picture (oder Pic)

Beschreibung: Ein Bereich, in dem ein Bild angezeigt wird (siehe die letzten beiden Absätze für unterstützte Dateitypen). Der letzte Parameter ist der Dateiname des Bildes, das sich im A_WorkingDir-Verzeichnis befindet, falls kein absoluter Pfad angegeben ist.

Beispiel: Gui, Add, Picture, w300 h-1, C:\Eigene Bilder\Firmenlogo.gif

Um die aktuelle Breite und/oder Höhe des Bildes zu erhalten, lasst die Optionen W und/oder H weg. Ansonsten wird das Bild auf die angegebene Breite und/oder Höhe skaliert (diese Breite und Höhe bestimmt auch, welches Icon aus einer ICO-Datei, das mehrere Icons enthält, geladen wird). Um das Bild unter Beibehaltung des Seitenverhältnisses zu verkleinern oder zu vergrößern, verwendet -1 bei einer Abmessung und eine positive Zahl bei der anderen. Zum Beispiel würde mit "w200 h-1" das Bild 200 Pixel breit gemacht und dessen Höhe automatisch gesetzt werden. Sobald das Bild nicht geladen oder angezeigt werden kann (z. B. Datei nicht gefunden), wird das Steuerelement leer gelassen und die Breite und Höhe auf 0 gesetzt.

Es ist möglich, ein g-Label wie gEigeneSubroutine in den Optionen des Steuerelements anzugeben. Dadurch würde das Label EigeneSubroutine automatisch ausgeführt, sobald der Benutzer das Bild anklickt. Ein Doppelklick kann erkannt werden, wenn überprüft wird, ob A_GuiEvent das Wort DoubleClick enthält.

Um ein Bild als Hintergrund bei anderen Steuerelementen zu verwenden, sollte es normalerweise vor diesen Steuerelementen eingefügt werden. Wenn diese Steuerelemente allerdings eingabefähig sind und das Bild ein g-Label hat, sollte das Bild erst nach diesen Steuerelementen erstellt und 0x4000000 (WS_CLIPSIBLINGS) in den Optionen von Picture eingefügt werden. Mit diesem Trick kann das Bild auch der Hintergrund eines Tab- oder ListView-Steuerelements sein.

Icon, Cursor und animierte Cursor: Icon und Cursor können von folgenden Dateitypen geladen werden: ICO, CUR, ANI, EXE, DLL, CPL, SCR und andere Typen, die Icon-Ressourcen enthalten. Um anstelle der ersten Icon-Gruppe eine andere zu verwenden, fügt in den Optionen das Wort Icon und danach die Gruppennummer ein. Im folgenden Beispiel wird das Standard-Icon aus der zweiten Icon-Gruppe verwendet: Gui, Add, Picture, Icon2, C:\Eigene Anwendung.exe.

Das Wort AltSubmit in den Optionen teilt dem Programm mit, zum Laden des Bildes die Microsoft-Datei GDIPlus.dll zu verwenden. Das könnte dazu führen, dass GIF-, BMP- und Icon-Bilder anders dargestellt werden. Zum Beispiel könnte dadurch mithilfe von BackgroundTrans der transparente Hintergrund eines ICO/GIF-Bildes wirksam gemacht werden. Wenn GDIPlus nicht verfügbar ist (siehe nächster Absatz), wird AltSubmit ignoriert und das Bild mit der normalen Methode geladen.

Alle Betriebssysteme unterstützen GIF, JPG, BMP, ICO, CUR und ANI. In Windows XP (oder höher) werden zusätzliche Bildformate wie PNG, TIF, Exif, WMF und EMF unterstützt. Zur Unterstützung von Betriebssystemen älter als XP kann die frei verfügbare GDIPlus.dll von Microsoft in den AutoHotkey-Ordner kopiert werden (bei kompilierten Scripts muss diese Datei jedoch in den Ordner des Scripts kopiert werden). Um diese Datei herunterzuladen, sucht mit Begriffen wie "gdi redistributable" auf www.microsoft.com.

In Windows 95/98/Me verursachen animierte Cursor größer als 90x90 einen Absturz des Scripts.

Animierte GIF-Bilder: Animierte GIF-Dateien werden zwar in einem Bild-Steuerelement angezeigt, aber nicht animiert. Zur Lösung dieses Problems kann die AniGIF.dll verwendet werden (frei verfügbar, solange sie nicht kommerziell genutzt wird). Siehe dazu "Show animated Gifs in your GUI".

Button

Beschreibung: Eine Schaltfläche, die gedrückt werden kann, um eine Aktion auszulösen. In diesem Fall ist der letzte Parameter der Name der Schaltfläche (auf der Schaltfläche selbst angezeigt), der LF-Zeichen (`n) enthalten kann, um neue Zeilen zu beginnen.

Beispiel: Gui, Add, Button, Default, OK

Das obige Beispiel enthält das Wort Default in den Optionen, um das "OK" als Standard-Schaltfläche zu definieren. Die Aktion der Standard-Schaltfläche wird automatisch ausgelöst, sobald der Benutzer Enter drückt, sofern der Tastaturfokus nicht auf einer anderen Schaltfläche oder auf ein mehrzeiliges Eingabefeld mit aktiviertem WantReturn-Style liegt. Die Standard-Schaltfläche kann jederzeit gewechselt werden, wenn wie folgt vorgegangen wird: GuiControl, +default, Cancel. Diese Eigenschaft kann später wieder wie folgt entfernt werden: GuiControl, -default, OK.

Falls der Name der Schaltfläche ein Und-Zeichen (&) enthält, wird der nachfolgende Buchstabe unterstrichen. Zum Beispiel:

Gui, Add, Button,, &Pause

Im obigen Beispiel wird der Buchstabe P unterstrichen, dass dem Benutzer erlaubt, Alt+P als Tastenkürzel zu verwenden. Um dieses Und-Zeichen stattdessen direkt anzuzeigen, verwendet zwei aufeinanderfolgende Und-Zeichen (&&).

Sollte der Schaltfläche ein g-Label fehlen, wird ihr automatisch ein Label zugewiesen. Wenn zum Beispiel das erste GUI-Fenster eine OK-Schaltfläche enthält, wird das ButtonOK-Label (falls vorhanden) ausgeführt, sobald die Schaltfläche gedrückt wird. Abgesehen vom ersten GUI-Fenster werden bei anderen GUI-Fenstern deren Nummer vor dem Label angefügt, wie zum Beispiel 2ButtonOK.

Wenn der Text auf der Schaltfläche Leerzeichen, &, `r, `n, `t oder ` enthält, werden diese automatisch beim dazugehörigen Label weggelassen. Zum Beispiel würde der Schaltfläche mit der Bezeichnung "&Pause" automatisch das Label ButtonPause zugewiesen. Ebenso würde der Schaltfläche mit dem Namen "Speichern && Beenden" das Label ButtonSpeichernBeenden bereitgestellt (das Und-Zeichen-Paar wird dazu verwendet, ein Und-Zeichen direkt anzuzeigen).

Bekannte Einschränkung: Es ist möglich, dass bestimmte Desktop-Designs den Text einer Schaltfläche nicht richtig darstellen. Daher könnte die Option -Wrap dabei helfen, dieses Problem zu umgehen. Allerdings ist dadurch der Text nicht länger mehrzeilig.

CheckBox

Beschreibung: Ein Kontrollkästchen, in der eine Markierung gesetzt oder entfernt werden kann, um An/Aus, Ja/Nein und so weiter darzustellen.

Beispiel: Gui, Add, Checkbox, vSendenAnRechnungsadresse, An Rechnungsadresse senden?

Der letzte Parameter ist eine Bezeichnung, die neben dem Kästchen angezeigt wird. Diese Bezeichnung dient normalerweise als Abfrage, oder beschreibt die Funktionsweise des Kontrollkästchens näher. Es können LF-Zeichen (`n) enthalten sein, um neue Zeilen zu beginnen. Wenn eine Breite (W) in den Optionen angegeben ist, aber keine Zeilen (R) oder Höhe (H), wird der Text des Steuerelements je nach Bedarf umbrochen und dessen Höhe automatisch gesetzt. Die zugewiesene Ausgabevariable des Kontrollkästchen (falls vorhanden) erhält eine 1 für markiert, 0 für nicht markiert, und -1 für grau/unbestimmt.

Mit dem Wort Check3 in den Optionen wird der dritte Status aktiviert, um anstelle des schwarzen ein graues Häkchen anzuzeigen (dieser Status zeigt an, dass die Markierung des Kontrollkästchens weder gesetzt noch entfernt wurde). Mit dem Wort Checked oder CheckedGray in den Optionen hat das Kontrollkästchen anfangs jeweils ein schwarzes oder graues Häkchen. Nach dem Wort Checked kann optional eine 0, 1 oder -1 für den Anfangsstatus angegeben werden. Das heißt, dass "Checked" und "Checked%VarMitEins%" identisch sind.

Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Diese Option bewirkt, dass EigeneSubroutine automatisch ausgeführt wird, sobald der Benutzer das Kontrollkästchen anklickt oder ändert.

Bekannte Einschränkung: Es ist möglich, dass bestimmte Desktop-Designs die Bezeichnung nicht richtig darstellen. Daher könnte die Option -Wrap dabei helfen, dieses Problem zu umgehen. Allerdings ist dadurch der Text nicht länger mehrzeilig.

Radio

Beschreibung: Ein Optionsfeld in Form eines kleinen leeren Kreises, in der eine Markierung gesetzt (An) oder entfernt (Aus) werden kann.

Beispiel: Gui, Add, Radio, vEigeneRadioGruppe, Warten, bis alle Artikel auf Lager sind, bevor sie abgeschickt werden.

Diese Steuerelemente befinden sich für gewöhnlich in Optionsfeldgruppen, bestehend aus 2 oder mehr Optionsfelder. Sobald der Benutzer ein Optionsfeld anklickt, um es zu aktivieren, werden alle anderen Optionsfelder in der Gruppe automatisch deaktiviert (innerhalb einer Gruppe kann der Benutzer außerdem mit Pfeiltasten navigieren). Eine Optionsfeldgruppe wird automatisch um alle hintereinander hinzugefügten Optionsfelder erstellt. Um eine neue Gruppe zu beginnen, verwendet das Wort Group in den Optionen des ersten Optionsfeldes - oder fügt dazwischen einfach ein anderes Steuerelement ein, dass automatisch eine neue Gruppe startet.

Der letzte Parameter ist eine Bezeichnung, die neben dem Optionsfeld angezeigt wird. Diese Bezeichnung dient normalerweise als Abfrage, oder beschreibt die Funktionsweise des Kontrollkästchens näher. Es können LF-Zeichen (`n) enthalten sein, um neue Zeilen zu beginnen. Wenn eine Breite (W) in den Optionen angegeben ist, aber keine Zeilen (R) oder Höhe (H), wird der Text des Steuerelements je nach Bedarf umbrochen und dessen Höhe automatisch gesetzt.

Mit dem Wort Checked in den Optionen wird die Schaltfläche anfangs aktiviert. Nach dem Wort Checked kann optional eine 0 oder 1 für den Anfangsstatus angegeben werden: 0 für markiert und 1 für nicht markiert. Das heißt, dass "Checked" und "Checked%VarMitEins%" identisch sind.

Die zugewiesene Ausgabevariable des Optionsfeldes (falls vorhanden) erhält eine 1 für markiert und 0 für nicht markiert. Wenn jedoch nur ein Optionsfeld in einer Optionsfeldgruppe eine Variable hat, wird in dieser Variable stattdessen die Nummer des aktuell ausgewählten Optionsfeldes gespeichert, dabei steht die 1 für das erste Optionsfeld (gemäß der originalen Erstellungsreihenfolge), 2 für das zweite und so weiter. Sollte kein Optionsfeld markiert sein, wird 0 gespeichert.

Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Diese Option bewirkt, dass EigeneSubroutine automatisch ausgeführt wird, sobald der Benutzer das Optionsfeld aktiviert. Im Gegensatz zum Einzelvariablenmodus vom vorherigen Abschnitt muss das g-Label für jedes Optionsfeld in einer Gruppe angegeben werden. Dadurch wird die Flexibilität gewährleistet, das Anklicken von bestimmten Optionsfeldern zu ignorieren. Ein Doppelklick kann erkannt werden, wenn überprüft wird, ob A_GuiEvent das Wort DoubleClick enthält.

Bekannte Einschränkung: Es ist möglich, dass bestimmte Desktop-Designs die Bezeichnung nicht richtig darstellen. Daher könnte die Option -Wrap dabei helfen, dieses Problem zu umgehen. Allerdings ist dadurch der Text nicht länger mehrzeilig.

DropDownList (oder DDL)

Beschreibung: Eine Auswahlliste, die angezeigt wird, wenn eine kleine Schaltfläche gedrückt wird. In diesem Fall ist der letzte Parameter eine Auswahlliste, deren Einträge durch senkrechte Striche getrennt sind.

Beispiel: Gui, Add, DropDownList, vFarbauswahl, Schwarz|Weiß|Rot|Grün|Blau

Damit einer der Einträge beim erstmaligen Anzeigen des Fensters vorausgewählt ist, fügt nach diesem Eintrag zwei senkrechte Striche an (z. B. Rot|Grün||Blau). Alternativ kann mit dem Wort Choose in den Optionen, unmittelbar gefolgt von der Nummer, ein Eintrag auch vorausgewählt werden. Zum Beispiel würde mit Choose5 der fünfte Eintrag vorausgewählt (wie bei anderen Optionen kann hierbei auch eine Variable angegeben werden, zum Beispiel Choose%Var%). Mit GuiControl können Einträge in der Auswahlliste nachträglich geändert oder hinzugefügt bzw. entfernt werden.

Mit dem Wort Uppercase oder Lowercase in den Optionen werden alle Einträge in der Liste jeweils automatisch in Groß- oder Kleinbuchstaben umgewandelt. Mit dem Wort Sort wird der Inhalt der Liste automatisch alphabetisch sortiert (das auch Auswirkung auf später hinzugefügte Einträge mittels GuiControl hat). Diese Option aktiviert zudem die inkrementelle Suche, sobald die Liste aufgeklappt wird. Das heißt, dass ein Eintrag ausgewählt wird, sobald die ersten Zeichen des Namens eingegeben werden.

Wenn der Befehl Gui Submit verwendet wird, wird in der zugewiesenen Ausgabevariable des Steuerelements (falls vorhanden) der Text des zurzeit ausgewählten Eintrags gespeichert. Falls das Steuerelement allerdings die AltSubmit-Eigenschaft hat, wird die Ausgabevariable stattdessen die Positionsnummer des Eintrags enthalten (der erste Eintrag ist 1, der zweite 2 und so weiter).

Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Diese Option bewirkt, dass EigeneSubroutine automatisch ausgeführt wird, sobald der Benutzer einen neuen Eintrag auswählt.

Mit der R- oder H-Option kann die Höhe der Aufklappliste bestimmt werden. Zum Beispiel wird mit R5 die Liste 5 Zeilen hoch gemacht. Wenn sowohl R als auch H weggelassen wird, erweitert sich die Liste automatisch, um die Vorteile der verfügbaren Desktophöhe des Benutzers zu übernehmen (allerdings zeigen Betriebssysteme älter als Windows XP standardmäßig 3 Zeilen).

Das Trennzeichen zwischen den Feldern muss nicht zwingend ein senkrechter Strich sein. Zum Beispiel kann das Trennzeichen mit Gui +Delimiter`n in ein Zeilenumbruchszeichen und mit Gui +DelimiterTab in ein Tabulatorzeichen (`t) geändert werden.

Falls das Steuerelement allerdings die AltSubmit-Eigenschaft hat, wird die Ausgabevariable stattdessen die Positionsnummer des Eintrags enthalten (der erste Eintrag ist 1, der zweite 2 und so weiter).

Zum Beispiel würde 1|2|3 bedeuten, dass die ersten drei Einträge ausgewählt sind. Um die Einträge einzeln aus den String zu extrahieren, verwendet eine Parsing-Schleife wie folgt:

Loop, Parse, EigenesListenfeld, |

Falls der Buchstabe T nur einmal verwendet wird, werden die Tabstopps auf je n Einheiten über die gesamte Breite des Steuerelements verteilt. Link

Beschreibung: Ein Textfeld, das anklickbare Links enthalten kann. Link-Steuerelemente verwenden die HTML-Sprache (Hypertext Markup Language), von der jedoch nur das A-Tag unterstützt wird. Innerhalb des öffnenden Tags kann ein Attribut im Format Attribut="Wert" angegeben werden. Normalerweise wird hierbei ein ID- oder HREF-Attribut verwendet, zum Beispiel:

Beispiel: Gui, Add, Link,, Das ist ein <a href="http://de.autohotkey.com">Link</a>
Beispiel: Gui, Add, Link,, Links können überall im Text angegeben werden, wie beispielsweise <a id="1">hier</a> oder <a id="2">dort</a>

Das Anklicken eines Links wird automatisch bearbeitet, falls das HREF-Attribut gesetzt wurde und ausführbar ist (z. B. eine URL oder ein Befehl). Wenn das HREF-Attribut nicht verwendet wird oder ausführbar ist, könnte ein bereitgestelltes g-Label wie gEigeneSubroutine ausgeführt werden. In diesem Fall enthält ErrorLevel den Attributswert, damit der angeklickte Link identifiziert werden kann.

Hotkey

Beschreibung: Sieht wie ein einzeiliges Eingabefeld aus, das aber nur eine gedrückte Tastenkombination vom Benutzer akzeptiert. Wenn der Benutzer zum Beispiel Strg+Alt+C auf einer deutschen Tastaturbelegung drückt, erscheint im Feld "STRG + ALT + C".

Beispiel: Gui, Add, Hotkey, vGewählterHotkey

Wenn der Befehl Gui Submit verwendet wird, wird in der zugewiesenen Ausgabevariable des Steuerelements (falls vorhanden) die Modifikatoren und der Name des Hotkeys gespeichert, die kompatibel mit dem Hotkey-Befehl sind. Beispiele: ^!C, +!Home, +^Down, ^Numpad1, !NumpadEnd. Falls kein Hotkey im Steuerelement zu sehen ist, wird die Ausgabevariable leer gemacht. Hinweis: Einige angezeigten Tasten zeigen das Gleiche an, obwohl sie unterschiedliche Namen haben. Zum Beispiel wird sowohl ^Numpad7 als auch ^NumpadHome als "STRG + 7 (ZEHNERTASTATUR)" angezeigt.

Standardmäßig ist das Steuerelement anfangs leer. Um das zu ändern, kann im letzten Parameter ein Hotkey angegeben werden, mit oder ohne Modifikatoren. Zum Beispiel: Gui, Add, Hotkey, vGewählterHotkey, ^!p.
Es werden nur die Modifikatoren ^ (Strg), ! (Alt) und + (Umschalt) unterstützt. In der Tastenliste können die verfügbaren Tasten eingesehen werden.

Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Diese Option bewirkt, dass EigeneSubroutine automatisch ausgeführt wird, sobald der Benutzer den Hotkey ändert. Zudem wird jedes mal, wenn das g-Label ausgeführt wird, der Hotkey in die zugewiesene Ausgabevariable gespeichert (falls vorhanden). Hinweis: Das g-Label wird auch dann ausgeführt, wenn ein unvollständiger Hotkey vorhanden ist. Sobald der Benutzer beispielsweise Strg gedrückt hält, wird das g-Label ausgeführt und ein Zirkumflex (^) in die Ausgabevariable gespeichert. Wenn der Benutzer dann die Tastenkombination vollendet, wird das g-Label nochmals ausgeführt und der vollständige Hotkey in die Variable gespeichert.

Um die Benutzereingabe auf bestimmte Hotkeys zu beschränken, fügt nach dem Wort Limit die Summe folgender Zahlen an:

1 Tasten ohne Modifikatoren ignorieren
2 Umschalt+Tasten ignorieren
4 Strg+Tasten ignorieren
8 Alt+Tasten ignorieren
16 Umschalt+Strg+Tasten ignorieren
32 Umschalt+Alt+Tasten ignorieren
64 Dieser Wert wird nicht unterstützt (verhält sich falsch)
128 Umschalt+Strg+Alt+Tasten ignorieren

Zum Beispiel wird mit Limit1 verhindert, dass Hotkeys ohne Modifikatoren wie Buchstaben und Zahlen eingegeben werden können. Ebenso wird mit Limit15 die Anforderung gestellt, mindestens zwei Modifikatortasten einzugeben. Sollte der Benutzer eine verbotene Modifikator-Kombination eingeben, wird diese automatisch und sichtbar mit "STRG + ALT" ersetzt.

Das Hotkey-Steuerelement hat nur begrenzte Möglichkeiten. Zum Beispiel werden Maus/Joystick-Hotkeys oder Windows-Tasten (LWin und RWin) nicht unterstützt. Dieses Problem kann zum Beispiel umgangen werden, wenn eine oder mehrere Kontrollkästchen bereitgestellt werden, die zusätzliche Modifikatoren anbieten.

DateTime

Beschreibung: Sieht wie ein einzeiliges Eingabefeld aus, das aber nur die Eingabe von Datum und/oder Zeit akzeptiert. Ein ausklappbarer Kalender ist ebenfalls vorhanden.

Beispiel: Gui, Add, DateTime, vEigenesDateTime, LongDate

Der letzte Parameter kann eine der folgenden Angaben enthalten:

(nicht vorhanden): Beim Weglassen wird das kurze Datumsformat verwendet, abhängig vom aktuellen Gebietsschema des Benutzers. Bei einigen Gebietsschemen könnte das wie folgt aussehen: 17.04.2011.

LongDate: Verwendet das lange Datumsformat, abhängig vom aktuellen Gebietsschema des Benutzers. Bei einigen Gebietsschemen könnte das wie folgt aussehen: 17.04.2011.

Time: Zeigt nur die Zeit an, abhängig vom aktuellen Gebietsschema des Benutzers. Auch wenn das Datum nicht angezeigt wird, ist es dennoch im Steuerelement vorhanden und wird zusammen mit der Zeit im YYYYMMDDHH24MISS-Format abgerufen.

(benutzerdefiniertes Format): Bestimmt eine beliebige Kombination von Datum- und Zeitformaten. Zum Beispiel würde d.M.yy HH:mm wie 17.4.11 12:19 aussehen. Ebenso würde dddd MMMM d, yyyy hh:mm:ss dann wie Sonntag April 17, 2011 12:22:45 dargestellt. Buchstaben und Zahlen, die direkt angezeigt werden sollen, müssen wie folgt in Hochkommas gesetzt werden: 'Datum:' dd.MM.yy 'Zeit:' hh:mm:ss tt. Nicht-alphanumerische Zeichen wie Leerzeichen, Tabulatoren, Schrägstriche, Doppelpunkte, Kommas und andere Satzzeichen müssen hingegen nicht von Hochkommas umschlossen werden. Die Ausnahme ist das Hochkomma selbst: Zur Darstellung dieses Zeichens müssen vier aufeinanderfolgende Hochkommas ('''') verwendet werden, oder nur zwei, wenn das Hochkomma sich bereits innerhalb von zwei äußeren Hochkommas befindet.

Verwendung

Damit anfangs ein beliebiges Datum außer Heute vorausgewählt ist, fügt in den Optionen das Wort Choose ein, unmittelbar gefolgt von einem Datum im YYYYMMDD-Format. Zum Beispiel würde mit Choose20110417 das Datum 17. April 2011 vorausgewählt (wie bei anderen Optionen kann hierbei auch eine Variable angegeben werden, zum Beispiel Choose%Var%). Mit ChooseNone wird kein Datum ausgewählt. Zudem erstellt diese Option innerhalb des Steuerelements ein Kontrollkästchen, dessen Markierung entfernt wird, sobald das Steuerelement kein Datum hat. Sollte das Steuerelement kein Datum haben, ermitteln Gui Submit und GuiControlGet einen leeren String.

Die Tageszeit kann mit angegeben werden. Allerdings muss sich davor ein Datum befinden, wenn es im Steuerelement gesetzt oder abgerufen wird. Das Zeitformat ist HH24MISS (Stunden, Minuten, Sekunden), wobei die Stunden im 24-Stunden-Format angegeben werden müssen. Folglich ist ein vollständiger Datum-Zeit-String im YYYYMMDDHH24MISS-Format.

Bei einem Datum im YYYYMMDDHH24MISS-Format können nachfolgende Elemente weggelassen werden, um diese mit Standardwerten zu ersetzen. Element MM wird mit 01 ersetzt, DD mit 01, HH24 mit 00, MI mit 00 und SS mit 00.

Innerhalb des ausklappbaren Kalenders kann unten der Heute-String angeklickt werden, um das heutige Datum auszuwählen. Darüber hinaus ist das Jahr und der Monatsname anklickbar, um die Navigation zwischen den Monaten oder Jahren zu vereinfachen.

Navigation mit der Tastatur:

  • Innerhalb des Eingabefeldes: Mit den Pfeiltasten ↑ / ↓ , den Ziffernblocktasten + / - und den Tasten Pos1 / Ende können die Werte erhöht oder verringert werden. Mit den Pfeiltasten ← und → ist es möglich, zwischen den Feldern zu wechseln.
  • Innerhalb des ausklappbaren Kalenders: Mit den Tasten Bild ↑ / Bild ↓ einen Monat zurück/vor, mit den Tastenkombinationen Strg+Bild ↑ / Bild ↓ ein Jahr zurück/vor und mit den Tasten Pos1 / Ende jeweils den ersten/letzten Tag im Monat auswählen. Mit den Pfeiltasten ← und → ist es möglich, zwischen den Tagen zu wechseln.

Wenn der Befehl Gui Submit verwendet wird, wird in der zugewiesenen Ausgabevariable des Steuerelements (falls vorhanden) das Datum und die Zeit im YYYYMMDDHH24MISS-Format gespeichert. Sowohl das Datum als auch die Zeit sind vorhanden, unabhängig davon, ob sie tatsächlich im Steuerelement sichtbar sind.

Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Diese Option bewirkt, dass EigeneSubroutine automatisch ausgeführt wird, sobald der Benutzer das Datum oder die Zeit ändert. Jedes mal, wenn das Label ausgeführt wird, wird die zugewiesene Ausgabevariable (falls vorhanden) des Steuerelements auf den aktuell ausgewählten Datums-/Zeitwert aktualisiert.

Windows 95 und NT4 benötigen mindestens Internet Explorer 3.0, damit DateTime unterstützt wird.

Optionen

Choose: Siehe oben.

Range: Beschränkt die Datumsauswahl auf einem bestimmten Zeitraum. Nach dem Wort Range kann das minimale und maximale Datum im YYYYMMDD-Format angegeben werden (mit einem Bindestrich dazwischen). Zum Beispiel wird mit Range20050101-20050615 das Datum auf die ersten 5,5 Monaten des Jahres 2005 beschränkt. Einer der beiden Werte kann weggelassen werden, um die Beschränkung in dieser Richtung aufzuheben. Zum Beispiel würde Range20010101 verhindern, dass ein Tag vor 2001 ausgewählt werden kann. Ebenso würde Range-20091231 (Bindestrich davor) verhindern, dass ein Tag nach 2009 ausgewählt werden kann. Ist diese Option nicht vorhanden, kann jedes Datum zwischen den Jahren 1601 und 9999 ausgewählt werden. Die Tageszeit ist nicht einschränkbar.

Right: Bewirkt, dass der Kalender auf der rechten Seite des Eingabefeldes aufgeklappt wird.

1: Ersetzt die Schaltfläche des ausklappbaren Kalenders mit einem UpDown, um Datum-Zeit-Werte bearbeiten zu können. Funktioniert nicht in Verbindung mit LongDate.

2: Ermöglicht ein Kontrollkästchen innerhalb des Steuerelements, dessen Markierung entfernt werden kann, um die Auswahl des Datums/der Zeit aufzuheben. Diese Option kann nach Erstellen des Steuerelements nicht mehr geändert werden.

MonthCal

Beschreibung: Ein hohes und breites Steuerelement, das alle Tage des Monats im Kalenderformat anzeigt. Der Benutzer kann ein einzelnes Datum oder einen Datumsbereich auswählen.

Beispiel: Gui, Add, MonthCal, vEigenerKalender

Damit anfangs ein beliebiges Datum außer Heute vorausgewählt ist, fügt im letzten Parameter ein Datum im YYYYMMDD-Format ein (z. B. 20050531). Zudem kann ein Datumsbereich mithilfe eines Bindestrichs vorausgewählt werden (z. B. 20050525-20050531).

Bei diesem Steuerelement sollte die Breite (W) und Höhe (H) weggelassen werden, weil die Größe automatisch auf ein Monat angepasst wird. Um mehrere Monate vertikal darzustellen, verwendet R2 oder höher in den Optionen. Um mehrere Monate horizontal darzustellen, verwendet W-2 oder höher in den Optionen. Diese Optionen können gleichzeitig vorhanden sein, um in beide Richtungen zu erweitern.

Im unteren Bereich des Steuerelements kann der Heute-String angeklickt werden, um das heutige Datum auszuwählen. Darüber hinaus ist das Jahr und der Monatsname anklickbar, um die Navigation zwischen den Monaten oder Jahren zu vereinfachen.

Im Gegensatz zum ausklappbaren Kalender von DateTime wird die Tastaturnavigation in MonthCal generell nicht unterstützt.

Wenn der Befehl Gui Submit verwendet wird, wird in der zugewiesenen Ausgabevariable des Steuerelements (falls vorhanden) das ausgewählte Datum im YYYYMMDD-Format (ohne Zeitangabe) gespeichert. Sollte allerdings die Mehrfachauswahl-Option aktiv sein, wird sowohl der Minimal- als auch Maximalwert mit einem Bindestrich dazwischen abgerufen (z. B. 20050101-20050108). Falls in einem Mehrfachauswahl-Kalender nur ein Datum ausgewählt wurde, wird auch ein Minimal- und Maximalwert abgerufen, jedoch sind beide identisch. StringSplit kann dazu verwendet werden, die Datumswerte zu trennen. Im folgenden Beispiel wird der Minimalwert in Datum1 und der Maximalwert in Datum2 gespeichert: StringSplit, Datum, EigenerMonatskalender, -.

Wenn MonthCal einen g-Label hat, wird in der zugewiesenen Ausgabevariable des Steuerelements (falls vorhanden) das aktuell ausgewählte Datum oder der Bereich gespeichert. Standardmäßig wird das Label nur aufgerufen, wenn: 1) der Benutzer die Auswahl ändert, oder 2) nur alle zwei Minuten, falls ein neuer Tag erreicht wurde (dieses Verhalten ist eine Eigenart des Betriebssystems). Wenn allerdings das Wort AltSubmit in den Optionen des Steuerelements vorkommt, wird g-Label öfters ausgeführt. Zudem wird die interne Variable A_GuiEvent auf "Normal" gesetzt, wenn das Datum geändert wurde; auf 1, falls ein Datum angeklickt wurde; oder auf 2, wenn MonthCal die "Mauserfassung" freigibt. Wenn der Benutzer zum Beispiel ein neues Datum zweimal anklickt, wird das Label fünfmal ausgeführt: Einmal mit Normal, zweimal mit 1 und zweimal mit 2. Das kann verwendet werden, um die Doppelklicks zwischen den Instanzen von Nummer 1 durch Messen der Zeit zu erkennen.

Wenn das Datum im YYYYMMDD-Format angegeben wird, kann der MM- und/oder DD-Teil weggelassen werden, welche dann als 1 angesehen werden. Zum Beispiel wird 200205 als 20020501, und 2005 als 20050101 angesehen.

Windows 95 und NT4 benötigen mindestens Internet Explorer 3.0, damit MonthCal unterstützt wird.

Zum Beispiel: Gui, Add, MonthCal, vEigenerKalender, 20050101-20050108. Diese Option kann nach Erstellen des Steuerelements nicht mehr geändert werden.

Zum Beispiel wird mit Range20050101-20050615 die Auswahl auf die ersten 5,5 Monaten des Jahres 2005 beschränkt. Slider

Beschreibung: Ein Schieberegler, den der Benutzer entlang einer vertikalen oder horizontalen Strecke bewegen kann. Zum Beispiel ist die Standard-Lautstärkeregelung in der Taskleiste ein Schieberegler.

Beispiel: Gui, Add, Slider, vEigenerSchieberegler, 50

Im letzten Parameter kann die Startposition des Schiebereglers angegeben werden. Wenn dieser Parameter weggelassen wird, beginnt der Schieberegler bei 0 oder bei der Bereichszahl, die am nächsten an der Null ist.

Der Benutzer kann den Regler wie folgt verschieben:

  1. Mit der Maus ziehen
  2. Auf den Bereich der Schiene klicken
  3. Das Mausrad drehen, während das Steuerelement den Fokus hat
  4. Folgende Tasten drücken, während das Steuerelement den Fokus hat: Pfeiltasten, Bild ↑, Bild ↓, Pos1 und Ende.

Wenn der Befehl Gui Submit verwendet wird, wird in der zugewiesenen Ausgabevariable des Steuerelements (falls vorhanden) die numerische Position des Schiebereglers gespeichert. Die Position ist auch dann enthalten, wenn das g-Label des Steuerelements ausgeführt wird.

Ein g-Label wird standardmäßig nur ausgeführt, wenn der Benutzer aufgehört hat, den Regler zu bewegen (z. B. durch Loslassen der Maustaste, nachdem die Maus gezogen wurde). Wenn allerdings das Wort AltSubmit in den Optionen des Steuerelements vorkommt, wird das g-Label bei allen Ereignissen des Schiebereglers ausgeführt und die interne Variable A_GuiEvent auf eine der folgenden Zahlen oder Strings gesetzt:

0
 Der Benutzer hat ← oder ↑ gedrückt.
1
 Der Benutzer hat → oder ↓ gedrückt.
2
 Der Benutzer hat Bild ↑ gedrückt.
3
 Der Benutzer hat Bild ↓ gedrückt.
4
 Der Benutzer hat den Schieberegler mit dem Mausrad oder mit Ziehen und Ablegen verschoben.
5
 Der Benutzer zieht den Schieberegler momentan mit der Maus. Das heißt, dass die Maustaste zurzeit gedrückt wird.
6
 Der Benutzer hat Pos1 gedrückt, um den Regler ganz nach links oder nach oben zu verschieben.
7
 Der Benutzer hat Ende gedrückt, um den Regler ganz nach rechts oder nach unten zu verschieben.
Normal
 Der Benutzer hat das Verschieben des Reglers entweder mit der Maus oder mit der Tastatur beendet. Hinweis: Auch wenn das g-Label bereits bei einem der oben genannten Zahlenereignissen ausgeführt wurde, wird es bei diesem Ereignis nochmals ausgeführt, außer bei Nummer 4 (Mausradbewegung).

Optionen

Buddy1/Buddy2: Bestimmt bis zu zwei vorhandene Steuerelemente, die automatisch an beiden Enden des Schiebereglers positioniert werden. Buddy1 wird auf der linken oder oberen Seite angezeigt (je nachdem, ob die Option Vertical vorhanden ist). Buddy2 wird auf der rechten oder unteren Seite angezeigt. Nach dem Wort Buddy1 oder Buddy2 kann der Variablenname eines vorhandenen Steuerelements angegeben werden. Zum Beispiel wird mit Buddy1EigenerObertext das Steuerelement mit dem Variablennamen EigenerObertext zugewiesen. Windows 95 und NT4 benötigen Internet Explorer 3.0 oder höher, um diese Option zu unterstützen.

Center: Der Einstellknopf (welcher durch den Benutzer verschoben wird) ist in keine Richtung ausgerichtet.

Invert: Dreht das Steuerelement um, damit der niedrigere Wert rechts/unten ist, anstatt links/oben. Das wird normalerweise benutzt, um einen vertikalen Schieberegler als traditionelle Lautstärkeregelung darzustellen. Hinweis: Die unten beschriebene ToolTip-Option berücksichtigt nicht die Umkehrung und sollte in diesem Fall nicht verwendet werden.

Left: Der Einstellknopf (welcher durch den Benutzer verschoben wird) wird nach oben, anstatt nach unten ausgerichtet. Ist jedoch die Vertical-Option vorhanden, wird der Einstellknopf nach links, anstatt nach rechts ausgerichtet.

Line: Bestimmt die Anzahl der Positionen, die verschoben werden, falls der Benutzer eine Pfeiltaste drückt. Nach dem Wort Line kann die Anzahl der zu bewegenden Positionen angegeben werden. Zum Beispiel: Line2

NoTicks: Entfernt die Skalenstriche entlang der Strecke.

Page: Bestimmt die Anzahl der Positionen, die verschoben werden, falls der Benutzer Bild ↑ oder Bild ↓ drückt. Nach dem Wort Page kann die Anzahl der zu bewegenden Positionen angegeben werden. Zum Beispiel: Page10

Range: Bestimmt einen benutzerdefinierten Bereich (standardmäßig von 0 bis 100). Nach dem Wort Range erfolgt der Minimalwert, ein Bindestrich und der Maximalwert. Zum Beispiel kann mit Range1-1000 eine Zahl zwischen 1 und 1000 ausgewählt werden, mit Range-50-50 eine Zahl zwischen -50 und 50 und mit Range-10--5 eine Zahl zwischen -10 und -5.

Thick: Bestimmt die Länge des Einstellknopfs (welcher durch den Benutzer verschoben wird). Nach dem Wort Thick erfolgt die Dicke in Pixel (z. B. Thick30). Um die Dicke in Windows XP oder höher zu erhöhen, ist es wahrscheinlich notwendig, die Center-Option anzugeben oder das Design des Steuerelements zu entfernen (dazu muss -Theme in den Optionen des Steuerelements angegeben werden).

TickInterval: Ermöglicht Skalenstriche entlang der Strecke in einem bestimmten Intervall. Nach dem Wort TickInterval erfolgt das Intervall, um zusätzliche Skalenstriche anzuzeigen (sollte der Intervall weggelassen werden, wird standardmäßig 1 verwendet). Zum Beispiel wird mit TickInterval10 alle 10 Positionen ein Skalenstrich angezeigt.

ToolTip: Erstellt ein Tooltip, der die numerische Position beim Verschieben des Reglers anzeigt. Um den Tooltip an einer anderen Position anzuzeigen, verwendet Folgendes: ToolTipLeft oder ToolTipRight (für horizontale Schieberegler); ToolTipTop oder ToolTipBottom (für vertikale Schieberegler). Windows 95 und NT4 benötigen Internet Explorer 3.0 oder höher, um diese Option zu unterstützen.

Vertical: Damit kann der Schieberegler nach oben oder nach unten verschoben werden, anstatt nach links oder nach rechts.

Die oben genannten Optionen können mit GuiControl nachträglich geändert werden.

Progress

Beschreibung: Ein Fortschrittsbalken in zwei Farben, der visuell darstellt, wie weit eine Operation voran geschritten ist.

Beispiel: Gui, Add, Progress, w300 h20 cBlue vEigenesProgress

Im letzten Parameter kann die Anfangsposition des Balkens angegeben werden (ist dieser nicht vorhanden, beginnt der Balken bei 0 oder bei der Bereichszahl, die am nächsten an der Null ist). Um die Position des Balkens später zu ändern, befolgt diese Beispiele, die sich auf einen Fortschrittsbalken mit dem zugewiesenen Variablennamen EigenesProgress beziehen:

GuiControl,, EigenesProgress, +20  ; Die Position wird um 20 erhöht.
GuiControl,, EigenesProgress, 50  ; Die Position wird auf 50 gesetzt.

Bei horizontalen Fortschrittsbalken wird die Dicke des Balkens durch die Höhe des Steuerelements bestimmt. Bei vertikalen Fortschrittsbalken durch die Breite des Steuerelements.

Optionen

Cn: Ändert die Farbe des Balkens. Ersetzt n mit einer HTML-Primärfarbe oder einem 6-stelligen RGB-Farbwert. Beispiele: cRed, cFFFF33, cDefault. Sollte die C-Option nie verwendet werden (oder cDefault angegeben sein), wird die Standardfarbe des Betriebssystems verwendet.

BackgroundN: Ändert die Hintergrundfarbe des Balkens. Ersetzt n mit einer HTML-Primärfarbe oder einem 6-stelligen RGB-Farbwert. Beispiele: BackgroundGreen, BackgroundFFFF33, BackgroundDefault. Sollte die Background-Option nie verwendet werden (oder BackgroundDefault angegeben sein), wird standardmäßig die Hintergrundfarbe des Fensters oder Tab-Steuerelements verwendet.

Range: Bestimmt einen benutzerdefinierten Bereich (standardmäßig von 0 bis 100). Nach dem Wort Range erfolgt der Minimalwert, ein Bindestrich und der Maximalwert. Zum Beispiel ermöglicht Range1-1000 eine Zahl zwischen 1 und 1000, Range-50-50 eine Zahl zwischen -50 und 50 und Range-10--5 eine Zahl zwischen -10 und -5. In Windows 95 und NT4 verhalten sich negative Bereiche und Bereiche über 65535 erst korrekt, wenn Internet Explorer 3.0 oder höher installiert ist.

-Smooth: Teilt den Balken im mehrere Segmente, anstatt einen durchgehenden Balken anzuzeigen. Zudem ist -Smooth eine der Voraussetzungen, um einen thematisch gestalteten Fortschrittsbalken in Windows XP oder höher anzuzeigen. Die andere Voraussetzung ist, dass der Balken keine benutzerdefinierten Farben enthalten darf. Das heißt, dass die Optionen C und Background weggelassen werden müssen. Windows 95 und NT4 benötigen Internet Explorer 3.0 oder höher, um diese Option zu unterstützen.

Vertical: Bewirkt, dass der Balken vertikal steigt oder fällt, anstatt sich auf horizontaler Ebene zu bewegen. Windows 95 und NT4 benötigen Internet Explorer 3.0 oder höher, um diese Option zu unterstützen.

Die oben genannten Optionen können mit GuiControl nachträglich geändert werden.

GroupBox

Beschreibung: Ein Gruppenfeld mit rechteckigem Rahmen, das dazu verwendet wird, die gemeinschaftliche Beziehung mehrere Steuerelemente darzustellen. In diesem Fall enthält der letzte Parameter den Titel des Rahmens, der in der oberen linken Ecke angezeigt wird.

Beispiel: Gui, Add, GroupBox, w400 h300, Geographische Kriterien

Standardmäßig ist der Titel des Gruppenfeldes nur einzeilig. Das kann mit Wrap in den Optionen des Steuerelements überschrieben werden.

Um die Anzahl der Zeilen innerhalb des Steuerelements zu bestimmen (oder die Höhe und Breite), siehe Position und Größe von Steuerelementen anpassen.

Tab2

Beschreibung: Ein großes Steuerelement mit mehreren Seiten, die jeweils weitere Steuerelemente enthalten. Von diesem Punkt an werden diese Seiten "Tabs" genannt.

Tab2 vs. Tab: In v1.0.47.05 wurde das Tab2-Steuerelement hinzugefügt, das im originalen Tab-Steuerelement seltene Probleme beim Neuzeichnen behebt (tritt zum Beispiel auf, wenn ein GUI-Fenster durch Anklicken bestimmter Bereiche der Steuerelemente, wie bei Fortschrittsbalken, aktiviert wird). Das ursprüngliche Tab-Steuerelement wird für die Abwärtskompatibilität beibehalten, weil Tab2 das Tab-Steuerelement erst einfügt, wenn es Steuerelemente in Navigationsreihenfolge von Tab enthält. Wenn möglich sollten neue Scripts immer Tab2 verwenden.

Beispiel: Gui, Add, Tab2,, Allgemein|Ansicht|Aussehen|Einstellungen

Der letzte Parameter enthält eine Liste mit Tabnamen, die durch senkrechte Striche getrennt sind. Damit einer der Tabs beim erstmaligen Anzeigen des Fensters vorausgewählt ist, fügt nach diesem zwei senkrechte Striche an (z. B. Rot|Grün||Blau). Alternativ kann mit dem Wort Choose in den Optionen, unmittelbar gefolgt von der Nummer, ein Eintrag sofort vorausgewählt werden. Zum Beispiel würde mit Choose5 der fünfte Tab vorausgewählt (wie bei anderen Optionen kann hierbei auch eine Variable angegeben werden, zum Beispiel Choose%Var%). Mit GuiControl kann der ausgewählte Tab nachträglich durch Hinzufügen oder Entfernen von Tabs geändert werden.

Steuerelemente, die nach dem Tab-Steuerelement hinzugefügt werden, gehören automatisch zum ersten Tab. Mit den folgenden Beispielen kann das jederzeit geändert werden:

Gui, Tab  ; Nachfolgende Steuerelemente gehören nicht mehr zum Tab-Steuerelement.
Gui, Tab, 3  ; Nachfolgende Steuerelemente werden dem dritten Tab im Tab-Steuerelement untergeordnet.
Gui, Tab, 3, 2  ; Nachfolgende Steuerelemente werden dem dritten Tab im zweiten Tab-Steuerelement untergeordnet.
Gui, Tab, Name  ; Nachfolgende Steuerelemente werden dem Tab mit der Bezeichnung Name untergeordnet (unabhängig von Groß-/Kleinschreibung).
Gui, Tab, Name,, Exact  ; Das Gleiche wie oben, aber mit exakter Übereinstimmung (und abhängig von Groß-/Kleinschreibung).

Die oben genannten Beispiele können auch dazu verwendet werden, Steuerelemente einem Tab zuzuweisen oder einem Tab-Steuerelement, das noch nicht vorhanden ist (außer bei der Name-Methode). In diesem Fall aber werden die unten genannten Optionen zur relativen Positionierung nicht unterstützt.

Positionierung: Sobald jeder Tab eines Tab-Steuerelements sein erstes Untersteuerelement erhält, hat dieses Untersteuerelement unter folgenden Bedingungen eine besondere Standardposition:

  1. Ist sowohl die X- als auch Y-Koordinate nicht vorhanden, wird das erste Untersteuerelement in die obere linke Ecke des Tab-Steuerelements positioniert (mit Standard-Abstand). Die restlichen Untersteuerelemente werden jeweils unter dem vorherigen Steuerelement positioniert.
  2. Falls die Positionierungsoptionen X+n und/oder Y+n angegeben sind, wird das Untersteuerelement in Bezug auf die linke obere Ecke des Innenbereichs vom Tab-Steuerelement positioniert. Zum Beispiel würde mit "x+10 y+10" das Steuerelement von der linken oberen Ecke 10 Pixel nach rechts und 10 Pixel nach unten positioniert.

Untersteuerelemente müssen nicht unbedingt in der Abgrenzung des Tab-Steuerelements vorhanden sein: Sie werden weiterhin versteckt und sichtbar gemacht, sobald ihr Tab ausgewählt oder abgewählt wird. Dieses Verhalten ist besonders für den unten beschriebenen Button-Style geeignet.

Wenn der Befehl Gui Submit verwendet wird, wird in der zugewiesenen Ausgabevariable des Steuerelements (falls vorhanden) der Name des zurzeit ausgewählten Tabs gespeichert. Hat das Steuerelement jedoch die AltSubmit-Eigenschaft, erhält die Ausgabevariable stattdessen die Positionsnummer des Tabs (der erste Tab ist 1, der zweite ist 2 und so weiter).

Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Diese Option bewirkt, dass EigeneSubroutine automatisch ausgeführt wird, sobald der Benutzer auf einen anderen Tab wechselt. Wenn das Tab-Steuerelement sowohl ein g-Label als auch eine Ausgabevariable hat, wird die Ausgabevariable beim Wechseln des Tabs auf den zuvor ausgewählten Tabnamen gesetzt (oder bei AltSubmit auf die Nummer).

Navigation mit der Tastatur: Der Benutzer kann mit Strg+Bild ↑ oder mit Strg+Bild ↓ zwischen den Tabs wechseln. Befindet sich der Tastaturfokus auf einem Steuerelement, das nicht zum Tab-Steuerelement gehört, wird das erste Tab-Steuerelement des Fensters angesteuert. Strg+Tab und Strg+Umschalt+Tab können auch verwendet werden, aber mit der Ausnahme, dass sie nicht funktionieren, wenn das aktuell fokussierte Steuerelement ein mehrzeiliges Eingabefeld ist.

Jedes Fenster kann nicht mehr als 255 Tab-Steuerelemente enthalten. Jedes Tab-Steuerelement kann nicht mehr als 256 Tabs (Seiten) haben. Darüber hinaus ist es nicht möglich, dass ein Tab-Steuerelement weitere Tab-Steuerelemente enthält.

Optionen

Choose: Siehe oben.

-Background: Überschreibt die benutzerdefinierte Hintergrundfarbe des Fensters und verwendet die Standardfarbe des Tab-Steuerelements vom Betriebssystem. Verwendet +Theme -Background, damit sich das Tab-Steuerelement am aktuellen Desktop-Design anpasst. Allerdings werden dadurch die meisten Steuerelementarten im Tab-Steuerlement merkwürdig dargestellt, da ihre Hintergründe nicht dem Hintergrund des Tab-Steuerelements entsprechen. Bei einigen Steuerelementarten (wie Text) kann das Problem behoben werden, wenn BackgroundTrans in deren Optionen eingefügt wird.

Buttons: Erstellt anstelle von Tabs mehrere Schaltflächen im oberen Bereich des Steuerelements (in diesem Fall wird standardmäßig kein Rahmen angezeigt, weil die Anzeigefläche normalerweise keine Steuerelemente enthält).

Left/Right/Bottom: Mit diesen Wörtern werden die Tabs an der linken, rechten oder unteren Seite angezeigt, anstatt an der oberen. Für Left und Right gelten Einschränkungen, die in TCS_VERTICAL eingesehen werden können.

-Wrap: Verhindert, dass die Tabs mehr als eine Reihe verbrauchen (sind hierbei zu viele Tabs vorhanden, werden Schaltflächen mit Pfeilen angezeigt, damit der Benutzer zwischen den Tabs wechseln kann).

Um die Anzahl der Textreihen innerhalb des Steuerelements zu bestimmen (oder deren Höhe und Breite), siehe Position und Größe von Steuerelementen anpassen.

Icons in Tabs: Mit SendMessage kann ein Icon neben dem Tabnamen angezeigt werden. Siehe dazu im Forumsbeitrag Icons in tabs.

StatusBar[v1.0.44+]

Beschreibung: Eine Statusleiste am unteren Rand des Fensters, in der sich mehrere Texte und/oder Icons befinden. Dieses Steuerelement wird normalerweise dazu verwendet, geänderte Zustände zu melden.

; Beispiel:
Gui, Add, StatusBar,, Starttext der Leiste (kann leer gelassen werden).
SB_SetText(RowCount . " Zeilen markiert.")

Die einfachste Anwendungsmöglichkeit der Statusleiste ist das Aufrufen von SB_SetText(), sobald sich etwas ändert, das dem Benutzer gemeldet werden soll. Um mehr als eine Information zu melden, kann die Leiste mittels SB_SetParts() in mehrere Abschnitte aufgeteilt werden. Mit SB_SetIcon() können Icons in der Leiste angezeigt werden.

Alle folgenden Funktionen gelten für das Standard-GUI-Fenster des aktuellen Threads (das zum Beispiel mit Gui, 2:Default geändert werden kann). Falls das Standardfenster weder vorhanden ist noch eine Statusleiste hat, geben alle Funktionen eine 0 zurück, um das Problem zu kennzeichnen.

SB_SetText(NeuerText [, Bereichsnummer, Style]): Zeigt NeuerText im angegebenen Bereich der Statusleiste an. Falls die Bereichsnummer weggelassen wird, gilt 1 als Standardwert. Andernfalls kann ein Integer zwischen 1 und 256 angegeben werden. Wenn Style weggelassen wird, gilt 0 als Standardwert, um den traditionellen Rahmen zu verwenden, der den Bereich aussehen lässt, als wäre er im Boden versunken. Ansonsten kann eine 1 angegeben werden, um den Rahmen zu entfernen oder eine 2, um den Bereich aussehen zu lassen, als würde er sich vom Boden abheben. Letztendlich können bis zu zwei Tabulatorzeichen (`t) in NeuerText angegeben werden, um den Text rechts neben dem ersten Tabulator im Bereich zu zentrieren und den Text rechts neben dem zweiten Tabulator rechts auszurichten. Diese Funktion gibt bei Erfolg eine 1 und bei Misserfolg eine 0 zurück.

SB_SetParts([Breite1, Breite2, ... Breite255]): Teilt die Leiste in mehrere Abschnitte auf, abhängig von der angegebenen Breite (in Pixel). Wenn alle Parameter weggelassen werden, wird ein einzelner langer Bereich wiederhergestellt. Andernfalls kann für jeden Bereich, außer dem letzten, die Breite angegeben werden (der letzte Bereich wird mit der verbliebenen Leistenbreite aufgefüllt). Zum Beispiel werden mit SB_SetParts(50, 50) drei Bereiche erstellt: Die ersten zwei mit einer Breite von 50 und der letzte mit der verbliebenen Breite. Hinweis: Jeder Bereich, der mit SB_SetParts() "gelöscht" wurde, startet beim nächsten Mal ohne Text (außerdem werden die Icons automatisch zerstört). Diese Funktion gibt bei Erfolg einen Wert ungleich 0 (HWND der Statusleiste) und bei Misserfolg eine 0 zurück.

SB_SetIcon(Dateiname [, Iconnummer, Bereichsnummer]): Zeigt im angegebenen Bereich ein kleines Icon links neben den Text an (falls die Bereichsnummer weggelassen wird, gilt 1 als Standardwert). Der Dateiname ist der Name einer ICO-, CUR- oder ANI-Datei (animierte Cursor werden in der Leiste zurzeit nicht animiert). Weitere Icons können in folgenden Dateitypen enthalten sein: EXE, DLL, CPL, SCR und andere Typen, die Icon-Ressourcen enthalten. Um anstelle der ersten Icon-Gruppe eine andere zu verwenden, fügt deren Nummer in Iconnummer ein. Zum Beispiel würde mit SB_SetIcon("Shell32.dll", 2) das Standard-Icon aus der zweiten Icon-Gruppe verwendet werden. Diese Funktion gibt bei Erfolg das HICON des Icons und bei Misserfolg eine 0 zurück. Das HICON ist eine Systemressource, die bei den meisten Scripts getrost ignoriert werden kann, weil es automatisch zerstört wird, sobald das Fenster der Statusleiste zerstört wird. Ebenso wird jedes alte Icon zerstört, sobald es mit einer neuen ersetzt wird. Das kann wie folgt verhindert werden:

Gui +LastFound
SendMessage, 0x40F, Bereichsnummer - 1, EigenesHicon, msctls_statusbar321  ; 0x40F ist SB_SETICON.

SB_SetProgress(): Erstellt und steuert einen Fortschrittsbalken innerhalb einer Statusleiste. Diese Funktion ist auf www.autohotkey.com/forum/topic37754.html verfügbar.

G-Label-Meldungen: Ein g-Label wie gEigeneSubroutine kann in den Optionen des Steuerelements angegeben werden. Diese Option bewirkt, dass EigeneSubroutine automatisch ausgeführt wird, sobald der Benutzer auf die Leiste klickt. Diese Subroutine kann auf die internen Variablen A_Gui und A_GuiControl zurückgreifen. Noch wichtiger ist, dass sie zudem auf die interne Variable A_GuiEvent zurückgreifen kann, die einen der folgenden Strings enthält (aus Kompatibilitätsgründen mit zukünftigen Versionen sollte ein Script nicht davon ausgehen, dass das die einzigen möglichen Werte sind):

  • Normal: Der Benutzer hat mit der linken Maustaste auf die Leiste geklickt. Die interne Variable A_EventInfo enthält die Bereichsnummer (die ein sehr großer Integer sein kann, falls der Benutzer in der Nähe des Größenziehpunktes rechts neben der Leiste klickt).
  • RightClick: Der Benutzer hat mit der rechten Maustaste auf die Leiste geklickt. Die interne Variable A_EventInfo enthält die Bereichsnummer. HINWEIS: GuiContextMenu wird bei der Statusleiste nicht aufgerufen, wenn ein g-Label vorhanden ist. Zudem sollte stattdessen das RightClick-Ereignis des g-Labels verwendet werden, wenn das Script wissen muss, auf welche Bereichsnummer der Benutzer geklickt hat (A_EventInfo).
  • DoubleClick: Der Benutzer hat mit der linken Maustaste zweimal auf die Leiste geklickt. Die interne Variable A_EventInfo enthält die Bereichsnummer.
  • R: Der Benutzer hat mit der rechten Maustaste zweimal auf die Leiste geklickt. Die interne Variable A_EventInfo enthält die Bereichsnummer.

Schrift und Farbe: Obwohl Größe, Art und Stil der Schrift mittels Gui Font eingestellt werden kann (wie bei einem normalen Steuerelement), kann die Textfarbe dennoch nicht geändert werden. Auch "Gui Color" wird nicht berücksichtigt; stattdessen kann die Hintergrundfarbe der Statusleiste mit dem Wort Background in den Optionen geändert werden, unmittelbar gefolgt vom Farbnamen (siehe Farbtabelle) oder RGB-Wert (0x-Präfix ist optional). Beispiele: BackgroundSilver, BackgroundFFDD99, BackgroundDefault.

Statusleiste verstecken: Bei ihrer Erstellung kann die Leiste mit Gui, Add, StatusBar, Hidden vEigeneStatusleiste versteckt werden. Nach ihrer Erstellung kann sie nur mit GuiControl, Hide, EigeneStatusleiste versteckt werden. Um sie wieder sichtbar zu machen, verwendet GuiControl, Show, EigeneStatusleiste. Hinweis: Das Verstecken der Leiste reduziert nicht die Fensterhöhe. Wenn das gewünscht ist, verwendet einfach Gui, Show, AutoSize.

Styles (selten verwendet): Siehe Style-Tabelle der Statusleiste.

Bekannte Einschränkungen:

  1. Jedes Steuerelement, das sich mit der Statusleiste überschneidet, wird manchmal darüber gezeichnet. Dieses Problem kann zum Beispiel verhindert werden, wenn solche Steuerelemente mit dem GuiSize-Label dynamisch verkleinert werden.
  2. Es kann nicht mehr als eine Statusleiste in einem Fenster vorhanden sein.

Beispiel: Im unteren Bereich der TreeView-Seite wird eine Statusleiste mit mehreren Bereichen demonstriert.

ActiveX [v1.1.03+]

ActiveX-Komponenten wie das MSIE-Browser-Steuerelement können wie folgt eingebettet werden:

Gui Add, ActiveX, w980 h640 vWB, Shell.Explorer  ; Der letzte Parameter ist der Name der ActiveX-Komponente.
WB.Navigate("http://www.autohotkey.com/forum/")  ; Speziell für das Web-Browser-Steuerelement.
Gui Show

Sobald das Steuerelement erstellt ist, wird ein ActiveX-Objekt in die zugewiesenen Variable des Steuerelements gespeichert, falls vorhanden. Zudem kann GuiControlGet verwendet werden, um das Objekt abzurufen.

Damit die Ereignisse des Objekts bearbeitet werden können, verwendet ComObjConnect wie unten gezeigt:

Gui Add, Edit, w930 r1 vURL, http://www.autohotkey.com/forum/
Gui Add, Button, x+6 yp w44 Default, Laden
Gui Add, ActiveX, xm w980 h640 vWB, Shell.Explorer
ComObjConnect(WB, WB_events)  ; Verbindet die WB-Ereignisse mit dem Klassenobjekt WB_events.
Gui Show
; Fortfahren, um die Startseite zu laden:
ButtonLaden:
Gui Submit, NoHide
WB.Navigate(URL)
return

class WB_events
{
    NavigateComplete2(wb, NewURL)
    {
        GuiControl,, URL, %NewURL%  ; URL-Eingabefeld aktualisieren.
    }
}

GuiClose:
ExitApp

ComObjType kann verwendet werden, um den Typ des Objekts zu bestimmen, das in der Variable des Steuerelements gespeichert ist.
