Else

Verweist auf Befehle, welche durchzuführen sind, wenn eine If-Bedingung nicht erfüllt wird. Wenn mehrere Befehle vorhanden sind, dann müssen sie von einem Block (geschweifte Klammern) umschlossen werden. 

Else

Bemerkungen

Jedes ELSE muss mit einer vorherigen IF-Anweisung assoziiert werden. Ein ELSE gehört immer einer nächstgelegenen nicht beanspruchten IF-Anweisung, es sei denn, Block wird dazu verwendet, dieses Verhalten zu ändern.

Auf der gleichen Zeile nach ELSE kann ein beliebiger Befehl folgen. Das wird häufig bei "Else If"-Konstruktionen verwendet (siehe Beispiele unten).

Wenn IF oder ELSE mehrere Zeilen enthalten, dann müssen diese von geschweiften Klammern umschlossen werden. Wenn allerdings nur eine Zeile vorhanden ist, dann sind die Klammern optional. Zum Beispiel:

If count > 0  ; Klammern sind in der nächsten Zeile nicht notwendig, weil es nur eine Zeile ist.
    MsgBox OK drücken`, um Prozess fortzufahren.
Else  ; Klammern müssen für den unteren Abschnitt verwendet werden, weil mehrere Zeilen vorhanden sind.
{
    WinClose Unbenannt - Editor
    MsgBox Keine Einträge vorhanden.
}

Der One True Brace (OTB)-Stil kann gegebenenfalls bei einem "Else" verwendet werden. Zum Beispiel:

If IsDone {
    ...
} Else If (x < y) {
    ...
} Else {
    ...
}

Verwandte Befehle

Siehe Blocks. Auch jeder IF-Befehl kann ELSE verwenden, einschließlich IfWinActive, IfWinExist, IfMsgBox, IfInString, IfBetween, IfIn, IF, und IF (Ausdruck).

Beispiele

IfWinExist, Unbenannt - Editor
{
    WinActivate
    Send Das ist ein Test.{Enter}
}
Else
{
    WinActivate, Beliebiges Fenster
    MouseClick, left, 100, 200
}

If x = 1
    Gosub, a1
Else If x = 2 ; "Else If"-Stil
    Gosub, a2
Else IfEqual, x, 3 ; alternativer Stil
{
    Gosub, a3
    Sleep, 1
}
Else Gosub, a4  ; jeder einzelne Befehl kann auf der gleichen Zeile nach ELSE folgen.

; Auch OK:
IfEqual, y, 1, Gosub, b1
Else {
    Sleep, 1
    Gosub, b2
}
