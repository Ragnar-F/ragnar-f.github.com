WinMove

Verändert die Position und/oder Größe des angegebenen Fensters.

WinMove, X, Y
WinMove, Fenstertitel, Fenstertext, X, Y [, Breite, Höhe, Titelausnahme, Textausnahme]

Parameter

X, Y

Die Koordinaten X und Y der oberen linken Ecke der neuen Position vom Zielfenster, die Ausdrücke sein können. Das obere linke Pixel des Bildschirms befindet sich auf 0, 0.

Wenn nur diese Parameter angegeben werden, dann wird das zuletzt gefundene Fenster als Zielfenster verwendet.

Ansonsten kann X und/oder Y weggelassen werden, um die aktuelle Position zu verwenden.
Fenstertitel Der Titel oder Teiltitel des Fensters (das Übereinstimmungsverhalten wird mit SetTitleMatchMode bestimmt). Wenn dieser und die anderen 3 Fensterparameter weggelassen werden, wird das zuletzt gefundene Fenster verwendet. Sobald dieser den Buchstaben A enthält und die anderen 3 Fensterparameter weggelassen werden, wird das aktive Fenster verwendet. Bei Verwendung einer Fensterklasse muss ahk_class ExakterKlassenname angegeben werden (wird von Windows Spy angezeigt). Bei Verwendung einer Prozess-Kennung (PID) muss ahk_pid %VariableMitPID% angegeben werden. Bei Verwendung einer Fenstergruppe muss ahk_group Gruppenname angegeben werden. Bei Verwendung einer eindeutigen ID-Nummer eines Fensters muss ahk_id %VariableMitID% angegeben werden. Die Suche kann durch Angabe mehrerer Kriterien eingeschränkt werden. Zum Beispiel: Eigene Datei.txt ahk_class Notepad
Fenstertext In diesem Parameter kann ein Teilstring angegeben werden, der in einem Textelement des Zielfensters vorkommt (werden mit Windows Spy angezeigt). Versteckte Textelemente werden auch erkannt, sofern DetectHiddenText aktiv ist.
Breite, Höhe Die neue Breite und Höhe des Fensters (in Pixel), die Ausdrücke sein können. Wenn einer der Parameter leer ist, weggelassen wird oder das Wort DEFAULT enthält, dann wird die Größe in dieser Abmessung nicht verändert.
Titelausnahme Ignoriert Fenster, deren Titel diesen Wert enthalten.
Textausnahme Ignoriert Fenster, deren Textelemente diesen Wert enthalten.

Bemerkungen

Wenn die Breite und Höhe klein ist (oder negativ), dann sind die meisten Fenster mit einer Titelleiste in der Regel nicht kleiner als 112 x 27 Pixel (jedoch haben einige Fensterarten verschiedene Mindestgrößen). Wenn die Breite und Höhe groß ist, dann überschreiten die meisten Fenster die Desktop-Abmessungen nicht mehr als ca. 12 Pixel.

Negative Werte sind für die Koordinaten X und Y erlaubt, um Systeme mit mehreren Monitoren zu unterstützen und ein Fenster außerhalb des Bildschirms zu verschieben.

Obwohl WinMove minimierte Fenster nicht verschieben kann, können jedoch versteckte Fenster verschoben werden, wenn DetectHiddenWindows aktiv ist.

Die Geschwindigkeit von WinMove wird von SetWinDelay beeinflusst.

Fenstertitel und -texte sind von der Groß- und Kleinschreibung abhängig. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.

Verwandte Befehle

ControlMove, WinGetPos, WinHide, WinMinimize, WinMaximize, WinSet

Beispiel

Run, calc.exe
WinWait, Rechner
WinMove, 0, 0 ; Verschiebt das zuletzt gefundene Fenster in die obere linke Ecke des Bildschirms.

SplashTextOn, 400, 300, Zwischenablage, Die Zwischenablage enthält:`n%clipboard%
WinMove, Zwischenablage, , 0, 0 ; Verschiebt das Splash-Fenster in die linke obere Ecke. 
Msgbox, OK drücken, damit der SplashText verschwindet.
SplashTextOff

; Die folgende Funktion zentriert das angegebene Fenster auf dem Bildschirm:
FensterZentrieren(Fenstertitel)
{
    WinGetPos,,, Breite, Höhe, %Fenstertitel%
    WinMove, %Fenstertitel%,, (A_ScreenWidth/2)-(Breite/2), (A_ScreenHeight/2)-(Höhe/2)
}
