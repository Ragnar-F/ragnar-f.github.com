#NoEnv [v1.0.43.08+]

Leere Variablen werden nicht länger überprüft, ob sie Umgebungsvariablen sind (für alle neuen Scripts empfohlen).

#NoEnv

#NoEnv irgendwo im Script verhindert, dass leere Variablen als potenzielle Umgebungsvariablen angesehen werden. Zum Beispiel:

#NoEnv
MsgBox %WinDir%

Das obige Beispiel würde nicht die WinDir-Umgebungsvariable abrufen (könnte aber mit WinDir := A_WinDir am Anfang des Scripts umgangen werden).

Es wird empfohlen, #NoEnv in jedem neuen Script zu verwenden, weil:

  1. diese Direktive die Performance deutlich verbessert, sobald leere Variablen in einem Ausdruck oder Befehl verwendet werden. Sie verbessert auch die Performance von DllCall, wenn Parametertypen ohne Anführungszeichen angegeben werden (z. B. Int anstelle von "Int").
  2. diese Direktive Script-Bugs verhindert, die von Umgebungsveriablen erzeugt werden, deren Namen unerwartet mit den Variablen des Scripts übereinstimmen.
  3. AutoHotkey v2 dieses Verhalten zum Standard machen wird.

Um den Übergang zu #NoEnv zu erleichtern, wurden die internen Variablen Comspec und ProgramFiles hinzugefügt. Sie enthalten die gleichen Strings wie die entsprechenden Umgebungsvariablen.

Falls #NoEnv aktiv ist, sollte das Script EnvGet verwenden, um Umgebungsvariablen abrufen zu können, oder interne Variablen wie A_WinDir verwenden.

Verwandte Befehle

EnvGet, Comspec, ProgramFiles, A_WinDir
