#NoEnv [v1.0.43.08+]

Verhindert die Überprüfung leerer Variablen, ob sie Umgebungsvariablen sind (für alle neuen Scripts empfohlen).

#NoEnv

#NoEnv irgendwo im Script verhindert, dass leere Variablen als potenzielle Umgebungsvariablen angesehen werden. Zum Beispiel:

#NoEnv
MsgBox %WinDir%

Das obige Beispiel ermittelt nicht die WinDir-Umgebungsvariable (könnte aber mit WinDir := A_WinDir am Anfang des Scripts umgangen werden).

Es wird empfohlen, #NoEnv in jedem neuen Script zu verwenden, weil:

  1. dadurch die Performance deutlich verbessert wird, wenn leere Variablen in einem Ausdruck oder Befehl verwendet werden. Es verbessert auch die Performance von DllCall, wenn Parametertypen ohne Anführungszeichen verwendet werden (z. B. Int anstelle von "Int").
  2. dadurch Script-Bugs durch Umgebungsvariablen verhindert werden, deren Namen unerwartet mit den Variablen des Scripts übereinstimmen.
  3. AutoHotkey v2 dieses Verhalten zum Standard machen wird.

Um den Übergang zu #NoEnv zu erleichtern, wurden die internen Variablen Comspec und ProgramFiles hinzugefügt. Sie enthalten die gleichen Strings wie die entsprechenden Umgebungsvariablen.

Falls #NoEnv aktiv ist, sollte das Script zum Abrufen von Umgebungsvariablen EnvGet verwenden, oder interne Variablen wie A_WinDir.

Verwandte Befehle

EnvGet, Comspec, ProgramFiles, A_WinDir
