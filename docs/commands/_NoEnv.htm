#NoEnv [v1.0.43.08+]

Verhindert die Überprüfung leerer Variablen, ob sie Umgebungsvariablen sind (für alle neuen Scripts empfohlen).

#NoEnv

Mit der Zeile #NoEnv irgendwo in einem Script wird verhindert, dass Variablen überprüft werden, ob sie Umgebungsvariablen sind. Zum Beispiel:

#NoEnv
MsgBox %WinDir%

Damit wird die WinDir-Umgebungsvariable nicht abgerufen (kann aber mit WinDir := A_WinDir am Anfang des Scripts umgangen werden).

#NoEnv wird für alle neuen Scripts empfohlen, weil:

  1. es deutlich die Performance verbessert, wenn leere Variablen in einem Ausdruck oder Befehl verwendet werden. Es verbessert auch die Performance von DllCall, wenn Parametertypen ohne Anführungszeichen verwendet werden (z. B. int anstelle von "int").
  2. es Script-Fehler verhindert, die durch Umgebungsvariablen verursacht werden, deren Namen unerwartet mit Variablen des Scripts übereinstimmen.
  3. AutoHotkey v2 dieses Verhalten zum Standard macht.

Die internen Variablen Comspec und ProgramFiles wurden hinzugefügt, um den Übergang zu #NoEnv zu erleichtern. Sie enthalten die gleichen Strings wie die entsprechenden Umgebungsvariablen.

Falls #NoEnv aktiv ist, sollte das Script interne Variablen wie A_WinDir und EnvGet verwenden, um Umgebungsvariablen abzurufen.

Verwandte Befehle

EnvGet, Comspec, ProgramFiles, A_WinDir
