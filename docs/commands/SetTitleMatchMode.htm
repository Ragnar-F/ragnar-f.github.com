SetTitleMatchMode

Bestimmt das Übereinstimmungsverhalten des Fenstertitel-Parameters in Befehlen wie WinWait.

SetTitleMatchMode, Übereinstimmungsmodus
SetTitleMatchMode, Fast|Slow

Parameter

Übereinstimmungsmodus

Eine der folgenden Zahlen oder das Wort RegEx:

1: Der Titel eines Fensters muss mit dem angegebenen Fenstertitel beginnen, damit eine Übereinstimmung erfolgt.
2: Innerhalb des Titels eines Fensters kann Fenstertitel überall vorkommen, damit eine Übereinstimmung erfolgt.
3: Der Titel eines Fensters muss genau mit dem angegebenen Fenstertitel übereinstimmen, damit eine Übereinstimmung erfolgt.

RegEx (v1.0.45+): Bewirkt, dass Fenstertitel, Fenstertext, Titelausnahme und Textausnahme reguläre Ausdrücke sein können. Solche Ausdrücke dürfen innerhalb von Befehlen nicht in Anführungszeichen gesetzt werden. Zum Beispiel: WinActivate Unbenannt.*Editor. RegEx gilt auch für ahk_class; zum Beispiel kann mit ahk_class IEFrame jedes Fenster gesucht werden, dessen Klassenname IEFrame irgendwo enthält (standardmäßig finden reguläre Ausdrücke eine Übereistimmung überall im Zielstring). Beachtet bei Fenstertext, dass jedes Textelement (z. B. Text von jedem Steuerelement) mit RegEx verglichen wird. Daher ist es nicht möglich, eine Übereinstimmung zu haben, die mehr als ein Textelement umfasst.

Die oberen Methoden haben auf Titelausnahme den gleichen Einfluss wie Fenstertitel. Beim dritten Modus muss der Titel eines Fensters beispielsweise genau mit Titelausnahme übereinstimmen, damit dieses Fenster ignoriert wird.
Fast|Slow

Fast: Standardverhalten. Es ist möglich, dass die Performance dadurch erheblich besser als Slow ist, aber mit dem Nachteil, dass mehrere Fensterbefehle nicht in der Lage sind, bestimmte Fenstertext-Elemente von einigen Fensterarten zu "sehen".

Slow: Möglicherweise deutlich langsamer, aber mit dem Vorteil, dass alle möglichen Fenstertext-Elemente eines Fensters abgerufen werden, das von einem Fensterbefehl durchsucht wird, um eine Übereinstimmung zu finden. Window Spy deckt auf, welche Teile eines Fenstertextes (falls vorhanden) den langsamen Modus benötigen.

Bemerkungen

Dieser Befehl beeinflusst das Verhalten aller Fenster-Befehle (z. B. IfWinExist und WinActivate).

Wenn dieser Befehl nicht vorhanden ist, gilt standardmäßig 1 und Fast.

Falls eine Fenstergruppe verwendet wird, dann gilt der aktuelle Übereinstimmungsmodus des Titels für jede einzelne Regel in der Gruppe.

Der Slow-Modus sollte nur verwendet werden, wenn das Zielfenster nicht eindeutig durch dessen Titel oder Fast-Modus-Text identifiziert werden kann. Das liegt daran, dass der Slow-Modus ziemlich langsam sein kann, wenn bestimmte Anwendungsfenster beschäftigt sind oder "nicht antworten".

Die benutzerdefinierte Version von Window Spy zeigt den Slow-Text in einem zusätzlichen Abschnitt an, so dass auf einfache Weise festgestellt werden kann, ob der Slow-Modus benötigt wird.

Falls beide Attribute geändert werden sollen, muss der Befehl wie folgt zweimal ausgeführt werden:
SetTitleMatchMode, 2
SetTitleMatchMode, slow

Die internen Variablen A_TitleMatchMode und A_TitleMatchModeSpeed enthalten die aktuellen Einstellungen.

Ungeachtet der aktuellen Einstellungen im Übereinstimmungsmodus unterscheiden Fenstertitel, Fenstertext, Titelausnahme und Textausnahme zwischen Groß- und Kleinschreibung. Die einzige Ausnahme ist die Zeichenungenauigkeits-Option im RegEx-Modus, zum Beispiel: i)unbenannt - editor

Jeder neu gestartete Thread (z. B. ein Hotkey, ein benutzerdefinierter Menüpunkt oder eine zeitlich festgelegte Subroutine) beginnt mit der Standardeinstellung des Befehls. Diese Standardeinstellung kann geändert werden, indem dieser Befehl im automatischen Ausführungsbereich (am Anfang des Scripts) verwendet wird.

Verwandte Befehle

SetWinDelay, IfWinExist, WinActivate, RegExMatch()

Beispiel

SetTitleMatchMode 2
; ODER:
SetTitleMatchMode RegEx

SetTitleMatchMode Slow  ; Slow/Fast können unabhängig von den anderen Modi gesetzt werden.
