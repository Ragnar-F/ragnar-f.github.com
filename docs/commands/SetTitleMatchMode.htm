SetTitleMatchMode

Bestimmt das Übereinstimmungsverhalten der Fenstertitel-Parameter in Befehlen wie WinWait.

SetTitleMatchMode, Übereinstimmungsmodus
SetTitleMatchMode, Fast|Slow

Parameter

Übereinstimmungsmodus

Eine folgende Zahl oder das Wort RegEx:

1: Der Fenstertitel muss mit dem angegebenen Fenstertitel beginnen, damit eine Übereinstimmung erfolgt.
2: Der Fenstertitel kann Fenstertitel überall enthalten, damit eine Übereinstimmung erfolgt.
3: Der Fenstertitel muss mit dem angegebenen Fenstertitel exakt übereinstimmen, damit eine Übereinstimmung erfolgt.

RegEx (v1.0.45+): Ändert Fenstertitel, Fenstertext, Titelausnahme und Textausnahme zu regulären Ausdrücken. Solche Ausdrücke dürfen nicht in Anführungszeichen eingeschlossen werden. Zum Beispiel: WinActivate Unbenannt.*Editor. RegEx gilt auch für ahk_class; zum Beispiel kann mit ahk_class IEFrame jedes Fenster gesucht werden, das den Klassenname IEFrame irgendwo enthält (denn die regulären Ausdrücke finden standardmäßig überall eine Übereinstimmung im Zielstring). Hinweis: Jedes Textelement im Fenstertext (z. B. in Steuerelementen) wird daraufhin untersucht, ob der gesuchte Text enthalten ist. Deshalb kann eine Suche auf Übereinstimmung nicht mehrere Textelemente beinhalten.

Die obigen Methoden beeinflussen auch Titelausnahme, sowie Fenstertitel. In Modus 3 bedeutet das, das ein Fenster nur ausgewählt wird, wenn sein Titel nicht komplett mit Titelausnahme übereinstimmt.
Fast|Slow

Fast: Standardwert. Die Performance ist erheblich besser als Slow, aber bestimmte Fenstertext-Elemente bei einigen Fensterarten werden möglicherweise von verschiedenen Fenster-Befehlen nicht "gesehen".

Slow: Möglicherweise deutlich langsamer, aber es werden alle möglichen Fenstertext-Elemente von jedem Fenster abgerufen, um eine Übereinstimmung zu finden. Window Spy zeigt, welcher Teil des Textes im Fenster (falls vorhanden) den langsamen Modus benötigt.

Bemerkungen

Dieser Befehl beeinflusst das Verhalten aller Fenster-Befehle (z. B. IfWinExist und WinActivate).

Wenn nicht angegeben, dann wird standardmäßig 1 und Fast verwendet.

Wenn eine Fenstergruppe verwendet wird, dann gilt der aktuelle Übereinstimmungsmodus des Titels für jede einzelne Regel in der Gruppe.

Die Methode Slow sollte nur gewählt werden, wenn das Ziel-Fenster nicht eindeutig durch den Titel oder Text der Fast-Methode identifiziert werden kann. Denn die Slow-Methode kann ziemlich langsam sein, wenn bestimmte Anwendungsfenster beschäftigt sind oder "nicht antworten".

Die benutzerdefinierte Version von Window Spy zeigt den Slow-Text in einem abgegrenzten Abschnitt, so kann leicht ermittelt werden, ob die Slow-Methode gebraucht wird.

Wenn es der Wunsch ist, beide Attribute zu ändern, führt den Befehl wie im diesen Beispiel zweimal aus:
SetTitleMatchMode, 2
SetTitleMatchMode, slow

Die internen Variablen A_TitleMatchMode und A_TitleMatchModeSpeed enthalten die aktuellen Einstellungen.

Ungeachtet der aktuellen Einstellungen in TitleMatchMode sind Fenstertitel, Fenstertext, Titelausnahme und Textausnahme von der Groß- und Kleinschreibung abhängig. Die einzige Ausnahme bildet die Option zur Unabhängigkeit von Groß- und Kleinschreibung im RegEx-Modus, zum Beispiel: i)unbenannt - editor

Jeder neu gestartete Thread (z. B. ein Hotkey, ein benutzerdefinierter Menüpunkt oder eine zeitlich festgelegte Subroutine) beginnt mit der Standardeinstellung des Befehls. Diese Standardeinstellung kann geändert werden, indem dieser Befehl im automatischen Ausführungsbereich (am Anfang des Scripts) verwendet wird.

Verwandte Befehle

SetWinDelay, IfWinExist, WinActivate, RegExMatch()

Beispiel

SetTitleMatchMode 2
; ODER:
SetTitleMatchMode RegEx

SetTitleMatchMode Slow  ; Slow/Fast können unabhängig von den anderen Modi gesetzt werden.
