Reload

Ersetzt die aktuell laufende Instanz des Scripts mit einer neuen Instanz.

Reload

Dieser Befehl ist für Scripts nützlich, die häufig geändert werden. Wenn diesem Befehl ein Hotkey zugewiesen wird, kann der Benutzer das Script einfach neu starten, nachdem er seine Änderungen in einem Editor abgespeichert hat.

Jeder Befehlszeilenparameter, der ans originale Script übergeben wurde, wird nicht an die neue Instanz übergeben. Damit solche Parameter übergeben werden, sollte nicht Reload, sondern Run in Verbindung mit A_AhkPath und A_ScriptFullPath (und A_IsCompiled, wenn das Script kompiliert ist) verwendet werden. Zudem kann der String /restart als erster Parameter eingefügt werden (d. h. nach dem Namen der ausführbaren Datei), um den Programm mitzuteilen, das gleiche Verhalten von Reload durchzuführen. Siehe auch: Befehlszeilenoptionen und Syntax.

Bei einem Neustart des Scripts wird es im originalen Arbeitsverzeichnis ausgeführt (das wirksam war, als das Script zum ersten Mal gestartet wurde). Das heißt, dass SetWorkingDir nicht in der Lage ist, das Arbeitsverzeichnis zu ändern, das für die neue Instanz verwendet wird.

Wenn das Script nicht neu geladen werden kann (z. B. wegen eines Syntax-Fehlers), wird die originale Instanz des Scripts fortgesetzt. Daher sollten nach dem Reload-Befehl bestimmte Aktionen erfolgen, die im Falle eines Fehlers durchgeführt werden sollen (z. B. Return, um die aktuelle Subroutine zu verlassen). Die originale Instanz kann den Fehler wie folgt erkennen:

Reload
Sleep 1000 ; Bei Erfolg wird Reload diese Instanz während Sleep schließen, so dass die Zeile darunter nie erreicht werden kann.
MsgBox, 4,, Das Script konnte nicht neu geladen werden. Möchten Sie das Script zur Bearbeitung öffnen?
IfMsgBox, Yes, Edit
Return

Verwandte Befehle

Edit

Beispiel

^!r::Reload  ; Strg+Alt+R als Hotkey zuweisen, um das Script neu zu starten.
