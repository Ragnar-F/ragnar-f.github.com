Reload

Ersetzt die aktuell ausgeführte Instanz des Scripts mit einer neuen Instanz.

Reload

Dieser Befehl ist für Scripts nützlich, die häufig geändert werden. Wenn diesem Befehl ein Hotkey zugewiesen wird, dann kann der Benutzer das Script einfach neu starten, nachdem er seine Änderungen in einem Editor abgespeichert hat.

Jeder Parameter einer Befehlszeile, der ans originale Script übergeben wurde, wird nicht an die neue Instanz übergeben. Um solche Parameter zu übergeben, verwendet nicht Reload. Verwendet stattdessen Run in Verbindung mit A_AhkPath und A_ScriptFullPath (und A_IsCompiled, wenn das Script jemals in kompilierter Form verwendet wird). Es kann auch der String /restart als erster Parameter eingefügt werden (d. h. nach dem Namen der ausführbaren Datei), um das gleiche Verhalten von Reload durchzuführen. Siehe auch: Befehlszeilenoptionen und Syntax.

Wenn das Script neu startet, dann wird dieses im originalen Arbeitsverzeichnis ausgeführt (welches beim ersten Start wirksam war). Das heißt, dass SetWorkingDir nicht das Arbeitsverzeichnis ändert, das für die neue Instanz verwendet wird.

Wenn das Script nicht neu geladen werden kann (z. B. wegen eines Syntax-Fehlers), dann wird die originale Instanz des Scripts fortgeführt. Daher sollten nach dem Reload-Befehl bestimmte Aktionen im Falle eines Fehlers angegeben werden (z. B. Return, um die aktuelle Subroutine zu verlassen). Damit die originale Instanz den Fehler erkennt, befolgt dieses Beispiel:

Reload
Sleep 1000 ; Wenn erfolgreich, dann schließt Reload diese Instanz während Sleep, die Zeile darunter wird also nie erreicht.
MsgBox, 4,, Das Script konnte nicht neu geladen werden. Möchten sie das Script öffnen, um es zu bearbeiten?
IfMsgBox, Yes, Edit
Return

Verwandte Befehle

Edit

Beispiel

^!r::Reload  ; Strg+Alt+R wird als Hotkey dem Script zugewiesen, um es neu zu starten.
