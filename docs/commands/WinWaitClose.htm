WinWaitClose

Wartet, bis das angegebene Fenster nicht mehr existiert.

WinWaitClose [, Fenstertitel, Fenstertext, Sekunden, Titelausnahme, Textausnahme]

Parameter

Fenstertitel Der Titel oder Teiltitel des Fensters (das Übereinstimmungsverhalten wird mit SetTitleMatchMode bestimmt). Wenn dieser und die anderen 3 Fensterparameter nicht vorhanden sind, dann wird das zuletzt gefundene Fenster verwendet. Bei Verwendung einer Fensterklasse muss ahk_class ExakterKlassenname angegeben werden (wird von Windows Spy angezeigt). Bei Verwendung einer Prozess-Kennung (PID) muss ahk_pid %VariableMitPID% angegeben werden. Bei Verwendung einer Fenstergruppe muss ahk_group Gruppenname angegeben werden. Bei Verwendung einer eindeutigen ID-Nummer eines Fensters muss ahk_id %VariableMitID% angegeben werden. Die Suche kann durch Angabe mehrerer Kriterien eingeschränkt werden. Zum Beispiel: Eigene Datei.txt ahk_class Notepad
Fenstertext Wenn vorhanden, dann muss dieser Parameter einen Teil der einzelnen Textelemente des Fensters enthalten (wird von Windows Spy angezeigt). Versteckte Textelemente werden erfasst, wenn DetectHiddenText aktiv ist.
Sekunden Die Wartezeit in Sekunden, bis der Befehl beendet und ErrorLevel auf 1 gesetzt wird. Kann leer gelassen werden, um unendlich lang zuwarten. Der Wert 0 entspricht 0.5. Dieser Parameter kann ein Ausdruck sein.
Titelausnahme Fenstertitel mit diesem Wert werden nicht berücksichtigt.
Textausnahme Fenstertexte mit diesem Wert werden nicht berücksichtigt.

ErrorLevel

Wenn die Wartezeit abgelaufen ist, dann wird ErrorLevel auf 1 gesetzt, ansonsten enthält ErrorLevel eine 0.

Bemerkungen

Wenn keine Instanzen des angegebenen Fensters vorhanden sind, dann wartet der Befehl nicht, bis die Sekunden abgelaufen sind. Stattdessen wird ErrorLevel sofort auf 0 gesetzt und das Script weiter ausgeführt.

Während der Befehl in einem Wartezustand ist, können neue Threads mittels Hotkey, benutzerdefinierte Menüeinträge oder Timer gestartet werden.

Wenn ein anderer Thread den Inhalt der Variable(n) verändert, die für die Parameter des Befehls verwenden wurden, dann sieht dieser Befehl die Veränderung nicht. Er verwendet weiterhin den Titel und Text, die original in den Variablen vorhandenen sind, wenn der Befehl wartet.

Fenstertitel und -texte sind von der Groß- und Kleinschreibung abhängig. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.

Verwandte Befehle

WinClose, WinWait, WinWaitActive, IfWinExist, IfWinActive, Process, SetTitleMatchMode, DetectHiddenWindows

Beispiel

Run, notepad.exe
WinWait, Unbenannt - Editor
WinWaitClose  ; Wartet auf das zuletzt gefundene Fenster.
MsgBox, Editor wurde geschlossen.
