#InputLevel [v1.1.06+]

Bestimmt, welche künstliche Tastatur- oder Mausereignisse von Hotkeys und Hotstrings ignoriert werden.

#InputLevel [, Level]

Parameter

Level Ein Integer zwischen 0 und 100. Falls weggelassen, standardmäßig 0.

Bemerkungen

Diese Direktive ist von der Position abhängig: alle Hotkeys and Hotstrings zwischen dieser und der nächsten #InputLevel-Direktive sind davon betroffen. Ist dieser Parameter nicht vorhanden, wird standardmäßig Level 0 verwendet.

Wenn ein künstliches Tastatur- oder Mausereignis auf gleichem oder niedrigerem Level empfangen wird, löst diese Direktive den Hotkey oder Hotstring nicht aus. Da diese Art von Filterung einen Tastatur- oder Maus-Hook benötigt, bewirkt ein Eingabelevel ungleich 0 zudem, dass #UseHook in Kraft tritt. Allerdings können Level-0-Hotkeys mit der Reg-Methode implementiert werden. In diesem Fall werden sie möglicherweise durch künstliche Eingaben jeden Levels ausgelöst (selbst Level 0).

Der Eingabelevel eines Hotkeys oder Hotstrings gilt auch als Standard-Sendelevel für jede künstliche Eingabe, die durch diesen Hotkey oder Hotstring erzeugt wird. Folglich bewirkt ein Eingabelevel ungleich 0, dass jede Eingabe, die von einem Hotkey erzeugt wird, vom Hotkey selbst und von Hotkeys mit niedrigerem Level ignoriert wird.

AutoHotkey-Versionen älter als v1.1.06 verhalten sich so, als wäre #InputLevel 0 und SendLevel 0 wirksam.

Verwandte Befehle

SendLevel, Hotkeys, Hotstrings

Beispiele

#InputLevel 1
Numpad0::LButton
#InputLevel 0
; Dieser Hotkey kann sowohl durch Numpad0 als auch LButton ausgelöst werden:
~LButton::Msgbox Gedrückt
