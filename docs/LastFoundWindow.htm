Das "zuletzt gefundene" Fenster

Das Fenster, dass zuletzt von IfWin[Not]Exist, IfWin[Not]Active, WinWait[Not]Active oder WinWait gefunden wurde. Dadurch können Scripts einfacher erstellt und verwaltet werden, da der Fenstertitel und Fenstertext des Zielfensters nicht für jeden Fensterbefehl angegeben werden muss. Darüber hinaus werden Scripts besser ausgeführt, weil sie nicht nochmals das Zielfenster suchen müssen, nachdem es zum ersten Mal gefunden wurde.

Das "zuletzt gefundene" Fenster kann von allen Fensterbefehlen außer WinWait, WinActivateBottom und GroupAdd verwendet werden. Es wird verwendet, sobald alle vier Fensterparameter weggelassen werden (Fenstertitel, Fenstertext, Titelausnahme und Textausnahme).

Jeder Thread behält seinen eigenen Wert des "zuletzt gefundenen" Fensters. Falls der aktuelle Thread also von einem anderen unterbrochen wird, während der originale Thread wiederaufgenommen wird, enthält er weiterhin den originalen Wert des "zuletzt gefundenen" Fensters, aber nicht den Wert, der vom unterbrochenen Thread gesetzt wurde.

Falls das zuletzt gefundene Fenster ein verstecktes GUI-Fenster ist, dann kann es auch dann verwendet werden, wenn DetectHiddenWindows deaktiviert ist. Das wird oft in Verbindung mit "Gui +LastFound" benutzt.

Beispiele

Run Notepad
WinWait Unbenannt - Editor
WinActivate  ; Verwendet das zuletzt gefundene Fenster.

IfWinExist, Unbenannt - Editor
{
    WinActivate  ; Verwendet das oben gefundene Fenster.
    WinMaximize  ; dasselbe
    Send, Dies ist ein Text.{Enter}
    Return
}

IfWinNotExist, Rechner
    Return
Else
{
    WinActivate  ; Das obere "IfWinNotExist" setzt das zuletzt gefundene Fenster für uns.
    WinMove, 40, 40  ; Zur neuen Position verschieben.
    Return
}

Weitere Verwendungsmöglichkeiten des Fenstertitel-Parameters

Sobald mehrere Fenster mit den Fenstertitel/text-Kriterien eines Fensterbefehls wie WinMove übereinstimmen, wird das oberste der Stapelreihenfolge verwendet. Falls zum Beispiel das aktive/vorderste Fenster den Kriterien entspricht, wird es auch dann verwendet, wenn sich andere passende Fenster darunter befinden.

Aktives Fenster (A): Nahezu alle Fensterbefehle können mit dem aktiven Fenster agieren, wenn der Buchstabe A im Fenstertitel-Parameter angegeben und der Fenstertext-, Titelausnahme- und Textausnahme-Parameter weggelassen wird. Im folgenden Beispiel kann mit Win+↑ das aktive Fenster maximiert werden: #Up::WinMaximize A

Fensterklasse (ahk_class): Alle Fensterbefehle können mit der Fensterklasse agieren, die im Windows Spy angezeigt wird oder mit WinGetClass abgerufen werden kann. Im folgenden Beispiel wird ein Explorerfenster in der Teilansicht aktiviert: WinActivate ahk_class ExploreWClass. Falls der Titelübereinstimmungsmodus RegEx aktiv ist, akzeptiert ahk_class auch einen regulären Ausdruck.

Eindeutige ID/HWND (ahk_id): Alle Fensterbefehle können mit einem bestimmten Fenster oder Steuerelement mithilfe einer eindeutigen ID-Nummer agieren. Zum Beispiel: WinActivate ahk_id %VariableMitID%. Die ID eines Fensters wird normalerweise mithilfe von WinExist() oder WinGet abgerufen. Die ID eines Steuerelements wird üblicherweise mittels ControlGet Hwnd, MouseGetPos oder DllCall abgerufen. Außerdem wird ahk_id auch dann mit Steuerelementen agieren, wenn sie versteckt sind; das heißt, die Einstellung von DetectHiddenWindows spielt keine Rolle.

Prozess-ID (ahk_pid): Alle Fensterbefehle können mit Fenster agieren, die eine bestimmte Prozesskennung (PID) haben. Zum Beispiel wird WinClose ahk_pid %VariableMitPID% das oberste Fenster des Prozesses schließen. Eine PID kann mithilfe von WinGet, Run oder Process abgerufen werden.

Prozess-Name/Pfad (ahk_exe) [v1.1.01+]: Während ahk_pid auf einem bestimmten Prozess begrenzt ist, bezieht ahk_exe alle Prozesse mit ein, deren Namen oder vollständigen Pfade mit dem angegebenen String übereinstimmen. Falls der Titelübereinstimmungsmodus RegEx aktiv ist, akzeptiert ahk_exe auch einen regulären Ausdruck. Ansonsten akzeptiert ahk_exe einen Namen oder Pfad ohne Beachtung der Groß- und Kleinschreibung; zum Beispiel schließt ahk_exe notepad.exe ahk_exe C:\Windows\Notepad.exe, ahk_exe C:\Windows\System32\Notepad.exe und andere Variationen mit ein.

Fenstergruppe (ahk_group): Alle Fensterbefehle können mithilfe von ahk_group EigenerGruppenname im Fenstertitel-Parameter mit Fenstern agieren, die in einer Fenstergruppe sind. Die Befehle WinMinimize, WinMaximize, WinRestore, WinHide, WinShow, WinClose und WinKill beeinflussen alle Fenster einer Gruppe. Andere Fensterbefehle wie WinActivate und IfWinExist bearbeiten hingegen nur das oberste Fenster einer Gruppe.

Mehrere Kriterien: Im Gegensatz zu ahk_group von oben (das die Suche erweitert), kann die Suche durch die Angabe von mehr als einem Kriterium im Fenstertitel-Parameter eingegrenzt werden. Im folgenden Beispiel wartet das Script auf ein Fenster mit dem Titel Eigene Datei.txt und mit der Klasse Notepad:

WinWait Eigene Datei.txt ahk_class Notepad
WinActivate  ; Aktiviert das zuletzt gefundene Fenster.

Beim Verwenden dieser Methode sollte der Text des Titels je nach Bedarf zuerst aufgelistet werden, bevor andere zusätzliche Kriterien erfolgen. Kriterien sollten mit einem Leerzeichen oder Tabulator voneinander getrennt werden (jedes weitere Leer- oder Tabulatorzeichen wird direkt behandelt).
