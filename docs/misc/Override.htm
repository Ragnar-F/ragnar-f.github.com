Überschreiben oder Deaktivieren von Hotkeys

Alle internen Windows-Hotkeys außer WIN+L und WIN+U können deaktiviert werden, wenn folgende Änderungen an der Registry vorgenommen werden (sollte normalerweise in allen Betriebssystemen funktionieren, erfordert aber möglicherweise ein Neustart):

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer
NoWinKeys REG_DWORD 0x00000001 (1)

Lest weiter, wenn Hotkeys mehr als nur deaktiviert werden sollen.

Hotkeys von anderen Anwendungen können einfach überschrieben oder deaktiviert werden, wenn ihnen eine Aktion im Script zugewiesen wird. Mit dieser Funktion werden am häufigsten Hotkeys geändert, die in Windows selbst integriert sind. Wenn zum Beispiel der Wunsch besteht, WIN+E (Windows Explorer starten) neu zu belegen, verwendet folgendes:

#e::
MsgBox Dieser Hotkey gehört nun zum Script.
Return

Im nächsten Beispiel wird der Hotkey WIN+R, der zum Öffnen des Ausführen-Fensters gedacht ist, vollständig deaktiviert:

#r::Return

Auf ähnliche Weise können die beiden Windows-Tasten wie folgt deaktiviert werden:

Lwin::Return
Rwin::Return

Um einen nicht-globalen Hotkey einer Anwendung zu deaktivieren oder zu ändern (das heißt, eine Tastenkombination, die nur funktioniert, wenn die Anwendung das aktive Fenster ist), verwendet folgendes Beispiel, das STRG+P (Drucken) nur für den Editor deaktiviert, aber nicht für die restlichen Fenster:

$^p::
IfWinActive ahk_class Notepad
    Return  ; nichts tun, damit STRG+P im Editor deaktiviert ist.
Send ^p
Return

Im oberen Beispiel wird das $-Präfix benötigt, damit der Hotkey sich selbst senden kann, ohne dabei aktiviert zu werden (ansonsten wird eine Fehlermeldung wegen einer Endlosschleife ausgelöst). Siehe auch: Kontextabhängige Hotkeys.

Die oben genannten Beispiele können ausprobiert werden, indem sie in einer neuen Textdatei z. B. namens "Override.ahk" kopiert werden und die Datei danach ausgeführt wird.
