Script-Performance

Für die maximale Performance sollte SetFormat vermieden (außer der schnellen Modus) und folgende Zeilen am Anfang des Scripts hinzugefügt werden:

#NoEnv
SetBatchLines -1
ListLines Off

Darüber hinaus kann die Performance auch durch folgende Befehle je nach Art des Scripts beeinflusst werden: SendMode, SetKeyDelay, SetMouseDelay, SetWinDelay, SetControlDelay und SetDefaultMouseSpeed.

Interne Performance-Funktionen

Jedes Script wird beim Laden und Überprüfen der Syntax teilweise kompiliert. Neben der Verringerung des Speicherverbrauchs wird dadurch auch die Laufzeit-Performance deutlich verbessert.

Hier die technischen Details des Optimierungsprozesses (Semi-Kompilierung):

  • Eingabe- und Ausgabevariablen (wenn deren Namen keine Verweise auf anderen Variablen enthalten) und Gruppennamen werden in Speicheradressen aufgelöst.
  • Schleifen, Blöcke, IF- und ELSE-Anweisungen erhalten die Speicheradressen in Bezug auf ihre Sprungpunkte im Script.
  • Das Ziel von jedem Hotkey-, Gosub- und Goto-Befehls wird in eine Speicheradresse aufgelöst, solange es keine Variable ist.
  • Jeder Befehlsname wird durch eine Adresse aus der Sprungtabelle ersetzt.
  • Jede Zeile wird vorher in eine Liste von Parametern aufgeteilt und jeder Parameter wird vorher in eine Liste von Variablen aufgeteilt.
  • Jeder Ausdruck wird tokenisiert und von Infix zu Postfix umgewandelt.
  • Jeder Verweis auf eine Variable oder Funktion wird in eine Speicheradresse aufgelöst.
  • Direkte Integer in Ausdrücken und Mathematik-/Vergleichsbefehle werden mit binäre Integer ersetzt.

Darüber hinaus werden binäre Zahlen beim Ausführen des Scripts in Variablen zwischengespeichert, um Umwandlungen zu/von Strings zu vermeiden. Siehe SetFormat für weitere Details.
