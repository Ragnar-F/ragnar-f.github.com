Script-Performance

Für die maximale Performance sollte man auf SetFormat (außer der schnelle Modus) verzichten und folgende Zeilen am Anfang des Scripts hinzufügen:

#NoEnv
SetBatchLines -1
ListLines Off

Die folgenden Befehle könnten auch Einfluss auf die Performance haben, je nachdem, welche Funktion das Script erfüllen soll: SendMode, SetKeyDelay, SetMouseDelay, SetWinDelay, SetControlDelay und SetDefaultMouseSpeed.

Integrierte Performance-Features

Jedes Script wird teilweise kompiliert, während es geladen und seine Syntax überprüft wird. Der Vorteil davon ist, dass es den Speicherverbrauch des Scripts reduziert und die Laufzeit-Performance deutlich verbessert.

Hier die technischen Details des Optimierungsprozesses (Semi-Kompilierung):

  • Eingabe- und Ausgabevariablen (wenn deren Namen keine Verweise auf anderen Variablen enthalten) und Gruppennamen werden in Speicheradressen aufgelöst.
  • Schleifen, Blöcke, IFs und ELSEs bekommen die Speicheradressen von ihren Sprungpunkten im Script.
  • Das Ziel von jedem Hotkey-, Gosub- und Goto-Befehls wird in eine Speicheradresse aufgelöst, sofern es keine Variable ist.
  • Jeder Befehlsname wird durch eine Adresse aus der Sprungtabelle ersetzt.
  • Jede Zeile wird vorab in Parameterlisten aufgeteilt, und jeder Parameter wird vorab in Variablenlisten aufgeteilt (falls vorhanden).
  • Jeder Ausdruck wird tokenisiert und von Infix in Postfix konvertiert.
  • Jeder Verweis auf eine Variable oder Funktion wird in eine Speicheradresse aufgelöst.
  • Literale Integer in Ausdrücken und Mathematik-/Vergleichsbefehlen werden mit binären Integern ersetzt.

Während das Script ausgeführt wird, werden binäre Zahlen in Variablen zwischengespeichert, um Umwandlungen zu/von Strings zu vermeiden. Siehe SetFormat für weitere Details.
