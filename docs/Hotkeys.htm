Einführung und einfache Beispiele

Hotkeys werden manchmal auch als Tastenkürzel bezeichnet, weil sie in der Lage sind, schnell und einfach eine Aktion auszulösen (z. B. das Ausführen eines Programms oder Tastatur-Makros). Im folgenden Beispiel wird der Hotkey Win+N dazu gebracht, den Editor zu starten. Die Raute [#] ist Win, die als Modifikatortaste bezeichnet wird:

#n::
Run Notepad
Return

Das Return in der letzten Zeile dient dazu, den Hotkey zu beenden. Falls ein Hotkey nur eine einzige Zeile ausführen muss, darf die Zeile rechts neben den Doppelpunkten geschrieben werden. In diesem Fall ist das Return inbegriffen:

#n::Run Notepad

Um mehrere Modifikatoren mit einem Hotkey zu kombinieren, listet diese nebeneinander auf (die Reihenfolge spielt keine Rolle). Das folgende Beispiel verwendet ^!s, das als Strg+Alt+S interpretiert wird:

^!s::
Send Mit freundlichen Grüßen,{Enter}John Smith  ; Diese Zeile sendet Tastatureingaben an das aktive (vorderste) Fenster.
Return

Folgende Modifikatorsymbole können zum Definieren von Hotkeys verwendet werden:

Symbol Beschreibung
# Win (Windows-Logo-Taste). Windows Vista und höher: Seit v1.0.48.01 warten Hotkeys, die die Win-Taste enthalten (z. B. #a), auf das Loslassen der Win-Taste, bevor ein beliebiger Text gesendet wird, der ein "L" enthält. Dadurch wird verhindert, dass bei Send der PC unbeabsichtigt gesperrt wird. Diese Änderung gilt für alle Sendmethoden, außer SendPlay (weil es das nicht braucht) und Blind-Modus.
! Alt
^ Strg
+ Umschalt
& Ein kaufmännisches Und dient dazu, zwei beliebige Tasten oder Maustasten zu kombinieren, um einen benutzerdefinierten Hotkey zu erstellen. Siehe unten für weitere Details. Solche Hotkeys werden in Windows 95/98/Me ignoriert.
< Verwendet die linke Version der angegebenen Modifikatortaste. Zum Beispiel ist <!a das Gleiche wie !a, nur dass die linke Alt-Taste verwendet wird. Dieses Symbol wird in Windows 95/98/Me ignoriert.
> Verwendet die rechte Version der angegebenen Modifikatortaste. Dieses Symbol wird in Windows 95/98/Me ignoriert.
<^>!

AltGr (alternativer Schriftsatz). Falls auf der aktuellen Tastaturbelegung AltGr vorhanden ist, kann diese Symbolenreihe normalerweise für die Taste verwendet werden (benötigt Windows NT/2k/XP oder höher). Zum Beispiel:

<^>!m::MsgBox Es wurde AltGr+m gedrückt.
<^<!m::MsgBox Es wurde SteuerungLinks+AltLinks+m gedrückt.

Alternativ kann die folgende Tastenkombination verwendet werden, um AltGr selbst in ein Hotkey zu verwandeln (jedoch ohne die oben genannten Hotkeys):

LControl & RAlt::MsgBox Es wurde AltGr selbst gedrückt.
*

Platzhalter: Der Hotkey wird auch dann ausgeführt, wenn zusätzliche Modifikatortasten gedrückt gehalten werden. Das wird oft in Verbindung mit der Neubelegung von Tasten verwendet. Zum Beispiel:

; Alle möglichen Kombinationen wie Win+C, Umschalt+Win+C und Strg+Win+C
; werden den folgenden Hotkey auslösen.
*#c::Run Calc.exe
; Beim Drücken von ScrollLock wird der folgende Hotkey auch dann ausgelöst,
; wenn beliebig viele Modifikatortasten gedrückt gehalten werden.
*ScrollLock::Run Notepad

Dieses Symbol wird in Windows 95/98/Me ignoriert.
~

Beim Drücken des Hotkeys wird seine ursprüngliche Funktion nicht blockiert (bzw. nicht im System versteckt). In den beiden folgenden Beispielen wird der Mausklick des Benutzers weiterhin an das aktive Fenster gesendet:

~RButton::MsgBox Es wurde die rechte Maustaste gedrückt.
~RButton & C::MsgBox Es wurde die rechte Maustaste + C gedrückt.

Hinweise:

  1. Im Gegensatz zu den anderen Präfix-Symbolen darf das Tilde-Präfix in einigen Hotkey-Varianten vorkommen und in anderen nicht.
  2. Spezielle Hotkeys, die etwas mit Alt-Tab zu tun haben, ignorieren immer das Tilde-Präfix.
  3. Das Tilde-Präfix wird in Windows 95/98/ME ignoriert.
$

Dieses Präfix ist normalerweise nur notwendig, wenn der Send-Befehl verwendet wird, um Tasten zu senden, die den Hotkey selbst enthalten. Ansonsten würde sich der Hotkey selbst auslösen. Das genaue Verhalten der möglichen Varianten mit dem $-Präfix ist vom Betriebssystem abhängig:

In Windows NT4/2k/XP oder höher: Das $-Präfix verwendet für die Implementierung des Hotkeys den Tastatur-Hook, wodurch als Nebeneffekt der Send-Befehl daran gehindert wird, den Hotkey auszulösen. Das heißt, als würde man #UseHook vor der Hotkey-Definition angeben.

In Windows 95/98/Me: Der Hotkey wird, während er als Thread ausgeführt wird, deaktiviert und danach wieder aktiviert. Dabei entsteht der Nebeneffekt, dass sobald #MaxThreadsPerHotkey auf ein Wert höher als 1 gesetzt wird, bei solchen Hotkeys ein Wert von 1 gilt.
UP

Das Wort UP kann nach dem Namen eines Hotkeys erfolgen, um den Hotkey beim Loslassen anstatt beim Drücken auszuführen. Im folgenden Beispiel wird die linke Win-Taste mit der linken Strg-Taste neu belegt:

*LWin::Send {LControl Down}
*LWin Up::Send {LControl Up}

"Up" kann auch mit normalen Hotkeys verwendet werden. Zum Beispiel:

^!r Up::MsgBox Es wurde Strg+Alt+R gedrückt und losgelassen.

Das funktioniert auch mit kombinierten Hotkeys (z. B. F1 & e Up::)

Einschränkungen:

  1. "Up" funktioniert nicht mit Joystick-Tasten.
  2. "Up" benötigt Windows NT4/2000/XP oder höher.
  3. Ein "Up"-Hotkey ohne entsprechenden normalen/gedrückten Gegenstück wird diese Taste komplett übernehmen, um zu verhindern, dass sie stecken bleibt. Dieses Problem kann zum Beispiel verhindert werden, wenn ein Tilde-Präfix hinzugefügt wird (z. B. ~LControl up::).

Zusätzlich kann mit einer ähnlichen Methode wie oben ein Hotkey in eine Präfix-Taste umgewandelt werden. Der Vorteil davon ist, dass der Hotkey beim Loslassen nur ausgeführt wird, wenn keine andere Taste gedrückt wird, während er gedrückt gehalten wird. Zum Beispiel:

; Verwandelt die linke STRG-Taste in ein Präfix,
; wenn sie mindestens einmal vor "&" verwendet wird.
LControl & F1::Return
LControl::MsgBox Es wurde LControl ohne Wirkung losgelassen.

(In der Tastenliste können alle möglichen Tasten von der Tastatur, Maus und des Joysticks gefunden werden)


Es können mehrere Hotkeys vertikal gestapelt werden, damit sie die gleiche Aktion ausführen. Zum Beispiel:

^Numpad0::
^Numpad1::
MsgBox Durch Drücken von Steuerung+Numpad0 oder Steuerung+Numpad1 wird diese Meldung angezeigt.
Return

Eine Taste oder Tastenkombination kann vollständig deaktiviert werden, wenn ihr nichts zugewiesen wird. Das folgende Beispiel deaktiviert die rechte Win-Taste:

RWin::Return

Kontextabhängige Hotkeys

Die Direktiven #IfWinActive/Exist und #If können ein Hotkey dazu bringen, eine andere (oder überhaupt keine) Aktion in Abhängigkeit eines bestimmten Zustandes auszuführen. Zum Beispiel:

#IfWinActive, ahk_class Notepad
^a::MsgBox Strg+A wurde gedrückt, während der Editor aktiv ist. Wenn Strg+A in einem anderen Fenster gedrückt wird, dann wird diese Tastatureingabe an dieses Fenster gesendet.
#c::MsgBox Win+C wurde während des aktiven Editors gedrückt.

#IfWinActive
#c::MsgBox Win+C wurde gedrückt, während ein Fenster außer dem Editor aktiv ist.

#If MouseIsOver("ahk_class Shell_TrayWnd")
WheelUp::Send {Volume_Up}     ; Mausrad über die Taskleiste: Lautstärke erhöhen/verringern.
WheelDown::Send {Volume_Down} ;

Benutzerdefinierte Kombinationen und andere Funktionen [Windows NT/2000/XP oder höher]

Es kann eine benutzerdefinierte Kombination von zwei Tasten mithilfe von " & " angegeben werden. Im folgenden Beispiel kann Numpad0 gedrückt gehalten und die zweite Taste gedrückt werden, um den Hotkey auszulösen:

Numpad0 & Numpad1::MsgBox, Numpad1 wurde gedrückt, während Numpad0 gedrückt gehalten wird.
Numpad0 & Numpad2::Run Notepad

Im obigen Beispiel wird Numpad0 eine Präfix-Taste; dadurch verliert Numpad0 aber seine ursprüngliche Funktion, wenn sie selbst gedrückt wird. Um das zu vermeiden, konfiguriert die Numpad0-Taste wie folgt, um eine neue Aktion durchzuführen:

Numpad0::WinMaximize A   ; Maximiert das aktive/vorderste Fenster.
Numpad0::Send {Numpad0}  ; Numpad0 erzeugt beim Loslassen eine Numpad0-Tastatureingabe. Siehe unteren Kommentar.

Sobald einer der oben genannten Hotkeys vorhanden ist, wird Numpad0 beim Loslassen die angegebene Aktion durchführen, solange keine andere Taste gedrückt wurde, während Numpad0 gedrückt gehalten wurde.

Num, Feststell und Rollen: Für diese Tasten kann "AlwaysOn" oder "AlwaysOff" erzwungen werden. Zum Beispiel: 

SetNumlockState AlwaysOn

Hotkeys vom Explorer überschreiben: Die internen Hotkeys von Windows wie Win+E (#e) und Win+R (#r) können individuell überschrieben werden, indem ihnen einfach eine Aktion im Script zugewiesen wird. Siehe "Überschreiben oder Deaktivieren von Hotkeys" für Details.

Ersatz für Alt-Tab: Hotkeys können eine alternative Methode für Alt-Tab bereitstellen. Zum Beispiel ermöglichen die folgenden zwei Hotkeys, Alt-Tab mit der rechten Hand aufzurufen:

RControl & RShift::AltTab  ; Rechte STRG-Taste gedrückt halten und danach rechte UMSCHALT-Taste wiederholend drücken, um durchzuschalten.
RControl & Enter::ShiftAltTab  ; Ohne Loslassen der rechten STRG-Taste Enter drücken, um die Richtung umzukehren.

Für weitere Details, siehe Alt-Tab.

Mausrad-Hotkeys [Windows NT/2000/XP oder höher]

Hotkeys, die beim Drehen des Mausrads ausgeführt werden, werden durch WheelDown und WheelUp unterstützt. WheelLeft und WheelRight werden auch seit v1.0.48 unterstützt, funktionieren aber nicht auf Betriebssystemen älter als Windows Vista. Hier einige Beispiele für Mausrad-Hotkeys:

MButton & WheelDown::MsgBox Das Mausrad wurde nach unten gedreht, während die mittlere Maustaste gedrückt gehalten wurde.
^!WheelUp::MsgBox Das Mausrad wurde nach oben gedreht, während Steuerung+Alt gedrückt gehalten wurde.

Seit v1.0.43.03 enthält die interne Variable A_EventInfo die Anzahl an gedrehten Rasterungen (standardmäßig eine 1). Allerdings kann A_EventInfo unter folgenden Umständen größer oder kleiner als 1 sein:

  • Wenn die Maus-Hardware eine Drehungsgröße meldet, die kleiner als eine Rasterung ist, enthält A_EventInfo eine 0;
  • Wenn das Mausrad schnell gedreht wird (je nach Typ der Maus), dann kann A_EventInfo größer als 1 sein. Zum Beispiel kann der folgende Hotkey dabei helfen, die eigene Maus zu analysieren: ~WheelDown::ToolTip %A_EventInfo%

Es gibt nützliche Hotkeys für das Mausrad, die alternative Methoden für das Scrollen eines Fenstertextes bereitstellen. Zum Beispiel werden die folgenden Hotkey-Paare horizontal anstatt vertikal scrollen, wenn das Mausrad gedreht wird, während die linke STRG-Taste gedrückt gehalten wird:

~LControl & WheelUp::  ; Nach links scrollen.
ControlGetFocus, fcontrol, A
Loop 2  ; <-- Erhöht diesen Wert, um das Scrollen zu beschleunigen.
    SendMessage, 0x114, 0, 0, %fcontrol%, A  ; 0x114 ist WM_HSCROLL und die 0 danach SB_LINELEFT.
Return

~LControl & WheelDown::  ; Nach rechts scrollen.
ControlGetFocus, fcontrol, A
Loop 2  ; <-- Erhöht diesen Wert, um das Scrollen zu beschleunigen.
    SendMessage, 0x114, 1, 0, %fcontrol%, A  ; 0x114 ist WM_HSCROLL und die 1 danach SB_LINERIGHT.
Return

Da Mausrad-Hotkeys nur Ereignisse beim Drücken erzeugen (nie beim Loslassen), können sie nicht als Hotkeys verwendet werden, die das Wort "UP" haben.

Tipps und Hinweise

Jede Ziffernblocktaste kann zwei verschiedene Hotkey-Subroutinen ausführen, abhängig vom Num-Status. Alternativ kann eine Ziffernblocktaste die gleiche Subroutine starten, unabhängig vom Num-Status. Zum Beispiel:

NumpadEnd::
Numpad1::
MsgBox, Dieser Hotkey wurde unabhängig vom Numlock-Status gestartet.
Return

Auch wenn der Tilde-Operator (~) nur einmal mit einer Präfix-Taste verwendet wird, wird dieses Präfix immer bis ans aktive Fenster gesendet. In den beiden folgenden Hotkeys wird zum Beispiel das aktive Fenster alle Rechtsklick-Ereignisse empfangen, obwohl nur in einer der Definitionen ein Tilde-Zeichen enthalten ist:

~RButton & LButton::MsgBox Es wurde die linke Maustaste gedrückt, während die rechte gedrückt gehalten wurde.
RButton & WheelUp::MsgBox Das Mausrad wurde nach oben gedreht, während die rechte Maustaste gedrückt gehalten wurde.

Der Suspend-Befehl kann vorübergehend alle Hotkeys deaktivieren, außer denen, die vom Benutzer ausgeschlossen wurden. Um Hotkeys abhängig vom Fenstertyp zu machen, verwendet #IfWinActive/Exist.

Mithilfe des Hotkey-Befehls können Hotkeys dynamisch erstellt werden, während das Script ausgeführt wird. Der Hotkey-Befehl kann auch bestehende Hotkeys im Script individuell bearbeiten, deaktivieren oder aktivieren.

Joystick-Hotkeys unterstützen zurzeit keine Modifikatortasten als Präfix, wie z. B. ^ (Strg) und # (Win). Allerdings kann GetKeyState wie im folgenden Beispiel verwendet werden, um diesen Effekt nachzuahmen:

Joy2::
If not GetKeyState("Control")  ; Weder die linke noch die rechte STRG-Taste wurde gedrückt.
    Return  ; also nichts tun.
MsgBox Es wurde die zweite Taste vom ersten Joystick gedrückt, während die STRG-Taste gedrückt gehalten wurde.
Return

Manchmal sollte ein Hotkey auf das Loslassen seiner Modifkatortasten warten, bevor er fortfährt. Betrachtet dazu das folgende Beispiel:

^!s::Send {Delete}

Beim Drücken von Strg+Alt+S denkt das System, dass Strg+Alt+Entf gedrückt wird (aufgrund der aggressiven Erkennung dieser Tastenkombination). Als Übergangslösung kann KeyWait verwendet werden, um damit auf das Loslassen der Tasten zu warten; zum Beispiel:

^!s::
KeyWait Control
KeyWait Alt
Send {Delete}
Return

Wenn ein Hotkey-Label wie #z:: ein Fehler wie "Ungültiger Hotkey" erzeugt, dann kennt die Tastaturbelegung/Sprache des Systems möglicherweise dieses bestimmte Zeichen nicht (in diesem Fall das "Z"). Verwendet versuchsweise Zeichen, die bereits in der aktuellen Tastaturbelegung bekannt sind.

Ein Hotkey-Label kann als Ziel von Gosub oder Goto verwendet werden. Zum Beispiel: Gosub ^!s

Häufig werden Hotkeys dazu verwendet, eine wiederholende Aktion zu starten und zu stoppen, wie z. B. mehrere Tastatureingaben oder Mausklicks. Ein Beispiel dafür kann in diesem FAQ-Thema gefunden werden.

Letztendlich ist jedes Script sozusagen multi-threaded, damit ein neuer Hotkey auch dann ausgeführt werden kann, wenn eine vorherige Hotkey-Subroutine noch läuft. Zum Beispiel können auch dann neue Hotkeys ausgeführt werden, während eine MsgBox durch den aktuellen Hotkey angezeigt wird.

Alt-Tab-Hotkeys

Jeder Alt-Tab-Hotkey muss eine Kombination von zwei Tasten sein, dass normalerweise mithilfe eines kaufmännischen Und-Zeichens (&) erreicht wird. Im folgenden Beispiel wird die rechte Alt-Taste gedrückt gehalten und mit J oder K durch das Alt-Tab-Menü navigiert:

RAlt & j::AltTab
RAlt & k::ShiftAltTab

AltTab und ShiftAltTab sind zwei spezielle Befehle, die nur erkannt werden, wenn sie auf der gleichen Zeile des Hotkeys verwendet werden. Hier die komplette Liste:

AltTab: Sobald das Alt-Tab-Menü sichtbar ist, wird es durchgeschaltet. Ansonsten wird das Menü angezeigt (nur wenn der Hotkey eine "&"-Kombination von zwei Tasten ist, sonst passiert nichts).

ShiftAltTab: Das Gleiche wie oben, nur in die entgegengesetzte Richtung.

AltTabAndMenu: Sobald das Alt-Tab-Menü sichtbar ist, wird es durchgeschaltet. Ansonsten wird das Menü angezeigt.

AltTabMenuDismiss: Schließt das Alt-Tab-Menü.

Zur Veranschaulichung dient das Mausrad als Ersatz für Alt-Tab. Mit den folgenden Hotkeys wird das Menü durch Klicken der mittlere Maustaste angezeigt und durch Drehen des Mausrads durchgeschaltet:

MButton::AltTabMenu
WheelDown::AltTab
WheelUp::ShiftAltTab

Mit dem folgenden Beispiel kann ein vom Hotkey aufgerufenes Alt-Tab-Menü geschlossen werden, ohne dabei das ausgewählte Fenster zu aktivieren. Es sind möglicherweise einige Anpassungen notwendig, je nachdem womit das Alt-Tab-Menü angezeigt wurde und ob das Script den Tastatur-Hook installiert hat.

LCtrl & CapsLock::AltTab
!MButton::  ; Mittlere Maustaste. Das !-Präfix führt es aus, während die ALT-Taste gedrückt wird (sobald das Alt-Tab-Menü sichtbar ist).
IfWinExist ahk_class #32771  ; Zeigt an, dass das Alt-Tab-Menü auf dem Bildschirm vorhanden ist.
    Send !{Escape}{Alt up}
Return

Zurzeit müssen alle speziellen Alt-Tab-Aktionen wie in den folgenden Beispielen direkt einem Hotkey zugewiesen werden (d. h. sie können nicht wie Befehle verwendet werden). Sie werden nicht von #IfWin oder #If beeinflusst.

Benutzerdefinierte Alt-Tab-Aktionen können auch mithilfe von Hotkeys erstellt werden. Im folgenden Beispiel wird F1 gedrückt, um das Menü anzuzeigen und durchzuschalten. Danach F2, um das ausgewählte Fenster zu aktivieren (oder Esc, um abzubrechen):

*F1::Send {Alt down}{tab} ; In diesem Fall ist das Sternchen notwendig.
!F2::Send {Alt up}  ; Lässt die ALT-Taste los, um das ausgewählte Fenster zu aktivieren.
~*Escape::
IfWinExist ahk_class #32771
    Send {Escape}{Alt up}  ; Schließt das Menü, ohne dabei das ausgewählte Fenster zu aktivieren.
Return
