Sprachsyntax

Wann werden im Zusammenhang mit Befehlen und Parametern Anführungszeichen verwendet?

Anführungszeichen (") haben nur innerhalb von Ausdrücken eine besondere Bedeutung. Überall woanders werden sie literal behandelt, als wären sie ganz normale Zeichen. Führt ein Script ein Programm oder ein Dokument aus, benötigt das Betriebssystem allerdings Anführungszeichen bei Befehlszeilenparameter, die Leerzeichen enthalten. Zum Beispiel: Run, Notepad.exe "C:\Meine Dateien\Adressenliste.txt".

Wann genau werden Variablen mit Prozentzeichen umschlossen (%)?

Variablennamen werden immer mit Prozentzeichen umschlossen, außer in den unteren Fällen, die fett hervorgehoben sind:

1) Bei Parametern, die Eingabe- oder Ausgabevariablen sind: StringLen, AusgabeVar, EingabeVar
2) Auf der linken Seite einer Zuweisung: Var = 123abc
3) Auf der linken Seite einer traditionellen If-Anweisung (kein Ausdruck): If Var1 < %Var2%
4) Überall in Ausdrücken. Zum Beispiel:

If (Var1 <> Var2)
    Var1 := Var2 + 100

Wann sollten Prozentzeichen und Kommas mit einem Escapezeichen versehen werden?

Literale Prozentzeichen müssen mit einem Escapezeichen versehen werden, durch Voranstellen eines Akzents / umgekehrten Häkchens. Zum Beispiel: MsgBox Der aktuelle Prozentwert liegt bei 25`%. Literale Kommas müssen auch mit einem Escapezeichen versehen werden (`,), es sei denn, sie werden in einer MsgBox oder im letzten Parameter eines Befehls verwendet (in diesem Fall ist das umgekehrte Häkchen zulässig, aber nicht notwendig).

Sind Kommas oder Prozentzeichen innerhalb eines Ausdrucks in Anführungszeichen gesetzt, ist das umgekehrte Häkchen zulässig, aber nicht notwendig. Zum Beispiel: Var := "15%".

Allgemeine Aufgaben

Wo finde ich den offiziellen Build, oder ältere Versionen?

Siehe Download-Seite von AutoHotkey.

Warum werden einige Zeilen in meinem Script nie ausgeführt?

Jede Zeile, die unmittelbar beim Starten des Scripts ausgeführt werden soll, muss ganz oben im Script vorkommen und vor dem ersten Hotkey, Hotstring oder Return stehen. Siehe automatischer Ausführungsbereich für weitere Details.

Bei einem Hotkey, der mehr als eine Zeile ausführt, muss seine erste Zeile unterhalb des Hotkeys geschrieben werden, nicht auf der gleichen Zeile. Zum Beispiel:

#space::  ; WIN+LEERTASTE
Run Notepad
WinWaitActive Unbenannt - Editor
WinMaximize
Return

Warum funktioniert mein Script problemlos in Windows XP, aber nicht in Windows Vista/7/8?

Da du das Betriebsystem gewechselt hast, ist es wahrscheinlich, dass auch etwas anderes verändert wurde, das dein Script beeinflussen könnte. Wenn du beispielsweise einen neuen Computer hast, können Abweichungen zum alten Computer auftreten. Wenn du AutoHotkey auf eine neue Version aktualisiert hast, finde heraus, welche Version zuvor installiert war, und überprüfe dann die Changelog und Kompatibilitätshinweise auf Änderungen, die deinen Script stören könnten.

Ein häufiges Problem beim Wechsel von XP auf Vista oder höher ist, dass Programme, deren Rechte "erhöht" sind (die als Administrator ausgeführt werden sollen), Hotkeys blockieren oder verweigern, automatisiert zu werden. Es gibt mindestens drei Übergangslösungen:

  • Modifiziere AutoHotkey.exe, so dass Interaktionen mit administrativen Programmen möglich sind. Allerdings hat das Nachteile. Für mehr Information, siehe http://www.autohotkey.com/board/topic/70449-enable-interaction-with-administrative-programs/
  • Führe das Script als Administrator aus. Beachte, dass dadurch Programme, die vom Script aus gestartet werden, auch als Administrator ausgeführt werden.
  • Deaktiviere Benutzerkontensteuerung (nicht empfohlen); in Windows 8 und höher müsstest du zusätzlich noch die Sicherheitseinstellung "Alle Administratoren im Administratorgenehmigungsmodus ausführen" deaktivieren (auch nicht empfohlen).

SoundGet, SoundSet, SoundGetWaveVolume und SoundSetWaveVolume verhalten sich in Windows Vista und höher anders als in früheren Versionen von Windows. Insbesondere sind die Gerätenummern unterschiedlich und einige Komponenten möglicherweise nicht verfügbar. Das Verhalten hängt von den Audio-Treibern ab, die sich zwangsläufig von den Treibern für Windows XP unterscheiden. Das Soundkartenanalyse-Script kann verwendet werden, um die korrekten Gerätenummern ermitteln zu können.

Ich kann mein Script nicht über das Tray-Icon bearbeiten, weil es sich aufgrund eines Fehlers nicht starten lässt. Kann ich mein Script noch irgendwo anders finden?

Versuche, eine der folgenden Dateien zu finden und zu bearbeiten (z. B. im Windows-Editor):

  • My Documents/AutoHotkey.ahk
  • My Documents/AutoHotkey.ini
  • Program Files/AutoHotkey/AutoHotkey.ahk
  • Program Files/AutoHotkey/AutoHotkey.ini

Verschiedene Versionen von AHK können unterschiedliche Dateien erstellen. Überprüfe alle Standorte - vielleicht findest du einen dieser Dateinamen.

AHK ist kein "normales" Programm; es ist nicht dazu gedacht, über das Startmenü ausgeführt zu werden. Startet man AHK direkt über das Startmenü oder AutoHotkey.exe, wird eine der oben genannten Dateien gelesen und als Script ausgeführt, falls möglich. Normalerweise sollte man Dateien mit der Endung .ahk erstellen und diese dann per Doppelklick starten.

Siehe auch Befehlszeilenparameter "Script-Dateiname" und Portabilität von AutoHotkey.exe.

Wie kann ich Fehler in meinem Code finden und beheben?

Für einfache Scripts, siehe Ein Script debuggen. Mit MsgBox oder ToolTip können Inhalte einer Variable angezeigt werden. Für komplexe Scripts, siehe Interaktives Debuggen.

Kann ich AHK von einem USB-Laufwerk ausführen?

Siehe Portabilität von AutoHotkey.exe. Hinweis: Wenn du automatisch einfügende Funktions-Libraries verwendest, müssen AutoHotkey.exe und der Lib-Ordner eine Ebene höher sein als Ahk2Exe.exe (z. B. \AutoHotkey.exe vs \Compiler\Ahk2Exe.exe). Beachte zudem, dass Ahk2Exe seine Einstellungen in den folgenden Registry-Key speichert: HKEY_CURRENT_USER\Software\AutoHotkey\Ahk2ExeCommand.

Warum ist der Run-Befehl nicht in der Lage, mein Spiel oder Programm auszuführen?

Einige Programme müssen in ihren eigenen Verzeichnissen gestartet werden (im Zweifelsfall der sicherste Weg). Zum Beispiel:

Run, %ProgramFiles%\Beliebige Anwendung\Anwendung.exe, %ProgramFiles%\Beliebige Anwendung

Wie kann die Ausgabe einer Befehlszeilenoperation abgefragt werden?

Tests zeigen, dass eine temporäre Datei aufgrund von Datei-Caching sehr schnell bei relativ kleinen Ausgaben sein kann. Tatsächlich wird sie gar nicht erst auf das Laufwerk geschrieben, wenn die Datei unmittelbar nach ihrer Verwendung gelöscht wird. Zum Beispiel:

RunWait %comspec% /c dir > C:\Meine Temp-Datei.txt
FileRead, AusgabeVar, C:\Meine Temp-Datei.txt
FileDelete, C:\Meine Temp-Datei.txt

Um temporäre Dateien zu vermeiden (besonders wenn die Ausgabe groß ist), könnte man CmdRet oder StdoutToVar in Betracht ziehen.

Wie kann ein Script andere Scripts schließen, pausieren oder sperren?

Zunächst ein Beispiel, das ein anderes Script schließt:

DetectHiddenWindows On  ; Ermöglicht die Erkennung des versteckten Hauptfensters eines Scripts.
SetTitleMatchMode 2  ; Verhindert, dass unten der vollständige Pfad der Datei angegeben werden muss.
WinClose Script-Dateiname.ahk - AutoHotkey  ; Verwende hier den Namen deines Scripts (zeichengenau).

Um ein anderes Script zu sperren oder zu pausieren, ersetze die oben genannte letzte Zeile mit eine der folgenden Zeilen:

PostMessage, 0x111, 65305,,, Script-Dateiname.ahk - AutoHotkey  ; Suspend (Sperren).
PostMessage, 0x111, 65306,,, Script-Dateiname.ahk - AutoHotkey  ; Pause.

Wie kann eine sich wiederholende Aktion gestoppt werden, ohne das Script beenden zu müssen?

Um bei Tastendruck das ganze Script zu pausieren oder fortzusetzen, weise dem Pause-Befehl einen Hotkey wie folgt zu:

^!p::Pause  ; Drücke STRG+ALT+P für Pause. Drückt es nochmal, um fortzufahren.

Um eine sich wiederholende Aktion innerhalb einer Schleife zu stoppen, verwende den folgenden Hotkey, der seine eigene wiederholende Aktion sowohl starten als auch beenden kann. Das heißt, dass durch einmaliges Drücken des Hotkeys die Schleife begonnen und durch nochmaliges Drücken diese wieder beendet wird.

#MaxThreadsPerHotkey 3
#z::  ; WIN+Z (passe diesen Hotkey nach Belieben an).
#MaxThreadsPerHotkey 1
If WinZLaufenLassen  ; Bedeutet, dass ein darunterliegender Thread bereits die untere Schleife ausführt.
{
    WinZLaufenLassen := false  ; Signalisiert dem Thread, die Schleife zu stoppen.
    Return  ; Beendet diesen Thread, so dass der darunterliegende fortgesetzt wird und die vorgenommene Änderung der oberen Zeile sehen kann.
}
; Ansonsten:
WinZLaufenLassen := true
Loop
{
    ; Die nächsten vier Zeilen sind Aktionen, die mehrfach ausgeführt werden sollen (passe sie nach Belieben an):
    ToolTip, Drücke erneut WIN+Z, um das hier zu stoppen.
    Sleep 1000
    ToolTip
    Sleep 1000
    ; Lasse den Rest aber unverändert.
    If not WinZLaufenLassen ; Der Benutzer hat die Schleife durch erneutes Drücken von WIN+Z gestoppt.
        break  ; Schleife unterbrechen.
}
WinZLaufenLassen := false  ; Zurücksetzen, damit der Hotkey wieder verwendet werden kann.
Return

Wie kann die Performance für Spiele oder zu einem anderen Zeitpunkt verbessert werden, wenn die CPU stark belastet wird?

Process, Priority, , High

Wie können Datums- oder Zeitwerte verglichen oder geändert werden?

Der EnvAdd-Befehl kann eine bestimmte Menge an Tagen, Stunden, Minuten oder Sekunden im YYYYMMDDHH24MISS-Format addieren oder subtrahieren. Das folgende Beispiel subtrahiert 7 Tage von der angegebenen Zeit: EnvAdd, VariableMitZeitstempel, -7, days.

Um die Zeitmenge zwischen zwei Datum- oder Zeitwerten zu ermitteln, siehe EnvSub. Die integrierte Variable A_Now enthält die aktuelle Ortszeit. Außerdem gibt es noch andere integrierte Datum/Zeit-Variablen, sowie auch den FormatTime-Befehl, um einen benutzerdefinierten Datum/Zeit-String erstellen zu können.

Warum funktionieren Hotstrings, Sends und Clicks in einigen Spielen nicht?

AHK kann nicht bei allen Spielen Tasten/Mausklicks senden oder Pixelfarben lesen.

Allerdings gibt es einige Alternativen - probiere alle unten erwähnten Lösungen aus. Wenn sie alle fehlschlagen, könnte es sein, dass AHK nicht für dein Spiel geeignet ist. Manche Spiele haben Maßnahmen integriert, um Hacks und Cheats zu verhindern, wie z. B. GameGuard und Hackshield. Wenn sie so etwas haben, ist es sehr wahrscheinlich, dass AutoHotkey nicht für dein Spiel funktionieren wird.

  • Verwende SendPlay via SendPlay-Befehl, SendMode Play und/oder Hotstring-Option SP.

    SendPlay, abc
    SendMode, Play
Send, abc
    :SP:bzw::beziehungsweise

; oder

#Hotstring SP
::bzw::beziehungsweise

    Hinweis: Es kann sein, dass SendPlay überhaupt nicht in Windows Vista oder höher funktioniert, wenn die Benutzerkontensteuerung (UAC) aktiv ist, selbst wenn das Script als Administrator läuft.

  • Erhöhe SetKeyDelay. Zum Beispiel:

    SetKeyDelay, 0, 50
SetKeyDelay, 0, 50, Play

  • Probiere ControlSend, das dort funktionieren könnte, wo andere Sendemethoden versagen:

    ControlSend,, abc, Spiel_Titel

  • Probiere das Down- und Up-Ereignis einer Taste mit verschiedenen Sendemethoden:

    Send {TASTE down}{TASTE up}

  • Probiere das Down- und Up-Ereignis einer Taste mit einem Sleep dazwischen:

    Send {TASTE down}
Sleep 10 ; probiere verschiedene Millisekunden
Send {TASTE Up}

Mein Antivirus-Programm hat AHK als Malware gekennzeichnet. Enthält es wirklich ein Virus?

Natürlich nicht. Antivirus-Programme könnten AHK fälschlicherweise als Malware erkennen (genannt False-Positive), wenn das Script mit einem Kompressor wie UPX (Standard in AHK 1.0, seit AHK 1.1 entfernt) oder MPRESS (optional in AHK 1.1) kompiliert wurde. Kompressoren können die Größe von kompilierten Scripts verringern. Wenn du das nicht unbedingt für deine kompilierte EXE-Datei brauchst, solltest du auf Kompressoren verzichten, um Falschmeldungen vom AV-Programm bei dir und möglichen anderen Benutzern zu verhindern.

Hotkeys, Hotstrings und Neubelegung

Wie können meine Hotkeys und Hotstrings automatisch bei jedem PC-Start ausgeführt werden?

Es gibt einen Ordner im Startmenü, der sich "Autostart" nennt. Fügst du eine Verknüpfung zu deinem Script in diesen Ordner ein, wird es jedes Mal gestartet, wenn du dein PC hochfährst. Um eine Verknüpfung zu erstellen:

  1. Finde die Script-Datei, markiere sie und drücke STRG+C.
  2. Klicke mit der rechten Maustaste auf Start (üblicherweise in der linken unteren Ecke des Bildschirms) oder im Startmenü auf "Alle Programme" (falls das neue Design aktiv ist) und wähle "Öffnen - Alle Benutzer" aus.
  3. Navigiere zum Autostart-Ordner innerhalb des Programmordners.
  4. Wähle über die Menüleiste Bearbeiten -> Verknüpfung einfügen aus. Die Verknüpfung des Scripts sollte nun im Autostart-Ordner sein.

Ich habe Schwierigkeiten, dass meine Maustasten wie Hotkeys funktionieren. Irgendwelche Hinweise?

Die linke und rechte Maustaste sollte ganz normal zuweisbar sein (zum Beispiel wäre #LButton:: der Hotkey WIN+MAUSTASTE-LINKS). Das gilt auch für die mittlere Maustaste und das Mausrad, außer bei Mäusen, deren Tasten direkt vom Treiber gesteuert werden.

Die vierte (XButton1) und fünfte Maustaste (XButton2) sind möglicherweise auch zuweisbar, sofern dein Maustreiber diese Klicks für das System erkennbar macht. Sind sie nicht erkennbar (oder hat die Maus mehr als 5 Tasten, die verwendet werden sollen), kann man versuchen, die mitgelieferte Software der Maus manuell zu konfigurieren (meistens in der Systemsteuerung oder im Startmenü aufrufbar), um eine Tastatureingabe zu senden, wann immer man eine dieser Tasten drückt. Dadurch ist es möglich, so eine Tastatureingabe als Hotkey zu definieren. Bringt man beispielsweise die vierte Maustaste dazu, STRG+F1 zu senden, könnte man sie als Hotkey verwenden, wenn ^F1:: im Script definiert wird.

Wenn man eine 5-Tasten-Maus besitzt, deren vierte und fünfte Maustaste nicht erkennbar sind, kann man versuchen, den Maustreiber mit dem Standardtreiber des Betriebssystems zu ersetzen. Das setzt voraus, dass so ein Treiber für diese bestimmte Maus vorhanden ist und dass man ohne den mitgelieferten Funktionen der Software leben kann.

Wie können TAB und LEERTASTE als Hotkeys definiert werden?

Verwende nicht die Zeichen der Tasten, sondern ihre englischen Namen (Tab und Space). Zum Beispiel ist #Space WIN+LEERTASTE und ^!Tab STRG+ALT+TAB.

Wie können Tastatur- oder Maustasten neu belegt werden, so dass sie zu anderen Tasten werden?

Das wird auf der Seite "Neubelegung von Tasten" beschrieben.

Wie kann ich einen doppelten Tastendruck oder Mausklick erkennen?

Verwende die integrierten Variablen für Hotkeys wie folgt:

~Ctrl::
    if (A_ThisHotkey = A_PriorHotkey && A_TimeSincePriorHotkey < 200)
        MsgBox Doppeldruck
return

Wie kann ein Hotkey oder Hotstring exklusiv für ein bestimmtes Programm bereitgestellt werden? Mit anderen Worten möchte ich eine bestimmte Taste dazu bringen, dass sie sich anders als normal verhält, falls ein bestimmtes Fenster aktiv ist.

Die bevorzugte Methode ist #IfWinActive. Zum Beispiel:

#IfWinActive, ahk_class Notepad
^a::MsgBox Sie haben STRG+A gedrückt, während der Editor aktiv ist.

Wie kann man das machen, dass eine Präfix-Taste ihre ursprüngliche Funktion durchführt, anstatt nichts zu tun?

Das folgende Beispiel macht Numpad0 zu einer Präfix-Taste:

Numpad0 & Numpad1::MsgBox, Sie haben Numpad1 gedrückt, während Numpad0 gedrückt wurde.

Um Numpad0 nun dazu zu bringen, immer dann eine echte Numpad0-Tastatureingabe zu senden, wenn sie nicht zum Starten eines Hotkeys wie oben benötigt wird, füge den folgenden Hotkey hinzu:

 $Numpad0::Send, {Numpad0}

Das $-Präfix ist notwendig, um eine Warnmeldung im Falle einer Endlosschleife zu unterdrücken (da der Hotkey "sich selbst sendet"). Darüber hinaus wird die obere Aktion erst ausgelöst, wenn die Taste losgelassen wird.
