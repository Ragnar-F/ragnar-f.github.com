#SingleInstance

Bestimmt, ob ein Script wieder ausgeführt werden kann, wenn es bereits läuft.

#SingleInstance [force|ignore|off]

Parameter

force|ignore|off

Dieser Parameter bestimmt, was geschieht, wenn ein Script gestartet wird, während eine frühere Instanz von sich selbst bereits aktiv ist:

Das Wort FORCE überspringt das Dialogfenster und ersetzt die alte Instanz automatisch, vergleichbar mit dem Reload-Befehl.

Das Wort IGNORE überspringt das Dialogfenster und lässt die alte Instanz aufrecht. Mit anderen Worten wird der Versuch ignoriert, ein laufendes Script zu starten.

Das Wort OFF erlaubt mehrere Instanzen des Scripts, damit sie gleichzeitig ausgeführt werden können.

Wird der Parameter weggelassen wird, dann wird ein Dialogfenster mit der Anfrage anzeigt, die alte Instanz zu erhalten oder mit einer neuen Instanz zu ersetzen.

Bemerkungen

Ein Skript mit Hotkeys, Hotstrings, #Persistent, OnMessage() oder Gui ist standardmäßig eine Einzelinstanz (Dialogfenster & Eingabeaufforderung). Andere Scripts erlauben standardmäßig mehrere Instanzen. Dieses Verhalten kann wie oben beschrieben deaktiviert oder geändert werden.

Verwandte Befehle

Reload, #Persistent

Beispiel

#SingleInstance force
#SingleInstance ignore
#SingleInstance off
