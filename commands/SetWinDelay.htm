SetWinDelay

Bestimmt die Verzögerung, die nach jedem Fenster-Befehl wie WinActivate auftreten wird.

SetWinDelay, Verzögerung

Parameter

Verzögerung Zeit in Millisekunden (kann ein Ausdruck sein). Verwendet -1 für keine Verzögerung und 0 für die kleinste mögliche Verzögerung. Wenn nicht gesetzt, dann wird eine Standardverzögerung von 100 verwendet.

Bemerkungen

Eine kurze Verzögerung (Ruhephase) erfolgt automatisch nach jedem Fenster-Befehl, außer bei IfWinActive oder IfWinExist. Das geschieht, um die Zuverlässigkeit der Scripts zu verbessern, weil ein Fenster manchmal eine kurze Ruhephase benötigt, nachdem es erstellt, aktiviert, minimiert usw. wurde, damit die Chance besteht, sich selbst zu aktualisieren und auf dem nächsten Befehl zu reagieren, der vom Script versuchsweise gesendet wird.

Obwohl eine Verzögerung von -1 (keine Verzögerung) erlaubt ist, wird mindestens ein Wert von 0 empfohlen, um das Vertrauen zu stärken, dass das Script auch unter hoher CPU-Auslastung noch richtig ausgeführt wird.

Eine Verzögerung von 0 führt Sleep(0) intern aus, die den Rest der Script-Zeitscheibe für einen anderen Prozess zurückgibt. Wenn es keine gibt, dann ermöglicht Sleep(0) keine Ruhephase.

Wenn die CPU langsam ist oder belastet wird, oder die Fensteranimation aktiviert ist, dann werden möglicherweise höhere Verzögerungswerte benötigt.

Die interne Variable A_WinDelay enthält die aktuelle Einstellung.

Jeder neu gestartete Thread (z. B. ein Hotkey, ein benutzerdefinierter Menüpunkt oder eine zeitlich festgelegte Subroutine) beginnt mit der Voreinstellung für diesen Befehl. Die Voreinstellung lässt sich ändern, wenn dieser Befehl im automatischen Ausführungsbereich (am Anfang des Scripts) ausgeführt wird.

Verwandte Befehle

SetControlDelay, SetKeyDelay, SetMouseDelay, SetBatchLines, SendMode

Beispiel

SetWinDelay, 10
