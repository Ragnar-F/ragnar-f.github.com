InputBox

Zeigt ein Fenster zur Eingabe eines Strings an.

InputBox, Ausgabevariable [, Titel, Anzeigetext, HIDE, Breite, Höhe, X, Y, Schriftart, Zeitüberschreitung, Standardwert]

Parameter

Ausgabevariable Der Name der Variable, in der der eingegebene Text des Benutzers gespeichert wird.
Titel Der Titel der InputBox. Wenn leer oder nicht vorhanden, dann wird standardmäßig der aktuelle Name des Scripts verwendet.
Anzeigetext Text des Eingabefensters, der darauf hinweist, was eingegeben werden soll. Wenn der Anzeigetext lang ist, dann kann die Liste in mehreren kurzen Zeilen aufgeteilt werden (Fortsetzungsbereich), die die Lesbarkeit und Wartbarkeit verbessern könnten.
HIDE Wenn dieser Parameter das Wort HIDE ist, dann wird die Benutzereingabe verborgen, welches nützlich für Passwörter ist.
Breite Wenn dieser Parameter nicht vorhanden oder leer ist, dann ist die Standardbreite des Fensters 375. Dieser Parameter kann ein Ausdruck sein.
Höhe Wenn dieser Parameter nicht vorhanden oder leer ist, dann ist die Standardhöhe des Fensters 189. Dieser Parameter kann ein Ausdruck sein.
X, Y Die X- und Y-Koordinaten des Fensters (mit 0,0 wird das Fenster in die linken oberen Ecke des Bildschirms verschoben), die Ausdrücke sein können. Wenn einer der Koordinaten leer- oder weggelassen wird, dann wird das Fenster in dieser Abmessung zentriert. Jede Koordinate kann negativ sein, d. h. das Fenster liegt somit teilweise außerhalb des Bildschirms.
Font Noch nicht implementiert (leer lassen). Dieser Parameter könnte zukünftig so etwas ähnliches wie verdana:8 akzeptieren.
Zeitüberschreitung Zeitüberschreitung in Sekunden (kann einen Dezimalpunkt oder einen Ausdruck enthalten).  Wenn dieser Wert 2147483 (24.8 Tage) übersteigt, dann wird dieser auf 2147483 gesetzt. Wenn die Zeit abgelaufen ist, wird das Eingabefenster automatisch geschlossen und ErrorLevel wird auf 2 gesetzt. Die Ausgabevariable enthält nach wie vor die Eingabe des Benutzers.
Standardwert Ein String, der anfänglich im Edit-Feld des Eingabefensters erscheint. Der Benutzer kann den String z. B. mit der Rücktaste ändern.

ErrorLevel

Siehe unten.

Bemerkungen

Das Fenster erlaubt dem Benutzer, einen Text einzugeben und auf die Schaltflächen OK oder ABBRECHEN zu drücken. Der Benutzer kann das Fenster durch das Ziehen der Rahmen skalieren.

Wenn der Benutzer die ABBRECHEN-Schaltfläche betätigt: Bei AutoItv2-Scripts (.aut) ist die Ausgabevariable leer und ErrorLevel wird nicht geändert. Bei allen anderen Script-Typen (z. B. .ahk, .ini) wird ErrorLevel auf 1 gesetzt (oder 0 für OK) und die Ausgabevariable enthält den eingegeben Wert. Dadurch kann die ABBRECHEN-Schaltfläche nebenbei eine andere Funktion ausführen, wenn es gewünscht ist.

Wenn die Wartezeit des Fensters abläuft, wird ErrorLevel bei allen Script-Typen auf 2 gesetzt. Daher sollten AutoItv2-Scripts den ErrorLevel direkt auf 0 setzen, bevor dieser Befehl mit einer Zeitüberschreitung verwendet wird.

Ein GUI-Fenster kann eine modale InputBox mittels Gui, +OwnDialogs anzeigen. Eine modale InputBox hindert den Benutzer solange daran, mit dem GUI-Fenster zu interagieren, bis die InputBox wieder geschlossen wird.

Verwandte Befehle

GUI, Input, MsgBox, FileSelectFile, FileSelectFolder, SplashTextOn, ToolTip

Beispiel

InputBox, Passwort, Passwort eingeben, (die Eingabe wird versteckt), hide
InputBox, Benutzereingabe, Telefonnummer, Bitte geben sie eine Telefonnummer ein., , 640, 480
If ErrorLevel
    MsgBox, ABBRECHEN wurde gedrückt.
Else
    MsgBox, Sie haben "%Benutzereingabe%" eingegeben.
