DetectHiddenWindows

Legt fest, ob unsichtbare Fenster vom Script erkannt werden können.

DetectHiddenWindows, On|Off

Parameter

On|Off

On: Unsichtbare Fenster werden erkannt.

Off: Das wird standardmäßig verwendet. Unsichtbare Fenster werden nicht erkannt, außer mit dem Befehl WinShow.

Bemerkungen

Das Aktivieren von DetectHiddenWindows kann das Programmieren manchmal erschweren, da einige unsichtbare Systemfenster versehentlich den gleichen Titel oder Text wie das gewünschte Fenster beinhalten. Die meisten Scripts sollten deshalb diese Einstellung ausgeschaltet lassen. Das Aktivieren von DetectHiddenWindows kann nützlich sein, wenn die Absicht besteht, mit unsichtbaren Fenstern direkt zu arbeiten, ohne sie vorher mit WinShow sichtbar zu machen.

Alle Fensterbefehle, außer WinShow, können durch diese Einstellung sowie auch durch WinActivate, IfWinActive, WinWait und IfWinExist beeinflusst werden. Im Gegensatz dazu macht WinShow ein unsichtbares Fenster immer sichtbar, selbst wenn unsichtbare Fenster nicht erkannt werden können.

Das Aktivieren von DetectHiddenWindows ist nicht notwendig, wenn auf ein Steuerelement oder anhängendes Fenster mit der ahk_id-Methode oder dem zuletzt gefundenen Fenster zugegriffen wird. Das Aktivieren ist ebenfalls nicht notwendig, wenn auf GUI-Fenster mit Gui +LastFound zugegriffen werden.

Die interne Variable A_DetectHiddenWindows enthält die aktuelle Einstellung (On oder Off).

Jeder neu gestartete Thread (z. B. ein Hotkey, ein benutzerdefinierter Menüpunkt oder eine zeitlich festgelegte Subroutine) beginnt mit der Voreinstellung für diesen Befehl. Die Voreinstellung lässt sich ändern, wenn dieser Befehl im automatischen Ausführungsbereich (am Anfang des Scripts) ausgeführt wird.

Verwandte Befehle

DetectHiddenText

Beispiel

DetectHiddenWindows, on
