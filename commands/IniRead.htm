IniRead

Liest einen Wert von einer INI-Datei im Standardformat.

IniRead, Ausgabevariable, Dateiname [, Sektion, Schlüssel, Standardwert]

Parameter

Ausgabevariable

Der Name der Variable, in der der ermittelte Wert gespeichert wird. Wenn der Wert nicht ermittelt werden kann, dann enthält die Variable den Standardwert (siehe unten).

[AHK_L 57+]: Wenn der Schlüssel weggelassen wird, dann wird der Inhalt der angegebenen Sektion abgerufen, ohne jegliche Kommentare. Sobald die Sektion auch weggelassen wird, enthält die Ausgabevariable eine Liste mit Sektionsnamen, die durch LF-Zeichen (`n) getrennt sind.
Dateiname Der Name der Datei, die sich im A_WorkingDir-Verzeichnis befindet, solange kein absoluter Pfad angegeben ist. 
Sektion Name der Sektion in der INI-Datei. Solche Sektionen werden mit eckigen Klammern dargestellt (die Klammern dürfen nicht in diesem Parameter stehen).
Schlüssel Name des Schlüssels in der INI-Datei.
Standardwert

Dieser Wert wird in die Ausgabevariable gespeichert, wenn der angeforderte Schlüssel nicht gefunden wird. Wenn weggelassen, dann wird ERROR als Standard verwendet. Als leeren Wert kann %A_Space% benutzt werden.

[AHK_L 57+]: Dieser Parameter wird nicht verwendet, wenn der Schlüssel nicht vorhanden ist.

Für AutoIt-Scripts (.aut): Aus Kompatibilitätsgründen wird dieser Parameter nicht unterstützt; Das Wort ERROR wird immer in der Ausgabevariable gespeichert, wenn ein Problem beim Lesen des Wertes entstanden ist.

ErrorLevel

ErrorLevel wird nicht durch diesen Befehl beeinflusst. Bei einem Problem wird die Ausgabevariable auf den oben genannten Standardwert gesetzt.

Bemerkungen

Das Betriebssystem entfernt automatisch führende oder nachfolgende Leerzeichen/Tabulatoren in der Ausgabevariable.

Eine typische INI-Datei sieht wie folgt aus:

[Sektionsname]
Schlüssel=Wert

Verwandte Befehle

IniDelete, IniWrite, RegRead, dateilesende Schleife, FileRead

Beispiel

IniRead, Ausgabevariable, C:\Temp\EigeneDatei.ini, Sektion2, Schlüssel
MsgBox, Der Wert ist %Ausgabevariable%.
