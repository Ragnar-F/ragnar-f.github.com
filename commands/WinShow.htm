WinShow

Macht das angegebene Fenster wieder sichtbar.

WinShow [, Fenstertitel, Fenstertext, Titelausnahme, Textausnahme]

Parameter

Fenstertitel Der Titel oder Teiltitel des Fensters (das Übereinstimmungsverhalten wird mit SetTitleMatchMode bestimmt). Wenn dieser und die anderen 3 Fensterparameter nicht vorhanden sind, dann wird das zuletzt gefundene Fenster verwendet. Bei Verwendung einer Fensterklasse muss ahk_class ExakterKlassenname angegeben werden (wird von Windows Spy angezeigt). Bei Verwendung einer Prozess-Kennung (PID) muss ahk_pid %VariableMitPID% angegeben werden. Um eine Fenstergruppe anzuzeigen, verwendet ahk_group EigenerGruppenname (in diesem Fall müssen Fenstertext, Titelausnahme und Textausnahme leer sein). Bei Verwendung einer eindeutigen ID-Nummer eines Fensters muss ahk_id %VariableMitID% angegeben werden. Die Suche kann durch Angabe mehrerer Kriterien eingeschränkt werden. Zum Beispiel: Eigene Datei.txt ahk_class Notepad
Fenstertext Wenn vorhanden, dann muss dieser Parameter einen Teil der einzelnen Textelemente des Fensters enthalten (wird von Windows Spy angezeigt). Versteckte Textelemente werden erfasst, wenn DetectHiddenText aktiv ist.
Titelausnahme Fenstertitel mit diesem Wert werden nicht berücksichtigt.
Textausnahme Fenstertexte mit diesem Wert werden nicht berücksichtigt.

Bemerkungen

WinShow ist normalerweise der einzige Befehl, der in jedem Fall verborgene Fenster findet. Andere Befehle können dies nur, wenn DetectHiddenWindows aktiv ist.

Dieser Befehl arbeitet immer nur mit dem ersten gefundenen Fenster, es sein denn, es befindet sich ahk_group Groupname in Fenstertitel. In dem Fall sind alle Fenster der Gruppe betroffen.

Verwandte Befehle

WinHide, SetTitleMatchMode, DetectHiddenWindows, Zuletzt gefundenes Fenster

Beispiel

Run, notepad.exe
WinWait, Unbenannt - Editor
Sleep, 500
WinHide  ; Da hier der Parameter weggelassen wird, verwendet es das oben gefundene Fenster.
Sleep, 1000
WinShow
